## 1. 바이브 코딩 시대가 열리다

### 1.1 바이브 코딩이란?

2025년 초, OpenAI 공동창업자 **Andrej Karpathy**가 "바이브 코딩(Vibe Coding)"이라는 용어를 만들었습니다. 정의는 간단합니다:

> **AI에게 자연어로 설명하면 코드를 생성한다 — 코드를 이해하지 않고도 결과물을 만들 수 있다.**

숫자로 보면 이 변화의 규모가 보입니다:

| 지표 | 수치 | 출처 |
|------|------|------|
| 미국 개발자 중 AI 코딩 도구 일상 사용 | 92% | Google Cloud, 2025 |
| AI 에이전트가 작성하는 코드량 (인간 대비) | 약 10배 | IBM, 2026 |
| Fortune 500 중 바이브 코딩 플랫폼 도입 | 87% | Wikipedia |
| GitHub Copilot 사용 시 코딩 속도 향상 | 55% | GitHub/arXiv, 2023 |
| Copilot 사용자 수 | 150만+ | GitHub, 2025 |
| DORA Report: 전문 개발자 AI 도구 사용 | 90% 이상 | DORA, 2025 |

하지만 **바이브 코딩에는 그림자**도 있습니다. 이 부분은 Anti-Pattern에서 자세히 다룹니다.

### 1.3 경험 수준별 도구 선택 플로우차트

```
"코딩을 할 수 있는가?"
  ├─ No → "어떤 앱을 만드나?"
  │        ├─ 간단한 웹앱 → Lovable
  │        ├─ 빠른 프로토타입 → Bolt.new
  │        └─ 복잡한 앱 → Replit Agent
  │
  └─ Yes → "어떤 환경을 선호하나?"
           ├─ IDE (VS Code 스타일) → Cursor
           ├─ 기존 IDE 유지 → GitHub Copilot
           ├─ 터미널 → Claude Code
           └─ 브라우저 → Replit
```

## 2. 핵심 개념 이해

### 2.1 AI 코딩 도구의 3가지 유형

**유형 1: 자동완성형 (코드 어시스턴트)**

기존 IDE에 플러그인으로 설치. 코드를 타이핑하면 AI가 다음 줄을 제안합니다.

- 예: GitHub Copilot, Windsurf
- 장점: 기존 작업 환경 유지
- 적합: 코딩 경험이 있는 학생

**유형 2: AI 네이티브 IDE**

에디터 전체가 AI 중심으로 설계. 코드베이스를 이해하고 대규모 리팩토링 가능.

- 예: Cursor
- 장점: 코드베이스 전체 맥락 이해, 멀티파일 수정
- 적합: 중급 이상 개발자

**유형 3: 노코드 AI 빌더**

자연어로 설명하면 전체 앱을 생성. 코딩 지식 불필요.

- 예: Replit Agent, Lovable, Bolt.new
- 장점: 코딩 경험 없이 앱 제작 가능
- 적합: 비전공자, 아이디어 검증

### 2.2 코딩 에이전트란?

2025~2026년 AI 코딩의 가장 큰 변화는 **코딩 에이전트**의 등장입니다. 기존 AI 코딩 도구가 "질문에 답하는" 수준이었다면, 에이전트는 **자율적으로 코딩 작업을 수행**합니다:

- GitHub 이슈에서 자동으로 PR(Pull Request) 생성
- 에러를 감지하고 자동으로 수정
- 테스트를 작성하고 실행하여 검증
- 여러 파일을 동시에 수정

GitHub Copilot, Cursor, Claude Code 모두 에이전트 기능을 탑재하고 있습니다.

## 3. 따라하기 실습: AI로 웹앱 만들기

### 실습 A: Replit으로 할일 관리 앱 만들기 ⭐

**사용 도구:** Replit (replit.com)

**Step 1:** replit.com에 접속하여 회원가입합니다 (Google 계정으로 가능).

**Step 2:** "Create Repl" 버튼을 클릭합니다.

**Step 3:** Agent에 다음 프롬프트를 입력합니다:

```
할일 관리 웹앱을 만들어줘.

기능:
1. 할일 추가 (텍스트 입력)
2. 완료 체크 (체크박스)
3. 삭제 버튼
4. 완료된 할일과 미완료 할일 분리 표시
5. 오늘 날짜 자동 표시

디자인:
- 깔끔한 모던 UI
- 모바일 반응형
- 다크모드 지원
```

**Step 4:** Replit Agent가 자동으로:
- 프로젝트 구조 생성
- 코드 작성
- 내장 브라우저에서 실시간 미리보기

**Step 5:** 기능을 추가하고 싶다면 채팅에 요청합니다:

```
마감일(due date) 기능을 추가해줘.
마감일이 지난 할일은 빨간색으로 표시해줘.
```

**Step 6:** 배포하기 — "Deploy" 버튼 클릭 → URL 생성 → 누구나 접속 가능

✅ **체크포인트:** 할일 추가/삭제가 동작하나요? 완료 체크가 작동하나요? 배포 URL이 생성되었나요?

🔧 **트러블슈팅:**
- **"에러가 발생합니다":** 에러 메시지를 그대로 Replit Agent 채팅에 붙여넣으면 자동 수정
- **"화면이 안 보여요":** 내장 브라우저의 "Refresh" 클릭
- **"느려요":** 무료 플랜은 성능 제한이 있음. 복잡한 앱은 Core 플랜 고려

> **왜 Replit인가?** 설치가 필요 없고, 브라우저에서 바로 시작할 수 있으며, AI Agent가 전체 앱을 자동으로 만들어줍니다. 비전공자가 AI 코딩을 경험하는 **가장 쉬운 경로**입니다.

### 실습 B: GitHub Copilot으로 코딩 생산성 높이기 ⭐⭐

**사용 도구:** VS Code + GitHub Copilot

**Step 1:** GitHub 학생 인증하기

1. github.com에 로그인
2. education.github.com/pack에 접속
3. "Get your pack" 클릭
4. 학교 이메일(.ac.kr)로 인증 또는 학생증 업로드
5. 승인 후 **Copilot Student 플랜 활성화** (무료, 200만+ 학생 사용 중)

**Step 2:** VS Code에 Copilot 설치

1. VS Code 열기
2. Extensions → "GitHub Copilot" 검색 → Install
3. GitHub 계정으로 로그인
4. 하단 상태바에 Copilot 아이콘 확인

**Step 3:** Copilot 자동완성 체험

```python
# Python 파일에서 다음을 입력합니다:
def calculate_gpa(grades):
    # 여기서 Tab을 누르면 Copilot이 나머지를 제안합니다
```

**Step 4:** Copilot Chat 활용

VS Code에서 `Ctrl+I` (Mac: `Cmd+I`)를 누르고:

```
이 함수에 에러 핸들링을 추가해줘.
빈 리스트가 입력되면 0.0을 반환하고,
유효하지 않은 학점이 있으면 ValueError를 발생시켜줘.
```

**Step 5:** 테스트 코드 자동 생성

```
# Copilot Chat에 요청:
calculate_gpa 함수의 유닛 테스트를 pytest로 작성해줘.
정상 케이스, 빈 리스트, 유효하지 않은 학점 케이스를 포함해줘.
```

✅ **체크포인트:** Copilot이 코드 제안을 회색 텍스트로 표시하나요? Tab으로 제안을 수락할 수 있나요? Copilot Chat에서 질문에 답변하나요?

🔧 **트러블슈팅:**
- **"Copilot이 제안을 안 해요":** 하단 상태바에서 Copilot 아이콘 확인, 비활성화 상태면 클릭하여 활성화
- **"학생 인증이 안 돼요":** .ac.kr 이메일이 없으면 학생증 사진으로 인증 가능 (1~7일 소요)
- **"제안이 영어만 나와요":** 한국어 주석을 쓰면 한국어 맥락의 코드를 제안합니다

### 실습 C: Cursor로 풀스택 프로젝트 개발하기 ⭐⭐

**사용 도구:** Cursor (cursor.com)

**Step 1:** cursor.com에서 Cursor를 다운로드하고 설치합니다.

**Step 2:** 새 프로젝트 폴더를 열고, `Ctrl+L` (Chat)을 열어 다음을 입력합니다:

```
Next.js + TypeScript로 개인 블로그를 만들어줘.

기능:
1. 마크다운으로 글 작성
2. 글 목록 페이지 (최신순)
3. 글 상세 페이지
4. 태그 필터링
5. 다크모드 토글

기술 스택: Next.js 14, TypeScript, Tailwind CSS, shadcn/ui
```

**Step 3:** Cursor의 **Plan Mode**를 활성화합니다:
- Chat에서 "Plan" 버튼 클릭
- AI가 코드 수정 계획을 제시
- 계획을 검토하고 "Apply" 클릭

**Step 4:** 코드베이스 인덱싱 활용:

```
# @codebase 태그를 사용하면 전체 프로젝트를 참조합니다
@codebase 이 프로젝트의 폴더 구조를 설명해줘.
중복되거나 사용하지 않는 파일이 있으면 알려줘.
```

**Step 5:** Agent 모드로 대규모 작업:

```
# Ctrl+I → Agent 모드
블로그에 댓글 기능을 추가해줘.
Supabase를 사용해서 댓글을 저장하고,
실시간으로 새 댓글이 표시되게 해줘.
```

✅ **체크포인트:** Cursor가 여러 파일을 동시에 수정하나요? Plan Mode에서 계획을 미리 확인할 수 있나요? @codebase로 전체 프로젝트를 참조할 수 있나요?

## 4. 프롬프트 진화 패턴

### AI 코딩 프롬프트의 진화

**🔴 Level 1 — 단순 요청:**

```
파이썬으로 계산기 만들어줘
```

→ 기본적인 콘솔 계산기가 나옴. 에러 핸들링, UI, 테스트 없음.

**🟡 Level 2 — 조건 추가:**

```
React로 할일 관리 앱을 만들어줘.
CRUD 기능, 로컬 스토리지 저장, 반응형 디자인.
```

→ 기능이 구체적이나, 기술 스택과 품질 기준이 없음.

**🟢 Level 3 — 완전한 스펙:**

```
Next.js 14 + TypeScript + Supabase로 팀 프로젝트 관리 앱:
■ 기능: 프로젝트 CRUD, 태스크 칸반보드, 팀원 초대, 실시간 알림
■ 인증: Supabase Auth (Google OAuth)
■ DB: Supabase PostgreSQL (RLS 적용)
■ UI: shadcn/ui + Tailwind CSS, 다크모드
■ 배포: Vercel
■ 추가: 에러 핸들링, 로딩 상태, 빈 상태(empty state) 포함
각 기능을 모듈별로 분리하고, 테스트 코드 포함.
```

→ 기술 스택, 인증, DB, UI, 배포, 품질 기준까지 모두 지정. AI가 프로덕션 수준의 코드를 생성.

| **코드 설명** | ★★★★☆ | ★★★★★ | ★★★★☆ | ★★★☆☆ |
| **에러 핸들링** | ★★★★☆ | ★★★★★ | ★★★☆☆ | ★★★☆☆ |
| **테스트 코드** | ★★★★☆ | ★★★★★ | ★★★☆☆ | ★★☆☆☆ |
| **디버깅 능력** | ★★★★☆ | ★★★★★ | ★★★★☆ | ★★★☆☆ |

**도구별 특징:**

- **ChatGPT:** 가장 대중적. 코드 생성 속도가 빠르고, Code Interpreter로 즉시 실행 가능. 하지만 긴 코드에서 맥락을 잃을 수 있음.
- **Claude:** 코드 품질이 가장 높음. 에러 핸들링, 엣지 케이스까지 고려. "이 코드의 잠재적 문제점은..."을 자발적으로 설명. 디버깅에 최적.
- **Gemini:** Google 생태계와 통합. 코드 실행 환경 내장. 다만 복잡한 로직에서 불안정할 수 있음.
- **Perplexity:** 코딩보다는 "어떤 도구/라이브러리를 써야 하나?" 같은 기술 조사에 적합. 코드 생성은 보조적.

**최적 조합:** ChatGPT/Claude(코드 생성) → Perplexity(기술 조사) → GitHub Copilot/Cursor(IDE에서 실행)

## 6. 실전 사례: AI 코딩의 빛과 그림자

### 사례 1: GitHub Copilot — Accenture 대규모 RCT로 입증된 생산성

Accenture 개발자를 대상으로 한 **무작위 대조 실험(RCT)**에서 GitHub Copilot의 효과가 과학적으로 입증되었습니다:

| 지표 | 결과 |
|------|------|
| PR(Pull Request) 증가 | +8.69% |
| PR 병합률 증가 | +15% |
| 성공적 빌드 증가 | +84% |
| 코드 예제 검색 시간 감소 | -54% |
| 반복 작업 정신적 부담 감소 | -70% |
| "코딩이 더 즐거워졌다" 응답 | 95% |
| 제안 수용률 | 약 30% |
| 수용된 코드 중 유지된 비율 | 88% |

특히 주목할 점은 **제안 수용률이 30%**라는 것입니다. 개발자들은 Copilot의 제안 중 70%를 거부합니다. AI 코딩 도구는 "모든 코드를 자동으로 생성하는 것"이 아니라, **"좋은 출발점을 빠르게 제공하는 것"**입니다.

### 사례 2: 바이브 코딩 — 비전공자도 개발하는 시대

Andrej Karpathy가 창시한 바이브 코딩은 2026년 가장 뜨거운 트렌드입니다. 하지만 데이터는 양면을 보여줍니다:

**밝은 면:**
- Y Combinator W2025: 스타트업 25%가 코드베이스 95%를 AI로 생성
- Fortune 500의 87%가 바이브 코딩 플랫폼 도입
- 코딩 경험 0에서 풀스택 SaaS 앱 완성 가능 (Udemy 코스 증명)

**어두운 면:**
- AI 공동 작성 코드: 주요 이슈 **1.7배**, 로직 오류 **75% 더 많음** (2025.12 분석)
- 전문 개발자 중 **12%만** 업무에 바이브 코딩 사용 (72%는 사용하지 않음)
- Copilot 제안 수용률 30% — 전문가는 70%를 거부

## 7. 꿀팁 & 주의사항

### 꿀팁 5선

**Tip 1: GitHub Student Developer Pack을 반드시 신청하라**

GitHub Copilot 무료 외에도: .tech 도메인 무료, Canva Pro 무료, GitHub Pages(포트폴리오 호스팅), Codespaces(클라우드 개발 환경)가 포함됩니다. education.github.com/pack에서 신청하세요.

**Tip 2: "코드를 설명해줘"가 최고의 학습법이다**

AI 코딩 도구의 가장 강력한 기능은 코드 **생성**이 아니라 **설명**입니다:

```
이 코드를 한 줄씩 설명해줘.
초보자가 이해할 수 있는 수준으로,
각 줄이 왜 필요한지 포함해서.
```

**Tip 3: 테스트 코드를 먼저 요청하라**

AI에게 코드를 요청하기 전에 테스트를 먼저 요청하세요:

```
학생 성적 관리 프로그램의 유닛 테스트를 먼저 작성해줘.
정상 케이스, 엣지 케이스, 에러 케이스를 포함해줘.
그 다음에 이 테스트를 통과하는 코드를 작성해줘.
```

**Tip 4: Cursor의 @codebase를 적극 활용하라**

Cursor에서 `@codebase`를 붙이면 전체 프로젝트를 맥락으로 참조합니다. 대규모 프로젝트에서 특히 유용합니다:

```
@codebase 이 프로젝트에서 사용자 인증 로직이 어디에 있어?
```

**Tip 5: AI 생성 코드를 커밋 전에 반드시 읽어라**

AI가 생성한 코드를 "돌아가니까 커밋"하지 마세요. 최소한 다음을 확인하세요:
- 하드코딩된 API 키나 비밀번호가 없는가?
- 불필요한 console.log가 남아 있지 않은가?
- 에러 핸들링이 적절한가?

### ⚠️ Anti-Pattern 1: "코드는 돌아가지만 품질은 위험" — 바이브 코딩의 그림자

**상황:** 학생 L이 Replit으로 웹앱을 완성. "AI가 다 해줬다! 완벽해!"

**잘못된 방법:** AI 생성 코드를 읽지 않고 그대로 배포. 테스트 없음, 코드 리뷰 없음.

**결과:**
- 보안 취약점(SQL 인젝션) 발견. 사용자 데이터 유출 위험
- 로직 오류로 계산 결과 불일치
- 서비스 신뢰도 붕괴

**올바른 방법:**
1. AI 생성 코드를 반드시 리뷰 — 특히 보안 취약점 확인
2. 테스트 코드를 AI에게 먼저 요청 — 코드보다 테스트가 먼저
3. 보안 검사 수행 — API 키, 인젝션, XSS 등 기본 체크

**차이:** 2025년 12월 대규모 분석에 따르면, AI가 공동 작성한 코드는 인간이 작성한 코드 대비 주요 이슈 **1.7배**, 로직 오류 **75%** 더 많습니다. 전문 개발자의 **72%가 프로덕션 코드에 바이브 코딩을 사용하지 않는** 이유입니다. **"돌아간다 ≠ 안전하다"**를 기억하세요.

<!-- IMAGE: infographic - Anti-Pattern 비교: "검증 없는 AI 코딩(이슈 1.7배, 로직 오류 75%)"(❌) vs "검증 있는 AI 코딩(리뷰+테스트+보안검사)"(✅). Part 6 테마 레드(#E74C3C) 적용 -->

---

### ⚠️ Anti-Pattern 2: "AI가 코딩해줘도 디버깅은 내가 해야 한다"

**상황:** 대학생 K가 Cursor로 웹앱을 완성. "AI가 다 해줬다."

**잘못된 방법:** 배포 후 에러 발생 → 에러 메시지를 AI에 복사 → AI 수정 코드 적용 → 새로운 에러 → 반복.

**결과:**
- 에러가 누적. 코드가 점점 복잡해지며 AI도 해결 불가
- 결국 처음부터 다시 시작
- 시간과 노력 모두 낭비

**올바른 방법:**
1. 에러 메시지를 직접 읽고 이해 — 에러의 유형과 위치 파악
2. AI에게 "이 에러가 왜 발생하는지 설명해줘"로 학습 → 직접 수정
3. 스택 트레이스를 이해하여 에러의 근본 원인 추적

**차이:** AI 코딩 도구는 "에러 없는 코드"를 만드는 게 아니라 "에러를 더 빠르게 만나게" 해줍니다. 기초 디버깅 능력(에러 메시지 읽기, 스택 트레이스 이해)은 AI 시대에도 필수입니다. 특히 "이 에러 고쳐줘"(❌)가 아닌 "이 TypeError가 왜 발생하는지 설명하고, 원인별로 해결법을 알려줘"(✅)로 AI에게 질문하는 것이 핵심입니다.

## 8. 워크플로우 일반화: AI 코딩의 보편 원칙

### 원칙 1: 경험 수준에 맞는 도구를 선택하라

| 코딩 경험 | 추천 도구 | 이유 |
|-----------|----------|------|
| 코딩 불가 (비전공자) | Lovable, Replit Agent | 코딩 없이 앱 제작 |
| 코딩 가능 (초보~중급) | Cursor, GitHub Copilot | 자동완성 + 코드 설명 + 학습 |
| 코딩 전문 (개발자) | Claude Code, Cursor, Copilot | 대규모 리팩토링, 멀티에이전트 |

### 원칙 2: AI 코딩 = 80% 가속 + 20% 검증

AI가 코드의 80%를 빠르게 생성해주지만, 나머지 20%는 인간의 검증이 필요합니다:

```
AI 코딩 워크플로우:
[1] AI에게 요구사항 전달 (프롬프트)
[2] AI 생성 코드 리뷰 (이해 + 검증)
[3] 테스트 작성 및 실행 (자동/수동)
[4] 보안 검사 (API 키, 취약점)
[5] 커밋 + 배포
```

<!-- IMAGE: concept - "AI 코딩 워크플로우 5단계: 프롬프트 → 코드 리뷰 → 테스트 → 보안 검사 → 배포. '검증 없는 코딩'과 '검증 있는 코딩'의 결과 비교(이슈 1.7배 차이)" Before-After 인포그래픽. 테마 컬러: #E74C3C 레드 -->

### 원칙 3: 학습 목적의 AI 코딩 vs 생산성 목적의 AI 코딩

| 목적 | 접근법 | 핵심 질문 |
|------|--------|----------|
| **학습** | AI 생성 코드를 한 줄씩 이해 | "왜 이렇게 동작하지?" |
| **생산성** | AI에게 반복 작업 위임 | "이 작업을 더 빠르게 할 수 있나?" |

프로그래밍 수업에서는 **학습 목적**으로, 프로젝트/창업에서는 **생산성 목적**으로 사용하세요. 두 목적을 혼동하면 "AI가 다 해줬는데 아무것도 모르겠다"는 상태가 됩니다.

### 원칙 4: 카테고리당 1개 도구만 마스터하라

도구가 너무 많습니다. 모든 도구를 써보려 하지 말고, 각 카테고리에서 1개만 깊이 익히세요:

- **자동완성:** GitHub Copilot (학생 무료)
- **AI IDE:** Cursor 또는 Windsurf (택 1)
- **노코드 빌더:** Replit 또는 Lovable (택 1)
- **UI 생성:** v0

> 본문에서 ChatGPT·GitHub Copilot으로 코딩과 개발을 보조하는 방법을 배웠습니다. 이번 보충에서는 **Claude Code가 코딩·개발 보조의 본령**임을 실습합니다. 본문의 핵심 통계가 AI 코딩의 현재를 보여줍니다: 미국 개발자 **92%**가 AI 코딩 도구를 일상 사용하고(Google Cloud, 2025), GitHub Copilot 사용 시 코딩 속도가 **55%** 향상되며(GitHub/arXiv), Accenture RCT에서 성공적 빌드 **+84%**, 코드 예제 검색 시간 **-54%**가 입증되었습니다. 핵심 차이: Copilot이 "코드 한 줄 자동완성"이라면, Claude Code는 **"프로젝트 전체를 이해하고 설계→구현→테스트→디버깅"을 일관되게 수행**합니다. 본문의 SWE-Bench 벤치마크에서 Copilot 해결률 **56.0%** vs Cursor **51.7%**이지만 속도는 Cursor가 30% 빠른 결과와 비교하면, CC는 **코드베이스 전체 이해 + 멀티파일 자동 수정**에서 차별화됩니다. 이 챕터는 CC 교재의 마지막으로, 지금까지 배운 모든 개념(에이전트, CLAUDE.md, 스크립트, 하네스)을 활용하여 **실제 웹 애플리케이션을 개발**하는 종합 실습입니다.

## 1. Claude Code로 개발하는 것의 의미

### 1-1. 전통 개발 보조 도구 vs Claude Code

| 항목 | GitHub Copilot | ChatGPT (코드 생성) | Claude Code |
|------|---------------|-------------------|-------------|
| 작동 방식 | IDE 내 자동완성 | 대화형 코드 생성 | **프로젝트 전체 이해 + 실행** |
| 맥락 범위 | 현재 파일 중심 | 대화창에 붙여넣은 코드 | **전체 코드베이스** |
| 파일 관리 | 한 파일 수정 | 복사-붙여넣기 | **직접 파일 생성·수정** |
| 실행 | 불가 | 불가 (Code Interpreter 제외) | **터미널 직접 실행** |
| 테스트 | 불가 | 코드만 생성 | **테스트 작성 + 실행 + 수정** |
| 디버깅 | 제한적 | 에러 붙여넣기 → 해결책 | **에러 확인 → 수정 → 재실행** |
| 프로젝트 구조 | 인식 제한 | 없음 | **폴더 구조 인식 + 설계** |
| 벤치마크 (SWE-Bench) | 해결률 56.0%, 속도 89.9초 | — | **코드베이스 전체 이해 + 멀티에이전트** |
| 제안 수용률 | 30% (70%는 개발자가 거부) | — | CLAUDE.md 규칙으로 품질 기준 강제 |

### 1-2. CC 개발 워크플로우

```
[요구사항 정의] → [프로젝트 설계] → [코드 작성] → [테스트] → [디버깅] → [배포]
       ↓              ↓              ↓            ↓           ↓           ↓
   자연어 설명      폴더 구조       파일 생성     테스트 실행   에러 수정     스크립트
   CLAUDE.md        아키텍처       + 주석 상세    + 결과 확인   + 재실행     자동화
```

**architect.md**:
```markdown
---
name: architect
description: "프로젝트 구조를 설계하고 기술 스택을 결정하는 에이전트"
---
# Architect — 아키텍트

## 핵심 역할
- 요구사항을 분석하여 프로젝트 구조를 설계합니다.
- 기술 스택(프레임워크, 라이브러리)을 선정합니다.
- API 설계, 데이터 모델, 폴더 구조를 정의합니다.

## 설계 원칙
- 초보자가 이해할 수 있는 단순한 구조 우선
- 기술 스택 선정 이유를 명시 (왜 이 기술인지)
- API는 RESTful 원칙 준수
- 폴더 구조는 기능별 분리 (feature-based)

## 추천 스택 (대학생용)
- 프론트엔드: HTML/CSS/JavaScript (또는 React 기초)
- 백엔드: Python Flask 또는 Node.js Express
- 데이터베이스: SQLite (설치 불필요)
- 배포: GitHub Pages (정적) 또는 Render (동적)

## 출력
- docs/architecture.md: 프로젝트 설계서
- docs/api_spec.md: API 명세
```

**developer.md**:
```markdown
---
name: developer
description: "설계를 바탕으로 코드를 구현하는 에이전트"
---
# Developer — 개발자

## 핵심 역할
- architect의 설계를 바탕으로 코드를 작성합니다.
- 프론트엔드와 백엔드를 모두 구현합니다.
- 코드에 학습용 주석을 상세히 작성합니다.

## 작업 원칙
- 모든 코드에 한글 주석 (이 코드가 무슨 역할인지)
- 함수당 주석 필수 (입력, 출력, 동작 설명)
- 변수명은 의미 있게 (x, y 금지)
- 에러 처리 포함 (try-except/try-catch)
- 파일당 200줄 이하 (가독성)
- 코드 작성 후 "이 코드의 학습 포인트" 요약

## 출력
- src/: 소스 코드
```

```markdown
- 단위 테스트: 함수별 정상/비정상 입력 테스트

## 출력
- tests/: 테스트 코드
- docs/test_report.md: 테스트 결과 리포트
```

```markdown
# 풀스택 개발 규칙

## 코딩 컨벤션
- 들여쓰기: 2칸 (JavaScript), 4칸 (Python)
- 변수명: camelCase (JS), snake_case (Python)

- 파일당 200줄 이하

## 보안 원칙
- 하드코딩 비밀번호/API키 금지 → 환경변수 사용
- SQL 인젝션 방지 (파라미터 바인딩)
- XSS 방지 (입력값 이스케이프)
- CORS 설정 명시

## 코드 품질 원칙 (본문 데이터 연계)
- AI 공동 작성 코드는 이슈 **1.7배**, 로직 오류 **75%** 더 많음 (2025.12 분석) — 반드시 리뷰 후 커밋
- Copilot 제안 수용률 **30%** — 전문가는 70%를 거부함. AI 코드를 무조건 수용하지 말 것
- 전문 개발자 **72%가 프로덕션에 바이브 코딩 미사용** — "돌아간다 ≠ 안전하다"
- 커밋 전 체크리스트: API 키 노출, console.log 잔재, 에러 핸들링 적절성

## 학습 지원
- 새로운 개념 사용 시 "📚 학습 포인트" 주석 추가
- 코드 블록마다 "이 코드가 하는 일" 한 줄 요약
- 에러 발생 시 "왜 에러가 발생했는지" 설명 포함
```

### ❌ 흔한 실수: AI 코딩의 함정

**상황**: Claude Code로 코딩 프로젝트를 할 때

---

**잘못된 방법 — AI가 생성한 코드를 이해하지 않고 사용**:

```
코드 전체를 만들어줘. 동작만 하면 돼.
```

**결과**: AI가 만든 코드가 동작하지만, 왜 이렇게 작동하는지 모릅니다. 과제 발표에서 "이 부분은 어떻게 동작하나요?"라는 질문에 답하지 못합니다. 본문 Anti-Pattern 2 "디버깅은 내가 해야 한다"와 동일 — 에러 메시지를 AI에 복사 → AI 수정 코드 적용 → 새로운 에러 → 반복의 악순환에 빠지면 코드가 점점 복잡해져 결국 처음부터 다시 시작하게 됩니다.

**올바른 방법**: developer 에이전트가 모든 코드에 **학습용 주석**("📚 학습 포인트")을 달도록 CLAUDE.md에 규칙을 설정합니다. 코드를 받은 후 주석을 읽고, 이해가 안 되는 부분은 CC에게 추가 설명을 요청합니다.

**차이**: CC는 **"코드를 대신 써주는 도구"**가 아니라, **"코드를 가르쳐주는 도구"**입니다.

---

**잘못된 방법 — 보안을 무시한 코드**:

```
빨리 동작하게만 만들어줘. 보안은 나중에.
```

**결과**: API 키가 코드에 하드코딩되고, SQL 인젝션에 취약한 코드가 생성됩니다. 배포 후 문제가 발생합니다.

**올바른 방법**: CLAUDE.md에 보안 원칙(환경변수, 파라미터 바인딩, XSS 방지)을 설정하면, developer가 처음부터 안전한 코드를 작성합니다.

**차이**: "나중에 고치자"는 대부분 "영원히 안 고친다"가 됩니다. CLAUDE.md로 **처음부터 안전하게**.

## 실습 과제

### 🎯 실습: 프로젝트 설계 + 기본 구현 ⭐

**목표**: 간단한 웹 앱을 설계하고 기본 구조를 구현합니다.
**소요 시간**: 10분
**준비물**: Claude Code, Python 설치
**사용 도구**: Claude Code

---

#### Step 1. 프로젝트 설계 ⏱️ 4분

```
"나의 할 일 관리" 웹 앱을 설계해줘.

기술 스택: Python Flask + SQLite + HTML/CSS/JS
기능: 할 일 추가, 완료 체크, 삭제
폴더 구조 + 데이터 모델 + API 설계.
각 기술을 "왜 선택했는지" 설명도 포함.
```

✅ **체크**: 폴더 구조 + 데이터 모델 + API 명세가 생성되면 성공

---

#### Step 2. 백엔드 구현 ⏱️ 3분

```
설계대로 Flask 백엔드를 구현해줘.
- API 3개: 할 일 추가(POST), 목록(GET), 삭제(DELETE)
- SQLite DB 연동
- 모든 코드에 한글 주석
- 📚 학습 포인트 주석 포함
```

✅ **체크**: Flask 서버가 실행되면 성공 (`python app.py`)

---

#### Step 3. 프론트엔드 구현 ⏱️ 3분

```
간단한 HTML 프론트엔드를 만들어줘.
- 할 일 입력 폼
- 할 일 목록 표시
- 완료/삭제 버튼
- CSS 기본 스타일링
```

✅ **체크**: 브라우저에서 할 일 앱이 동작하면 성공

> 🔧 **안 되나요?**: Flask가 설치되지 않았다면 → `pip install flask` 실행. Python이 없다면 → "HTML+JavaScript만으로 동작하는 버전으로 만들어줘 (서버 없이)"라고 요청하세요.

### 🎯 실습: 기능 확장 + 테스트 ⭐⭐

**목표**: 기본 앱에 기능을 추가하고 테스트를 작성합니다.
**소요 시간**: 25분
**준비물**: 이전 실습의 기본 앱
**사용 도구**: Claude Code

---

#### Step 1. 기능 확장 ⏱️ 10min

```
할 일 앱에 기능을 추가해줘:
1. 우선순위 (높음/중간/낮음) 설정
2. 마감일 설정
3. 카테고리별 필터링
4. 완료율 표시 (진행 바)

기존 코드를 수정하고, 변경점에 주석을 달아줘.
```

✅ **체크**: 4가지 기능이 추가되고 동작하면 성공

---

#### Step 2. 테스트 작성 + 실행 ⏱️ 8분

```
앱의 테스트를 작성하고 실행해줘.

테스트:
1. API 테스트: 각 엔드포인트 정상/에러 케이스
2. 시나리오 테스트: "사용자가 할 일을 추가하고 완료하는" 전체 흐름
3. 경계 케이스: 빈 입력, 특수문자, 긴 텍스트

테스트 실행 결과를 리포트로 정리.
```

✅ **체크**: 테스트가 실행되고 결과 리포트가 생성되면 성공

---

#### Step 3. 버그 수정 + 코드 리뷰 ⏱️ 7분

```
테스트에서 발견된 버그를 수정해줘.
그리고 전체 코드를 리뷰해줘:

체크 항목:
1. 보안: SQL 인젝션, XSS, 하드코딩 비밀번호
2. 가독성: 변수명, 주석, 파일 구조
3. 성능: 불필요한 DB 호출, N+1 쿼리
4. 학습 포인트: "이 코드에서 배울 수 있는 것" 요약
```

✅ **체크**: 버그 수정 + 코드 리뷰 리포트가 생성되면 성공

> 🔧 **안 되나요?**: 테스트가 실패하면 → "이 에러의 원인과 해결 방법을 설명해줘. 그리고 수정해줘"라고 요청하세요. CC가 에러를 읽고 직접 수정합니다.

---

#### 🎉 완성! — 결과물 미리보기

기능 확장 → 테스트 → 디버깅 → 코드 리뷰까지 **개발 사이클**을 경험했습니다.

> 📊 **효과**:
> - 기존 방식: 코딩 초보가 웹 앱 만들기 — 검색 + 에러 해결에 80% 시간 소비
> - CC 활용: 설계→구현→테스트→디버깅 자동화 — 학습에 집중

### 🎯 실습: 종합 풀스택 개발 하네스 구축 ⭐⭐⭐

**목표**: 설계, 개발, 테스트를 에이전트로 분업한 풀스택 개발 하네스를 구축합니다.
**소요 시간**: 50분
**준비물**: Claude Code, Python, Node.js (선택)
**사용 도구**: Claude Code

---

#### Step 1. 개발 하네스 구축 ⏱️ 12분

```
~/fullstack-app/ 에 풀스택 개발 하네스를 구축해줘.

폴더 구조: [위의 구조 참조]
에이전트 3개 + CLAUDE.md + 스크립트 3개
프로젝트: "대학생 스터디 관리 웹앱"
```

✅ **체크**: 전체 구조 + 에이전트 3개 + CLAUDE.md + 스크립트가 완성되면 성공

---

#### Step 2. 에이전트 순차 개발 ⏱️ 20분

```
스터디 관리 웹앱을 에이전트 순차 실행으로 개발해줘.

1단계 @architect: 설계 (폴더 구조, API, 데이터 모델)
2단계 @developer: 구현 (백엔드 + 프론트엔드 + 학습 주석)
3단계 @tester: 테스트 (단위 + API + 시나리오)

기능:
- 스터디 그룹 생성/관리
- 일정 등록 (날짜, 시간, 장소)
- 멤버 출석 체크
- 학습 기록 메모

scripts/setup.sh 실행 → scripts/dev.sh 실행으로 바로 동작.
```

✅ **체크**: `bash scripts/dev.sh`로 앱이 실행되면 성공

> ⚠️ **주의**: AI가 생성한 코드를 반드시 읽고 이해하세요. "📚 학습 포인트" 주석을 확인하고, 이해되지 않는 부분은 CC에게 설명을 요청하세요.

---

#### Step 3. 디버깅 + 코드 리뷰 ⏱️ 10분

```
테스트 결과를 바탕으로:
1. 모든 버그 수정
2. 보안 점검 (CLAUDE.md 규칙 기반)
3. 코드 리뷰 (가독성, 성능, 학습 가치)
4. "이 프로젝트에서 배운 것" 요약 문서 작성

docs/에 정리.
```

✅ **체크**: 버그 수정 + 보안 점검 + 코드 리뷰가 완료되면 성공

---

#### Step 4. 배포 준비 ⏱️ 8분

```
웹앱을 배포할 수 있도록 준비해줘.

1. README.md 작성 (프로젝트 소개, 설치·실행 방법, 스크린샷 위치)
2. requirements.txt / package.json 생성
3. 배포 옵션 안내:
   - GitHub Pages (정적 프론트엔드만)
   - Render / Railway (풀스택)
   - 대학 서버 (학교 제공 시)
4. scripts/deploy.sh 작성

⚠️ 배포 전 보안 체크리스트:
   - [ ] API 키/비밀번호 환경변수 처리
   - [ ] DEBUG 모드 OFF
   - [ ] CORS 설정 확인
```

✅ **체크**: README + 의존성 파일 + 배포 스크립트가 생성되면 성공

### 📊 하이브리드 전략 비교표: 본문 도구 vs CC 하네스

| 개발 단계 | 본문 도구 조합 | CC 하네스 | 최적 전략 |
|----------|--------------|----------|----------|
| **1. 설계** | ChatGPT/Claude로 아키텍처 상담 | architect 에이전트(폴더 구조·API·데이터 모델 자동) | CC architect가 **파일까지 직접 생성**하여 압도적 |
| **2. UI 프로토타입** | **v0(React UI 최강)** + Lovable(풀스택 노코드) | developer 에이전트(HTML/CSS/JS 코드 생성) | **v0/Lovable로 시각적 UI** → CC로 백엔드 연동·확장 |
| **3. 코드 구현** | GitHub Copilot(자동완성) + Cursor(코드베이스 이해) | developer 에이전트(학습 주석 + 보안 규칙 자동 적용) | Copilot/Cursor로 IDE 내 코딩 → CC로 멀티파일 리팩토링 |
| **4. 테스트** | Copilot Chat으로 테스트 코드 생성 | tester 에이전트(단위·API·시나리오 + **실행까지**) | CC tester가 **테스트 작성+실행+수정** 일관 수행으로 우위 |
| **5. 디버깅** | 에러를 ChatGPT에 붙여넣기 → 해결책 | CC가 **에러 확인→수정→재실행** 직접 수행 | CC의 **터미널 직접 실행**이 압도적 |
| **6. 배포** | Vercel/Render 수동 설정 | scripts/deploy.sh 자동화 | CC 스크립트로 **원클릭 배포** |

**핵심**: 본문의 "경험 수준별 도구 선택" 플로우차트 — 코딩 불가(Lovable/Replit) → 초보~중급(Cursor/Copilot) → 전문가(Claude Code) — 는 CC 하네스가 **전문가 수준의 개발 프로세스**를 제공한다는 것을 의미합니다. Accenture RCT의 "코딩이 더 즐거워졌다" **95%** 응답은 AI 코딩 도구 전체에 해당하지만, CC의 차별화는 **"학습 주석 규칙"**입니다. 본문의 "학습 목적 vs 생산성 목적" 구분에서 CC의 "📚 학습 포인트" 주석은 두 목적을 동시에 달성합니다.

3. **스크립트는 반복을 자동화한다** — 같은 작업을 두 번 수동으로 하지 않음
4. **input → _workspace → output** — 데이터 흐름이 명확함

> **핵심 정리**: Ch 20은 Claude Code 교재의 **피날레**입니다. 지금까지 배운 모든 개념 — 에이전트 팀, CLAUDE.md 규칙, 스크립트 자동화, 하네스 구조 — 을 활용하여 **실제 웹 애플리케이션을 개발**합니다. CC로 코딩하는 것의 의미는 "코드를 대신 써주는 것"이 아닙니다. architect가 설계하고, developer가 구현하고, tester가 검증하는 **소프트웨어 공학의 기본 사이클**을 체험하는 것입니다. CLAUDE.md의 **학습 주석 규칙**이 핵심 — AI가 생성한 모든 코드에 "📚 학습 포인트"가 포함되어, 코드를 읽는 것 자체가 학습이 됩니다. 이것이 Claude Code가 다른 코딩 도구와 다른 점이며, **"AI 시대의 프로그래밍 교육"**이 나아갈 방향입니다.
