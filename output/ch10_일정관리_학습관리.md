## 1. 이런 분이 읽으면 좋아요

**이런 분에게 추천합니다:**

- 시험 기간에 항상 **벼락치기**를 하게 되는 분
- 과제 마감일을 잊어서 **감점당한 경험**이 있는 분
- 학습 계획은 세우지만 **3일 만에 포기**하는 분
- AI로 과제는 하는데, **학습 자체에 AI를 활용**하는 방법은 모르는 분
- 5과목 이상 수강하면서 일정 관리가 **점점 복잡해지는** 분

**이런 분은 건너뛰어도 됩니다:**

- 이미 Notion이나 전자 플래너로 체계적인 학습 관리를 하고 있는 분 → Section 4(간격 반복) 부분만 확인
- 일정 관리보다 특정 도구 사용법이 궁금한 분 → 해당 도구 공식 가이드 참조

## 2. 이 챕터에서 배우는 것

### 학습목표

이 챕터를 마치면 다음을 할 수 있습니다:

1. **AI 학습 플래너**(Notion AI, ChatGPT Tasks, Trevor AI)의 특성을 비교하고 자신에게 맞는 도구를 선택할 수 있다
2. **ChatGPT로 학기 학습 계획**을 생성하고, 주간/일간 단위로 실행 가능한 일정을 만들 수 있다
3. **간격 반복(Spaced Repetition)** 원리를 이해하고, AI로 시험 대비 플래시카드를 자동 생성할 수 있다
4. **AI 기반 자기 피드백 시스템**으로 학습 패턴을 분석하고 개선할 수 있다
5. AI 학습 도구의 **의존성 함정**을 인지하고, 능동적 학습 전략을 설계할 수 있다

### 완성 후 얻는 결과물

> 이 챕터의 실습을 마치면, **이번 학기 맞춤형 학습 계획표**(주간 시간표 + 시험 대비 플랜)와 **AI 생성 플래시카드 세트**(시험 과목 핵심 개념 20장)를 갖게 됩니다.

## 4. 핵심 개념

### 간격 반복(Spaced Repetition) — 과학적으로 증명된 학습법

간격 반복은 **"까먹을 만할 때 다시 보면 기억이 강화된다"**는 원리입니다. 에빙하우스의 망각 곡선에 따르면, 한 번 학습한 내용은 1일 후 74%, 1주 후 90% 이상 잊습니다. 하지만 적절한 간격으로 복습하면 **기억 유지율을 90%까지 높일 수 있습니다.**

**학술 검증 (SAGE Journals, 2025):**
- Anki 기반 간격 반복을 사용한 의대생이 비사용 학생 대비 시험 성적 **6.2~12.9% 향상**
- 성숙 카드 수(반복 학습 완료한 카드)와 시험 성적 간 **유의미한 양의 상관관계**
- 종합 시험(CBSE)에서 **+12.9%** 향상 효과

> 📌 **시사점**: 벼락치기보다 **매일 20분씩 간격 반복**하는 것이 시험 성적을 더 올립니다. AI가 이 과정을 자동화해줍니다.

### AI 학습 관리의 3축

| 축 | 내용 | AI 역할 |
|---|------|---------|
| **계획(Plan)** | 학기/주간/일간 학습 일정 | ChatGPT로 맞춤 계획 생성 |
| **실행(Do)** | 간격 반복, 능동적 회상, 자가 테스트 | Anki+ChatGPT로 플래시카드 자동 생성 |
| **피드백(Check)** | 학습량 분석, 부족 과목 식별, 개선 제안 | ChatGPT로 주간 학습 패턴 분석 |

이 3축을 AI로 **자동화**하면, 학습 관리에 드는 시간을 최소화하고 **실제 학습에 집중**할 수 있습니다.

### 🔬 프롬프트 진화 실험

같은 작업을 3단계 프롬프트로 시도하여, 학습 계획 프롬프트 품질의 차이를 직접 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
시험 공부 계획 짜줘
```

**결과**: "1. 범위를 파악합니다. 2. 중요한 부분을 정리합니다. 3. 반복 학습합니다." — 누구나 아는 일반론. 실행 불가능한 추상적 조언.

📋 **평가**:
- 정확성: ★★☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 과목, 시험일, 가용 시간, 현재 수준 정보가 전혀 없음

---

#### 🟡 Level 2 — 중급 프롬프트

```
경영전략론 중간고사가 2주 후입니다.
범위는 1~7장이고 사례 분석형 서술 시험입니다.
학습 계획을 짜줘.
```

**결과**: 14일 계획이 나옵니다. 하루에 1장씩 + 마지막 7일은 복습. 구조는 있지만 하루 몇 시간인지, 다른 과목과 어떻게 조율하는지, 본인의 약점은 무엇인지 반영되지 않습니다.

📋 **평가**:
- 정확성: ★★★☆☆
- 활용도: ★★★☆☆
- 개선점: 과목, 범위, 시험 형태가 명확해짐
- 아쉬운 점: 가용 시간, 다른 과목 일정, 현재 이해도 미반영

---

#### 🟢 Level 3 — 전문가 프롬프트

```
다음 주 수요일에 경영전략론 중간고사가 있습니다.

시험 정보:
- 범위: 1~7장 (포터의 5 Forces, 가치사슬, SWOT, BCG 매트릭스 등)
- 형태: 사례 분석형 서술 (2문항, 각 50점)
- 시간: 90분
- 교수님 스타일: 이론을 실제 기업 사례에 적용하는 것을 중시

나의 상황:
- 하루 자습 가능 시간: 2시간 (알바와 다른 수업 때문)
- 현재 이해도: 1~4장은 괜찮은데, 5~7장은 아직 수업 내용을 제대로 못 들었음
- 약점: 이론은 알지만 사례에 적용하는 연습이 부족

다음을 만들어줘:
1. 7일간 학습 계획 (하루 2시간 기준)
2. 각 장의 핵심 개념 요약 (시험 출제 가능성 기준)
3. 예상 사례 분석 문제 3개 + 모범답안 골격
4. 자가 점검 퀴즈 10문항
5. 시험 당일 30분 마지막 리뷰 체크리스트
```

**결과**: 7일간 세부 계획(1~2일: 5~7장 정리, 3~4일: 사례 적용 연습, 5~6일: 예상 문제 풀기, 7일: 총복습)이 나옵니다. 약점(5~7장, 사례 적용)에 더 많은 시간을 배분하고, 예상 문제와 모범답안까지 포함된 완전한 학습 패키지.

📋 **평가**:
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: **나의 상황(가용 시간, 이해도, 약점) + 시험 특성(교수 스타일) 명시**가 결정적

---

#### 💡 교훈: 학습 계획 프롬프트의 마법

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 과목/범위 | ✗ | ✓ | ✓ |
| 시험 형태 | ✗ | ✓ | ✓ |
| 가용 시간 | ✗ | ✗ | ✓ |
| 현재 이해도 | ✗ | ✗ | ✓ |
| 약점 | ✗ | ✗ | ✓ |
| 출력 형식 | ✗ | ✗ | ✓ (5가지 요청) |
| 결과 품질 | 15% | 50% | 95% |

> 📌 **핵심 원칙**: AI 학습 플래너의 핵심은 **"나의 현실적 제약을 정확히 알려주는 것"**입니다. AI는 당신의 알바 일정도, 수학이 약한 것도, 하루 5시간밖에 못 공부하는 것도 모릅니다. 맥락을 줘야 현실적인 계획이 나옵니다.

## 5. 따라하기 실습

### 🎯 실습 A: ChatGPT로 학기 학습 계획 세우기 ⭐

**목표**: ChatGPT를 활용하여 이번 학기 전체 학습 계획을 생성합니다.
**소요 시간**: 15분
**준비물**: ChatGPT 계정 (Free 가능), 이번 학기 수강 과목 정보
**사용 도구**: ChatGPT

---

#### Step 1. 수강 과목 정보 정리 ⏱️ 5분

먼저 본인의 수강 과목 정보를 정리합니다:

```
과목명 / 수업 시간 / 평가 비율(중간, 기말, 과제, 출석 등)
```

> 💡 수강 편람이나 실라버스에서 확인하세요. 정확한 평가 비율이 중요합니다.

✅ **체크**: 최소 3과목 이상의 정보가 정리되었으면 다음 단계로.

---

#### Step 2. 학습 계획 프롬프트 입력 ⏱️ 3분

ChatGPT에 다음 프롬프트를 입력합니다:

```
나는 [학과] [학년] 학생입니다.
이번 학기에 다음 과목을 수강합니다:

1. [과목명] ([요일/시간]) - [평가 비율]
2. [과목명] ([요일/시간]) - [평가 비율]
3. [과목명] ([요일/시간]) - [평가 비율]
4. [과목명] ([요일/시간]) - [평가 비율]
5. [과목명] ([요일/시간]) - [평가 비율]

나의 상황:
- 하루 자습 가능 시간: [N]시간
- [기타 제약: 알바, 동아리, 통학 시간 등]

다음을 만들어주세요:
1. 주간 학습 시간표 (수업 시간 + 자습 시간 포함)
2. 각 과목별 학습 전략 (평가 방법에 맞춤)
3. 중간고사 4주 전부터의 시험 준비 계획
4. 매일 할일 체크리스트 템플릿
```

> 💡 **커스텀 포인트**: 자습 시간은 **현실적으로** 설정하세요. AI에게 "하루 8시간 공부"라고 하면 비현실적 계획이 나옵니다.

✅ **체크**: AI가 주간 시간표를 표 형태로 제시하고, 과목별 학습 전략이 포함되었으면 성공.

---

#### Step 3. 계획 수정 및 현실화 ⏱️ 5분

AI의 계획에서 비현실적인 부분을 수정합니다:

```
몇 가지 수정해줘:
1. 수요일은 알바가 있어서 자습 시간 0시간이야
2. 통계학이 가장 약해서 다른 과목보다 30% 더 시간을 배분해줘
3. 금요일 저녁은 동아리 활동이라 빼줘
```

> 🔧 **안 되나요?**: 계획이 너무 빡빡하면 "전체적으로 80% 강도로 줄여줘. 여유 시간도 포함해줘."라고 요청하세요.

✅ **체크**: 수정된 계획이 본인의 생활 패턴과 맞으면 완료. **캘린더에 옮기거나 프린트하세요.**

---

#### Step 4. ChatGPT Tasks로 자동 알림 설정 (Plus 사용자) ⏱️ 2분

ChatGPT Plus 사용자라면 **Tasks** 기능으로 반복 알림을 설정합니다:

```
매일 오전 8시에 "오늘의 학습 목표"를 알려줘.
매주 일요일 저녁 8시에 "다음 주 학습 계획 요약"을 보내줘.
```

> 📍 **화면 안내**: ChatGPT 왼쪽 메뉴 → "Tasks" → "Create a task" → 반복 주기와 내용 설정

✅ **체크**: 설정한 시간에 알림이 오면 성공.

### 🎯 실습 B: AI로 시험 대비 플래시카드 만들기 ⭐⭐

**목표**: ChatGPT로 시험 범위의 핵심 개념 플래시카드를 자동 생성합니다.
**소요 시간**: 15분
**준비물**: ChatGPT 계정, 시험 범위 교재/노트
**사용 도구**: ChatGPT (+ Anki 또는 Quizlet 선택)

---

#### Step 1. 시험 범위 핵심 개념 추출 ⏱️ 5분

ChatGPT에 다음 프롬프트를 입력합니다:

```
[과목명] 시험 범위: [1~7장 또는 구체적 토픽]

이 범위에서 시험에 나올 가능성이 높은 핵심 개념 20개를 추출해줘.
각 개념에 대해:
- 앞면(질문): 핵심 질문 1개
- 뒷면(답변): 간결한 답변 (3줄 이내)
- 암기 팁: 연상 기억법이나 키워드

Anki 플래시카드 형식으로 만들어줘.
(앞면/뒷면이 탭으로 구분된 텍스트)
```

> 💡 **고급 팁**: 수업 노트나 교재 PDF가 있다면 ChatGPT에 업로드하세요. AI가 실제 수업 내용을 바탕으로 더 정확한 카드를 생성합니다.

✅ **체크**: 20개의 플래시카드가 "질문 - 답변" 형태로 생성되었으면 성공.

---

#### Step 2. 플래시카드 앱에 가져오기 ⏱️ 5분

**방법 A (Anki — 간격 반복 최강):**
1. ChatGPT 결과를 텍스트 파일(.txt)로 저장 (탭 구분 형식)
2. Anki → "파일" → "가져오기" → 텍스트 파일 선택
3. 구분자를 "탭"으로 설정 → "가져오기"

**방법 B (Quizlet — 간편):**
1. Quizlet(https://quizlet.com) → "만들기" → "학습 세트"
2. ChatGPT 결과를 "가져오기" 기능으로 붙여넣기
3. 구분자 설정 (앞/뒤: 탭, 카드 간: 줄바꿈)

✅ **체크**: 앱에서 카드를 넘기면서 학습할 수 있으면 성공.

---

#### Step 3. 간격 반복 학습 시작 ⏱️ 5분

가져온 카드로 **간격 반복 학습**을 시작합니다:

1. 카드 앞면(질문)을 보고 **답을 먼저 머릿속으로 떠올리세요** (능동적 회상)
2. 뒷면을 확인하고 **난이도를 평가**합니다 (쉬움/보통/어려움)
3. Anki가 난이도에 따라 **다음 복습 일정을 자동 조정**합니다

> 📌 **핵심**: SAGE Journals 연구에 따르면, 이 방법으로 학습한 학생은 시험 성적이 **최대 12.9% 향상**되었습니다. 핵심은 **매일 20분씩 꾸준히** 하는 것입니다.

✅ **체크**: 20장의 카드를 1회 학습 완료했으면 성공.

### 🎯 실습 C: AI 자기 피드백 시스템 만들기 ⭐⭐

**목표**: AI에게 주간 학습 기록을 입력하여 패턴 분석과 개선 제안을 받습니다.

#### Step 1. 주간 학습 기록 입력 ⏱️ 5분

```
이번 주 내 학습 기록입니다:

- 월: 경영전략론 2시간, 통계 1시간
- 화: 마케팅 1.5시간
- 수: 경영전략론 1시간, AI실습 2시간
- 목: 없음 (동아리 활동)
- 금: 통계 2시간
- 토: 영어 에세이 3시간
- 일: 없음

다음을 분석해줘:
1. 총 학습 시간과 과목별 분배 평가
2. 부족한 과목과 추가 학습 필요량
3. 시간 활용 패턴의 문제점
4. 다음 주 개선된 학습 계획 제안

```

✅ **체크**: AI가 총 학습 시간, 과목별 불균형, 개선 제안을 포함한 분석을 제시하면 성공.

```
마케팅이 부족하니 화/목에 마케팅 시간을 추가하고,
목요일에도 최소 1시간은 공부할 수 있게 배분해줘.
```

## 6. 실전 사례

### 사례 1: 간격 반복으로 시험 점수 12.9% 올린 연구

SAGE Journals(2025)에 게재된 연구에 따르면, Anki 기반 간격 반복을 사용한 의대 전임상 학생들이 비사용 학생 대비 **4개 시험 모두에서 유의미하게 높은 점수**를 받았습니다.

- Course I: **+6.4%** (p < 0.001)
- Course II: **+6.2%** (p = 0.002)
- Course III: **+7.0%** (p = 0.002)
- 종합 시험(CBSE): **+12.9%** (p = 0.003)

성숙 카드 수(matured cards)가 많을수록 성적이 높았으며, 총 학습 시간과도 유의미한 상관관계가 있었습니다.

> 📌 **실행 방법**: 매일 20분씩 Anki 카드를 복습하세요. 2~3주 후부터 효과를 체감하고, 2~3개월 후 본격적 성과가 나타납니다.
> — 출처: SAGE Journals "Exploring the Impact of Spaced Repetition Through Anki Usage" (2025)

### 사례 2: 기억 유지율 90% 향상 — 간격 반복의 과학

PMC와 Frontiers in Medicine에 게재된 연구들에 따르면, 간격 반복은 전통적 학습법 대비 **기억 유지율을 최대 90%까지 향상**시킵니다. 2025년 연구에서는 AI가 복습 간격을 개인화하고, 피드백 품질을 개선하며, 지식 통합을 촉진한다는 결과가 나왔습니다.

> 📌 **핵심**: 하루 전체를 벼락치기하는 것보다, **5일간 하루 30분씩** 간격을 두고 복습하는 것이 기억 유지에 6배 이상 효과적입니다.

### 사례 3: 대학생 51%가 AI를 시간 절약에 사용

HEPI 2025 설문에 따르면, 대학생 AI 도구 사용 목적 1위는 **시간 절약(51%)**이었습니다. 3년 연속 추적 설문에서 AI 사용률은 66% → 92%로 급등했으며, 더 이상 선택이 아닌 필수 도구가 되었습니다.

> 📌 **시사점**: AI를 "과제 대행"이 아닌 **"학습 효율화"**에 사용하는 학생이 늘고 있습니다. 일정 관리와 학습 관리가 가장 실용적인 AI 활용 영역입니다.

### 사례 4: Notion AI 학생 무료 계정의 활용

Notion은 .edu 이메일로 Plus 플랜 기능을 무료 제공하며, AI 기능으로 생산성 **평균 35% 향상**(Notion 자체 데이터)을 보고했습니다. 학기 대시보드(수업/과제/시험 데이터베이스), 캘린더/칸반/타임라인 뷰, AI 주간 요약 등을 올인원으로 관리할 수 있습니다.

## 7. 꿀팁 & 주의사항

### 전문가 팁

**팁 1: "AI에게 현실적 제약을 반드시 알려줘라"**

AI 학습 플래너의 가장 흔한 문제는 **비현실적 계획**입니다. "매일 12시간 공부" 같은 계획은 첫날에 무너집니다. 프롬프트에 "하루 자습 가능 시간", "알바 일정", "가장 약한 과목"을 반드시 포함하세요.

**팁 2: "AI로 공부한다" ≠ "AI가 대신 공부한다"**

ChatGPT에 "이 내용 요약해줘"를 반복하면 시간은 절약되지만, **시험에서는 아무것도 쓸 수 없습니다.** AI를 "답을 구하는 도구"가 아닌 **"질문을 만드는 도구"**로 사용하세요.

- ❌ "이 챕터 요약해줘" → 수동적 읽기
- ✅ "이 챕터로 나를 테스트해줘" → 능동적 회상

**팁 3: 주간 피드백 루틴을 만드세요**

매주 일요일 저녁에 10분만 투자하여 AI에게 주간 학습 기록을 분석받으세요. 이 피드백 루프가 학기 전체의 학습 품질을 결정합니다.

### ❌ Anti-Pattern 1: "AI 의존 학습의 역설" — 시간은 아꼈는데 실력은?

**상황**: AI로 과제를 빠르게 처리하고 시간을 절약했습니다. 하지만 시험에서 AI 없이 답을 쓸 수 없습니다.

**잘못된 방법**:
```
ChatGPT한테 매번 답을 구해서 과제 제출
→ 시험 때 백지 상태
```

**결과**: HEPI 2025 설문에 따르면 88%가 평가 과제에 AI를 사용하지만, Turnitin 분석에서는 "학습"과 "의존"의 경계가 모호하다고 경고합니다. AI가 답을 줄 때 학생은 **수동적 소비자**가 되기 쉽습니다.

**올바른 방법**:
```
1. AI로 "과제 답"을 구하지 말고 "연습 문제"를 생성
2. "이 내용으로 나를 테스트해줘" → 능동적 회상
3. AI 요약본 대신 "AI가 만든 플래시카드로 자가 테스트"
4. 틀린 문제를 AI에게 "왜 틀렸는지 설명해줘"
```

**차이**: AI를 **"교사"**로 사용하면 학습이 되고, **"대필자"**로 사용하면 의존이 됩니다.

### ❌ Anti-Pattern 2: "AI 학습 계획이 작심삼일인 이유"

**상황**: ChatGPT에 학습 계획을 요청하여 완벽한 14일 플랜을 받았습니다. 첫날 실행했지만 3일 만에 포기합니다.

**잘못된 방법**:
```
시험 2주 전, 5과목 학습 계획 짜줘.
[→ AI가 매일 12시간, 과목당 2.4시간 배분]
[→ 첫날 3시간 만에 포기]
```

**결과**: AI는 "가능한 최적의 계획"을 만들지만, **당신의 체력, 집중력, 약속, 돌발 상황**을 모릅니다. 비현실적 계획은 작심삼일로 끝나고, 오히려 자괴감만 줍니다.

**올바른 방법**:
```
나는 하루 최대 4시간만 공부할 수 있어.
목요일은 동아리, 토요일 오전은 알바.
통계가 가장 약해서 40% 더 배분해줘.
"80% 강도"의 현실적 계획을 짜줘.
매일 20분의 여유 시간도 포함해줘.
```

**차이**: **"현실적 제약을 프롬프트에 포함"**하면 실행 가능한 계획이 나옵니다. 완벽한 계획보다 **실행 가능한 계획**이 100배 낫습니다.

### 🔄 이 워크플로우를 다른 상황에 적용하기

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 학기 학습 계획 | 학기 → 자격증 시험 | 토익/한능검/컴활 준비 계획 |
| 시험 플래시카드 | 시험 → 면접 | Ch 14 면접 예상 질문 카드 |
| 주간 피드백 | 학습 패턴 → 취준 활동 | Ch 13~16 취업 준비 일정 관리 |
| ChatGPT Tasks 알림 | 학습 → 프로젝트 마감 | Ch 17 프로젝트 일정 관리 |

**핵심 패턴**: **"AI로 계획을 세우고, AI로 실행을 돕고, AI로 피드백을 받는다"** — 이 Plan-Do-Check 사이클은 학습뿐 아니라 모든 자기 관리에 적용됩니다.

# 🔧 Claude Code & 하네스 활용 — Chapter 10 보충

> **대상 독자**: Chapter 9까지 데이터·문서 자동화를 경험한 학생
> 본문에서 **Plan-Do-Check 3축**(계획→실행→피드백)과 **간격 반복(Spaced Repetition)** 기반 학습 관리를 배웠습니다. ChatGPT Tasks로 반복 알림을, Notion AI로 일정 관리를, Anki로 복습 카드를 만드는 방법을 익혔죠. 이번 보충에서는 같은 목표를 **cron과 셸 스크립트 + Claude Code**로 구현합니다. 본문의 ChatGPT Tasks가 "클릭 한 번으로 반복 알림"이라면, cron은 **완전한 로컬 자동화** — 인터넷 없이도, AI 구독 없이도 정해진 시간에 자동으로 작업을 수행합니다. 일정 리마인더, 간격 반복 기반 복습 문제, 학습 분석까지 — "내가 잊어도 AI가 챙겨주는" 시스템을 구축합니다.

---

## 1. 자동화의 마지막 퍼즐 — 스케줄링

### 1-1. 본문 도구 vs Claude Code 자동화

본문에서는 ChatGPT Tasks, Notion AI, Anki 등 **클라우드 기반 도구**로 Plan-Do-Check 3축을 구현했습니다. Claude Code + cron은 같은 3축을 **로컬 자동화**로 구현합니다:

| Plan-Do-Check 축 | 본문 도구 | CC + cron 대응 |
|-----------------|----------|---------------|
| **Plan**(계획) | ChatGPT로 학습 계획 수립 | schedule-manager 에이전트 + CLAUDE.md 규칙 |
| **Do**(실행) | Anki 간격 반복, ChatGPT Tasks 반복 알림 | quiz-generator + cron 자동 실행 |
| **Check**(피드백) | ChatGPT로 학습 분석 | study-tracker + 주간 KPT 리뷰 자동화 |

### 1-2. 수동 실행 vs 자동 실행

| 수동 실행 | 자동 실행 |
|----------|----------|
| 사용자가 명령 입력 | 정해진 시간에 자동 실행 |
| 잊으면 안 됨 | 잊어도 실행됨 |
| 매번 같은 명령 반복 | 한 번 설정하면 반복 |
| Part 3 Ch 7~9 방식 | **이번 챕터의 방식** |

### 1-3. cron이란?

**cron**은 macOS/Linux에 기본 내장된 작업 스케줄러입니다. 정해진 시간에 명령을 자동 실행합니다:

```bash
# cron 표현식: 분 시 일 월 요일
# 매일 아침 8시에 실행
0 8 * * * /path/to/script.sh

# 매주 월요일 9시에 실행
0 9 * * 1 /path/to/script.sh

# 매일 오후 6시에 실행
0 18 * * * /path/to/script.sh
```

| 필드 | 의미 | 범위 |
|------|------|------|
| 첫 번째 | 분 | 0-59 |
| 두 번째 | 시 | 0-23 |
| 세 번째 | 일 | 1-31 |
| 네 번째 | 월 | 1-12 |
| 다섯 번째 | 요일 | 0-7 (0,7=일요일) |

### 1-4. Claude Code + cron = 자동 AI 비서

```
cron (시간 트리거) → 셸 스크립트 → Claude Code -p (비대화형) → 결과 파일/알림
```

예시 시나리오:
- **매일 아침 8시**: 오늘의 할 일 목록 생성
- **매주 일요일 저녁**: 이번 주 학습 요약 보고서
- **시험 2주 전부터**: 매일 복습 문제 자동 생성
- **매월 1일**: 동아리 회비 정리 및 알림

### 3-2. CLAUDE.md 학습 관리 규칙

```markdown
# 학습 관리 시스템 규칙

## 일정 형식
schedule.md의 일정은 다음 형식을 따릅니다:
- YYYY-MM-DD | 과목명 | 유형(과제/시험/발표) | 설명

## 학습 기록 형식
study_log.md의 기록은 다음 형식을 따릅니다:
- YYYY-MM-DD | 과목명 | 시간(분) | 내용

## 원칙
- **현실적 제약을 반드시 반영**: 통학 시간, 아르바이트, 수면 등 개인 상황을 프롬프트에 포함해야 실행 가능한 계획이 나온다 (본문 핵심 교훈)
- 동기부여 메시지는 강요하지 않고 격려 톤으로
- 학습 시간 분석 시 수면/휴식의 중요성도 언급
- 비현실적인 계획 제안 금지 (하루 최대 학습 8시간, 계획 강도는 80% 수준으로 — "작심삼일" 방지)
```

### ❌ 흔한 실수: 자동화 만능주의 + cron 함정

**상황 1 — "AI 의존 학습의 역설"이 자동화에도 적용된다**

본문에서 경고한 "AI가 교사가 아니라 대필자가 되는 순간" — 이것은 cron 자동화에도 그대로 적용됩니다. quiz-generator가 매일 문제를 만들어줘도, **직접 풀지 않으면 의미 없습니다**. 자동 생성된 복습 문제를 "읽기만" 하면 본문의 간격 반복 효과(6.2~12.9% 향상)를 전혀 얻을 수 없습니다. 반드시 **직접 답을 작성한 후** 정답을 확인하세요.

**상황 2 — "작심삼일" 자동화 버전**

본문의 "작심삼일" 경고: 비현실적 계획은 3일 만에 포기합니다. cron도 마찬가지 — 매일 아침 8시 + 저녁 9시 + 일요일 리뷰를 한꺼번에 등록하면, 알림 피로로 1주일 만에 전부 무시하게 됩니다. **처음에는 일일 리마인더 1개만** 등록하고, 습관이 잡히면 점진적으로 추가하세요. 계획 강도는 본문 조언대로 "내가 가능하다고 생각하는 양의 80%"로 설정합니다.

---

**상황 3 — cron 환경 변수 함정**

cron으로 Claude Code 자동 실행을 설정할 때:

---

**잘못된 방법 — 환경 변수와 경로를 무시**:

```bash
# crontab에 이렇게 등록
0 8 * * * daily_reminder.sh
```

**결과**: cron은 사용자의 셸 환경(.zshrc 등)을 로드하지 않습니다. `claude` 명령어 경로를 찾지 못하고, 환경 변수도 없어서 스크립트가 실패합니다. 오류 로그도 확인하기 어렵습니다.

**올바른 방법**:

```bash
# 절대 경로 사용 + 환경 변수 설정 + 로그 기록
0 8 * * * /bin/bash -l $HOME/study-manager/scripts/daily_reminder.sh >> $HOME/study-manager/logs/cron.log 2>&1
```

스크립트 내부에서도:
```bash
#!/bin/bash
# 환경 변수 명시적 설정
export PATH="/usr/local/bin:/opt/homebrew/bin:$PATH"
export HOME="/Users/username"

# claude 명령어의 절대 경로 확인
CLAUDE=$(which claude)
```

**차이**: cron 환경에서는 경로와 환경 변수를 명시적으로 설정해야 합니다. `/bin/bash -l`로 로그인 셸을 사용하거나, 스크립트 내에서 직접 설정하세요. 로그 파일로 오류를 추적할 수 있게 하는 것도 중요합니다.
