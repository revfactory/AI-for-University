### 1.3 AI 탐지 도구 — 알아야 피할 수 있다

| 도구 | 탐지 정확도 | 특징 |
|------|-----------|------|
| **GPTZero** | 70~80% | 가장 널리 사용, perplexity + burstiness 분석 |
| **카피킬러** | 한국어 특화 | 국내 대학·기업에서 사용 |

## 2. 핵심 개념: AI 자소서의 빛과 그림자

### 2.1 채용담당자는 AI 자소서를 어떻게 볼까?

충격적인 데이터부터 확인합시다:

- **Resume Now(2025) 조사:** 채용담당자의 **62%**가 AI로 작성된 지원서를 **부정적으로 평가**
- **Oxford 대학 연구(2025):** 채용담당자의 **50%**가 AI 지원서에 **거부감** 표시
- **Harvard 커리어 센터:** "AI는 어시스턴트일 뿐, 대필 작가가 아니다"

이 데이터가 말하는 것은 명확합니다: **AI가 써준 그대로 제출하면 위험합니다.**

### 2.2 그렇다면 AI를 어떻게 활용해야 할까?

**AI 자소서 안전 활용 3단계:**

| 단계 | 방법 | 핵심 |
|------|------|------|
| **1단계: AI로 구조 잡기** | 경험 정리, STAR 프레임워크 적용, 초안 생성 | AI = 코치 |
| **2단계: 나의 언어로 다시 쓰기** | 구체적 에피소드, 감정, 고유한 표현으로 교체 | 나 = 작가 |
| **3단계: AI로 검토하기** | 맞춤법, 논리 흐름, 기업 인재상 매칭 확인 | AI = 편집자 |

> **Harvard 커리어 센터의 원칙:** "AI를 브레인스토밍과 편집에 활용하되, 핵심 내용과 진정성은 반드시 본인의 것이어야 한다."

### 2.3 STAR 기법 — AI와 함께 경험을 구조화하기

STAR 기법은 면접과 자소서에서 경험을 설득력 있게 전달하는 프레임워크입니다:

| 요소 | 의미 | 예시 |
|------|------|------|
| **S**ituation (상황) | 언제, 어디서, 어떤 상황이었는가 | "3학년 여름, 마케팅 인턴으로..." |
| **T**ask (과제) | 무엇을 해결해야 했는가 | "SNS 참여율이 3개월째 하락하는 문제를..." |
| **A**ction (행동) | 구체적으로 무엇을 했는가 | "데이터 분석으로 최적 게시 시간을 찾고..." |
| **R**esult (결과) | 어떤 성과를 냈는가 (수치화!) | "참여율 47% 증가, 팔로워 2,000명 증가" |

## 3. 따라하기 실습

### 실습 A: ChatGPT로 자기소개서 초안 작성하기 ⭐

> **목표:** STAR 기법을 활용해 자소서 한 문항의 초안을 AI와 함께 작성합니다.
> **소요 시간:** 30분
> **준비물:** ChatGPT 계정, 지원하려는 기업의 채용공고 또는 자소서 문항

---

**Step 1: 나의 경험 정리하기**

먼저 AI에게 줄 재료를 준비합니다. 다음을 메모하세요:

```
나의 경험 카드:
- 전공: ___________
- 학년: ___________
- 지원 기업/직무: ___________
- 핵심 경험 3가지:
  1. ___________
  2. ___________
  3. ___________
- 각 경험의 구체적 성과(숫자): ___________
```

📍 **화면 가이드:** 메모장이나 노션에 미리 정리해두면 AI에게 한 번에 전달할 수 있습니다.

✅ **체크포인트:** 경험 3가지와 각각의 구체적 숫자/성과가 준비되었나요?

---

**Step 2: STAR 기법으로 경험 구조화 요청하기**

ChatGPT에 다음 프롬프트를 입력합니다:

```
다음 자소서 문항에 답변하려고 합니다:
"지원 동기와 입사 후 이루고 싶은 목표를 서술하세요." (500자)

나의 정보:
- 경영학과 4학년, 카카오 마케팅 직무 지원
- 경험 1: 교내 마케팅 학회에서 SNS 캠페인 기획, 참여율 47% 증가
- 경험 2: 스타트업 마케팅 인턴 3개월, 콘텐츠 제작 50건
- 경험 3: Google Analytics 자격증 보유

각 경험을 STAR 기법으로 구조화해주세요:
1. 각 경험의 S(상황)-T(과제)-A(행동)-R(결과) 정리
2. 이 문항에 가장 적합한 경험 조합 추천
3. 500자 초안 작성 (기업 인재상 반영)
4. 초안에서 "나만의 표현으로 바꿔야 할 부분" 표시 [★개인화 필요★]
```

📍 **화면 가이드:** ChatGPT가 STAR 구조를 정리한 후 초안을 생성합니다. `[★개인화 필요★]` 표시된 부분을 확인하세요.

✅ **체크포인트:** STAR 구조표와 500자 초안이 생성되었나요? `[★개인화 필요★]` 부분이 표시되어 있나요?

---

**Step 3: 나의 언어로 개인화하기**

AI가 생성한 초안에서 `[★개인화 필요★]` 부분을 본인의 실제 경험과 감정으로 교체합니다.

**개인화 체크리스트:**
- [ ] 실제 프로젝트명, 팀명, 숫자가 정확한가?
- [ ] "~하면서 느꼈습니다" 같은 감정 표현이 진짜 내 감정인가?
- [ ] 다른 지원자와 겹칠 수 없는 나만의 에피소드가 있는가?
- [ ] 기업 인재상과 연결되는 구체적 행동이 포함되었는가?

---

**Step 4: AI로 최종 검토하기**

개인화한 자소서를 다시 ChatGPT에 입력합니다:

```
다음 자소서 답변을 검토해주세요:

[개인화한 자소서 전문 붙여넣기]

검토 항목:
1. 논리 흐름: 문장 간 연결이 자연스러운가?
2. 구체성: 추상적 표현은 없는가?
3. 기업 매칭: 카카오 인재상("일하기 좋은 함께하기 좋은")과 연결되는가?
4. 글자 수: 500자 이내인가?
5. AI 탐지 위험: 너무 정형화된 표현은 없는가?
6. 수정 제안을 구체적으로 해줘
```

📍 **화면 가이드:** AI가 항목별 피드백을 제공합니다. 수정 제안을 반영하되, 핵심 경험은 변경하지 마세요.

✅ **체크포인트:** 5개 항목 모두에 대한 피드백을 받았나요? 수정할 부분을 반영했나요?

🔧 **트러블슈팅:**
- AI가 너무 일반적인 피드백만 준다면 → "구체적으로 어떤 문장을 어떻게 수정해야 하는지 예시를 보여줘"로 후속 요청
- 글자 수가 초과한다면 → "500자 이내로 줄이되, STAR 구조와 핵심 성과 수치는 유지해줘"

> **준비물:** Notion 계정, ChatGPT (콘텐츠 생성용)

---

**Step 1: ChatGPT로 포트폴리오 콘텐츠 생성**

```

```

## 4. 프롬프트 진화 패턴

같은 목적이라도 프롬프트의 구체성에 따라 결과 품질이 완전히 달라집니다.

**🔴 Level 1 (초보):**
```
자기소개서 써줘
```
→ 누구에게나 해당되는 일반적인 자소서가 나옵니다. 채용담당자가 보면 바로 "AI가 썼다"고 느낍니다.

**🟡 Level 2 (중급):**
```
삼성전자 마케팅 직무 자소서를 써줘.
나는 경영학과 4학년이고 SNS 마케팅 인턴 경험이 있어.
```
→ 기업과 직무에 맞춘 자소서가 나오지만, 아직 개인 경험의 구체성이 부족합니다.

**🟢 Level 3 (고급):**
```
삼성전자 DX부문 마케팅 직무 지원 (자소서 문항 2번, 600자):
"팀 프로젝트에서 갈등을 해결한 경험"

■ 나의 경험:
  - 마케팅 학회 '브랜딩' 팀장 (2025.03~11, 6명)
  - 갈등 상황: 온라인 vs 오프라인 캠페인 방향 대립
  - 나의 행동: 데이터 기반 A/B 테스트 제안 → 양측 의견 통합
  - 결과: 캠페인 참여율 47% 증가, 학회 역대 최고 성과

■ 요청:
1. STAR 기법으로 구조화
2. 삼성 인재상 "함께가는 인재" 반영
3. 구체적 숫자와 행동 중심 서술
4. [★개인화 필요★] 표시로 내가 수정할 부분 안내
5. 600자 ±20자 이내
```
→ 나의 구체적 경험이 STAR 구조로 정리되고, 개인화할 부분이 명확히 표시됩니다.

## 5. 실전 사례

### 사례 1: NACE — 대학이 도입하는 AI 커리어 도구 (A급)

미국 대학·고용주 협의체인 **NACE(National Association of Colleges and Employers)**는 AI를 활용한 커리어·라이프 디자인 프로그램을 공식 운영하고 있습니다. 학생이 AI로 미래 자기를 매핑하고, 이력서와 자소서를 개선하는 체계적 프로그램입니다.

**시사점:** AI 자소서 작성은 "꼼수"가 아니라, 선진 대학이 공식으로 가르치는 **커리어 역량**입니다.

### 사례 2: Harvard — "AI는 어시스턴트, 대필 작가가 아니다" (A급)

**Harvard 커리어 센터**는 AI 활용 가이드에서 명확한 원칙을 제시합니다:
- AI를 **브레인스토밍과 구조화**에 활용 (O)
- AI가 생성한 텍스트를 **그대로 제출** (X)
- 핵심 원칙: **"진정성과 나만의 목소리"**가 최우선

### 사례 3: ChatGPT + STAR 기법 — 81% 자신감 향상 (B급)

**Parakeet AI(2026)** 조사에 따르면, AI를 활용해 STAR 기법으로 답변을 연습한 구직자의 **81%**가 면접 자신감이 향상되었다고 응답했습니다. 핵심은 AI가 경험을 구조화하는 **코칭 역할**을 한다는 점입니다.

### 사례 4: 채용담당자 62%가 AI 자소서에 부정적 (A급)

**Resume Now(2025)** 조사: 채용담당자의 **62%**가 AI로 작성된 지원서를 **부정적으로 평가**. 주된 이유는 "개성이 없다", "진정성이 느껴지지 않는다", "다 비슷하다"였습니다.

**핵심 교훈:** AI를 활용하되, **나만의 경험과 언어로 반드시 개인화**해야 합니다.

## 6. 꿀팁 & 주의사항

### 💡 꿀팁 5가지

1. **경험 카드를 먼저 만들어라:** AI에게 줄 재료(경험, 숫자, 감정)를 미리 정리하면 결과가 10배 좋아집니다
2. **기업 인재상을 프롬프트에 포함하라:** "삼성 '함께가는 인재'"처럼 구체적으로 넣으면 AI가 인재상에 맞춤 작성합니다
3. **한 문항씩 각개격파하라:** 자소서 전체를 한 번에 요청하지 말고, 문항별로 나눠서 깊이 있게 작업하세요
4. **다중 도구 크로스체크:** ChatGPT 초안 → Claude 논리 검토 → 최종 본인 작성이 최적 워크플로우입니다
5. **자소설닥터로 한국 특화 검증:** 한국 기업 지원 시, 합격 자소서 분석 기능으로 내 자소서의 경쟁력을 확인하세요

### ⚠️ Anti-Pattern 1: "다 똑같아 보인다" — AI 자소서의 가장 큰 함정

**상황:** 대학생 A가 ChatGPT에 "삼성전자 지원 동기 써줘"라고 요청. 깔끔한 자소서가 나와서 그대로 제출.

**잘못된 방법:** AI가 생성한 초안을 수정 없이 복사-붙여넣기.

**결과:**
- 채용담당자: "이번에도 '글로벌 시장을 선도하는 삼성의 비전에 공감하여...'로 시작하는 자소서가 50개째..."
- **Resume Now 조사:** 62%의 채용담당자가 AI 자소서에 부정적 (사례 4)
- **Oxford 연구:** 50%가 AI 지원서에 거부감 표시

**올바른 방법:**
1. AI에게 **나의 구체적 경험**을 입력 → STAR 구조화 요청
2. AI 초안에서 **[★개인화 필요★]** 부분을 실제 에피소드로 교체
3. "글로벌 시장을 선도하는" 같은 **상투적 표현을 내 언어로 교체**
4. AI 탐지 도구(GPTZero)로 **사전 체크**

**차이:** AI에게 "써줘"가 아니라 "내 경험을 구조화해줘"로 접근하면, 채용담당자가 원하는 **개성과 진정성**이 살아납니다.

---

### ⚠️ Anti-Pattern 2: "멋진 포트폴리오인데... 면접에서 설명을 못한다"

**상황:** 대학생 B가 AI로 화려한 포트폴리오 웹사이트를 제작. 프로젝트 설명도 AI가 작성.

**잘못된 방법:** 프로젝트 설명, 기술 선택 이유, 문제 해결 과정을 모두 AI에게 맡김.

**결과:**
- 면접관: "이 프로젝트에서 왜 React를 선택했나요?"
- 학생 B: "..." (AI가 써준 대로 React가 최적이라고 했는데, 실제로는 비교 검토를 안 해봄)
- 면접관: "포트폴리오는 인상적인데, 본인이 직접 한 것 같지 않네요."

**올바른 방법:**
1. AI로 포트폴리오 **레이아웃과 텍스트 초안** 생성 (O)
2. 프로젝트 설명은 **본인이 직접 작성** — 왜 이 기술을 선택했는지, 어떤 문제를 어떻게 해결했는지
3. 면접 전 **모든 포트폴리오 내용을 구두로 설명하는 연습**
4. AI에게 "이 프로젝트에 대해 면접관이 물어볼 질문 10가지"를 생성 → 미리 대비

**차이:** 포트폴리오의 **외형은 AI의 도움**을 받되, **내용에 대한 이해와 설명 능력**은 반드시 본인의 것이어야 합니다.

## 7. 워크플로우 일반화: AI 자소서·포트폴리오 파이프라인

이 챕터에서 배운 과정을 일반화하면 다음 워크플로우가 됩니다:

```
[Phase 1: 재료 준비]
  → 경험 카드 작성 (STAR 프레임워크)
  → 지원 기업 인재상 + 채용공고 분석
  → AI에게 줄 "나의 데이터" 정리

[Phase 2: AI 초안 생성]
  → ChatGPT로 문항별 STAR 구조화 + 초안
  → Claude로 논리 흐름 검토 + 보완
  → 다중 도구 비교로 최적 버전 선택

[Phase 3: 개인화]
  → [★개인화 필요★] 부분을 실제 경험으로 교체
  → 상투적 표현 → 나만의 언어로 변환
  → 구체적 숫자, 감정, 에피소드 추가

[Phase 4: 검증]
  → AI 탐지 도구(GPTZero)로 사전 체크
  → 기업 인재상 매칭도 재확인
  → 주변 사람(친구, 선배, 교수)에게 피드백

[Phase 5: 포트폴리오 연계]
  → 자소서 핵심 프로젝트를 포트폴리오에 상세 기술
  → Notion/Super.so 또는 GitHub Pages로 배포
  → 면접 전 포트폴리오 내용 구두 설명 연습
```

이 파이프라인은 자소서뿐 아니라, **이력서, 경력기술서, 프로젝트 보고서** 등 모든 종류의 자기 PR 문서에 적용할 수 있습니다. 핵심은 항상 **"AI = 구조화 도구, 나 = 콘텐츠 소유자"**라는 역할 분담입니다.

## 8. 도전 과제

### ⭐ 기본 과제: 자소서 1문항 AI 활용 작성

1. 관심 있는 기업의 채용공고에서 자소서 문항 1개를 선택하세요
2. 실습 A의 4단계(경험 정리 → STAR 구조화 → 개인화 → AI 검토)를 따라 자소서를 완성하세요
3. 완성된 자소서를 GPTZero에 검사해보세요 — "AI가 작성한 확률"이 30% 이하인가요?

**제출물:** 자소서 원문 + AI 활용 과정 스크린샷 + GPTZero 검사 결과

### ⭐⭐ 심화 과제: 4개 도구 비교 실험 보고서

1. 실습 B를 수행하고, 4개 AI 도구의 비교 분석표를 완성하세요
2. 각 도구의 강점/약점을 정리하고, 최적 조합 전략을 서술하세요
3. 최종 자소서(600자)를 작성하고, 어떤 도구의 어떤 부분을 활용했는지 표시하세요

**제출물:** 비교 분석표 + 최적 조합 전략 + 최종 자소서(도구 활용 표시 포함)

### ⭐⭐⭐ 도전 과제: 포트폴리오 웹사이트 제작 + 자소서 연계

1. 실습 C를 따라 포트폴리오 웹사이트를 제작하세요 (Notion+Super.so 또는 대안 도구)
2. 포트폴리오에 최소 3개 프로젝트를 포함하되, 각 프로젝트를 STAR 기법으로 서술하세요
3. 자소서에서 언급한 프로젝트와 포트폴리오 내용이 일관성 있게 연결되도록 구성하세요
4. 면접 시뮬레이션: ChatGPT에 "이 포트폴리오를 보고 면접관이 물어볼 질문 10가지"를 생성하고, 각 질문에 답변을 준비하세요

**제출물:** 포트폴리오 URL + 자소서(포트폴리오 연계 표시) + 면접 Q&A 10문항

> 본문에서 ChatGPT·Claude로 자기소개서를 작성하고 포트폴리오를 구성하는 방법을 배웠습니다. 본문의 핵심은 **"AI 자소서 안전 활용 3단계"** — AI=코치(구조 잡기) → 나=작가(나의 언어로) → AI=편집자(검토)라는 역할 분담입니다. 이번 보충에서는 같은 3단계를 **Claude Code + Git 버전관리**로 체계화합니다. 핵심 차이: 웹 UI에서 자소서를 수정하면 이전 버전이 사라지지만, Claude Code는 **모든 수정 이력을 Git으로 추적** — "1차 초안 → 피드백 반영 → 기업 맞춤 수정"의 전 과정이 기록됩니다. Resume Now(2025) 조사에서 채용담당자의 **62%**가 AI 자소서에 부정적이라는 결과가 말해주듯, "AI가 써준 그대로 제출"은 위험합니다. 하나의 자소서 원본에서 기업별 맞춤 버전을 자동 생성하되, **개인화 과정은 반드시 사람이** — 이것이 자소서 팩토리의 원칙입니다.

## 1. 자소서 버전관리의 필요성

### 1-1. 웹 UI 작성 vs CC 버전관리

| 항목 | 웹 UI (ChatGPT 등) | Claude Code + Git |
|------|-------------------|-------------------|
| 수정 이력 | 대화 히스토리에 묻힘 | Git 커밋으로 완전 추적 |
| 기업별 맞춤 | 매번 처음부터 요청 | 원본 → 기업별 브랜치 자동 생성 |
| 피드백 반영 | 복사-붙여넣기 반복 | diff로 변경점 시각화 |
| 여러 버전 관리 | 파일명에 v1, v2... | Git 브랜치로 체계적 관리 |
| AI 탐지 관리 | 수동 GPTZero 체크 | CLAUDE.md 규칙으로 자동 경고 |
| 협업 | 불가 | 공유 가능 (선배·멘토 피드백) |

> **본문 연계**: 본문에서 GPTZero의 perplexity(예측 가능성) + burstiness(문장 변화) 원리를 배웠습니다. CC에서는 CLAUDE.md에 "상투적 표현 자동 경고" 규칙을 설정하여, AI 탐지에 걸릴 수 있는 정형화된 표현을 사전에 차단합니다.

### 1-2. 자소서 팩토리 개념

```
원본 자소서 (master)
├── 삼성전자 맞춤 버전 (branch: samsung)
├── LG에너지솔루션 맞춤 버전 (branch: lg-energy)
├── 네이버 맞춤 버전 (branch: naver)
└── 스타트업 A 맞춤 버전 (branch: startup-a)

각 브랜치에서:
  원본 내용 + 기업 분석 + 직무 키워드 + 톤 조정 = 맞춤 자소서
```

**resume-writer.md**:
```markdown
---
name: resume-writer
description: "자기소개서를 작성하고 기업별로 맞춤화하는 에이전트"
---
# Resume Writer — 자소서 작성기

## 핵심 역할
- my_profile.md의 경험·역량을 바탕으로 자소서 초안을 작성합니다.
- 기업별 채용 공고와 인재상에 맞춰 자소서를 맞춤화합니다.
- STAR 기법(상황-과제-행동-결과)으로 경험을 구조화합니다.

## 작업 원칙
- 추상적 표현("열심히", "최선을") 대신 구체적 수치와 행동으로 작성
- 기업의 핵심 가치/인재상 키워드를 자연스럽게 포함
- 글자 수 제한을 엄격히 준수 (공백 포함/미포함 구분)
- 각 문항별 핵심 메시지를 한 줄로 먼저 정의
- AI가 "만들어낸" 경험을 절대 추가하지 않음 — my_profile.md에 있는 사실만 활용

## 출력
- output/resumes/기업명_자소서_YYYYMMDD.md: 맞춤 자소서
```

```markdown
---
name: feedback-coach
description: "자소서를 다각도로 평가하고 개선점을 제안하는 에이전트"
---
# Feedback Coach — 피드백 코치

## 핵심 역할
- 작성된 자소서를 인사담당자 관점에서 평가합니다.
- 구체성, 직무 적합성, 차별화, 가독성 4개 축으로 점수화합니다.
- 문장 단위로 개선 제안을 합니다.

## 작업 원칙
- 평가 기준: 구체성(30%), 직무 적합성(30%), 차별화(20%), 가독성(20%)
- 각 항목 10점 만점, 총 100점
- "좋은 점"과 "개선할 점"을 균형 있게 제시
- 개선 제안은 "왜 문제인지 + 어떻게 고칠지" 구체적으로
- 글자 수 초과/미달 경고

## 출력
- _workspace/feedback_기업명_YYYYMMDD.md: 피드백 보고서
```

### 2-2. CLAUDE.md 자소서 작성 규칙

```markdown
# 자소서 관리 시스템 규칙

## 작성 원칙
- STAR 기법 필수: 상황(Situation) → 과제(Task) → 행동(Action) → 결과(Result)
- 수치화 가능한 성과는 반드시 수치로 ("매출 증가" → "매출 15% 증가")
- 한 문항에 핵심 메시지 1개만 — 여러 이야기를 넣지 않기
- AI가 경험을 창작하지 않음 — my_profile.md의 사실만 활용
- Harvard 커리어 센터 원칙 준수: "AI는 브레인스토밍과 편집에 활용하되, 핵심 내용과 진정성은 반드시 본인의 것"

## AI 탐지 방지
- "글로벌 시장을 선도하는", "혁신적인 비전에 공감하여" 등 상투적 표현 자동 경고
- 문장 길이 변화(burstiness)가 균일하면 경고 — 의도적으로 짧은 문장과 긴 문장 혼합
- 최종 출력에 "⚠️ AI 탐지 위험 표현" 태그 표시

## 글자 수 관리
- 공백 포함/미포함을 채용 공고에 따라 자동 구분
- 제한의 90~100% 범위로 작성 (너무 짧으면 성의 부족)
- 글자 수 카운트를 매 출력 끝에 표시

## 버전 관리
- 모든 자소서는 Git으로 추적
- 커밋 메시지: "[기업명] 문항N 초안/수정/피드백반영"
- 기업별로 별도 브랜치 사용
```

### ❌ 흔한 실수: 자소서 AI 활용의 함정

**상황**: Claude Code로 자소서를 작성할 때

---

**잘못된 방법 — AI가 만든 경험을 사용**:

```
claude -p "삼성전자 자소서를 써줘. 경험이 부족하면 적당히 만들어줘"
```

**결과**: AI가 "해커톤에서 1등을 수상했다", "인턴에서 매출 30% 기여" 등 허위 경험을 생성합니다. 면접에서 구체적 질문을 받으면 바로 탄로납니다.

**올바른 방법**:

```
my_profile.md에 있는 실제 경험만 사용해서 자소서를 작성해줘.
경험이 부족한 부분은 "보완 필요" 표시를 해줘.
수업 프로젝트, 동아리 활동, 아르바이트 경험도 활용 가능하니
my_profile.md에서 관련 경험을 찾아서 연결해줘.
```

**차이**: CLAUDE.md에 "AI가 경험을 창작하지 않음" 규칙을 설정하면, 허위 경험 생성을 원천 차단합니다. 본문 Anti-Pattern 1의 "다 똑같아 보인다" — Resume Now 62% + Oxford 50% 데이터가 경고하듯, **AI가 써준 그대로 제출은 서류 탈락의 지름길**입니다. CC에서는 이 위험을 CLAUDE.md 규칙으로 **시스템 수준에서** 방지합니다.

---

**잘못된 방법 — 한 버전으로 모든 기업에 지원**:

```
자소서 잘 쓴 거 하나 만들어줘. 이걸로 10군데 지원할 거야.
```

**결과**: 기업별 인재상과 직무가 다른데 같은 자소서를 사용하면, "이 지원자는 우리 회사에 대해 알아보지도 않았구나"로 판단됩니다.

**올바른 방법**: 원본 자소서(master) + 기업별 브랜치 전략. `generate_resume.sh samsung`처럼 기업명만 바꿔 실행하면 맞춤 버전이 자동 생성됩니다.

**차이**: "1원본 N맞춤" 전략이 시간과 품질을 동시에 잡습니다.

## 실습 과제

### 🎯 실습: 나의 프로필 정리 + 자소서 초안 ⭐

**목표**: 자신의 경험을 구조화하고 Claude Code로 자소서 초안을 생성합니다.
**소요 시간**: 10분
**준비물**: Claude Code
**사용 도구**: Claude Code

---

#### Step 1. 프로필 작성 ⏱️ 5분

```
나의 취업 프로필을 정리하는 템플릿을 만들어줘.
input/my_profile.md에 저장.

포함 항목:
- 기본 정보 (전공, 학년, GPA)
- 경험 목록 (수업 프로젝트, 동아리, 아르바이트, 인턴, 대외활동)
- 각 경험을 STAR 형식으로 간략 정리
- 보유 스킬/자격증
- 지원 희망 직무/기업

샘플도 채워줘 (나중에 내 정보로 수정할 거야).
```

✅ **체크**: `input/my_profile.md`에 STAR 형식 경험이 포함된 프로필이 생성되면 성공

---

#### Step 2. 자소서 초안 생성 ⏱️ 3분

```
input/my_profile.md를 바탕으로
"지원 동기"와 "성장 과정" 2개 문항의 자소서 초안을 써줘.
각 문항 500자 이내 (공백 포함).
STAR 기법 적용하고, 글자 수를 끝에 표시해줘.
output/resumes/초안_자소서.md에 저장.
```

✅ **체크**: 2개 문항이 STAR 구조로 작성되고, 글자 수가 500자 이내이면 성공

> 🔧 **안 되나요?**: 글자 수가 초과하면 → "500자에 맞춰 줄여줘. 핵심 메시지를 유지하면서 부가 설명을 줄여줘"라고 요청하세요.

---

#### Step 3. 피드백 요청 ⏱️ 2분

```
output/resumes/초안_자소서.md를 평가해줘.
구체성(30점), 직무적합성(30점), 차별화(20점), 가독성(20점)
총 100점 기준으로 점수와 개선 제안을 해줘.
```
