## 1. 이런 분이 읽으면 좋아요

**이런 분에게 추천합니다:**

- "ChatGPT 써봤는데, 뭔가 원하는 대로 안 나와서 포기했다"는 분
- AI 도구가 너무 많아서 뭘 써야 할지 모르겠는 분
- "프롬프트 엔지니어링"이라는 말은 들어봤지만 구체적으로 뭘 어떻게 하라는 건지 감이 안 오는 분
- AI를 과제에 활용하고 싶은데, 제대로 된 방법을 배운 적이 없는 분

**이런 분은 건너뛰어도 됩니다:**

- 이미 AI 도구를 일상적으로 활용하고 있고, 프롬프트 작성에 자신 있는 분 → Ch 3부터 시작하세요
- AI의 기술적 원리(트랜스포머, 어텐션 메커니즘 등)를 깊이 알고 싶은 분 → 이 교재의 범위 밖입니다

## 2. 이 챕터에서 배우는 것

### 학습목표

이 챕터를 마치면 다음을 할 수 있습니다:

1. **ChatGPT, Gemini, Claude, Perplexity** 4대 AI 도구의 특성과 차이를 설명할 수 있다
2. 같은 질문이라도 **프롬프트를 어떻게 쓰느냐**에 따라 결과가 극적으로 달라진다는 것을 체감한다
3. **역할 부여, 조건 명시, 출력 형식 지정** 등 프롬프트 엔지니어링 핵심 기법 3가지를 적용할 수 있다
4. AI 답변의 **신뢰성을 검증하는 방법**(교차검증, 팩트체킹)을 실행할 수 있다
5. **학업 상황별 최적 도구**를 선택할 수 있다

### 완성 후 얻는 결과물

> 이 챕터의 실습을 마치면, 동일한 과제 프롬프트를 4개 AI 도구에 입력하고 결과를 비교 분석한 **"나만의 AI 도구 비교 보고서"**를 갖게 됩니다. 이 보고서는 앞으로 어떤 과제를 만나도 "어떤 AI 도구를 어떻게 쓸까?"를 판단하는 나침반이 됩니다.

### 🔍 도구 비교 실험: "경영학 레포트 개요"를 4개 도구에 물어보면?

동일한 프롬프트를 4개 도구에 입력했을 때 어떤 차이가 나는지 직접 비교해봅시다.

**실험 프롬프트:**

```
당신은 경영학 교수입니다. 대학교 3학년 경영학과 학생이
"디지털 트랜스포메이션이 중소기업 경쟁력에 미치는 영향"이라는 주제로
15페이지 분량의 학기말 레포트를 작성하려고 합니다.

다음 조건에 맞는 레포트 개요를 작성해주세요:
1. 서론-본론-결론 구조
2. 각 섹션별 핵심 논점 3개 이상
3. 참고할 수 있는 이론/프레임워크 제안
4. 각 섹션 예상 분량 (페이지 수)
5. 한국 중소기업 사례 포함 권장
```

**비교 결과:**

| 항목 | ChatGPT | Gemini | Claude | Perplexity |
|------|---------|--------|--------|------------|
| **구조 체계성** | ★★★★★ | ★★★★☆ | ★★★★★ | ★★★★☆ |
| **이론/프레임워크** | ★★★★☆ | ★★★★☆ | ★★★★★ | ★★★★☆ |
| **한국 맥락 반영** | ★★★★☆ | ★★★☆☆ | ★★★★☆ | ★★★★★ |
| **출처/참고문헌** | ★★☆☆☆ | ★★☆☆☆ | ★★☆☆☆ | ★★★★★ |
| **실행 가능 구체성** | ★★★★★ | ★★★★☆ | ★★★★★ | ★★★★☆ |

**도구별 특징 요약:**

- **ChatGPT**: Canvas 기능으로 개요를 즉시 편집·발전시킬 수 있습니다. 포터의 5 Forces, TAM 모델 등 이론을 풍부하게 제안하지만, 출처가 구체적이지 않습니다.
- **Gemini**: Google Scholar 연동으로 실제 논문·보고서를 찾아 제안합니다. 다만 한국어 맥락 이해가 상대적으로 약합니다.
- **Claude**: 가장 세밀하고 학술적인 개요를 제공합니다. "왜 → 무엇을 → 어떻게 → 그래서"의 논리적 흐름이 뛰어나고, 분량 배분이 가장 현실적입니다.
- **Perplexity**: 모든 주장에 출처(URL)를 포함합니다. 최신 한국 중소기업 디지털 전환 통계를 실시간 검색하여 제시하지만, 개요 자체의 구조적 완성도는 상대적으로 낮습니다.

**결론 — 상황별 최적 도구:**

> 하나의 도구가 모든 면에서 최고는 아닙니다. **자료 조사는 Perplexity**, **구조 설계는 Claude**, **초안 작성은 ChatGPT**, **교차 검증은 Gemini**로 조합하는 것이 가장 효과적입니다.

## 4. 핵심 개념

### AI는 "검색"이 아니라 "생성"입니다

생성형 AI를 처음 쓰는 분들이 가장 많이 하는 실수는 **Google 검색하듯이** 사용하는 것입니다.

| | Google 검색 | 생성형 AI |
|--|-----------|----------|
| **작동 방식** | 이미 있는 웹페이지를 찾아줌 | 학습한 패턴을 바탕으로 새로운 텍스트를 **생성**함 |
| **출처** | 원본 웹페이지 링크 제공 | 출처 없이 답변 (Perplexity 제외) |
| **정확도** | 원본 문서의 정확도에 의존 | **할루시네이션** 가능 — 없는 정보를 그럴듯하게 지어낼 수 있음 |
| **최적 질문** | 키워드 나열: "디지털 트랜스포메이션 중소기업 논문" | 맥락+조건+형식 구조화: "경영학 3학년이 15페이지 레포트를 쓰려고 합니다..." |

### 프롬프트 엔지니어링 — AI에게 "잘 말하는 기술"

프롬프트(prompt)란 AI에게 보내는 입력 텍스트입니다. 같은 AI라도 프롬프트를 어떻게 쓰느냐에 따라 결과물의 품질이 **하늘과 땅 차이**입니다.

프롬프트 엔지니어링의 핵심 요소 5가지:

| 요소 | 설명 | 예시 |
|------|------|------|
| **역할 부여** | AI에게 특정 전문가 역할을 부여 | "당신은 경영학 교수입니다" |
| **맥락 제공** | 본인의 상황과 배경 설명 | "3학년 경영학과, 학기말 레포트" |
| **조건 명시** | 구체적인 요구사항 나열 | "서론-본론-결론, 15페이지, 한국 사례 포함" |
| **출력 형식** | 원하는 결과물의 형태 지정 | "마크다운 표와 계층적 번호 매기기로" |
| **예시 제공** | 참고할 수 있는 예시 힌트 | "포터의 경쟁전략, TAM 모델 등" |

### 🔬 프롬프트 진화 실험

같은 작업을 3단계 프롬프트로 시도하여, 프롬프트 품질의 차이를 직접 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
경영학 레포트 개요 써줘
```

**결과**: 일반적인 경영학 레포트 구조를 나열합니다. "서론-본론-결론"이라는 뻔한 틀에, 주제도 학년도 분량도 모른 채 작성한 막연한 개요가 나옵니다.

📋 **평가**:
- 정확성: ★★☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 맥락 없음, 주제 불명확, 분량 기준 없음. 이대로는 쓸 수 없음

---

#### 🟡 Level 2 — 중급 프롬프트

```
디지털 트랜스포메이션이 중소기업 경쟁력에 미치는 영향이라는 주제로
경영학 레포트 개요를 작성해줘. 서론-본론-결론 구조로, 15페이지 분량이야.
```

**결과**: 주제에 맞는 개요가 나옵니다. 본론에 2~3개 소주제가 포함되고, 디지털 전환의 정의·사례·시사점이 언급됩니다. 하지만 이론적 깊이가 부족하고, 한국 맥락이 없습니다.

📋 **평가**:
- 정확성: ★★★★☆
- 활용도: ★★★☆☆
- 개선점: Level 1 대비 주제와 구조가 명확해짐
- 아쉬운 점: 이론 프레임워크 부재, 한국 사례 미포함, 각 섹션 분량 미제시

---

#### 🟢 Level 3 — 전문가 프롬프트

```
당신은 경영학 교수입니다. 대학교 3학년 경영학과 학생이
"디지털 트랜스포메이션이 중소기업 경쟁력에 미치는 영향"이라는 주제로
15페이지 분량의 학기말 레포트를 작성하려고 합니다.

다음 조건에 맞는 레포트 개요를 작성해주세요:
1. 서론-본론-결론 구조
2. 각 섹션별 핵심 논점 3개 이상
3. 참고할 수 있는 이론/프레임워크 제안 (포터의 경쟁전략, TAM 모델 등)
4. 각 섹션 예상 분량 (페이지 수)
5. 한국 중소기업 사례 포함 권장 (중소벤처기업부 자료 참고)

출력 형식: 마크다운 표와 계층적 번호 매기기로 구성
```

**결과**: 즉시 활용 가능한 체계적 개요가 나옵니다. 포터의 5 Forces, TAM 모델, 자원기반관점(RBV) 등 이론이 적절히 배치되고, 한국 중소기업 사례가 포함되며, 각 섹션별 페이지 수까지 제시됩니다.

📋 **평가**:
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: 역할 부여 + 구체적 조건 5가지 + 예시 힌트 + 출력 형식까지 지정

---

#### 💡 교훈: 프롬프트의 마법

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 역할 부여 | ✗ | ✗ | ✓ ("경영학 교수") |
| 맥락 제공 | ✗ | △ (주제만) | ✓ (학년, 과목, 분량) |
| 조건 명시 | ✗ | △ (구조만) | ✓ (5가지 상세 조건) |
| 출력 형식 | ✗ | ✗ | ✓ (마크다운 표) |
| 예시/참고 | ✗ | ✗ | ✓ (포터, TAM) |
| **결과 품질** | **20%** | **60%** | **95%** |

> 📌 **이 프롬프트가 좋은 이유:**
> - **역할 부여** ("경영학 교수") → 학술적 톤과 전문성을 유도합니다
> - **구체적 맥락** (3학년, 15페이지, 학기말) → AI가 적절한 수준과 분량을 조절합니다
> - **5가지 제약조건** → AI가 빠짐없이 체크하며 답변을 구성합니다
> - **예시 힌트** (포터, TAM) → AI가 같은 수준의 다른 이론도 추가합니다
> - **출력 형식 지정** → 바로 사용할 수 있는 깔끔한 결과가 나옵니다

### 도구별 프롬프트 최적화 전략

같은 프롬프트라도 도구마다 조금씩 다르게 쓰면 더 좋은 결과를 얻을 수 있습니다:

| 도구 | 프롬프트 팁 | 이유 |
|------|-----------|------|
| **ChatGPT** | 역할 부여 + 출력 형식 지정이 가장 효과적. Canvas 모드에서 반복 수정 활용 | 지시를 충실히 따르는 스타일 |
| **Gemini** | "최신 자료를 검색하여~"라고 명시. 이미지/PDF 업로드 후 분석 요청 활용 | 검색 연동이 강점 |
| **Claude** | "단계별로 생각하여~" 명시. 긴 문서를 통째로 붙여넣고 분석 요청. XML 태그(`<context>...</context>`) 활용 | 분석적이고 신중한 스타일 |
| **Perplexity** | "~에 대한 최신 연구/통계를 찾아줘" 형태가 최적. Focus 모드(Academic) 활용 | 검색 특화, 사실 중심 |

### 할루시네이션 — AI의 "자신감 있는 거짓말"

생성형 AI의 가장 큰 위험은 **할루시네이션(hallucination)**입니다. 틀린 정보를 마치 사실인 것처럼, 때로는 더 확신에 찬 어조로 말합니다.

**2025~2026년 할루시네이션 현황:**

| 모델 | 할루시네이션률 | 특징 |
|------|-------------|------|
| Gemini 2.0 Flash | **0.7%** | 가장 낮음 |
| Perplexity Sonar | ~1.5% | 검색 기반으로 보완 |
| Claude Opus 4 | ~2.5% | "모르겠다"고 인정하는 경향 |
| GPT-5 | ~3% | 범용 성능 우수 |
| OpenAI o3 (추론 특화) | **33%** | 추론 모델의 역설 |

> ⚠️ **주의**: 최고 성능 모델의 할루시네이션률이 0.7%까지 떨어졌지만, **완전히 0%가 될 수는 없습니다.** 2025년 수학적 증명으로 현재 LLM 아키텍처에서 할루시네이션 완전 제거가 불가능하다는 것이 확인되었습니다. AI가 "확실히~", "분명히~"라고 말할 때 오히려 더 의심해야 합니다 — Zhang et al. (2025)의 연구에 따르면 AI가 할루시네이션할 때 34% 더 높은 확률로 확신 표현을 사용합니다.

## 5. 따라하기 실습 (교재의 핵심)

### 🎯 실습: 4개 AI 도구 비교 체험 — "나만의 AI 도구 비교 보고서" 만들기 ⭐⭐

**목표**: 동일한 프롬프트를 4개 AI 도구에 입력하여 결과를 비교하고, 상황별 최적 도구를 선택하는 안목을 기릅니다.
**소요 시간**: 30~40분
**준비물**: 웹 브라우저, 각 도구의 무료 계정 (회원가입만 하면 됩니다)
**사용 도구**: ChatGPT, Gemini, Claude, Perplexity

---

#### Step 1. AI 도구 4개에 회원가입하기 ⏱️ 10분

아직 가입하지 않은 도구가 있다면, 아래 주소에서 무료 계정을 만듭니다.

| 도구 | 주소 | 가입 방법 |
|------|------|----------|
| ChatGPT | chat.openai.com | Google/Apple/이메일 계정 |
| Gemini | gemini.google.com | Google 계정 |
| Claude | claude.ai | Google/이메일 계정 |
| Perplexity | perplexity.ai | Google/Apple/이메일 계정 |

> 📍 **화면 안내**: 각 사이트에 접속하면 "Sign up" 또는 "시작하기" 버튼이 화면 중앙 또는 우측 상단에 있습니다. Google 계정이 있다면 "Continue with Google"을 클릭하면 가장 빠릅니다.

> 💡 **학생 혜택 챙기기**: 가입 후 Perplexity(perplexity.ai/settings → Education)와 Gemini(gemini.google/students)에서 .edu/.ac 이메일로 학생 인증을 하면 무료로 Pro/Premium 플랜을 받을 수 있습니다.

✅ **체크**: 4개 도구 모두 로그인하여 대화 화면이 보이면 성공입니다.

---

#### Step 2. 본인의 과제 주제로 프롬프트 작성하기 ⏱️ 5분

아래 프롬프트 템플릿에서 `[대괄호]` 안의 내용을 본인 상황에 맞게 수정하세요:

```
당신은 [전공] 교수입니다. 대학교 [학년]학년 [학과] 학생이
"[레포트 주제]"라는 주제로 [분량] 분량의 [과제 유형]을 작성하려고 합니다.

다음 조건에 맞는 레포트 개요를 작성해주세요:
1. 서론-본론-결론 구조
2. 각 섹션별 핵심 논점 3개 이상
3. 참고할 수 있는 이론/프레임워크 제안
4. 각 섹션 예상 분량 (페이지 수)
5. 한국 관련 사례 포함 권장

출력 형식: 마크다운 표와 계층적 번호 매기기로 구성
```

> 💡 **커스텀 포인트**: 실제 이번 학기 과제가 있다면 그 주제를 사용하세요. 없다면 아래 예시 중 하나를 선택하세요:
> - 경영학: "디지털 트랜스포메이션이 중소기업 경쟁력에 미치는 영향"
> - 심리학: "소셜 미디어 사용이 대학생 정신건강에 미치는 영향"
> - 교육학: "인공지능 시대의 대학 교육과정 혁신 방향"
> - 사회학: "한국 사회의 디지털 격차와 세대 간 불평등"

✅ **체크**: 프롬프트를 메모장이나 노트 앱에 작성하여 저장해두세요. 4개 도구에 동일한 프롬프트를 입력해야 비교가 의미 있습니다.

---

#### Step 3. ChatGPT에 프롬프트 입력하기 ⏱️ 3분

1. chat.openai.com에 접속합니다
2. 화면 하단의 입력창에 Step 2에서 작성한 프롬프트를 **그대로 복사**하여 붙여넣습니다
3. Enter를 누르거나 전송 버튼(↑)을 클릭합니다

> 📍 **화면 안내**: 입력창은 화면 최하단 중앙에 있습니다. "Message ChatGPT"라고 적힌 곳입니다.

✅ **체크**: AI가 서론-본론-결론 구조의 개요를 생성했나요? 이론/프레임워크가 언급되었나요? 결과를 복사하여 별도 문서에 저장합니다.

> 🔧 **안 되나요?**: "I'm unable to..." 메시지가 나오면 프롬프트가 너무 길 수 있습니다. 조건을 3개로 줄여 다시 시도하세요.

---

#### Step 4. Gemini에 동일 프롬프트 입력하기 ⏱️ 3분

1. gemini.google.com에 접속합니다
2. 화면 하단의 입력창에 **동일한 프롬프트**를 붙여넣습니다
3. 전송합니다

> 📍 **화면 안내**: Gemini의 입력창도 화면 하단 중앙에 있습니다. "Enter a prompt here"라고 적혀 있습니다.

✅ **체크**: ChatGPT와 비교하여, Gemini의 답변에서 다른 점이 보이나요? 특히 **웹 검색 결과**가 포함되었는지 확인하세요. 결과를 저장합니다.

---

#### Step 5. Claude에 동일 프롬프트 입력하기 ⏱️ 3분

1. claude.ai에 접속합니다
2. 화면 하단의 입력창에 **동일한 프롬프트**를 붙여넣습니다
3. 전송합니다

> 📍 **화면 안내**: Claude의 입력창은 화면 하단에 "Reply to Claude..."라고 적혀 있습니다.

✅ **체크**: Claude의 답변이 다른 도구보다 **더 구조적이고 학술적**으로 느껴지나요? 논리 흐름(왜→무엇을→어떻게→그래서)이 자연스러운지 확인하세요. 결과를 저장합니다.

---

#### Step 6. Perplexity에 동일 프롬프트 입력하기 ⏱️ 3분

1. perplexity.ai에 접속합니다
2. 화면 중앙의 검색창에 **동일한 프롬프트**를 붙여넣습니다
3. 전송합니다

> 📍 **화면 안내**: Perplexity의 입력창은 화면 상단~중앙의 큰 검색바입니다. "Ask anything..."이라고 적혀 있습니다.

✅ **체크**: Perplexity의 답변에 **출처 링크(숫자 표시)**가 포함되어 있나요? 다른 3개 도구와 가장 큰 차이점입니다. 결과를 저장합니다.

---

이제 저장한 4개의 결과를 비교합니다. 아래 표를 복사하여 채워보세요:

## 6. 실전 사례

### 사례 1. 하버드 물리학 AI 튜터 — 학습 효과 2배 (✅ A급)

하버드대학교 최대 물리학 수업(Physical Sciences 2)에서 194명의 학생을 대상으로 AI 튜터와 전통적 능동학습 교실을 비교하는 무작위 대조 실험이 진행되었습니다.

**결과:**
- AI 튜터 그룹의 사후검사 중앙값: **4.5** vs 능동학습 교실: **3.5**
- 학습 효과 **2배 이상**, 소요 시간은 오히려 단축
- 효과 크기 0.73~1.3 표준편차 (교육학에서 0.4 이상이면 유의미)
- 학생들의 참여도, 동기, 만족도 모두 향상

> 출처: Kestin et al. (2025), Harvard University, PubMed (pubmed.ncbi.nlm.nih.gov/40537565)

### 사례 2. 메타분석으로 확인된 AI의 학업 효과 (✅ A급)

51개 연구를 종합 분석한 메타분석(Wang & Fan, 2025) 결과:

| 영역 | 효과 크기 (g) | 해석 |
|------|-------------|------|
| 학습 성과 | **0.867** | 큰 효과 |
| 학습 인식 향상 | **0.456** | 중간 효과 |
| 고차원적 사고 능력 | **0.457** | 중간 효과 |

AI 활용이 단순히 성적만 올리는 것이 아니라, **비판적 사고력**까지 향상시킨다는 것이 학술적으로 확인되었습니다.

> 출처: Wang & Fan (2025), *Humanities and Social Sciences Communications*, Nature (nature.com/articles/s41599-025-04787-y)

### 사례 3. 대학생의 AI 활용 실태 — "이미 일상이 되었다" (✅ A급)

Anthropic이 574,740건의 대학생 Claude 대화를 분석한 결과:

- **콘텐츠 생성 39.3%**: 연습 문제 설계, 에세이 편집, 학술 자료 요약
- **기술적 문제 해결 33.5%**: 코드 디버깅, 알고리즘 설명, 수학 문제
- **데이터 분석 11.0%**, 연구 지원 6.5%
- 컴퓨터과학 전공이 전체 대화의 **36.8%** 차지 (학위 비중 5.4%의 7배)

하지만 **47%의 대화가 "직접적" 패턴** — 최소한의 노력으로 즉각적 답변만 요구하는 방식이었습니다. 이것이 바로 "제대로 배워야 하는" 이유입니다.

> 출처: Anthropic 교육 보고서 (2025), anthropic.com/news/anthropic-education-report-how-university-students-use-claude

## 7. 꿀팁 & 주의사항

### 전문가 팁

1. **프롬프트는 대화다**: 한 번에 완벽한 결과를 기대하지 마세요. "여기에 한국 사례를 추가해줘", "본론 2번을 더 구체화해줘"처럼 **후속 대화**로 결과를 발전시키세요.

2. **Temporary Chat을 활용하세요**: 개인 정보가 포함된 과제를 작업할 때, ChatGPT의 Temporary Chat 모드를 사용하면 대화 내용이 학습에 사용되지 않습니다.

3. **Perplexity는 "검증 도구"로 쓰세요**: 다른 AI의 답변에서 의심스러운 내용이 있으면, Perplexity에 "~이 사실인지 확인해줘"라고 물어보세요. 출처까지 확인해줍니다.

### ❌ 흔한 실수: AI 답변을 검증 없이 믿기

**상황**: 레포트에 넣을 통계 수치를 AI에게 물어봤습니다.

**잘못된 방법**:
```
한국 중소기업의 디지털 전환율이 몇 %인지 알려줘
```
→ AI가 "2024년 기준 한국 중소기업의 디지털 전환율은 42.3%입니다"라고 자신있게 답변합니다.
→ 이 수치를 그대로 레포트에 인용합니다.

**결과**: 이 수치는 할루시네이션일 수 있습니다. 변호사들조차 AI가 만들어낸 가짜 판례를 법정에 인용하여 제재를 받은 사례가 **전 세계 486건**이나 됩니다. 128명의 변호사와 2명의 판사가 실제로 징계를 받았습니다.

**올바른 방법**:
```
한국 중소기업의 디지털 전환 현황에 대한 최신 통계를 찾아줘.
반드시 출처(기관명, 보고서명, 발행연도)를 함께 알려줘.
```
→ Perplexity에서 검색하면 출처가 포함된 답변을 받을 수 있습니다.
→ 제시된 출처 링크를 직접 클릭하여 원본 보고서에서 수치를 확인합니다.

**차이**: "출처를 요구"하는 한 줄을 추가하고, Perplexity를 활용한 것만으로 할루시네이션 위험을 크게 줄였습니다. 출처를 직접 확인하는 마지막 단계가 핵심입니다.

### AI 답변 교차검증 5단계 워크플로우

AI가 알려준 정보를 레포트에 쓰기 전, 반드시 이 순서로 검증하세요:

```
Step 1. 동일 질문을 최소 2개 도구에 입력하여 답변 비교
Step 2. Perplexity로 핵심 주장의 출처 확인
Step 3. 구체적 수치/통계는 원본 출처(논문, 보고서)에서 직접 확인
Step 4. AI가 "확실히~", "분명히~" 표현을 사용하면 오히려 의심하기
Step 5. 최신 정보일수록 검증 필수 (AI 학습 데이터 마감일 이후 정보는 부정확 가능)
```

### 도구 선택 가이드 — 상황별 의사결정

| 학업 상황 | 1순위 | 2순위 | 이유 |
|----------|-------|-------|------|
| 레포트 주제 리서치 | Perplexity | Gemini | 출처 포함 검색 |
| 레포트 개요/구조 설계 | Claude | ChatGPT | 논리적 구조화 |
| 레포트 초안 작성 | ChatGPT | Claude | Canvas 편집 용이 |
| 논문/PDF 분석 | Claude | Gemini | 긴 문서 처리 탁월 |
| 수식/계산 문제 | ChatGPT | Gemini | 수학 추론 최강 |
| 코딩 과제 | Claude | ChatGPT | Artifacts 코드 미리보기 |
| 팩트체킹/교차검증 | Perplexity | Gemini | 실시간 검색 |
| 이미지/시각자료 생성 | ChatGPT | Gemini | DALL-E 이미지 생성 |

### 🔄 이 워크플로우를 다른 상황에 적용하기

이 실습에서 배운 "동일 프롬프트 → 다중 도구 비교 → 최적 조합 선택" 패턴은 다음 상황에도 적용할 수 있습니다:

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 레포트 개요 비교 | 프롬프트를 "면접 예상 질문 생성"으로 변경 | Ch 14: 면접 준비 |
| 4개 도구에 동일 입력 | 프롬프트를 "논문 요약"으로 변경 | Ch 4: 논문 분석 |
| 결과물 비교표 작성 | 비교 항목을 "코드 품질/실행 가능성"으로 변경 | Ch 20: 코딩 보조 |

**핵심 패턴**: "같은 질문을 여러 도구에 넣고, 각 도구의 강점을 조합하여 최적의 결과물을 만든다."

## 1. Claude Code 설치와 기본 사용법

### 1-1. Claude Code란?

본문에서 ChatGPT, Gemini, Claude 웹, Perplexity 4개 도구를 비교해봤습니다. 이 도구들은 모두 **웹 브라우저(GUI)**에서 작동합니다. Claude Code는 이와 다르게 **터미널(명령줄, CLI)**에서 Claude AI와 대화하는 도구입니다. 브라우저를 열지 않아도 터미널 창 하나에서 질문하고, 로컬 파일을 직접 읽고, 코드를 실행하고, 반복 작업을 자동화할 수 있습니다.

### 1-2. 설치 방법

#### macOS

```bash
# 1. Node.js 설치 (Homebrew 이용)
brew install node

# 2. Claude Code 설치
npm install -g @anthropic-ai/claude-code

# 3. 설치 확인
claude --version
```

✅ **체크**: `claude v1.x.x` 형태의 버전 번호가 출력되면 정상입니다.

> 🔧 **안 되나요?**
> - `permission denied` 오류 → `sudo npm install -g @anthropic-ai/claude-code`로 재시도하거나, nvm을 설치하여 Node.js를 관리하면 sudo 없이 사용 가능합니다.
> - `brew: command not found` → Homebrew 먼저 설치: `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`
> - `node --version`으로 Node.js 18 이상인지 확인하세요. 18 미만이면 `brew upgrade node`로 업그레이드하세요.

#### Windows

```powershell
# 1. Node.js 설치 — https://nodejs.org 에서 LTS 버전 다운로드 후 설치
# 또는 winget 사용:
winget install OpenJS.NodeJS.LTS

# 2. Claude Code 설치 (PowerShell 또는 명령 프롬프트)
npm install -g @anthropic-ai/claude-code

# 3. 설치 확인
claude --version
```

✅ **체크**: `claude v1.x.x` 형태의 버전 번호가 출력되면 정상입니다.

> 🔧 **안 되나요?**
> - `npm: command not found` → Node.js 설치 후 **터미널을 새로 열어야** PATH가 적용됩니다.
> - PowerShell 실행 정책 오류 (`cannot be loaded because running scripts is disabled`) → 관리자 PowerShell에서 `Set-ExecutionPolicy RemoteSigned` 실행 후 재시도하세요.
> - `claude: command not found` → `npx @anthropic-ai/claude-code`로 실행하거나, `npm config get prefix` 경로가 시스템 PATH에 포함되어 있는지 확인하세요.

#### Linux (Ubuntu/Debian)

```bash
# 1. Node.js 설치 (nvm 권장 — sudo 불필요)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
nvm install --lts

# 2. Claude Code 설치
npm install -g @anthropic-ai/claude-code

# 3. 설치 확인
claude --version
```

✅ **체크**: `claude v1.x.x` 형태의 버전 번호가 출력되면 정상입니다.

> ⚠️ **주의**: Node.js 18 이상이 필요합니다. `node --version`으로 버전을 확인하세요.
>
> 🔧 **안 되나요?**
> - `nvm: command not found` → `source ~/.bashrc` (또는 `source ~/.zshrc`) 실행 후 재시도하세요.
> - 글로벌 설치 권한 오류 → nvm을 사용하면 sudo 없이 설치할 수 있습니다.

### 1-3. 처음 실행하기

```bash
# 터미널에서 Claude Code 시작
claude
```

처음 실행 시 Anthropic 계정 인증이 필요합니다. 터미널에 표시되는 안내를 따라 로그인하면 됩니다.

### 🔬 프롬프트 진화 실험: Claude Code에 질문하기

같은 작업을 3단계 프롬프트로 시도하여, 프롬프트 품질의 차이를 직접 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
파일 요약해줘
```

**결과**: "어떤 파일을 요약할까요?"라는 되물음이 돌아옵니다. 파일이 지정되지 않아 AI가 작업을 수행할 수 없습니다.

📋 **평가**:
- 정확성: ★☆☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 대상 파일, 요약 기준, 출력 형식이 모두 불명확

---

#### 🟡 Level 2 — 중급 프롬프트

```
report.pdf를 읽고 내용을 요약해줘
```

**결과**: 파일을 읽고 전체 내용을 장황하게 요약합니다. 길이 조절이 안 되고, 어떤 관점에서 요약해야 하는지 불명확합니다.

📋 **평가**:
- 정확성: ★★★☆☆
- 활용도: ★★★☆☆
- 개선점: Level 1 대비 파일이 특정되어 실제 작업이 수행됨
- 아쉬운 점: 요약 분량, 관점, 출력 형식이 미지정

---

#### 🟢 Level 3 — 전문가 프롬프트

```
report.pdf를 읽고 다음 기준으로 요약해줘:
- 분량: 5줄 이내의 핵심 요약
- 관점: 대학생 과제 활용 관점
- 형식: 번호 매기기
- 포함 사항: 핵심 주장, 사용된 방법론, 주요 결론
- 결과를 summary.md로 저장
```

**결과**: 정확히 5줄 이내로, 과제에 바로 활용할 수 있는 구조화된 요약이 생성되고, 파일로 저장됩니다.

## 3. GUI(ChatGPT/Gemini/Claude 웹/Perplexity) vs CLI(Claude Code) 비교

본문에서 4개 도구를 "상황별 최적 도구"로 조합하는 워크플로우를 배웠습니다. Claude Code는 이 조합에 **자동화 레이어**를 추가합니다.

| 기준 | GUI (웹 기반 채팅) | CLI (Claude Code) |
|------|-------------------|-------------------|
| **접근성** | 브라우저만 있으면 사용 가능 | 터미널 설치 필요 |
| **파일 처리** | 파일을 직접 업로드/다운로드 | 로컬 파일 시스템에 직접 접근 |
| **대량 작업** | 파일을 하나씩 업로드해야 함 | 폴더 전체를 한 번에 처리 가능 |
| **자동화** | 수동 반복 작업 | 스크립트로 자동화 가능 |
| **코드 실행** | 제한적 (일부 플랫폼만) | 로컬 환경에서 바로 실행 |
| **버전 관리** | 대화 기록만 저장 | Git과 연동, 변경 이력 추적 |
| **프로젝트 규모** | 단일 대화 중심 | 프로젝트 전체를 맥락으로 활용 |

### 언제 GUI가 유리한가?

- 간단한 질문/답변이 필요할 때
- 이미지 생성이나 시각적 결과물이 필요할 때
- 터미널에 익숙하지 않은 초보 사용자
- 모바일 환경에서 빠르게 접근할 때

### 언제 CLI(Claude Code)가 유리한가?

- 여러 파일을 동시에 처리해야 할 때
- 반복 작업을 자동화하고 싶을 때
- 코딩 프로젝트를 진행할 때
- 로컬 파일을 직접 읽고 수정해야 할 때
- 워크플로우를 재사용 가능하게 만들고 싶을 때

## 4. 간단한 자동화 예시

### 4-1. 파일 일괄 이름 변경

학기 중에 "과제1_홍길동.docx", "과제1_김철수.docx" 같은 파일들의 이름 형식을 통일하고 싶을 때:

```
> 현재 폴더의 모든 docx 파일 이름을 "2024_AI과목_학번_이름.docx" 형식으로 변경해줘.
> 파일 내용에서 학번과 이름을 추출해서 사용해.

Claude: [파일들을 분석하고 이름을 변경합니다]
        - 과제1_홍길동.docx → 2024_AI과목_20210001_홍길동.docx
        - 과제1_김철수.docx → 2024_AI과목_20210042_김철수.docx
        ...
```

### 4-2. 텍스트 요약 자동화

여러 논문 PDF를 한 번에 요약하고 싶을 때:

```
> papers 폴더에 있는 모든 PDF 파일을 각각 읽고,
> 각 논문의 제목, 저자, 핵심 주장, 방법론을 표로 정리해서
> summary.md 파일로 저장해줘

Claude: [papers 폴더를 스캔합니다]
        [5개의 PDF 파일을 발견했습니다]
        [각 파일을 분석하는 중...]
        [summary.md 파일을 생성했습니다]
```

### 4-3. 비대화형 모드로 자동화 파이프라인 만들기

Claude Code는 `-p` (print) 플래그를 사용하면 비대화형 모드로 실행됩니다. 이를 활용하면 셸 스크립트에서 자동화가 가능합니다.

```bash
# 단일 명령 실행
claude -p "현재 디렉터리의 파일 목록을 알려줘"

# 파일을 읽고 결과를 다른 파일로 저장
claude -p "report.txt를 읽고 영어로 번역해줘" > report_en.txt

# 셸 스크립트로 여러 파일 일괄 처리
for file in papers/*.pdf; do
  claude -p "$file 을 읽고 3문장으로 요약해줘" > "summaries/$(basename "$file" .pdf)_summary.txt"
done
```

### 4-4. 과제 초안 작성 보조

```
> 주제: "AI가 대학 교육에 미치는 영향"
> 이 주제로 보고서 초안의 개요를 작성하고,
> 각 섹션에 들어갈 핵심 논점을 정리해서
> draft_outline.md로 저장해줘

Claude: [보고서 개요를 구성합니다]
        [draft_outline.md를 생성했습니다]
```

### ❌ 흔한 실수: 터미널에서 AI에게 질문할 때 흔한 실수

**상황**: Claude Code를 처음 사용하면서 기대한 결과가 나오지 않을 때

---

**잘못된 방법 1 — 파일 경로 미지정**:
```
> 파일 요약해줘
```

**결과**: AI가 "어떤 파일인가요?"라고 되묻습니다. 현재 폴더에 파일이 여러 개 있어도 어떤 파일을 원하는지 알 수 없습니다.

**올바른 방법**:
```
> report.pdf를 읽고 핵심 내용을 3줄로 요약해줘
```

**차이**: 파일명을 명시하면 AI가 바로 작업을 시작합니다.

---

**잘못된 방법 2 — 맥락 없는 요청**:
```
> 이거 고쳐줘
```

**결과**: "무엇을 고칠까요?"라는 되물음. "이거"가 무엇인지, 어떻게 고칠지 알 수 없습니다.

**올바른 방법**:
```
> essay.md의 3번째 문단에서 문법 오류를 찾아서 수정해줘
```

**차이**: 대상(파일명), 범위(3번째 문단), 작업(문법 오류 수정)을 명확히 지정합니다.

---

**잘못된 방법 3 — 작업 디렉터리 무시**:
```
> ~/Documents/학교/2024/AI과목/과제/report.pdf를 요약해줘
```

**결과**: 작동은 하지만, 매번 긴 경로를 입력하는 것은 비효율적입니다.

**올바른 방법**:
```bash
# 먼저 해당 폴더로 이동한 뒤 Claude Code 시작
cd ~/Documents/학교/2024/AI과목/과제
claude
```
```
> report.pdf를 요약해줘
```

**차이**: Claude Code를 작업 폴더에서 시작하면, 해당 폴더의 모든 파일에 파일명만으로 접근할 수 있습니다.

## 5. 더 나아가기: 하네스 맛보기

> Ch 2 본문에서 "에이전트 AI"가 2024~2025년 사이 문의량 **1,445% 급증**한 핵심 트렌드임을 배웠습니다. Claude Code의 **하네스(Harness)**는 바로 이 에이전트 AI를 직접 체험할 수 있는 기능입니다.

**하네스**란 여러 AI 에이전트를 팀처럼 구성하여 복잡한 작업을 분담시킬 수 있는 환경입니다. 본문에서 배운 "Perplexity→Claude→ChatGPT→Gemini" 4개 도구 조합 워크플로우를, AI 에이전트가 자동으로 수행하는 것이라고 생각하면 됩니다.

예를 들어, 교재 한 챕터를 작성할 때:
- **리서처 에이전트**: 관련 자료를 웹에서 수집 (Perplexity 역할)
- **작성 에이전트**: 수집된 자료를 바탕으로 본문 작성 (ChatGPT 역할)
- **편집 에이전트**: 작성된 본문의 문법과 흐름 검수 (교차검증 역할)

이런 구성을 `.claude/agents/` 폴더에 마크다운 파일로 정의하면, Claude Code가 각 역할을 자동으로 수행합니다. 자세한 내용은 이후 챕터에서 다룹니다.

```bash
#!/bin/bash
# summarize.sh — docs 폴더의 모든 텍스트 파일을 일괄 요약

mkdir -p summaries

for file in docs/*.txt; do

  echo "✓ 완료 → summaries/${filename}_summary.txt"
done

echo "🎉 모든 파일 요약 완료!"
```


### 🔄 이 워크플로우를 다른 상황에 적용하기

이 실습에서 배운 패턴은 다음 상황에도 적용할 수 있습니다:

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 텍스트 파일 요약 | PDF, DOCX 파일로 변경 | 논문 일괄 요약 |
| 2문장 요약 | "키워드 5개 추출"로 변경 | 자료 분류/태깅 |
| 한국어 요약 | "영어로 번역"으로 변경 | 다국어 문서 번역 |

**핵심 패턴**: `claude -p "[파일] 을 읽고 [작업]해줘" > [결과파일]` — 이 한 줄이 모든 파일 자동화의 기본 단위입니다.
