## 1. 이런 분이 읽으면 좋아요

**이런 고민이 있다면, 바로 이 챕터입니다:**

- 데이터 분석에 관심은 있지만 파이썬이나 R을 배울 시간이 없는 분
- 과제로 데이터 분석 보고서를 내야 하는데 코딩을 모르는 분
- 공공데이터를 활용한 프로젝트를 해보고 싶지만 어디서 시작해야 할지 모르는 분
- 엑셀로 차트는 만들 수 있지만 "전문가 수준의 분석"은 어렵다고 느끼는 분
- 데이터 분석 대회에 참가하고 싶지만 코딩 실력이 부족한 분

**이런 분은 건너뛰어도 괜찮습니다:**
- 파이썬 pandas와 matplotlib을 이미 능숙하게 다루는 분 (다만, AI로 분석 속도를 높이는 팁은 유용할 수 있습니다)

## 2. 이 챕터에서 배우는 것

### 학습목표

1. ChatGPT Code Interpreter에 데이터 파일을 업로드하여 탐색적 데이터 분석(EDA)을 수행할 수 있다
2. 공공데이터 포털(data.go.kr)에서 데이터를 다운로드하고, AI로 분석할 수 있다
3. Claude Artifacts로 인터랙티브 대시보드를 만들 수 있다
4. AI가 생성한 분석 결과의 한계를 인식하고, 비판적으로 검증할 수 있다
5. 분석 결과를 시각화하고, 인사이트를 도출하여 보고서에 활용할 수 있다

### 완성 후 결과물 미리보기

> - 시간대별 승하차 추이 꺾은선 그래프
> - 노선별 이용량 상위 20 수평 막대그래프
> - 주중 vs 주말 패턴 비교 차트

> - 핵심 인사이트 5가지 + 정책 제안

### ChatGPT Code Interpreter — 핵심 원리

ChatGPT Code Interpreter는 **사용자 대신 파이썬 코드를 자동으로 작성하고 실행**합니다. 사용자는 코드를 볼 필요 없이, 자연어로 분석을 요청하면 됩니다.

| 분석 유형 | 능력 | 설명 |
|----------|------|------|
| 기술통계 | ★★★★★ | 평균/중앙값/표준편차 자동 계산 |
| 데이터 정제 | ★★★★★ | 결측치 처리, 이상치 탐지, 타입 변환 |
| 시각화 | ★★★★★ | 10+ 차트 유형, 인터랙티브 전환 |
| 통계 검정 | ★★★★☆ | t-검정, ANOVA, 상관분석, 회귀분석 |
| 결과 해석 | ★★★★★ | 비전공자 눈높이 설명 |

> 💡 **핵심:** "코딩 능력"이 아닌 **"질문 능력(프롬프트)"**이 데이터 분석의 핵심 역량이 되었습니다.

### ChatGPT Code Interpreter vs Claude Artifacts

| 항목 | ChatGPT Code Interpreter | Claude Artifacts |
|------|-------------------------|------------------|
| 분석 방식 | **Python 자동 실행** (서버사이드) | JavaScript/React 기반 (클라이언트사이드) |
| 통계 엄밀성 | ★★★★★ (scipy/statsmodels) | ★★★★☆ (JS 라이브러리) |
| 시각화 | Matplotlib/Plotly (정적→인터랙티브) | **React 컴포넌트** (네이티브 인터랙티브) |
| 대용량 처리 | 100MB 제한 | **1M 토큰** (텍스트 기반) |
| 결과 재사용 | ⚠️ 세션 종료 시 삭제 | ✅ Artifacts로 저장·공유 |
| **적합 용도** | **정확한 통계 분석** | **인터랙티브 대시보드** |

> 📌 **최적 조합:** ChatGPT Code Interpreter로 정확한 분석 수행 → Claude Artifacts로 인터랙티브 대시보드 제작

## 4. 핵심 개념

### "파이썬 없이 데이터 분석" — 무엇이 바뀌었나?

| 과거 (코딩 필요) | 현재 (AI 활용) |
|-----------------|---------------|
| Python 설치 + 라이브러리 세팅 | **파일 업로드 한 번** |
| pandas 문법 학습 | "이 데이터 분석해줘" 한 줄 |
| Matplotlib 차트 코드 작성 | "막대그래프 만들어줘" |
| 에러 디버깅 수시간 | AI가 자동 코드 수정·재실행 |
| 통계 공식 직접 구현 | "t-검정 해줘" → 결과 + 해석 |

> 💡 **시장이 증명한 변화:** No-code AI 분석 시장은 $185억(2023) → **$2,361억(2033)** 예상, 연평균 성장률 29%(AIMultiple, 2026). MIT도 "No Code AI and Machine Learning" 공식 교육 과정을 운영하고 있습니다.

### 5단계 노코드 데이터 분석 파이프라인

```
[1단계] 데이터 획득
  → 공공데이터포털(data.go.kr) CSV 다운로드
  → 또는 직접 수집한 데이터

[2단계] 데이터 탐색 (ChatGPT Code Interpreter)
  → 파일 업로드 → "이 데이터를 탐색해줘"
  → 구조 파악, 결측치 확인, 기초 통계

[3단계] 데이터 정제 (ChatGPT/Claude)
  → "결측치를 처리하고 이상치를 제거해줘"
  → 정제된 데이터 다운로드

[4단계] 시각화 + 인사이트 (Claude Artifacts / ChatGPT)
  → "인터랙티브 대시보드를 만들어줘" (Claude)
  → 또는 "차트 5개를 만들어줘" (ChatGPT)

[5단계] 보고서 통합
  → 차트 이미지 다운로드 → Word/PPT 삽입
  → AI 작성 분석 문장을 보고서에 활용
```

### 공공데이터, 어디서 찾나?

| 데이터셋 | 출처 | 분석 예시 |
|---------|------|----------|
| 서울시 대중교통 이용량 | data.seoul.go.kr | 시간대별 승하차 패턴 |
| 전국 대학교 현황 | data.go.kr | 지역별 대학 수, 학생 수 비교 |
| 기상 데이터 | data.kma.go.kr | 연도별 기온 변화 추이 |
| 소비자물가지수 | kostat.go.kr | 품목별 물가 변동 시각화 |
| 취업률 통계 | kess.kedi.re.kr | 전공별 취업률 비교 |
| 교통사고 통계 | taas.koroad.or.kr | 지역별/시간대별 사고 패턴 |

> 📍 **접근 방법:** data.go.kr → 검색 → "파일 데이터" 탭 → CSV 또는 Excel 다운로드 (로그인 불필요)

### 🔬 프롬프트 진화 실험

같은 데이터 분석 작업을 3단계 프롬프트로 시도하여, 결과 품질의 차이를 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
이 엑셀 파일 분석해줘
```

**결과:** AI가 데이터 구조만 요약하고, "어떤 분석을 원하시나요?"라고 되물어봅니다. 기본 통계 몇 줄이 전부입니다.

📋 **평가**:
- 정확성: ★★☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 분석 방향, 시각화 유형, 해석 수준 모두 불명확

---

#### 🟡 Level 2 — 중급 프롬프트

```
이 CSV 데이터의 트렌드를 분석하고 차트를 만들어줘
```

**결과:** 기본 트렌드 차트와 간략한 분석이 나옵니다. 하지만 어떤 변수의 트렌드인지, 차트 형식은 무엇인지 AI가 임의로 선택합니다.

📋 **평가**:
- 정확성: ★★★☆☆
- 활용도: ★★★☆☆
- 개선점: Level 1보다 목적(트렌드)이 명확
- 아쉬운 점: 분석 변수, 차트 유형, 비교 기준 불명확

---

#### 🟢 Level 3 — 전문가 프롬프트

```
서울시 대중교통 데이터(CSV 첨부)를 분석해주세요:

1. 시간대별(0~23시) 평균 승차 인원 → 꺾은선 그래프
2. 노선별 상위 20개 이용량 → 수평 막대그래프
3. 주중 vs 주말 이용 패턴 차이 → 그룹화된 막대그래프
4. 월별 이용량 추이 → 영역 차트
5. 환승역 네트워크 분석 → 히트맵

모든 차트: 한국어 레이블, 인터랙티브, 인사이트 코멘트 포함
최종 요약: 핵심 발견 5가지 + 정책 제안 2가지
```

**결과:** 5개 시각화와 상세한 인사이트가 한 번에 생성됩니다. 각 차트 아래에 데이터 기반 해석이 포함되고, 마지막에 정책 제안까지 도출됩니다.

📋 **평가**:
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: 분석 대상·차트 유형·비교 기준·해석 수준·최종 산출물까지 명시

---

#### 💡 교훈: 노코드 분석에서 프롬프트의 중요성

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 파일 지정 | ✗ | △ | ✓ (구체적 데이터명) |
| 분석 변수 | ✗ | ✗ | ✓ (시간대, 노선 등) |
| 차트 유형 | ✗ | ✗ | ✓ (꺾은선, 막대, 히트맵) |
| 비교 기준 | ✗ | ✗ | ✓ (주중 vs 주말) |
| 최종 산출물 | ✗ | ✗ | ✓ (인사이트 + 제안) |
| 결과 품질 | 15% | 45% | 95% |

> 📌 **핵심:** "코딩을 모르니까 AI에 맡긴다"가 아니라, **"무엇을 분석할지 정확히 지시한다"**가 노코드 데이터 분석의 진짜 실력입니다.

**목표:** 공공데이터 포털에서 데이터를 다운로드하고, ChatGPT Code Interpreter로 탐색적 데이터 분석(EDA)을 수행한다

**준비물:** ChatGPT 계정 (Free 가능), 인터넷 접속
**사용 도구:** 공공데이터포털(data.go.kr), ChatGPT

#### Step 1. 공공데이터 다운로드 ⏱️ 5분

> 📍 **화면 안내**: data.go.kr 접속 → 상단 검색창에 "서울시 대중교통" 또는 원하는 키워드 입력

검색 결과에서 **"파일데이터"** 탭을 클릭합니다.

> 📍 **화면 안내**: 원하는 데이터셋 → "다운로드" 버튼 → CSV 또는 Excel 선택

> 💡 **추천 데이터셋 (실습용):**
> - "서울특별시 지하철호선별 역별 승하차 인원 정보" (data.seoul.go.kr)
> - "전국 대학 현황 통계" (data.go.kr)
> - 또는 관심 있는 아무 CSV 데이터

✅ **체크**: CSV 또는 Excel 파일이 다운로드되었으면 성공!

> 🔧 **안 되나요?** 원하는 데이터를 못 찾으면, ChatGPT에 "서울시 월별 평균 기온 데이터를 30년치 표로 만들어줘"라고 요청하여 실습용 데이터를 생성할 수 있습니다.

---

#### Step 2. ChatGPT에 업로드 + 데이터 탐색 ⏱️ 5분

> 📍 **화면 안내**: ChatGPT 새 대화 → 📎 클립 아이콘 → 다운로드한 파일 첨부

파일 첨부 후, 다음 프롬프트를 입력합니다:

```
이 데이터 파일을 탐색해주세요.

1. 전체 행/열 수와 각 컬럼명
2. 각 컬럼의 데이터 타입과 결측치 비율
3. 수치형 컬럼의 기초 통계 (평균, 중앙값, 최솟값, 최댓값, 표준편차)
4. 범주형 컬럼의 고유값 개수와 상위 5개 빈도
5. 데이터 품질 이슈 (결측치, 이상치, 중복행) 리포트
6. 이 데이터로 할 수 있는 흥미로운 분석 5가지를 제안해줘
```

✅ **체크**: 데이터 구조 요약, 기초 통계, 분석 제안이 출력됩니다. 특히 6번의 분석 제안은 이후 실습에서 활용합니다.

---

#### Step 3. 기초 시각화 요청 ⏱️ 5분

AI가 제안한 분석 중 흥미로운 것을 선택하여 시각화를 요청합니다:

```
이 데이터로 다음 차트를 만들어줘:

1. [수치형 변수] 분포 확인 — 히스토그램 + 박스플롯
2. [범주형 변수] 비교 — 수평 막대그래프 (상위 10개)
3. [시간 변수]별 추이 — 꺾은선 그래프
4. 주요 수치형 변수 간 상관관계 — 산점도 (추세선 포함)

차트 조건:
- 한국어 제목과 축 레이블
- 색상은 파랑~초록 계열 (시인성 높게)
- 각 차트 아래 1~2줄 해석 코멘트 추가
```

> 💡 **커스텀 포인트:** `[대괄호]` 안의 내용은 본인의 데이터에 맞게 수정하세요. 예: "승차인원 분포", "노선별 비교", "월별 추이"

✅ **체크**: 4개 차트와 각각의 해석 코멘트가 출력됩니다

---

#### Step 4. 인사이트 도출 ⏱️ 5분

```
지금까지 분석한 결과를 바탕으로:

1. 핵심 인사이트 5가지를 정리해줘
2. 각 인사이트에 근거 데이터(수치)를 포함
3. 대학생 리포트에 쓸 수 있는 "분석 결과" 문단(300자)을 작성해줘
4. 한계점과 추가 분석이 필요한 부분도 명시
```

✅ **체크**: 5가지 인사이트, 근거 수치, 보고서용 문단, 한계점이 출력됩니다

> 📊 **효과 비교**:
> - 파이썬으로 EDA: 환경 세팅 30분 + 코드 작성 60분 + 디버깅 30분 = 약 2시간
> - ChatGPT 활용: 데이터 다운로드 5분 + 프롬프트 입력 5분 + 결과 확인 10분 = **약 20분** (83% 시간 절약)

### 🎯 실습 B: 데이터 정제 + 심화 분석 ⭐⭐

**목표:** 데이터 정제 과정을 AI에게 맡기고, 심화 분석(트렌드, 그룹 비교, 예측)을 수행한다
**소요 시간:** 약 25분
**준비물:** 실습 A에서 사용한 ChatGPT 대화 (이어서 진행)
**사용 도구:** ChatGPT

---

#### Step 1. 데이터 정제 요청 ⏱️ 5분

```
이 데이터를 분석에 적합하게 정제해주세요:

1. 결측치 처리
   - 수치형: 중앙값으로 대체 (극단값 영향 최소화)
   - 범주형: 최빈값으로 대체
   - 결측치가 30% 이상인 컬럼은 삭제 이유와 함께 보고

2. 이상치 탐지
   - IQR 방법으로 이상치 식별
   - 이상치 목록과 처리 방법 제안

3. 데이터 변환
   - 날짜 컬럼을 연/월/요일로 분리
   - 필요시 범주형 인코딩

4. 정제 결과 요약
   - 처리 전/후 행 수 비교
   - 정제된 데이터를 CSV로 다운로드 가능하게

각 단계에서 "무엇을 왜 했는지" 설명도 포함해주세요.
```

✅ **체크**: 정제 과정 설명 + 정제된 데이터 파일 다운로드 링크가 제공됩니다

> 💡 **이것이 핵심입니다:** 코딩을 모르더라도 "결측치는 중앙값으로, 이상치는 IQR로"라는 **분석 전략**을 알면 AI에게 정확히 지시할 수 있습니다. 코드가 아닌 **분석 설계 능력**이 중요합니다.

---

#### Step 2. 그룹 비교 분석 ⏱️ 7분

```
정제된 데이터로 다음 그룹 비교 분석을 수행해주세요:

1. [그룹 변수]별 [수치 변수] 비교
   - 그룹별 평균, 중앙값, 표준편차
   - 박스플롯으로 시각화
   - 그룹 간 차이가 통계적으로 유의한지 검정 (t-검정 또는 ANOVA)

2. 시간대별 패턴 분석
   - 시간대별 추이 → 꺾은선 그래프
   - 피크 시간대와 비수기 시간대 식별
   - 전년 대비 변화율 계산

3. 카테고리별 상위/하위 분석
   - 상위 10개와 하위 10개를 나란히 시각화
   - 상위·하위 간 격차 분석

각 분석에 보고서용 해석 문장을 포함해주세요.
```

✅ **체크**: 그룹 비교 차트, 시간대 패턴, 상하위 분석 결과가 출력됩니다

---

#### Step 3. 트렌드 예측 ⏱️ 8분

```
이 데이터의 시간적 추이를 바탕으로 향후 트렌드를 예측해주세요:

1. 과거 데이터의 추세선(선형/다항식) 그리기
2. 향후 6개월 예측값 산출
3. 예측의 불확실성(95% 신뢰구간) 시각화
4. 예측의 한계점과 주의사항 명시

차트에 실제값(실선)과 예측값(점선)을 함께 표시해주세요.
```

✅ **체크**: 추세선 + 예측 차트가 생성되고, 신뢰구간과 한계점이 포함됩니다

> ⚠️ **주의:** AI 예측은 "과거 패턴이 유지된다"는 가정 하에 작동합니다. 코로나19 같은 외부 충격은 반영되지 않으므로, 보고서에 반드시 "예측의 한계"를 명시하세요.

---

#### Step 4. 분석 결과 다운로드 ⏱️ 5분

```
지금까지 생성한 모든 차트와 분석 결과를 정리해주세요:

1. 모든 차트를 고해상도 PNG로 다운로드 가능하게
2. 분석 결과 요약을 Markdown 표로 정리
3. "데이터 분석 보고서" 초안을 작성 (서론-방법-결과-결론 구조)
```

✅ **체크**: 차트 이미지 + 보고서 초안이 생성됩니다

> 📊 **효과 비교**:
> - 파이썬으로 심화 분석: 코드 작성 2시간 + 시각화 1시간 + 디버깅 1시간 = 약 4시간
> - ChatGPT 활용: 프롬프트 입력 10분 + 결과 확인 15분 = **약 25분** (90% 시간 절약)

### 🎯 실습 C: Claude Artifacts 인터랙티브 대시보드 ⭐⭐

**목표:** ChatGPT 분석 결과를 Claude Artifacts로 인터랙티브 대시보드로 변환하고, 최종 보고서를 완성한다
**소요 시간:** 약 30분
**준비물:** Claude 계정 (Free 가능), 실습 B의 분석 결과 요약
**사용 도구:** Claude

#### Step 1. Claude에 데이터 제공 ⏱️ 3분

Claude를 열고, 실습 A~B에서 분석한 데이터의 핵심 요약을 제공합니다:

> 📍 **화면 안내**: claude.ai → 새 대화 → 📎 파일 첨부 (CSV 파일) 또는 텍스트로 요약 데이터 입력

```
다음은 서울시 대중교통 데이터 분석 결과입니다.
이 데이터를 인터랙티브 대시보드로 만들어주세요.

또는 첨부한 CSV 파일을 직접 분석하여 대시보드를 만들어도 됩니다.
```

✅ **체크**: Claude가 데이터를 이해하고 대시보드 생성을 시작합니다

#### Step 2. 인터랙티브 대시보드 요청 ⏱️ 10분

```
이 데이터로 Artifacts를 활용해 인터랙티브 대시보드를 만들어주세요:

대시보드 구성:
1. 상단: KPI 카드 4개 (총 이용량, 일평균, 최대 노선, 전월 대비 변화율)
2. 좌측: 시간대별 이용량 꺾은선 그래프 (호버하면 수치 표시)
3. 우측: 노선별 상위 10개 수평 막대그래프 (클릭하면 상세 정보)
4. 하단: 주중/주말 비교 그룹 막대그래프

디자인:
- 한국어 레이블 전체 적용
- 색상 테마: 파랑~초록 계열
- 반응형 레이아웃
- 각 차트 아래 1줄 인사이트 표시
```

✅ **체크**: Artifacts 사이드 패널에 인터랙티브 대시보드가 렌더링됩니다. 차트 위에 마우스를 올리면 수치가 표시되어야 합니다.

> 🔧 **안 되나요?** Artifacts가 생성되지 않으면, "이 대시보드를 React 컴포넌트로 만들어서 Artifacts에 표시해줘"라고 명시적으로 요청하세요.

---

#### Step 3. 대시보드 수정 및 개선 ⏱️ 7분

```
대시보드를 다음과 같이 개선해주세요:

1. 필터 추가: 드롭다운으로 노선 선택 → 해당 노선만 하이라이트
2. 타임라인 슬라이더: 월별 범위를 조절할 수 있게
```

✅ **체크**: 필터, 슬라이더, 요약 패널이 추가된 향상된 대시보드가 표시됩니다

> 💡 **팁:** Claude Artifacts의 강점은 **"자연어로 디자인을 수정"**할 수 있다는 것입니다. "막대그래프 색상을 주황색으로 바꿔줘", "KPI 카드를 2줄로 배치해줘" 등 세부 디자인도 자연어로 조정합니다.

```
   - 발견 1: [시간대별 패턴] + 근거 수치
   - 발견 2: [노선별 차이] + 근거 수치
   - 발견 3: [주중/주말 비교] + 근거 수치

4. 시사점 및 제안 (300자)
5. 한계점 및 향후 과제 (200자)

학술 보고서 톤, APA 스타일, 한국어
```

✅ **체크**: 완성된 분석 보고서가 출력됩니다

> - 전통 방식 (Python + Tableau): 코딩 3시간 + 대시보드 2시간 + 보고서 1시간 = 약 6시간

### 사례 1: ChatGPT 데이터 분석 학술 평가 — "코딩 없이 어디까지?"

**출처:** PMC (Koçak, 2025)

ChatGPT-4o의 데이터 분석 성능을 체계적으로 평가한 학술 연구입니다.

| 분석 유형 | ChatGPT 성능 | 판정 |
|----------|-------------|------|
| 기술통계 (평균, 표준편차) | ★★★★★ 효과적 | ✅ 코딩 불필요 |
| 히스토그램, 산점도 | ★★★★★ 효과적 | ✅ 코딩 불필요 |
| 회귀분석, 상관분석 | ★★★★☆ 대체로 정확 (오차율 5%) | △ 검증 권장 |

```
- 해상도: 300dpi
- 크기: 10×6 인치
- 폰트: 한국어 지원 (NanumGothic 또는 Malgun Gothic)
```

**팁 3: 데이터 정제를 AI에게 맡길 때 반드시 확인할 3가지**

1. **결측치 처리 방법:** "어떤 방법으로 처리했는지" 항상 물어보세요 (삭제 vs 대체)
2. **처리 전후 행 수:** 데이터가 예상 외로 많이 줄었다면 처리 기준을 재검토
3. **정제 코드:** "사용한 코드를 보여줘"라고 요청하면, 나중에 동일한 전처리를 재현할 수 있습니다

### ❌ Anti-Pattern 1: "쉽다 ≠ 정확하다" — No-code 분석의 과신

**상황:** ChatGPT에 CSV를 올리고 "분석해줘"라고 했더니 멋진 차트와 인사이트가 나왔습니다.

**잘못된 방법:**
```
ChatGPT 결과를 검증 없이 보고서에 그대로 사용:

"이 데이터에 따르면 A지역의 교통량이 B지역보다 2.3배 높으며,
이는 향후 5년간 지속될 것으로 예측된다."
```

**결과:** 교수 피드백: "2.3배라는 수치의 근거가 불명확합니다. 어떤 기간의 데이터인가요? 계절 효과는 고려했나요? 예측의 신뢰구간은요?"

BBC 조사에 따르면 **AI 쿼리의 45%가 오류**를 포함합니다. 특히 수치 계산에서 **열 이름을 잘못 매핑**하거나 **집계 기준을 임의로 설정**하는 경우가 빈번합니다.

**올바른 방법:**
```
ChatGPT에게:
"이 분석에서:
1. 사용한 데이터 범위와 기간을 명시해줘
2. 계절성이나 외부 요인의 영향을 고려했는지 설명해줘
3. 2.3배라는 수치의 계산 과정(사용한 코드)을 보여줘
4. 예측의 95% 신뢰구간과 한계점을 포함해줘"

+ 핵심 수치는 직접 엑셀/Sheets에서 수식으로 교차 검증
```

**차이:** "AI가 멋진 차트를 만들어줬다"와 "AI 분석을 검증하고 보강했다"는 보고서 점수에서 큰 차이를 만듭니다.

### ❌ Anti-Pattern 2: "멋진 차트 = 좋은 분석"이라는 착각

**상황:** 동아리 설문 데이터 50건을 ChatGPT로 분석하여 컬러풀한 차트 5개와 인사이트를 얻었습니다.

**잘못된 방법:**
```
ChatGPT:
"만족도 평균 4.2/5.0으로 높은 수준입니다.
유용성 인식과 만족도 간 상관계수 r=0.89로 매우 강한 양의 상관이 있습니다."

→ 이 결과를 "전문가 수준 분석"으로 제출
```

**결과:** 교수 피드백: "표본 50명으로 소수점 한 자리까지 제시하는 건 과도한 정밀도(overprecision)입니다. 신뢰구간은? 표본의 대표성은? r=0.89는 다중공선성을 의심해야 하지 않나요?"

**올바른 방법:**
```
ChatGPT에게:
"이 분석 결과에 대해 비판적 검토를 해주세요:
1. 표본 크기(50명)에서 이 정밀도가 적절한가?
2. 95% 신뢰구간은 얼마인가?
3. r=0.89의 경우 다중공선성 문제는 없는가?
4. 이 결과의 외적 타당도(일반화 가능성)는?
5. 보고서에 포함해야 할 '분석의 한계' 섹션을 작성해줘"
```

**차이:** "AI가 만든 멋진 차트"는 출발점이지, 도착점이 아닙니다. **비판적 검토를 AI에게 요청하는 한 줄이** 보고서의 질을 결정합니다.

### 🔄 이 워크플로우를 다른 상황에 적용하기

이 챕터에서 배운 "공공데이터 → ChatGPT 분석 → Claude 대시보드 → 보고서" 패턴은 다음 상황에도 적용할 수 있습니다:

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 서울시 대중교통 데이터 | 전국 대학교 현황 데이터 | 교육학 과제 보고서 |
| 시간대별 이용량 분석 | 연도별 물가 변동 분석 | 경제학 레포트 |
| 노선별 비교 | 지역별 비교 | 지방자치론 프로젝트 |
| ChatGPT 분석 | Julius AI 분석 | 학생 할인 활용 |
| Claude 대시보드 | Google Looker Studio | 무료 대시보드 |
| 보고서 작성 | 발표 자료 제작 | PPT로 변환 (Ch 7 참조) |

**핵심 패턴:** 데이터 획득 → AI 탐색(EDA) → 정제 → 시각화 → 인사이트 → 보고서 — 이 6단계는 **모든 데이터 분석 프로젝트**에 적용되는 범용 패턴입니다.

## 1. "파이썬 없이" 분석의 두 가지 경로

### 1-1. 본문 도구 vs Claude Code 비교

본문에서는 ChatGPT Code Interpreter(정확한 분석) + Claude Artifacts(인터랙티브 대시보드)를 조합했습니다. Claude Code는 이 둘의 장점을 로컬에서 통합합니다:

| 항목 | ChatGPT Code Interpreter | Claude Artifacts | Claude Code |
|------|------------------------|-----------------|-------------|
| 접근 방식 | 웹 UI 파일 업로드 | 웹 UI 대화 | 로컬 CLI 자연어 지시 |
| 분석 엔진 | Python (서버사이드) | JavaScript/React | **Python (로컬)** |
| 시각화 | 정적 이미지 | 인터랙티브 대시보드 | 로컬 PNG/PDF + 스크립트 |
| 코드 확인 | 접힌 블록 클릭 | 코드 숨김 | **항상 보임** — 학습 효과 |
| 반복 실행 | 매번 대화에서 요청 | 매번 재생성 | 스크립트로 재실행 |
| 대용량 | 100MB 제한 | 1M 토큰 | **제한 없음** |

### 1-2. Claude Code의 "코드 투명성" 장점

Claude Code는 분석을 수행할 때 **파이썬 코드를 터미널에 그대로 보여줍니다**. 이것이 학습자에게 큰 장점입니다:

```
사용자: "이 CSV의 평균과 표준편차를 구해줘"

Claude Code가 실행하는 코드:
───────────────────────────────
import pandas as pd
df = pd.read_csv('input/data.csv')
print(df.describe())
───────────────────────────────
```

**잘못된 방법 — 본문의 "쉽다≠정확하다"가 CC에도 적용**:


**결과**: 분석 결과는 나오지만, "왜 이런 결과가 나왔는지" 검증할 수 없습니다. 본문에서 경고한 **"쉽다≠정확하다"** 그대로 — AI가 잘못된 컬럼을 분석하거나, 집계 기준을 임의로 설정해도 발견하지 못합니다. BBC 조사에 따르면 AI 쿼리의 45%가 오류를 포함하며, "열 이름을 잘못 매핑"하는 것이 가장 흔한 원인입니다.

**올바른 방법**:

```
데이터 분석해주고, 각 단계의 코드도 보여줘.

```

**차이**: 코드를 "이해하려는 노력" 자체가 학습입니다. 파이썬을 배우지 않아도, "df.groupby()가 그룹별 분석이구나"를 알게 되면 다음에 AI에게 더 정확한 지시를 내릴 수 있습니다.

---

**잘못된 방법 — 본문의 "멋진 차트≠좋은 분석" 패턴**:

```

```

**결과**: 본문 Anti-Pattern 2에서 경고한 그대로 — 표본 50명으로 소수점 한 자리까지 제시하는 "과도한 정밀도", 신뢰구간 없는 수치, 검증되지 않은 기준(전월 대비인지 전년 대비인지). Claude Code의 멋진 matplotlib 차트도 **내용이 틀리면 소용없습니다**.

**올바른 방법**: "23% 증가의 기준 기간은? 95% 신뢰구간은? 이상치를 제외하면 달라지는지도 확인해줘"라고 검증 요청을 추가합니다. CLAUDE.md에 "모든 수치에 기준과 신뢰구간 명시" 규칙을 포함합니다.

**차이**: AI 분석 결과에 대한 **"정말?" 습관**이 데이터 리터러시의 핵심입니다. 본문이 "비판적 검토를 AI에게 요청하는 한 줄이 보고서의 질을 결정한다"고 했듯, CC에서는 이 한 줄을 CLAUDE.md에 영구 규칙으로 넣습니다.
