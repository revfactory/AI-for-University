## 2. 핵심 개념: AI + 간격 반복 = 시험 효율 극대화

### 2.1 간격 반복 + AI의 과학적 효과

**SAGE Journals(2025) 연구:**
- Anki 간격 반복 사용자: 시험 성적 **6.2~12.9% 향상**
- AI가 복습 일정을 자동 최적화하면 효과 극대화
- ChatGPT에게 "GSAT 수리논리 유형별 핵심 개념을 Anki 플래시카드로 만들어줘" → 자동 복습 시스템 구축

### 2.2 AI 문제 생성의 원리와 한계

| 장점 | 한계 |
|------|------|
| 원하는 유형의 문제를 **무한 생성** | 실제 시험 난이도와 **차이 가능** |
| 풀이 과정을 **단계별로 설명** | 고난도 추론 문제에서 **오류 발생** |
| 약점 영역 **집중 출제** 가능 | 출제 패턴(함정 선지) 재현 **불완전** |
| 오답 분석 + 유사 문제 **자동 재생성** | **기출 문제 데이터** 접근 제한 |

> **핵심 전략:** AI로 개념 이해 + 기출 문제로 실전 감각 → **"AI 학습 + 기출 검증"** 투트랙

### 2.3 잡다(JOBDA) AI 역량검사란?

- **750개 이상 기업**에서 활용하는 AI 기반 역량검사
- 생물학/신경과학 + 데이터 사이언스 기반 **잠재역량 측정**
- 총 **8개 게임**으로 구성 — 논리력, 집중력, 기억력, 창의력, 문제해결력 평가
- AI가 **게임 성공률 + 반응 속도**를 분석하여 역량 평가
- 한번 응시한 결과를 다른 기업 지원 시 **"불러오기"** 가능

**연습 방법:**
1. 잡다 튜토리얼(무료): 실제 검사와 동일한 형태 연습
2. YouTube 연습 영상: 게임 유형별 공략법
3. 반복 연습: **반응 속도가 핵심**이므로 게임 유형에 익숙해지는 것이 중요

## 3. 따라하기 실습

### 실습 A: ChatGPT로 인적성 문제 생성 + 풀이 학습 ⭐

> **목표:** AI로 인적성 시험 문제를 생성하고 풀이 과정을 학습합니다.
> **소요 시간:** 40분
> **준비물:** ChatGPT 계정

---

**Step 1: 수리논리 문제 생성**

ChatGPT에 다음 프롬프트를 입력합니다:

```
삼성 GSAT 수리논리 영역의 자료해석 문제를 5개 만들어줘.

조건:
1. 난이도: 중~상 (대기업 필기시험 수준)
2. 표나 그래프 데이터를 포함한 문제
3. 5지선다 객관식
4. 각 문제에 풀이 과정을 단계별로 설명
5. 시간 목표: 문제당 2분
6. 함정 선지(오답 유인)를 포함하고, 왜 틀리기 쉬운지 설명

출력 형식:
- 문제 → 선지 → 정답 → 풀이 → 오답 분석
```

📍 **화면 가이드:** ChatGPT가 표 데이터를 포함한 자료해석 문제를 생성합니다. 문제를 풀기 전에 먼저 시간을 재며 직접 풀어보세요.

✅ **체크포인트:** 5개 문제가 표/데이터와 함께 생성되었나요? 각 문제에 풀이와 오답 분석이 포함되어 있나요?

---

**Step 2: 논리 추론 문제 생성**

```
NCS 문제해결능력 영역의 논리 추론 문제를 5개 만들어줘.

유형:
- 명제 논리 (참/거짓 판단) 2문제
- 배열/순서 문제 (조건 기반 배치) 2문제
- 상황판단 (최적 의사결정) 1문제

각 문제에:
1. 5지선다 선지
2. 정답과 풀이
3. 이 유형의 빠른 풀이 전략(팁)
```

📍 **화면 가이드:** 유형별로 문제가 분류되어 생성됩니다. "빠른 풀이 전략"을 메모해두세요.

✅ **체크포인트:** 3가지 유형(명제, 배열, 상황판단) 문제가 모두 생성되었나요?

---

**Step 3: 시간 측정 + 자기 평가**

생성된 10개 문제를 **타이머를 켜고** 풀어봅니다:
- 수리논리 5문제: 목표 시간 **10분**
- 논리추론 5문제: 목표 시간 **10분**

풀이 후 채점하고, 틀린 문제를 기록하세요:
- 맞힌 문제 수: ___/10
- 시간 내 완료 여부: ___
- 가장 어려웠던 유형: ___

🔧 **트러블슈팅:**
- AI가 너무 쉬운 문제를 만든다면 → "실제 GSAT 수준의 고난도 문제로 만들어줘. 정답률 40% 수준"
- 풀이 과정이 이해가 안 된다면 → "이 풀이를 고등학교 수학 수준으로 더 쉽게 설명해줘"

### 실습 B: AI 오답 노트 자동화 ⭐⭐

> **목표:** 틀린 문제를 AI로 분석하고, 약점별 맞춤 학습 계획을 세웁니다.
> **소요 시간:** 30분
> **준비물:** ChatGPT 계정, 실습 A에서 틀린 문제

---

**Step 1: 오답 분석 요청**

```
다음은 내가 틀린 인적성 시험 문제들입니다:

[틀린 문제 + 내가 고른 답 + 정답 붙여넣기]

각 문제에 대해:
1. 내가 틀린 이유 분석 (계산 실수? 개념 오해? 시간 부족?)
2. 이 유형의 핵심 풀이 공식/전략
3. 유사 문제 2개씩 추가 생성 (같은 유형, 다른 숫자)
4. 내 약점 패턴 종합 분석
5. 약점 보완을 위한 1주일 학습 계획

출력:
- 약점 유형별 분류표
- 우선순위 학습 리스트 (가장 자주 틀리는 유형부터)
```

📍 **화면 가이드:** ChatGPT가 오답 패턴을 분석하고, 약점 유형별로 분류합니다. **유사 문제**가 자동 생성됩니다.

✅ **체크포인트:** 약점 유형이 분류되고, 유사 문제가 생성되었나요? 1주일 학습 계획이 나왔나요?

---

**Step 2: 간격 반복용 플래시카드 생성**

```
위의 약점 유형별로 Anki 플래시카드를 만들어줘.

형식:
앞면: [문제 유형 + 핵심 질문]
뒷면: [풀이 공식/전략 + 예시]

카드 10장으로 만들어줘.
```

📍 **화면 가이드:** 생성된 카드를 Anki 앱에 수동으로 입력하거나, 텍스트 파일로 저장하여 일괄 임포트할 수 있습니다.

✅ **체크포인트:** 10장의 플래시카드가 생성되었나요?

🔧 **트러블슈팅:**
- 오답 분석이 너무 일반적이라면 → 틀린 문제의 **내가 고른 답과 고른 이유**를 함께 입력
- 유사 문제가 원래 문제보다 너무 쉽다면 → "난이도를 원래 문제와 동일하게 유지해줘"

```
삼성 GSAT 수리영역 대비:
1. 자료해석 5문제 (표+그래프 포함, 난이도 상)
```

## 5. 실전 사례

### 사례 1: 간격 반복 + AI — 시험 성적 6.2~12.9% 향상 (A급)

**SAGE Journals(2025)** 연구에 따르면, Anki 간격 반복을 사용한 학생들의 시험 성적이 **6.2~12.9% 향상**되었습니다. 인적성 시험도 반복 학습이 핵심이며, AI가 복습 일정을 자동 최적화하면 효과가 극대화됩니다.

**실전 적용:** ChatGPT에게 "GSAT 수리논리 유형별 핵심 개념을 Anki 플래시카드 형식으로 만들어줘" → 간격 반복 시스템 구축

### 사례 2: 링커리어 CBT — 대기업 인적성 온라인 모의시험 (A급)

**링커리어 CBT**는 삼성 GSAT, SKCT, LG Way Fit, SSAFY 등 **실제 기출 기반 CBT(Computer Based Test)**를 제공합니다. AI 기반 정답/해설이 포함되며, 실전 감각 훈련에 필수적입니다.

**핵심:** AI로 문제를 만들어 개념을 공부하고, **CBT 모의시험으로 실전 속도**를 훈련하는 투트랙 전략이 효과적입니다.

### 사례 3: 삼성 GSAT 2025 공식 정보 (A급)

**Samsung Newsroom(2025)** 공식 발표: 2025년 GSAT은 온라인 시험 방식으로 진행. 수리논리 20문항(30분) + 추리 30문항(30분) 구성. AI로 대비하되, **시간 제약 하에서의 풀이 속도**가 합격을 결정합니다.

### 사례 4: GSAT 기출 + ChatGPT 4단계 학습법 (B급)

커뮤니티에서 공유되는 효과적인 학습법:
1. GSAT 기출문제 PDF 다운로드
2. ChatGPT에 업로드 → "이 문제의 풀이 과정을 단계별로 설명해줘"
3. 오답 유형 분석 → "내가 틀린 유형과 비슷한 문제를 5개 더 만들어줘"
4. 간격 반복 앱에 오답 노트 등록 → 자동 복습

## 6. 꿀팁 & 주의사항

### 💡 꿀팁 5가지

1. **AI는 개념 학습, 실전은 CBT:** AI로 풀이 과정을 이해하고, 링커리어 CBT로 시간 내 풀기를 연습하세요
2. **오답 데이터를 누적하라:** 틀린 문제를 계속 ChatGPT에 입력하면, AI가 약점 패턴을 더 정확히 분석합니다
3. **영역별 도구를 나눠라:** 수리→ChatGPT, 의사소통→Claude, 시사→Perplexity로 최적 도구 조합
4. **잡다 튜토리얼은 반드시 해보라:** 750+ 기업이 사용하는 AI 역량검사 — 연습만으로 반응 속도가 크게 향상
5. **AI-POT 자격증 병행:** 한국생산성본부 주관 AI 프롬프트 활용능력 자격증(2026년 2·5·8·11월 시행) — AI 활용 역량 증명에 유용

### ⚠️ Anti-Pattern 1: "AI로 개념만 공부하고 실전 속도 훈련을 안 한다"

**상황:** 대학생 E가 ChatGPT로 GSAT 문제 풀이를 열심히 공부. AI의 단계별 설명으로 개념은 완벽히 이해.

**잘못된 방법:** AI 풀이 설명만 반복 학습, 타이머 없이 느긋하게 문제 풀기.

**결과:**
- 시험장에서 문제당 30~60초 제한에 당황
- 개념은 알지만 **시간 내에 풀지 못함**
- 결과: 시간 부족으로 절반도 못 풀고 종료

**올바른 방법:**
1. AI로 **풀이 과정 이해** (개념 학습 단계)
2. 이해한 후 **타이머 켜고 기출 CBT** 반복 (속도 훈련 단계)
3. 문제당 목표 시간: GSAT 수리 **1분 30초**, 추리 **1분**
4. AI에게 "시간 관리 전략"도 함께 요청

**차이:** AI 학습만으로는 시험을 볼 수 없습니다. **"개념 이해(AI) + 속도 훈련(CBT) + 간격 반복(Anki)"** 3가지가 모두 필요합니다.

---

### ⚠️ Anti-Pattern 2: "AI 생성 문제만 풀고 기출을 안 본다"

**상황:** 대학생 F가 ChatGPT에 "GSAT 수리논리 문제 20개 만들어줘" 요청. 매일 AI 문제만 풀며 준비.

**잘못된 방법:** AI 생성 문제만으로 시험 준비. 기출문제는 "AI가 더 많이 만들어주니까" 스킵.

**결과:**
- AI가 만든 문제: 10개는 너무 쉽고 10개는 너무 어려움
- 실제 GSAT: 중간 난이도가 대부분이나 **함정 선지**가 핵심
- AI 문제에 없는 **출제 패턴**(함정, 오답 유인)에 취약
- 결과: 쉬운 문제는 맞지만 함정 문제에 연속 실수

**올바른 방법:**
1. **기출 문제를 먼저** 풀어 실제 난이도와 출제 패턴 파악
2. AI에게 기출을 업로드 → "이 기출과 비슷한 난이도의 문제를 만들어줘"
3. AI 문제는 **약점 영역 집중 연습용**으로 활용
4. 실전 대비는 반드시 **CBT 모의시험**으로

**차이:** AI 문제는 실제 시험의 난이도와 출제 패턴을 정확히 재현하지 못합니다. **기출이 기준, AI는 보조** — 이 순서를 지켜야 합니다.

## 7. 워크플로우 일반화: AI 시험 대비 파이프라인

```
[Phase 1: 시험 분석] (D-30)
  → 시험 유형 파악 (GSAT/NCS/SKCT 등)
  → 영역별 출제 비중, 시간 배분 파악
  → 기출 문제 확보 (링커리어 CBT, SpecUp 등)

[Phase 2: 기출 + AI 학습] (D-30~D-14)
  → 기출 문제 풀이 → 오답 수집
  → ChatGPT에 오답 분석 요청 → 약점 유형 파악
  → AI로 약점 유형 집중 문제 생성
  → 간격 반복 플래시카드 구축 (Anki)

[Phase 3: 속도 훈련] (D-14~D-7)
  → CBT 모의시험으로 실전 속도 연습
  → 문제당 목표 시간 설정 + 타이머
  → 오답 → AI 재분석 → 유사 문제 재풀이

[Phase 4: 약점 집중 보완] (D-7~D-3)
  → AI 약점 분석 결과 기반 집중 학습
  → 가장 자주 틀리는 유형 Top 3 반복
  → 잡다 ACCA 튜토리얼 연습 (해당 시)

[Phase 5: 최종 점검] (D-3~D-1)
  → 실전과 동일한 조건에서 CBT 모의시험 1회
  → 간격 반복 플래시카드 최종 복습
  → 시간 관리 전략 재확인
```

이 파이프라인은 **모든 종류의 객관식 시험**(자격증, 전공 필기, 공무원 시험 등)에 적용할 수 있습니다. 핵심은 **"기출 분석 → AI 약점 보완 → 속도 훈련 → 간격 반복"** 사이클입니다.

## 1. 시험 대비 자동화의 장점

### 1-1. 수동 연습 vs CC 자동화

| 항목 | 수동 연습 (웹 UI) | Claude Code 자동화 |
|------|-----------------|-------------------|
| 문제 생성 | 매번 프롬프트 입력 | 스크립트로 자동 생성 |
| 난이도 조절 | 직접 요청 | CLAUDE.md 규칙으로 자동 |
| 오답 분석 | 직접 정리 | 에이전트가 자동 분석 |
| 취약 영역 | 직접 파악 | 오답 패턴 기반 자동 집중 출제 |
| 반복 학습 | 같은 문제 재풀이 | 유사 유형 새 문제 자동 생성 |
| 간격 반복 | Anki 수동 등록 | 오답 자동 → 플래시카드 생성 |
| 학습 이력 | 기록 없음 | 일별 성적 추적 |

> **본문 연계**: 본문에서 AI 문제 생성의 한계를 강조합니다 — 실제 시험 난이도와 차이, 고난도 추론 오류, 함정 선지 재현 불완전. CC의 CLAUDE.md에 이 한계를 **영구 규칙**으로 설정하여, "AI 문제만으로 실전 대비 완료"라는 착각을 시스템 수준에서 방지합니다.

```markdown
# Problem Generator — 문제 생성기

## 핵심 역할
- exam_scope.md에 정의된 시험 유형에 맞는 문제를 생성합니다.
- 난이도별(쉬움/보통/어려움) 균형 있게 출제합니다.
- 이전에 출제한 문제와 중복되지 않도록 합니다.

## 문제 유형
- 언어 추론: 독해, 논리적 오류, 문장 배열, 빈칸 완성
- 수리 추론: 자료 해석, 응용 계산, 도표 분석
- 추리·논리: 명제 추론, 조건 추론, 수열, 도형 추론
- 상황 판단: 업무 상황 판단, 윤리적 딜레마
- 전공 필기: exam_scope.md에 정의된 전공 범위

## 작업 원칙
- 각 문제에 정답, 해설, 난이도, 유형 태그 포함
- 보기(선지)는 매력적인 오답 포함 — 쉽게 소거되지 않도록
- 실제 시험과 유사한 형식 유지
- 문제 수: 기본 세트 20문제 (유형별 4~5문제)

## 출력
- output/problems/문제세트_YYYYMMDD.md: 문제 파일
- output/problems/정답_YYYYMMDD.md: 정답·해설 파일 (별도)
```

```markdown
# Weakness Trainer — 취약 영역 집중 훈련기

## 핵심 역할
- output/analysis/의 오답 분석을 읽고 취약 유형을 식별합니다.
- 취약 유형의 문제를 집중 생성합니다 (기본 배율의 2배).
- 취약 유형의 핵심 개념 정리도 함께 제공합니다.

## 작업 원칙
- 정답률 70% 미만 유형을 "취약"으로 분류
- 취약 유형: 문제 수 2배 + 개념 정리 포함
- 강점 유형(90%+): 어려운 난이도 위주로 출제
- 3회 연속 90%+ 달성 시 "마스터" 표시 → 출제 비중 감소

## 출력
- output/problems/취약집중_YYYYMMDD.md: 취약 영역 집중 문제
```

### 2-2. CLAUDE.md 시험 대비 규칙

```markdown
# 시험 대비 시스템 규칙

## 문제 생성 원칙
- 실제 시험(GSAT, NCS, 전공 필기)과 동일한 형식
- 난이도 비율: 쉬움 30% / 보통 50% / 어려움 20%
- 문제와 정답·해설 파일을 반드시 분리 (정답 미리 보기 방지)
- 매 세트마다 새로운 문제 (이전 문제 재사용 금지)
- ⚠️ AI 생성 문제의 한계 인지: 실전 난이도·함정 선지 재현 불완전
- 기출 기반 보정: input/에 기출 패턴 파일이 있으면 반드시 참조

## 기출 연계 원칙
- 기출 70% + AI 보충 30% 비율 권장 (본문 투트랙 전략)
- 링커리어 CBT, 잡다 ACCA 등 실전 도구와 병행 필수
- AI-POT(한국생산성본부 AI 프롬프트 활용능력 자격증) 병행 추천

## 채점 원칙
- 정답률은 소수점 첫째 자리까지
- 시간 측정: 시작~종료 시간 기록 (문제당 평균 소요 시간)
- 오답 원인은 반드시 4가지 중 하나로 분류

## 학습 관리
- 일별 정답률 변화 추적
- 3회 연속 전체 정답률 80%+ 시 난이도 상향
- 유형별 목표: 전 유형 70%+가 기본 목표
```

### 🔬 프롬프트 진화 실험: 인적성 시험 대비

#### 🔴 Level 1 — 초보 프롬프트

```
인적성 문제 만들어줘
```

**결과**: 일반적인 상식 퀴즈 수준의 문제가 나옵니다. 실제 GSAT/NCS와 형식이 다르고, 난이도 조절이 안 됩니다.

📋 **평가**:
- 정확성: ★★☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 시험 유형 미지정, 형식 부정확, 해설 없음

---

#### 🟡 Level 2 — 중급 프롬프트

```
GSAT 언어추론 문제 5개 만들어줘.
4지선다, 정답과 해설 포함.
```

**결과**: 형식은 맞지만 난이도가 불균형하고, 오답 선지가 너무 쉽게 소거됩니다. 매번 수동 요청이 필요합니다.

📋 **평가**:
- 정확성: ★★★☆☆
- 활용도: ★★★☆☆
- 개선점: 시험 유형과 형식 지정
- 아쉬운 점: 난이도 불균형, 오답 분석 없음

---

#### 🟢 Level 3 — 전문가 프롬프트 (하네스 활용)

```
@problem-generator exam_scope.md 기준으로 20문제 생성해줘.
유형별 균형, 난이도 30/50/20 비율.
문제와 정답은 별도 파일로.

[풀이 완료 후]
@answer-checker 내 답안을 채점하고 오답 분석해줘.
@weakness-trainer 취약 유형 집중 문제 10개 추가 생성.
```

📋 **평가**:
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: 자동 출제 + 채점 + 오답 분석 + 취약 집중의 완전 자동화

---

#### 💡 교훈: 프롬프트의 마법

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 시험 유형 | ✗ | ✓ | ✓ (scope 파일) |
| 난이도 조절 | ✗ | ✗ | ✓ (비율 설정) |
| 오답 분석 | ✗ | ✗ | ✓ (에이전트) |
| 취약 집중 | ✗ | ✗ | ✓ (에이전트) |
| 자동화 | ✗ | ✗ | ✓ (스크립트) |
| 결과 품질 | 10% | 45% | 95% |

### ❌ 흔한 실수: 시험 대비 자동화의 함정

**상황**: Claude Code로 시험 대비 문제를 만들 때

---

**잘못된 방법 — AI 문제만 풀고 실전 준비 끝**:

```
매일 20문제씩 풀면 실전도 괜찮겠지
```

**결과**: AI 생성 문제는 실제 시험과 출제 패턴, 함정 유형, 시간 압박이 다릅니다. AI 문제로만 연습하면 "AI 문제에 최적화된 사람"이 되어 실전에서 당황합니다.

**올바른 방법**: AI 문제(70%) + 기출문제(30%) 조합. exam_scope.md에 "기출 문제 패턴을 참고하여 출제" 규칙을 설정합니다. 가능하면 실제 기출 문제 파일을 input/에 포함하여 유사 유형으로 출제하도록 합니다.

**차이**: AI 문제는 **"기출을 보충하는 추가 연습"**이지, 기출의 대체가 아닙니다.

---

**잘못된 방법 — 정답만 확인하고 넘기기**:

```
정답만 알려줘. 해설은 필요 없어.
```

**결과**: 왜 틀렸는지 모르면 같은 유형에서 계속 틀립니다. "맞으면 넘기고 틀리면 정답 확인"은 학습이 아닙니다.

**올바른 방법**: answer-checker 에이전트가 오답 원인을 4가지(개념/함정/부주의/시간)로 분류하고, weakness-trainer가 같은 유형을 재출제합니다.

**차이**: "틀린 문제 반복"이 아니라 **"틀린 이유 이해 + 유사 문제 연습"**이 실력을 키웁니다.

## 실습 과제

### 🎯 실습: 문제 생성 + 풀이 + 채점 ⭐

**목표**: Claude Code로 시험 문제를 생성하고, 풀이 후 자동 채점을 받습니다.
**소요 시간**: 10분
**준비물**: Claude Code
**사용 도구**: Claude Code

---

#### Step 1. 시험 범위 정의 ⏱️ 2분

```
인적성 시험 대비 범위를 정의해줘.
input/exam_scope.md에 저장.

포함 유형:
- 언어 추론 (독해, 논리)
- 수리 추론 (자료 해석, 응용 계산)
- 추리 (명제, 수열, 도형)
- 상황 판단

난이도: 쉬움 30% / 보통 50% / 어려움 20%
```

✅ **체크**: `input/exam_scope.md`에 시험 범위가 정의되면 성공

---

#### Step 2. 문제 생성 ⏱️ 3분

```
exam_scope.md 기준으로 10문제 세트를 만들어줘.
유형별 균형 있게, 4지선다 형식.
문제: output/problems/문제세트_01.md
정답: output/problems/정답_01.md (별도 파일)
```

✅ **체크**: 문제 파일과 정답 파일이 별도로 생성되면 성공

---

#### Step 3. 풀이 + 채점 ⏱️ 5분

문제를 풀고 답안을 입력합니다:

```
내 답안: 1-③, 2-①, 3-④, 4-②, 5-③, 6-①, 7-④, 8-②, 9-③, 10-①
채점하고 오답을 분석해줘.
오답 원인: 개념부족/함정/부주의/시간 중 분류.
```

✅ **체크**: 정답률 + 오답 원인 분류가 포함되면 성공

> 🔧 **안 되나요?**: 답안 형식이 다르면 → "1번부터 10번까지 내 답을 쉼표로 나열해줄게: 3,1,4,2,3,1,4,2,3,1"처럼 간단하게 입력하세요.

---

#### 🎉 완성! — 결과물 미리보기

문제 생성 → 풀이 → 채점 + 오답 분석의 기본 사이클을 완성했습니다.

> 📊 **효과**:
> - 핵심 이해: AI가 문제를 무한 생성하고, 오답 원인까지 분석
> - 다음 실습(⭐⭐)에서 취약 영역 집중과 일일 연습 자동화를 추가합니다

---

### 🎯 실습: 취약 영역 집중 + 일일 자동화 ⭐⭐

**목표**: 오답 분석 기반으로 취약 영역 집중 문제를 생성하고, 매일 자동 연습을 설정합니다.
**소요 시간**: 20분
**준비물**: 이전 실습의 채점 결과
**사용 도구**: Claude Code

---

#### Step 1. 취약 영역 분석 ⏱️ 5분

```
이전 채점 결과를 바탕으로 취약 영역을 분석해줘.
유형별 정답률, 오답 원인 패턴, 집중 필요 영역을 정리.
output/analysis/취약분석.md에 저장.
```

✅ **체크**: 유형별 정답률 + 취약 영역이 식별되면 성공

---

#### Step 2. 취약 집중 문제 생성 ⏱️ 5분

```
취약 영역(정답률 70% 미만)의 문제를 10개 추가 생성해줘.
각 문제에 핵심 개념 정리도 포함.
기존보다 난이도는 한 단계 낮춰서 기초부터.
```

✅ **체크**: 취약 유형 집중 문제 + 개념 정리가 생성되면 성공

---

#### Step 3. 일일 연습 스크립트 ⏱️ 5분

```
매일 자동으로 연습 문제를 생성하는 스크립트를 만들어줘.
scripts/daily_practice.sh

기능:
1. 이전 결과의 취약 영역 확인
2. 취약 영역 문제 비중 높여서 10문제 생성
3. 문제/정답 파일 자동 생성
4. (선택) cron으로 매일 아침 8시 자동 실행

오늘 날짜로 테스트 실행도 해줘.
```

✅ **체크**: 스크립트가 생성되고, 테스트 실행으로 문제가 자동 생성되면 성공

> 🔧 **안 되나요?**: cron 등록은 Ch10에서 배운 방법을 참고하세요. `0 8 * * * /bin/bash ~/exam-prep/scripts/daily_practice.sh`

---

#### Step 4. 성적 추이 확인 ⏱️ 5분

```
output/results/ 의 모든 채점 결과를 종합해서
성적 추이 리포트를 만들어줘.

포함 내용:
- 전체 정답률 변화 (날짜별)
- 유형별 정답률 변화
- "마스터" 달성 유형
- 다음 목표 제안

output/analysis/성적추이.md에 저장.
```

✅ **체크**: 성적 추이 + 유형별 변화가 포함된 리포트가 생성되면 성공

---

#### 🎉 완성! — 결과물 미리보기

오답 분석 → 취약 집중 → 일일 자동화 → 성적 추적까지 **적응형 학습 시스템**을 구축했습니다.

> 📊 **효과**:
> - 기존 방식: 문제집 구매 + 전 유형 무차별 풀기 — 효율 낮음
> - CC 자동화: 취약 영역 자동 식별 + 집중 출제 — 학습 효율 극대화

### 🎯 실습: 종합 시험 대비 하네스 구축 ⭐⭐⭐

**목표**: 문제 생성, 채점, 오답 분석, 취약 집중을 통합한 시험 대비 하네스를 구축합니다.
**소요 시간**: 40분
**준비물**: Claude Code, 시험 범위 정보
**사용 도구**: Claude Code

---

#### Step 1. 시험 대비 하네스 구축 ⏱️ 12분

```
~/exam-prep/ 에 시험 대비 하네스를 구축해줘.

폴더 구조: [위의 구조 참조]
에이전트 3개 + CLAUDE.md + 스크립트 2개
input/ 에 GSAT 범위 정의 + 샘플 기출 패턴도 만들어줘.
```

✅ **체크**: 전체 구조 + 에이전트 3개 + 스크립트가 완성되면 성공

---

#### Step 2. 모의시험 세션 ⏱️ 15분

```
GSAT 모의시험을 시작해줘.
20문제, 30분 제한 (타이머 표시).
유형별 균형 출제.

문제를 순서대로 보여주고, 내가 답을 입력하면 다음 문제로.
마지막에 일괄 채점 + 오답 분석.
```

✅ **체크**: 20문제 풀이 + 채점 + 오답 분석이 완료되면 성공

> ⚠️ **주의**: AI 생성 문제의 정확성을 확인하세요. 특히 수리 추론 문제의 계산이 맞는지, 논리 추론 문제의 정답이 유일한지 점검이 필요합니다.

---

#### Step 3. 취약 집중 + 반복 ⏱️ 8분

```
모의시험 결과에서 취약 유형을 식별하고
@weakness-trainer로 취약 집중 문제 10개 생성.
개념 정리도 포함.

풀이 후 재채점해서 개선 여부 확인.
```

✅ **체크**: 취약 집중 문제 풀이 후 정답률이 개선되면 성공

---

#### Step 4. 학습 대시보드 ⏱️ 5분

```
모든 연습 결과를 종합한 학습 대시보드를 만들어줘.

포함:
1. 전체 정답률 (현재 vs 목표)
2. 유형별 레이더 차트 데이터
3. 마스터 달성 현황
4. D-Day 카운트다운 (시험일 설정)
5. 주간 학습 계획 제안

output/analysis/dashboard.md에 저장.
```

✅ **체크**: 학습 대시보드가 생성되면 성공

#### 🎉 완성! — 결과물 미리보기

출제 → 풀이 → 채점 → 오답 분석 → 취약 집중 → 성장 추적까지 **적응형 시험 대비 시스템**을 구축했습니다.

> 📊 **효과**:

| 오답 분석 → 집중 출제 | 취약 문법 → 집중 연습 | 외국어 학습 |
| 일일 자동 출제 (cron) | 매일 단어 테스트 | 암기 자동화 |

**핵심 패턴**: `범위 정의(scope) + 자동 출제(에이전트) + 채점·분석(에이전트) + 취약 집중(에이전트) + 스크립트 자동화 = 적응형 학습 시스템` — 반복 학습이 필요한 모든 시험에 이 패턴을 적용할 수 있습니다.
