## 1. 이런 분이 읽으면 좋아요

**이런 분에게 추천합니다:**

- PPT 만들 때 **디자인에서 막혀서** 내용보다 꾸미는 데 시간이 더 걸리는 분

## 2. 이 챕터에서 배우는 것

### 학습목표

이 챕터를 마치면 다음을 할 수 있습니다:

1. **AI PPT 도구**(Gamma, Canva AI, ChatGPT, Gemini)의 특성을 비교하고 상황에 맞는 도구를 선택할 수 있다
2. AI에게 **효과적인 PPT 프롬프트**를 작성하여 원하는 구조와 디자인의 슬라이드를 생성할 수 있다
3. **ChatGPT로 발표 구성을 기획**하고, **Gamma로 시각화**하는 2단계 워크플로우를 수행할 수 있다
4. AI가 생성한 슬라이드를 **디자인 원칙에 따라 검수·수정**할 수 있다
5. **발표 스크립트와 Q&A 예상 질문**을 AI로 준비할 수 있다

### 완성 후 얻는 결과물

> 이 챕터의 실습을 마치면, 수업 발표 주제를 입력한 지 **15분 만에 완성된 10슬라이드 발표자료 + 슬라이드별 발표 스크립트 + 예상 Q&A 5문항**을 갖게 됩니다.

### Gamma 심층 안내

Gamma는 대학생이 가장 많이 쓰는 AI PPT 도구입니다. 핵심 특징을 정리합니다.

**주요 기능:**
- **프롬프트 → 슬라이드**: 텍스트를 입력하면 60초 내에 10페이지 슬라이드 초안을 생성합니다
- **문서 업로드**: PDF, Word 파일을 업로드하면 자동으로 슬라이드로 변환합니다
- **Gamma Agent 3.0** (2025.09~): "3번 슬라이드에 차트 추가해줘"처럼 자연어로 수정을 지시할 수 있습니다

- **실시간 협업**: 팀원과 동시 편집 가능

**가격 구조:**

## 4. 핵심 개념

### 발표자료의 3요소: 구조 + 디자인 + 스토리

AI PPT 도구를 잘 활용하려면, 좋은 발표자료의 구성 요소를 먼저 이해해야 합니다.

**1. 구조(Structure)** — 무엇을 어떤 순서로 말할 것인가
- 도입(Hook) → 문제 제기 → 분석 → 해결안 → 결론 → Q&A
- AI가 가장 잘 하는 부분입니다. ChatGPT에게 주제만 주면 논리적 구조를 즉시 생성합니다.

**2. 디자인(Design)** — 시각적으로 어떻게 보여줄 것인가
- 한 슬라이드 한 메시지, 6×6 규칙(6줄, 줄당 6단어 이하), 고대비 색상
- AI PPT 도구(Gamma, Canva)가 자동으로 처리합니다. 단, **기본 템플릿을 그대로 쓰면 안 됩니다.**

**3. 스토리(Story)** — 청중의 마음을 어떻게 움직일 것인가
- 오프닝 멘트, 전환 연결, 클로징 — 이것은 **인간이 직접 다듬어야** 합니다.
- AI는 초안을 생성하지만, 본인만의 경험과 관점을 넣는 것은 발표자의 몫입니다.

### AI PPT 디자인 체크리스트

발표자료를 AI로 만든 후 **반드시 확인**해야 할 항목입니다:

```
✅ 한 슬라이드에 핵심 메시지가 하나만 있는가?
✅ 텍스트가 6줄 이하인가? (줄당 6단어 이하)
✅ 색상이 일관되고, 배경과 텍스트가 고대비인가?
✅ 폰트가 제목 1종 + 본문 1종으로 통일되어 있는가?
✅ 여백이 충분한가? (슬라이드의 30% 이상 여백)
✅ 이미지/아이콘이 내용과 관련 있는가?
❌ 워드아트, 클립아트, 과도한 애니메이션은 없는가?
❌ AI 기본 템플릿을 그대로 사용하지 않았는가?
```

### 🔬 프롬프트 진화 실험

같은 작업을 3단계 프롬프트로 시도하여, 프롬프트 품질의 차이를 직접 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
AI 관련 발표자료 만들어줘
```

**결과**: "AI의 정의", "AI의 역사", "AI의 활용"… 교과서 복사 같은 일반적 내용. 청중도 시간도 모르니 분량 조절 불가. 디자인 지시가 없어서 기본 템플릿 그대로.

📋 **평가**:
- 정확성: ★★☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 주제가 넓고, 맥락(청중, 시간, 목적)이 없어서 어디서나 볼 수 있는 내용

---

#### 🟡 Level 2 — 중급 프롬프트

```
"대학생의 AI 활용 현황" 주제로 10슬라이드 발표자료를 만들어줘.
```

**결과**: 구조가 생겼습니다(표지, 목차, 현황, 사례, 결론). 하지만 청중이 교수인지 동기인지에 따라 어조가 달라야 하고, 15분인지 5분인지에 따라 슬라이드 밀도가 달라져야 합니다. 디자인 스타일 지시가 없어 AI가 임의 선택.

📋 **평가**:
- 정확성: ★★★☆☆
- 활용도: ★★★☆☆
- 개선점: 슬라이드 수와 주제가 명확해짐
- 아쉬운 점: 청중, 시간, 디자인 맥락 부재

---

#### 🟢 Level 3 — 전문가 프롬프트

```
주제: "대학생의 AI 활용 현황과 미래"
목적: 교양수업 기말 팀 발표
청중: 대학생 40명 + 교수님
시간: 15분

슬라이드 구성:
1. 표지 (주제, 팀명, 날짜)
2. 목차
3. AI 도구 사용 현황 (통계 차트 — HEPI 2025 설문 인용)
4. 주요 활용 사례 ① 과제·보고서 작성
5. 주요 활용 사례 ② 논문 검색·자료 정리
6. 주요 활용 사례 ③ 발표자료 제작
7. 문제점과 윤리적 이슈
8. 해결 방안 제안
9. 결론 및 시사점
10. Q&A / 참고문헌

디자인 요청:
- 색상: 네이비+화이트 배경 (학술적)
- 각 슬라이드에 관련 아이콘 또는 인포그래픽 포함
- 텍스트 최소화, 키워드 + 시각자료 중심
```

**결과**: 정확히 요청한 구조대로 10슬라이드가 생성됩니다. 통계 차트 슬라이드에는 데이터 시각화가 포함되고, 디자인도 네이비+화이트로 통일됩니다. 청중과 시간에 맞춘 적절한 텍스트 밀도.

📋 **평가**:
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: **슬라이드별 구조 지정 + 디자인 요청 + 데이터 출처 명시**가 결정적

---

#### 💡 교훈: PPT 프롬프트의 마법

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 주제 명시 | △ (너무 넓음) | ✓ | ✓ |
| 슬라이드 수 | ✗ | ✓ | ✓ |
| 청중/시간 | ✗ | ✗ | ✓ |
| 슬라이드별 구성 | ✗ | ✗ | ✓ |
| 디자인 지시 | ✗ | ✗ | ✓ |
| 데이터 출처 | ✗ | ✗ | ✓ |
| 결과 품질 | 20% | 55% | 95% |

> 📌 **핵심 원칙**: PPT 프롬프트는 "슬라이드별 구조를 직접 설계"하는 것이 가장 큰 차이를 만듭니다. AI는 디자인과 콘텐츠를 채워주지만, **전체 흐름(스토리보드)은 인간이 설계해야** 합니다.

## 5. 따라하기 실습

### 🎯 실습 A: ChatGPT로 발표 기획하기 ⭐

**목표**: ChatGPT를 활용하여 발표 구성(아웃라인)과 슬라이드별 핵심 메시지를 설계합니다.
**소요 시간**: 10분
**준비물**: ChatGPT 계정 (Free 가능)
**사용 도구**: ChatGPT

---

#### Step 1. ChatGPT에 접속하기 ⏱️ 1분

ChatGPT(https://chat.openai.com)에 접속하여 로그인합니다.

> 📍 **화면 안내**: 화면 하단의 메시지 입력창을 클릭합니다.

✅ **체크**: 메시지 입력창이 활성화되어 커서가 깜빡이면 준비 완료.

---

#### Step 2. 발표 구성 프롬프트 입력 ⏱️ 3분

다음 프롬프트를 **그대로 복사**하여 붙여넣으세요:

```
나는 [학과] [학년] 학생입니다.
[수업명] 수업에서 "[발표 주제]"라는 주제로 발표해야 합니다.

발표 조건:
- 청중: 같은 수업 학생 [N]명 + 교수님
- 시간: [N]분 발표 + [N]분 Q&A
- 슬라이드: [N]장
- 분위기: [학술적 / 캐주얼 / 설득적]

다음을 만들어주세요:
1. 슬라이드별 구성 (제목 + 핵심 메시지 1~2문장)
2. 각 슬라이드에 넣으면 좋을 시각자료 유형 (차트/이미지/아이콘/인용)
3. 전체 발표의 스토리라인 (도입→전개→결론의 논리적 흐름 설명)
```

> 💡 **커스텀 포인트**: `[대괄호]` 안의 내용은 본인 상황에 맞게 수정하세요.
> 예시: "나는 경영학과 3학년 학생입니다. 마케팅원론 수업에서 'MZ세대의 소비 트렌드 변화'라는 주제로 발표해야 합니다."

✅ **체크**: AI가 슬라이드별 구성을 1번부터 N번까지 번호를 매겨 응답했으면 성공.

---

#### Step 3. 결과 검토 및 수정 ⏱️ 3분

AI의 응답에서 다음을 확인합니다:

1. **논리적 흐름**: 도입 → 문제 제기 → 분석 → 해결 → 결론 순서가 자연스러운가?
2. **슬라이드 밀도**: 15분 발표라면 슬라이드당 약 1.5분. 10장이면 적절합니다.
3. **시각자료 제안**: 텍스트만 가득한 슬라이드는 없는지 확인합니다.

수정이 필요하면 추가 프롬프트를 입력합니다:

```
3번 슬라이드의 내용이 너무 넓어요. "국내 사례"로 한정해서 수정해줘.
그리고 7번 슬라이드 뒤에 "해외 사례 비교" 슬라이드를 추가해줘.
```

> 🔧 **안 되나요?**: AI가 너무 일반적인 내용을 생성했다면, "구체적인 통계 데이터나 연구 결과를 인용해서 다시 작성해줘"라고 추가 요청하세요.

✅ **체크**: 본인이 만족하는 슬라이드별 구성이 완성되었으면 이 결과를 **복사**해 둡니다.

---

#### Step 4. 발표 스크립트 생성 ⏱️ 3분

같은 대화에서 이어서 다음 프롬프트를 입력합니다:

```
위 구성을 바탕으로 슬라이드별 발표 스크립트를 작성해줘.

조건:
1. 각 슬라이드당 1~2분 분량의 구어체 스크립트
2. 청중 시선을 잡는 오프닝 멘트 포함
3. 슬라이드 전환 연결 문구 포함 ("이어서~", "다음으로~")
4. 예상 질문 5개 + 답변 전략 (STAR 기법 적용)
5. 클로징 멘트 (핵심 메시지 3줄 요약)
```

✅ **체크**: 슬라이드 1번부터 마지막까지 구어체 스크립트가 생성되고, 마지막에 예상 질문 5개가 포함되어 있으면 성공.

### 🎯 실습 B: Gamma로 슬라이드 시각화하기 ⭐

**목표**: 실습 A에서 만든 구성을 Gamma에 입력하여 시각적인 슬라이드를 생성합니다.
**소요 시간**: 10분
**준비물**: Gamma 계정 (Free — gamma.app에서 Google/이메일로 가입)
**사용 도구**: Gamma

---

#### Step 1. Gamma에 접속하고 새 프레젠테이션 만들기 ⏱️ 2분

Gamma(https://gamma.app)에 접속하여 로그인합니다.

> 📍 **화면 안내**: 로그인 후 화면 상단의 **"Create new"** 또는 **"새로 만들기"** 버튼을 클릭합니다. "Presentation"(프레젠테이션)을 선택합니다.

✅ **체크**: "프롬프트를 입력하세요" 또는 "Paste in text" 입력창이 나타나면 준비 완료.

---

#### Step 2. ChatGPT 결과를 Gamma에 붙여넣기 ⏱️ 2분

실습 A의 Step 3에서 복사한 **슬라이드별 구성**을 Gamma 입력창에 **그대로 붙여넣기**합니다.

> 📍 **화면 안내**: "Paste in text" 옵션을 선택한 뒤, ChatGPT에서 복사한 내용을 Ctrl+V(Mac: Cmd+V)로 붙여넣습니다.

> 💡 **팁**: Gamma에 직접 프롬프트를 입력하는 것보다, ChatGPT에서 구조를 먼저 설계하고 붙여넣는 것이 **훨씬 정교한 결과**를 얻습니다. ChatGPT는 구조와 논리에 강하고, Gamma는 시각화에 강하기 때문입니다.

✅ **체크**: Gamma가 "Generate" 또는 "생성" 버튼을 표시하면 다음 단계로 진행.

---

#### Step 3. 테마 선택 및 생성 ⏱️ 2분

1. Gamma가 테마 옵션을 제시합니다. **학술적 분위기**에 맞는 테마를 선택합니다 (보통 "Professional" 또는 "Minimal" 계열).
2. **"Generate"** 버튼을 클릭합니다.
3. 약 30~60초 후 슬라이드가 생성됩니다.

> 📍 **화면 안내**: 생성이 완료되면 왼쪽에 슬라이드 썸네일 목록이, 오른쪽에 현재 선택된 슬라이드가 표시됩니다.

✅ **체크**: 슬라이드가 10장 내외로 생성되고, 각 슬라이드에 제목과 내용이 들어 있으면 성공.

---

#### Step 4. Gamma Agent로 수정하기 ⏱️ 3분

생성된 슬라이드가 마음에 안 드는 부분이 있다면, **Gamma Agent**를 활용합니다.

Gamma 화면 하단의 AI 채팅창에 다음과 같이 입력합니다:

```
3번 카드의 텍스트를 줄이고 핵심 키워드 3개만 남겨줘.
5번 카드에 비교표를 추가해줘.
전체 색상을 네이비+화이트 톤으로 변경해줘.
```

> 💡 **팁**: Gamma Agent 3.0은 자연어로 편집을 지시할 수 있습니다. "이 슬라이드를 더 깔끔하게 만들어줘"처럼 막연한 요청보다, **"텍스트를 3줄로 줄여줘"**처럼 구체적으로 요청하세요.

> 🔧 **안 되나요?**: Agent 기능이 보이지 않으면, Gamma 화면 하단의 AI 아이콘(✨)을 클릭하세요. Free 플랜에서도 기본 편집은 가능합니다.

---

#### Step 5. 내보내기 및 발표 준비 ⏱️ 1분

완성된 슬라이드를 내보냅니다.

> 📍 **화면 안내**: 화면 우상단 **"Share"** 또는 **"Export"** 버튼 클릭 → 원하는 형식 선택

- **웹 공유** (기본 추천): 링크를 복사하여 교수님이나 팀원에게 공유
- **PDF 내보내기**: 인쇄용으로 가장 안정적
- **PPTX 내보내기**: ⚠️ 레이아웃이 깨질 수 있으므로 반드시 다운로드 후 확인

> ⚠️ **주의**: PPT 파일 제출이 필수라면, PPTX로 내보낸 후 PowerPoint에서 열어 레이아웃을 확인하세요. 깨진 부분은 수동으로 수정합니다. 또는 **Canva에서 처음부터 만드는 것**이 PPT 호환성이 더 좋습니다.

✅ **체크**: 내보낸 파일을 열어보았을 때 레이아웃이 정상이면 완성.

### 🎯 실습 C: 발표 리허설 — AI 예상 질문으로 준비하기 ⭐⭐

**목표**: AI가 생성한 예상 질문으로 Q&A 시간을 대비합니다.
**소요 시간**: 10분
**준비물**: 실습 A에서 생성한 스크립트
**사용 도구**: ChatGPT 또는 Claude

---

#### Step 1. Q&A 시뮬레이션 프롬프트 입력 ⏱️ 5분

```
내 발표 내용은 다음과 같습니다:
[실습 A에서 생성한 슬라이드 구성 + 핵심 메시지 붙여넣기]

너는 이 수업의 교수님 역할을 해줘.
발표 후 교수님이 물을 만한 질문 5개를 난이도 순으로 제시하고,

```

✅ **체크**: 교수 질문 5개 + 학생 질문 3개가 난이도 순으로 생성되고, 각각에 답변 전략이 포함되어 있으면 성공.

---

#### Step 2. 약점 보완 ⏱️ 5분

AI의 예상 질문 중 **답변하기 어려운 것**이 있다면, 추가로 요청합니다:

```
3번 질문("이 연구의 표본이 편향되어 있지 않나요?")에 대해
더 자세한 답변을 만들어줘. 구체적인 통계나 반박 논리를 포함해줘.
```

> 💡 **팁**: 실제 발표에서 예상 질문에 바로 답할 수 있으면, 교수님에게 **"이 학생은 충분히 준비했구나"**라는 인상을 줄 수 있습니다.

## 6. 실전 사례

### 사례 1: 대학생 92%가 AI를 사용하는 시대 (HEPI 2025 설문)

HEPI(Higher Education Policy Institute)가 2025년에 영국 풀타임 학부생 1,041명을 대상으로 실시한 설문에 따르면, **92%가 AI 도구를 사용**하고 있습니다(2024년 66%에서 급등). 평가 과제에 AI를 사용하는 비율은 53%에서 **88%**로 급증했습니다.

주요 사용 목적은 **시간 절약(51%)**과 **과제 품질 향상(50%)**이었습니다. 개념 설명(58%), 기사 요약(48%), 연구 아이디어 제안(41%) 등에 AI를 활용하고 있으며, PPT·발표자료 제작도 이 흐름의 일부입니다.

> 📌 **시사점**: "AI를 쓸까 말까"는 이미 지난 질문입니다. "AI를 **어떻게 잘 쓸까**"가 핵심입니다.
> — 출처: HEPI Student Generative AI Survey 2025

### 사례 2: Gamma로 3분 만에 10페이지 — 무료로 어디까지?

복수의 리뷰 사이트(24slides, GamsGo, Max-Productive)가 Gamma의 성능을 교차 검증한 결과, **프롬프트 입력 후 3분 이내에 10페이지 슬라이드 초안이 생성**되는 것으로 확인되었습니다. 무료 플랜(400크레딧)으로도 약 40회 생성이 가능하며, PDF 업로드 → 슬라이드 자동 변환, 실시간 팀 협업 기능도 지원합니다.

다만, **PPTX 내보내기 시 레이아웃이 깨지는 문제**가 보고되었고, 세밀한 디자인 커스터마이징에는 한계가 있습니다. 2025년 9월에 출시된 **Gamma Agent 3.0**으로 자연어 편집이 가능해져 사용성이 크게 개선되었습니다.

> 📌 **교재 핵심**: 무료로 시작하되, PPT 제출이 필요한 경우 **내보내기 후 반드시 확인**하세요.

### 사례 3: 미국 대학 공식 AI PPT 워크숍

UTHSC(University of Tennessee Health Science Center)는 교수학습센터에서 **"AI in 30"** 시리즈를 운영하며, Gamma를 활용한 30분 워크숍을 공식 프로그램으로 채택했습니다. 교수와 학생 모두를 대상으로, 강의 준비·학생 과제·연구 발표 세 가지 맥락에서 AI PPT 활용법을 교육하고 있습니다.

> 📌 **시사점**: AI PPT는 이미 대학 공식 교육 프로그램의 일부입니다.
> — 출처: UTHSC Teaching and Learning Center (2025)

### 사례 4 (Before/After): ChatGPT+Gamma 2단계 워크플로우의 위력

**Before** (AI 없이):
- PPT 만드는 과정: 자료 조사(1시간) → 개요 작성(30분) → 슬라이드 디자인(2시간) → 스크립트(1시간) = **4시간 30분**
- 디자인에서 가장 많은 시간 소요. 정작 내용 준비 시간은 부족.

**After** (ChatGPT+Gamma):
- ChatGPT 기획(10분) → Gamma 시각화(10분) → 검수·수정(15분) → 스크립트(10분) = **45분**
- 발표 시간 절반 이상을 **리허설**에 투자 가능.

> 📊 시간 절약: **4시간 30분 → 45분 (약 83% 단축)**

## 7. 꿀팁 & 주의사항

### 전문가 팁

**팁 1: "기획은 AI, 디자인은 AI, 스토리는 나"**

AI가 구조와 디자인을 빠르게 만들어주지만, 발표의 **차별화 포인트**는 여전히 인간의 몫입니다. "이 데이터가 내가 조사한 결과와 어떻게 연결되는지", "내가 이 주제에 관심을 갖게 된 개인적 경험" 같은 요소는 AI가 넣어줄 수 없습니다.

**팁 2: Gamma Free → Canva Free 릴레이**

Gamma에서 빠르게 초안을 만든 후, 디자인을 더 다듬고 싶으면 **Canva**로 옮기세요. Canva는 교육용 무료 계정(학교 이메일로 가입)으로 Pro 기능을 쓸 수 있는 경우가 많습니다. PPT 내보내기 호환성도 Canva가 더 좋습니다.

**팁 3: "1:1:1 법칙"으로 검수하기**

AI가 생성한 슬라이드를 검수할 때 이 기준을 적용하세요:
- **1슬라이드** = **1메시지** = **1분 발표**
- 이 기준에서 벗어나는 슬라이드가 있으면 수정합니다.

### ❌ Anti-Pattern 1: "Wall of Words" — 텍스트 폭탄 슬라이드

**상황**: AI PPT 도구에 주제를 입력했더니 슬라이드마다 텍스트가 빼곡합니다. "내용이 충실하니 좋겠지"라고 생각하고 그대로 사용합니다.

**잘못된 방법**:
```
디지털 마케팅 트렌드에 대해 10슬라이드 PPT 만들어줘.
내용을 상세하게 넣어줘.
```

**결과**: 슬라이드당 300자 이상의 텍스트. 청중은 슬라이드를 읽느라 발표자의 말을 듣지 않습니다. 4개 이상의 출처에서 이 문제를 "AI PPT의 가장 흔한 실수 1위"로 꼽았습니다. (SmallPPT, SlidesAI, SlideSpeek, Insight7 — 교차검증)

**올바른 방법**:
```
디지털 마케팅 트렌드에 대해 10슬라이드 PPT 만들어줘.
각 슬라이드는 핵심 키워드 3~5개와 간단한 설명 1문장만 포함해줘.
상세 내용은 발표자 노트에 넣어줘.
```

**차이**: "상세하게"를 요청하면 AI는 텍스트를 늘립니다. "키워드 + 발표자 노트"로 분리하면 **청중은 핵심만, 발표자는 상세 내용을 참고**할 수 있습니다.

### ❌ Anti-Pattern 2: "평균의 함정" — AI PPT로 좋은 점수를 못 받는 이유

**상황**: Gamma로 5분 만에 깔끔한 PPT를 만들었습니다. 디자인도 좋고 구조도 탄탄해 보여서 자신감 있게 발표했습니다.

**잘못된 방법**:
```
[Gamma에서 프롬프트 입력 → 생성 → 그대로 발표]
```

**결과**: 교수 피드백: "이 PPT는 인터넷 어디서나 볼 수 있는 내용입니다. 본인만의 관점이 없습니다." AI가 생성한 PPT는 **"평균 수준의 완성도"**를 빠르게 제공하지만, 그것은 누구나 5분이면 만들 수 있다는 뜻입니다.

**올바른 방법**:
```
1. AI로 초안 생성 (5분)
2. 본인의 관점·분석·경험 추가 (30분)
   - "나는 이 데이터를 보고 ~라고 생각한다"
   - "우리 학교/지역의 사례를 조사해본 결과..."
   - "기존 연구와 다른 점은..."
3. AI로 디자인 다듬기 (10분)
```

**차이**: AI가 만든 "80점짜리 초안"에 인간만의 분석을 더해 **"95점짜리 발표자료"**로 만드는 것이 핵심입니다. AI는 출발점이지, 도착점이 아닙니다.

### 상황별 도구 선택 가이드

| 상황 | 추천 도구 | 이유 |
|------|----------|------|
| 내일 수업 발표 (빠른 제작) | ChatGPT + Gamma Free | 무료, 30분 완성, 웹 링크 공유 |
| 팀 발표 (PPT 파일 제출) | ChatGPT + Canva | 팀 공동 편집 + PPT 호환 양호 |
| 학술 발표 (학회, 졸업발표) | ChatGPT + Beautiful.ai | 깔끔한 디자인, 정확한 PPT 내보내기 |
| Google Slides 사용 환경 | Gemini + SlidesAI | Google 생태계 네이티브 통합 |
| 포트폴리오/웹 공유 | Gamma Pro | 웹사이트형 프레젠테이션, 인터랙티브 |

### 🔄 이 워크플로우를 다른 상황에 적용하기

이 챕터에서 배운 **"AI 기획 → AI 시각화 → 인간 검수"** 패턴은 다음 상황에도 적용할 수 있습니다:

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 수업 발표 PPT | 발표 주제 → 면접 자기소개 | Ch 14 면접 발표 슬라이드 |
| 10슬라이드 구성 | 슬라이드 → 포스터 섹션 | Ch 8 행사 포스터 레이아웃 |
| Q&A 예상 질문 | 교수 질문 → 면접관 질문 | Ch 14 면접 Q&A 대비 |
| 디자인 체크리스트 | PPT → 보고서 서식 | Ch 3 보고서 최종 점검 |

**핵심 패턴**: **"AI에게 구조를 맡기고, AI에게 시각화를 맡기고, 인간이 스토리와 검수를 맡는다"** — 이 3단계 분업은 PPT뿐 아니라 모든 콘텐츠 제작에 적용됩니다.

## 1. 왜 마크다운으로 PPT를 만드는가?

### 1-1. 기존 발표자료 제작의 문제점

| 기존 방식 | 문제점 |
|----------|--------|
| PPT에서 직접 디자인 | 내용보다 디자인에 시간 소모 (본문의 "4시간 30분→45분" 비교 참조) |
| Gamma 등 AI PPT 도구 | 빠르지만 PPTX 내보내기 시 레이아웃 깨짐, 세밀한 수정 제한적 |
| 템플릿 활용 | 내용에 맞게 커스터마이즈 번거로움 |

### 1-2. Marp — 마크다운 기반 프레젠테이션 도구

**Marp**(Markdown Presentation Ecosystem)는 마크다운 문서를 슬라이드로 변환하는 도구입니다:

```markdown
---
marp: true
theme: default
paginate: true
---

# 첫 번째 슬라이드

발표 내용을 **마크다운**으로 작성합니다.

---

# 두 번째 슬라이드

- 글머리 기호도 자동 정렬
- 코드 블록, 표, 이미지 모두 지원
```

핵심 장점:
- **내용에 집중**: 디자인은 테마가 처리, 발표자는 내용만 작성
- **버전 관리**: 텍스트 파일이므로 수정 이력 추적 가능
- **자동화 가능**: Claude Code가 마크다운을 생성 → Marp가 PDF/HTML로 변환
- **일관성**: 같은 테마를 적용하면 모든 슬라이드가 통일된 디자인

### 1-3. Claude Code + Marp 워크플로우

```
[보고서/자료] → Claude Code(내용 구조화) → Marp 마크다운 → PDF/HTML 슬라이드
```

Claude Code가 기존 보고서나 노트를 분석하여 Marp 형식의 마크다운을 자동 생성하고, Marp CLI가 이를 PDF나 HTML 슬라이드로 변환합니다.

## 출력
- _workspace/slide_outline.md: 슬라이드 구성안

## 협업
- 완료 후 content-extractor에게 전달합니다.
