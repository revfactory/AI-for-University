## 2. 이 챕터에서 배우는 것

### 학습목표

이 챕터를 마치면 다음을 할 수 있습니다

1. **AI PPT 도구**(Gamma, Canva AI, ChatGPT, Gemini)의 특성을 비교하고 상황에 맞는 도구를 선택할 수 있다
2. AI에게 **효과적인 PPT 프롬프트**를 작성하여 원하는 구조와 디자인의 슬라이드를 생성할 수 있다
3. **ChatGPT로 발표 구성을 기획**하고, **Gamma로 시각화**하는 2단계 워크플로우를 수행할 수 있다
4. AI가 생성한 슬라이드를 **디자인 원칙에 따라 검수·수정**할 수 있다
5. **발표 스크립트와 Q&A 예상 질문**을 AI로 준비할 수 있다

### 완성 후 얻는 결과물

> 이 챕터의 실습을 마치면, 수업 발표 주제를 입력한 지 **15분 만에 완성된 10슬라이드 발표자료 + 슬라이드별 발표 스크립트 + 예상 Q&A 5문항**을 갖게 됩니다.

### Gamma 심층 안내

Gamma는 대학생이 가장 많이 쓰는 AI PPT 도구입니다. 핵심 특징을 정리합니다.

**주요 기능:**
- **프롬프트 → 슬라이드**: 텍스트를 입력하면 60초 내에 10페이지 슬라이드 초안을 생성합니다
- **문서 업로드**: PDF, Word 파일을 업로드하면 자동으로 슬라이드로 변환합니다
- **Gamma Agent 3.0** (2025.09~): "3번 슬라이드에 차트 추가해줘"처럼 자연어로 수정을 지시할 수 있습니다
- **다중 출력**: 프레젠테이션, 문서, 웹사이트 3가지 형태로 출력 가능
- **실시간 협업**: 팀원과 동시 편집 가능

**가격 구조:**

| 플랜 | 가격 | AI 크레딧 | 핵심 차이 |
|------|------|----------|----------|
| Free | $0 | 400 | Gamma 워터마크 표시 |
| Plus | $8/월(연간) | 무제한 | 워터마크 제거, 커스텀 폰트 |

> 💡 무료 플랜의 400크레딧이면 약 40개의 프레젠테이션을 만들 수 있습니다. 한 학기 발표에 충분한 양입니다.

**Gamma의 한계:**
- PPT(PPTX) 파일로 내보낼 때 레이아웃이 깨질 수 있습니다
- 세밀한 디자인 커스터마이징은 제한적입니다
- 교수님이 PPT 파일 제출을 요구하면, **Canva로 마무리하는 것이 안전**합니다

## 4. 핵심 개념

### 발표자료의 3요소: 구조 + 디자인 + 스토리

AI PPT 도구를 잘 활용하려면, 좋은 발표자료의 구성 요소를 먼저 이해해야 합니다.

**1. 구조(Structure)** — 무엇을 어떤 순서로 말할 것인가
- 도입(Hook) → 문제 제기 → 분석 → 해결안 → 결론 → Q&A
- AI가 가장 잘 하는 부분입니다. ChatGPT에게 주제만 주면 논리적 구조를 즉시 생성합니다.

**2. 디자인(Design)** — 시각적으로 어떻게 보여줄 것인가
- 한 슬라이드 한 메시지, 6×6 규칙(6줄, 줄당 6단어 이하), 고대비 색상
- AI PPT 도구(Gamma, Canva)가 자동으로 처리합니다. 단, **기본 템플릿을 그대로 쓰면 안 됩니다.**

**3. 스토리(Story)** — 청중의 마음을 어떻게 움직일 것인가
- 오프닝 멘트, 전환 연결, 클로징 — 이것은 **인간이 직접 다듬어야** 합니다.
- AI는 초안을 생성하지만, 본인만의 경험과 관점을 넣는 것은 발표자의 몫입니다.

### AI PPT 디자인 체크리스트

발표자료를 AI로 만든 후 **반드시 확인**해야 할 항목입니다

```
✅ 한 슬라이드에 핵심 메시지가 하나만 있는가?
✅ 텍스트가 6줄 이하인가? (줄당 6단어 이하)
✅ 색상이 일관되고, 배경과 텍스트가 고대비인가?
✅ 폰트가 제목 1종 + 본문 1종으로 통일되어 있는가?
✅ 여백이 충분한가? (슬라이드의 30% 이상 여백)
✅ 이미지/아이콘이 내용과 관련 있는가?
❌ 워드아트, 클립아트, 과도한 애니메이션은 없는가?
❌ AI 기본 템플릿을 그대로 사용하지 않았는가?
```


### 🔬 프롬프트 진화 실험

같은 작업을 3단계 프롬프트로 시도하여, 프롬프트 품질의 차이를 직접 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
AI 관련 발표자료 만들어줘
```

**결과**: "AI의 정의", "AI의 역사", "AI의 활용"… 교과서 복사 같은 일반적 내용. 청중도 시간도 모르니 분량 조절 불가. 디자인 지시가 없어서 기본 템플릿 그대로.

📋 **평가**
- 정확성: ★★☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 주제가 넓고, 맥락(청중, 시간, 목적)이 없어서 어디서나 볼 수 있는 내용

---

#### 🟡 Level 2 — 중급 프롬프트

```
"대학생의 AI 활용 현황" 주제로 10슬라이드 발표자료를 만들어줘.
```

**결과**: 구조가 생겼습니다(표지, 목차, 현황, 사례, 결론). 하지만 청중이 교수인지 동기인지에 따라 어조가 달라야 하고, 15분인지 5분인지에 따라 슬라이드 밀도가 달라져야 합니다. 디자인 스타일 지시가 없어 AI가 임의 선택.

📋 **평가**
- 정확성: ★★★☆☆
- 활용도: ★★★☆☆
- 개선점: 슬라이드 수와 주제가 명확해짐
- 아쉬운 점: 청중, 시간, 디자인 맥락 부재

---

#### 🟢 Level 3 — 전문가 프롬프트

```
주제: "대학생의 AI 활용 현황과 미래"
목적: 교양수업 기말 팀 발표
청중: 대학생 40명 + 교수님
시간: 15분

슬라이드 구성:
1. 표지 (주제, 팀명, 날짜)
2. 목차
3. AI 도구 사용 현황 (통계 차트 — HEPI 2025 설문 인용)
4. 주요 활용 사례 ① 과제·보고서 작성
5. 주요 활용 사례 ② 논문 검색·자료 정리
6. 주요 활용 사례 ③ 발표자료 제작
7. 문제점과 윤리적 이슈
8. 해결 방안 제안
9. 결론 및 시사점
10. Q&A / 참고문헌

디자인 요청:
- 색상: 네이비+화이트 배경 (학술적)
- 각 슬라이드에 관련 아이콘 또는 인포그래픽 포함
- 텍스트 최소화, 키워드 + 시각자료 중심
```

**결과**: 정확히 요청한 구조대로 10슬라이드가 생성됩니다. 통계 차트 슬라이드에는 데이터 시각화가 포함되고, 디자인도 네이비+화이트로 통일됩니다. 청중과 시간에 맞춘 적절한 텍스트 밀도.

📋 **평가**
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: **슬라이드별 구조 지정 + 디자인 요청 + 데이터 출처 명시**가 결정적

---

#### 💡 교훈: PPT 프롬프트의 마법

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 주제 명시 | △ (너무 넓음) | ✓ | ✓ |
| 슬라이드 수 | ✗ | ✓ | ✓ |
| 청중/시간 | ✗ | ✗ | ✓ |
| 슬라이드별 구성 | ✗ | ✗ | ✓ |
| 디자인 지시 | ✗ | ✗ | ✓ |
| 데이터 출처 | ✗ | ✗ | ✓ |
| 결과 품질 | 20% | 55% | 95% |

> 📌 **핵심 원칙**: PPT 프롬프트는 "슬라이드별 구조를 직접 설계"하는 것이 가장 큰 차이를 만듭니다. AI는 디자인과 콘텐츠를 채워주지만, **전체 흐름(스토리보드)은 인간이 설계해야** 합니다.

## 5. 따라하기 실습

### 🎯 실습 A: ChatGPT로 발표 기획하기 ⭐

**목표**: ChatGPT를 활용하여 발표 구성(아웃라인)과 슬라이드별 핵심 메시지를 설계합니다.
**소요 시간**: 10분
**준비물**: ChatGPT 계정 (Free 가능)
**사용 도구**: ChatGPT

---

#### Step 1. ChatGPT에 접속하기 ⏱️ 1분

ChatGPT(https://chat.openai.com)에 접속하여 로그인합니다.

> 📍 **화면 안내**: 화면 하단의 메시지 입력창을 클릭합니다.

✅ **체크**: 메시지 입력창이 활성화되어 커서가 깜빡이면 준비 완료.

---

#### Step 2. 발표 구성 프롬프트 입력 ⏱️ 3분

다음 프롬프트를 **그대로 복사**하여 붙여넣으세요

```
나는 [학과] [학년] 학생입니다.
[수업명] 수업에서 "[발표 주제]"라는 주제로 발표해야 합니다.

발표 조건:
- 청중: 같은 수업 학생 [N]명 + 교수님
- 시간: [N]분 발표 + [N]분 Q&A
- 슬라이드: [N]장
- 분위기: [학술적 / 캐주얼 / 설득적]

다음을 만들어주세요:
1. 슬라이드별 구성 (제목 + 핵심 메시지 1~2문장)
2. 각 슬라이드에 넣으면 좋을 시각자료 유형 (차트/이미지/아이콘/인용)
3. 전체 발표의 스토리라인 (도입→전개→결론의 논리적 흐름 설명)
```

> 💡 **커스텀 포인트**: `[대괄호]` 안의 내용은 본인 상황에 맞게 수정하세요.
> 예시: "나는 경영학과 3학년 학생입니다. 마케팅원론 수업에서 'MZ세대의 소비 트렌드 변화'라는 주제로 발표해야 합니다."

✅ **체크**: AI가 슬라이드별 구성을 1번부터 N번까지 번호를 매겨 응답했으면 성공.

---

#### Step 3. 결과 검토 및 수정 ⏱️ 3분

AI의 응답에서 다음을 확인합니다

1. **논리적 흐름**: 도입 → 문제 제기 → 분석 → 해결 → 결론 순서가 자연스러운가?
2. **슬라이드 밀도**: 15분 발표라면 슬라이드당 약 1.5분. 10장이면 적절합니다.
3. **시각자료 제안**: 텍스트만 가득한 슬라이드는 없는지 확인합니다.

수정이 필요하면 추가 프롬프트를 입력합니다

```
3번 슬라이드의 내용이 너무 넓어요. "국내 사례"로 한정해서 수정해줘.
그리고 7번 슬라이드 뒤에 "해외 사례 비교" 슬라이드를 추가해줘.
```

> 🔧 **안 되나요?**: AI가 너무 일반적인 내용을 생성했다면, "구체적인 통계 데이터나 연구 결과를 인용해서 다시 작성해줘"라고 추가 요청하세요.

✅ **체크**: 본인이 만족하는 슬라이드별 구성이 완성되었으면 이 결과를 **복사**해 둡니다.

---

#### Step 4. 발표 스크립트 생성 ⏱️ 3분

같은 대화에서 이어서 다음 프롬프트를 입력합니다

```
위 구성을 바탕으로 슬라이드별 발표 스크립트를 작성해줘.

조건:
1. 각 슬라이드당 1~2분 분량의 구어체 스크립트
2. 청중 시선을 잡는 오프닝 멘트 포함
3. 슬라이드 전환 연결 문구 포함 ("이어서~", "다음으로~")
4. 예상 질문 5개 + 답변 전략 (STAR 기법 적용)
5. 클로징 멘트 (핵심 메시지 3줄 요약)
```

✅ **체크**: 슬라이드 1번부터 마지막까지 구어체 스크립트가 생성되고, 마지막에 예상 질문 5개가 포함되어 있으면 성공.

### 🎯 실습 B: Gamma로 슬라이드 시각화하기 ⭐

**목표**: 실습 A에서 만든 구성을 Gamma에 입력하여 시각적인 슬라이드를 생성합니다.
**소요 시간**: 10분
**준비물**: Gamma 계정 (Free — gamma.app에서 Google/이메일로 가입)
**사용 도구**: Gamma

---

#### Step 1. Gamma에 접속하고 새 프레젠테이션 만들기 ⏱️ 2분

Gamma(https://gamma.app)에 접속하여 로그인합니다.

> 📍 **화면 안내**: 로그인 후 화면 상단의 **"Create new"** 또는 **"새로 만들기"** 버튼을 클릭합니다. "Presentation"(프레젠테이션)을 선택합니다.

✅ **체크**: "프롬프트를 입력하세요" 또는 "Paste in text" 입력창이 나타나면 준비 완료.

#### Step 2. ChatGPT 결과를 Gamma에 붙여넣기 ⏱️ 2분

실습 A의 Step 3에서 복사한 **슬라이드별 구성**을 Gamma 입력창에 **그대로 붙여넣기**합니다.

> 📍 **화면 안내**: "Paste in text" 옵션을 선택한 뒤, ChatGPT에서 복사한 내용을 Ctrl+V(Mac: Cmd+V)로 붙여넣습니다.

> 💡 **팁**: Gamma에 직접 프롬프트를 입력하는 것보다, ChatGPT에서 구조를 먼저 설계하고 붙여넣는 것이 **훨씬 정교한 결과**를 얻습니다. ChatGPT는 구조와 논리에 강하고, Gamma는 시각화에 강하기 때문입니다.

✅ **체크**: Gamma가 "Generate" 또는 "생성" 버튼을 표시하면 다음 단계로 진행.

---

#### Step 3. 테마 선택 및 생성 ⏱️ 2분

1. Gamma가 테마 옵션을 제시합니다. **학술적 분위기**에 맞는 테마를 선택합니다 (보통 "Professional" 또는 "Minimal" 계열).
2. **"Generate"** 버튼을 클릭합니다.
3. 약 30~60초 후 슬라이드가 생성됩니다.

> 📍 **화면 안내**: 생성이 완료되면 왼쪽에 슬라이드 썸네일 목록이, 오른쪽에 현재 선택된 슬라이드가 표시됩니다.

✅ **체크**: 슬라이드가 10장 내외로 생성되고, 각 슬라이드에 제목과 내용이 들어 있으면 성공.

---

#### Step 4. Gamma Agent로 수정하기 ⏱️ 3분

생성된 슬라이드가 마음에 안 드는 부분이 있다면, **Gamma Agent**를 활용합니다.

Gamma 화면 하단의 AI 채팅창에 다음과 같이 입력합니다

```
3번 카드의 텍스트를 줄이고 핵심 키워드 3개만 남겨줘.
5번 카드에 비교표를 추가해줘.
전체 색상을 네이비+화이트 톤으로 변경해줘.
```

> 💡 **팁**: Gamma Agent 3.0은 자연어로 편집을 지시할 수 있습니다. "이 슬라이드를 더 깔끔하게 만들어줘"처럼 막연한 요청보다, **"텍스트를 3줄로 줄여줘"**처럼 구체적으로 요청하세요.

> 🔧 **안 되나요?**: Agent 기능이 보이지 않으면, Gamma 화면 하단의 AI 아이콘(✨)을 클릭하세요. Free 플랜에서도 기본 편집은 가능합니다.

---

#### Step 5. 내보내기 및 발표 준비 ⏱️ 1분

완성된 슬라이드를 내보냅니다.

> 📍 **화면 안내**: 화면 우상단 **"Share"** 또는 **"Export"** 버튼 클릭 → 원하는 형식 선택

- **웹 공유** (기본 추천): 링크를 복사하여 교수님이나 팀원에게 공유
- **PDF 내보내기**: 인쇄용으로 가장 안정적
- **PPTX 내보내기**: ⚠️ 레이아웃이 깨질 수 있으므로 반드시 다운로드 후 확인

> ⚠️ **주의**: PPT 파일 제출이 필수라면, PPTX로 내보낸 후 PowerPoint에서 열어 레이아웃을 확인하세요. 깨진 부분은 수동으로 수정합니다. 또는 **Canva에서 처음부터 만드는 것**이 PPT 호환성이 더 좋습니다.

✅ **체크**: 내보낸 파일을 열어보았을 때 레이아웃이 정상이면 완성.

주요 사용 목적은 **시간 절약(51%)**과 **과제 품질 향상(50%)**이었습니다. 개념 설명(58%), 기사 요약(48%), 연구 아이디어 제안(41%) 등에 AI를 활용하고 있으며, PPT·발표자료 제작도 이 흐름의 일부입니다.

> 📌 **시사점**: "AI를 쓸까 말까"는 이미 지난 질문입니다. "AI를 **어떻게 잘 쓸까**"가 핵심입니다.

### 사례 2: Gamma로 3분 만에 10페이지 — 무료로 어디까지?

복수의 리뷰 사이트(24slides, GamsGo, Max-Productive)가 Gamma의 성능을 교차 검증한 결과, **프롬프트 입력 후 3분 이내에 10페이지 슬라이드 초안이 생성**되는 것으로 확인되었습니다. 무료 플랜(400크레딧)으로도 약 40회 생성이 가능하며, PDF 업로드 → 슬라이드 자동 변환, 실시간 팀 협업 기능도 지원합니다.

다만, **PPTX 내보내기 시 레이아웃이 깨지는 문제**가 보고되었고, 세밀한 디자인 커스터마이징에는 한계가 있습니다. 2025년 9월에 출시된 **Gamma Agent 3.0**으로 자연어 편집이 가능해져 사용성이 크게 개선되었습니다.

> 📌 **교재 핵심**: 무료로 시작하되, PPT 제출이 필요한 경우 **내보내기 후 반드시 확인**하세요.

UTHSC(University of Tennessee Health Science Center)는 교수학습센터에서 **"AI in 30"** 시리즈를 운영하며, Gamma를 활용한 30분 워크숍을 공식 프로그램으로 채택했습니다. 교수와 학생 모두를 대상으로, 강의 준비·학생 과제·연구 발표 세 가지 맥락에서 AI PPT 활용법을 교육하고 있습니다.

> 📌 **시사점**: AI PPT는 이미 대학 공식 교육 프로그램의 일부입니다.
> — 출처: UTHSC Teaching and Learning Center (2025)

### 사례 4 (Before/After): ChatGPT+Gamma 2단계 워크플로우의 위력

**Before** (AI 없이)
- PPT 만드는 과정: 자료 조사(1시간) → 개요 작성(30분) → 슬라이드 디자인(2시간) → 스크립트(1시간) = **4시간 30분**

**After** (ChatGPT+Gamma)
- ChatGPT 기획(10분) → Gamma 시각화(10분) → 검수·수정(15분) → 스크립트(10분) = **45분**
- 발표 시간 절반 이상을 **리허설**에 투자 가능.

> 📊 시간 절약: **4시간 30분 → 45분 (약 83% 단축)**

## 7. 꿀팁 & 주의사항

### 전문가 팁

**팁 1: "기획은 AI, 디자인은 AI, 스토리는 나"**

AI가 구조와 디자인을 빠르게 만들어주지만, 발표의 **차별화 포인트**는 여전히 인간의 몫입니다. "이 데이터가 내가 조사한 결과와 어떻게 연결되는지", "내가 이 주제에 관심을 갖게 된 개인적 경험" 같은 요소는 AI가 넣어줄 수 없습니다.

**팁 2: Gamma Free → Canva Free 릴레이**

Gamma에서 빠르게 초안을 만든 후, 디자인을 더 다듬고 싶으면 **Canva**로 옮기세요. Canva는 교육용 무료 계정(학교 이메일로 가입)으로 Pro 기능을 쓸 수 있는 경우가 많습니다. PPT 내보내기 호환성도 Canva가 더 좋습니다.

**팁 3: "1:1:1 법칙"으로 검수하기**

AI가 생성한 슬라이드를 검수할 때 이 기준을 적용하세요
- **1슬라이드** = **1메시지** = **1분 발표**
- 이 기준에서 벗어나는 슬라이드가 있으면 수정합니다.

### ❌ Anti-Pattern 1: "Wall of Words" — 텍스트 폭탄 슬라이드

**상황**: AI PPT 도구에 주제를 입력했더니 슬라이드마다 텍스트가 빼곡합니다. "내용이 충실하니 좋겠지"라고 생각하고 그대로 사용합니다.

**잘못된 방법**
```
디지털 마케팅 트렌드에 대해 10슬라이드 PPT 만들어줘.
내용을 상세하게 넣어줘.
```

**결과**: 슬라이드당 300자 이상의 텍스트. 청중은 슬라이드를 읽느라 발표자의 말을 듣지 않습니다. 4개 이상의 출처에서 이 문제를 "AI PPT의 가장 흔한 실수 1위"로 꼽았습니다. (SmallPPT, SlidesAI, SlideSpeek, Insight7 — 교차검증)

**올바른 방법**
```
디지털 마케팅 트렌드에 대해 10슬라이드 PPT 만들어줘.
각 슬라이드는 핵심 키워드 3~5개와 간단한 설명 1문장만 포함해줘.
상세 내용은 발표자 노트에 넣어줘.
```

**차이**: "상세하게"를 요청하면 AI는 텍스트를 늘립니다. "키워드 + 발표자 노트"로 분리하면 **청중은 핵심만, 발표자는 상세 내용을 참고**할 수 있습니다.

### ❌ Anti-Pattern 2: "평균의 함정" — AI PPT로 좋은 점수를 못 받는 이유

**상황**: Gamma로 5분 만에 깔끔한 PPT를 만들었습니다. 디자인도 좋고 구조도 탄탄해 보여서 자신감 있게 발표했습니다.

**잘못된 방법**
```
[Gamma에서 프롬프트 입력 → 생성 → 그대로 발표]
```

**결과**: 교수 피드백: "이 PPT는 인터넷 어디서나 볼 수 있는 내용입니다. 본인만의 관점이 없습니다." AI가 생성한 PPT는 **"평균 수준의 완성도"**를 빠르게 제공하지만, 그것은 누구나 5분이면 만들 수 있다는 뜻입니다.

**올바른 방법**
```
1. AI로 초안 생성 (5분)
2. 본인의 관점·분석·경험 추가 (30분)
   - "나는 이 데이터를 보고 ~라고 생각한다"
   - "우리 학교/지역의 사례를 조사해본 결과..."
   - "기존 연구와 다른 점은..."
3. AI로 디자인 다듬기 (10분)
```

**차이**: AI가 만든 "80점짜리 초안"에 인간만의 분석을 더해 **"95점짜리 발표자료"**로 만드는 것이 핵심입니다. AI는 출발점이지, 도착점이 아닙니다.

### 상황별 도구 선택 가이드

| 상황 | 추천 도구 | 이유 |
|------|----------|------|
| 내일 수업 발표 (빠른 제작) | ChatGPT + Gamma Free | 무료, 30분 완성, 웹 링크 공유 |
| 팀 발표 (PPT 파일 제출) | ChatGPT + Canva | 팀 공동 편집 + PPT 호환 양호 |
| 학술 발표 (학회, 졸업발표) | ChatGPT + Beautiful.ai | 깔끔한 디자인, 정확한 PPT 내보내기 |
| Google Slides 사용 환경 | Gemini + SlidesAI | Google 생태계 네이티브 통합 |
| 포트폴리오/웹 공유 | Gamma Pro | 웹사이트형 프레젠테이션, 인터랙티브 |

### 🔄 이 워크플로우를 다른 상황에 적용하기

이 챕터에서 배운 **"AI 기획 → AI 시각화 → 인간 검수"** 패턴은 다음 상황에도 적용할 수 있습니다

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 수업 발표 PPT | 발표 주제 → 면접 자기소개 | Ch 14 면접 발표 슬라이드 |
| 10슬라이드 구성 | 슬라이드 → 포스터 섹션 | Ch 8 행사 포스터 레이아웃 |
| Q&A 예상 질문 | 교수 질문 → 면접관 질문 | Ch 14 면접 Q&A 대비 |
| 디자인 체크리스트 | PPT → 보고서 서식 | Ch 3 보고서 최종 점검 |

**핵심 패턴**: **"AI에게 구조를 맡기고, AI에게 시각화를 맡기고, 인간이 스토리와 검수를 맡는다"** — 이 3단계 분업은 PPT뿐 아니라 모든 콘텐츠 제작에 적용됩니다.

## 1. 왜 마크다운으로 PPT를 만드는가?

### 1-1. 기존 발표자료 제작의 문제점

| 기존 방식 | 문제점 |
|----------|--------|
| PPT에서 직접 디자인 | 내용보다 디자인에 시간 소모 (본문의 "4시간 30분→45분" 비교 참조) |
| Gamma 등 AI PPT 도구 | 빠르지만 PPTX 내보내기 시 레이아웃 깨짐, 세밀한 수정 제한적 |
| 템플릿 활용 | 내용에 맞게 커스터마이즈 번거로움 |

### 1-2. Marp — 마크다운 기반 프레젠테이션 도구

**Marp**(Markdown Presentation Ecosystem)는 마크다운 문서를 슬라이드로 변환하는 도구입니다

```markdown
---
marp: true
theme: default
paginate: true
---

# 첫 번째 슬라이드

발표 내용을 **마크다운**으로 작성합니다.

---

# 두 번째 슬라이드

- 글머리 기호도 자동 정렬
- 코드 블록, 표, 이미지 모두 지원
```

핵심 장점
- **내용에 집중**: 디자인은 테마가 처리, 발표자는 내용만 작성
- **버전 관리**: 텍스트 파일이므로 수정 이력 추적 가능
- **자동화 가능**: Claude Code가 마크다운을 생성 → Marp가 PDF/HTML로 변환
- **일관성**: 같은 테마를 적용하면 모든 슬라이드가 통일된 디자인

### 1-3. Claude Code + Marp 워크플로우

```
[보고서/자료] → Claude Code(내용 구조화) → Marp 마크다운 → PDF/HTML 슬라이드
```

Claude Code가 기존 보고서나 노트를 분석하여 Marp 형식의 마크다운을 자동 생성하고, Marp CLI가 이를 PDF나 HTML 슬라이드로 변환합니다.

**slide-planner.md**
```markdown
---
name: slide-planner
description: "발표 목적과 청중에 맞춰 슬라이드 구성을 기획하는 에이전트"
---
# Slide Planner — 슬라이드 기획자

## 핵심 역할
- 발표 목적, 청중, 시간에 맞는 슬라이드 구성안을 설계합니다.
- 각 슬라이드의 핵심 메시지와 시각 자료 유형을 결정합니다.
- 스토리라인(도입→전개→결론)을 설계합니다.

## 작업 원칙
- 본문의 **1:1:1 법칙** 준수: 1슬라이드 = 1메시지 = 1분 발표
- 발표 시간 기준: 슬라이드 장수 = 발표시간(분) × 0.8
- 텍스트 최소화, 키워드 중심으로 구성
- 데이터는 표/차트로, 프로세스는 다이어그램으로 표현 제안

## 출력
- workspace/slide_outline.md: 슬라이드 구성안

## 협업
- 완료 후 content-extractor에게 전달합니다.
```

name: marp-writer

---
# Marp Writer — Marp 마크다운 작성자

## 핵심 역할
- slide-planner의 구성안을 Marp 형식 마크다운으로 변환합니다.

## 4. 핵심 개념

### 이미지 프롬프트의 7요소

AI 이미지 생성에서 프롬프트는 **그림의 설계도**입니다. 다음 7가지 요소를 조합하면 원하는 이미지를 정확하게 얻을 수 있습니다.

| 요소 | 설명 | 예시 |
|------|------|------|
| **1. 주제(Subject)** | 무엇을 그릴 것인가 | "카메라를 든 대학생" |
| **2. 스타일(Style)** | 어떤 화풍/형식으로 | "수채화 스타일", "포토리얼리스틱" |
| **3. 구도(Composition)** | 카메라 앵글과 배치 | "정면 클로즈업", "와이드 앵글" |
| **4. 조명(Lighting)** | 빛의 방향과 분위기 | "자연광", "골든아워 빛" |
| **5. 색감(Color)** | 전체 색상 톤 | "파스텔 핑크 + 화이트" |
| **6. 분위기(Mood)** | 감정적 느낌 | "밝고 활기찬", "차분하고 학술적" |
| **7. 해상도/비율(Format)** | 기술적 사양 | "3:4 비율", "고해상도" |

> 📌 **핵심 원칙**: 요소를 많이 지정할수록 원하는 결과에 가까워집니다. 하지만 너무 많은 지시는 AI를 혼란시킬 수 있으니 **핵심 5가지 이상**을 지정하는 것이 적절합니다.

### AI 이미지 저작권 — 반드시 알아야 할 3가지

AI 이미지는 편리하지만, **저작권 문제**를 모르고 사용하면 위험합니다.

**1. AI 단독 생성물은 저작권 보호를 받지 못합니다**

미국 저작권청(U.S. Copyright Office)의 공식 입장에 따르면, AI가 독자적으로 생성한 이미지는 저작권 보호 대상이 아닙니다. 프롬프트 작성 자체도 "창작"으로 인정되지 않습니다. 단, AI 출력물을 인간이 **편집, 합성, 변형**하여 새로운 창작물을 만들면 저작권 인정 가능성이 있습니다.

**2. "○○ 스타일로 그려줘"는 위험합니다**

IEEE Spectrum의 보도에 따르면, "미야자키 하야오 스타일로 그려줘", "뱅크시 스타일로" 같은 프롬프트는 **시각적 표절(visual plagiarism)**에 해당할 수 있습니다. 특히 공모전 출품이나 출판물에 사용할 때 법적 리스크가 있습니다.

**3. 대학 과제에서는 "AI 생성"을 명시해야 합니다**

여러 대학이 AI 이미지 사용 시 출처를 명시하도록 가이드라인을 발표하고 있습니다. "이 이미지는 DALL-E 3로 생성하였음"과 같이 표기하는 것이 안전합니다.


### 🔬 프롬프트 진화 실험

같은 작업을 3단계 프롬프트로 시도하여, 이미지 프롬프트 품질의 차이를 직접 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
대학교 포스터 만들어줘
```

**결과**: "대학교"와 "포스터"라는 키워드만으로는 AI가 방향을 잡지 못합니다. 일반적인 대학 건물 이미지가 나오거나, 의미 없는 추상적 디자인이 생성됩니다.

📋 **평가**
- 정확성: ★☆☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 주제, 스타일, 색감, 용도 모두 불명확

---

#### 🟡 Level 2 — 중급 프롬프트

```
대학교 봄 축제 포스터를 만들어줘. 벚꽃과 캠퍼스가 있는 밝은 느낌으로.
```

**결과**: 벚꽃과 캠퍼스가 포함된 이미지가 나옵니다. 하지만 텍스트 배치 공간이 없고, 비율도 불명확하며, 실제 포스터로 쓰기에는 아직 부족합니다.

📋 **평가**
- 정확성: ★★★☆☆
- 활용도: ★★☆☆☆
- 개선점: 주제와 분위기가 명확해짐
- 아쉬운 점: 텍스트 배치, 비율, 스타일 지정 부재

---

#### 🟢 Level 3 — 전문가 프롬프트

```
대학교 봄 축제 포스터를 디자인해줘.

내용:
- 상단: "2026 봄 축제" 텍스트 (큰 볼드체)
- 중앙: 벚꽃이 만개한 캠퍼스 일러스트 (수채화 스타일)
- 하단: "4월 15일~17일 | 중앙광장" 텍스트
- 전체 색감: 파스텔 핑크 + 화이트 + 연두색
- 스타일: 한국 대학교 동아리 포스터 느낌, 깔끔하고 현대적
- 비율: 3:4 (인스타그램 최적)
- 텍스트는 반드시 정확하게 렌더링해줘
```

**결과**: 지정한 텍스트가 포함된 포스터 이미지가 생성됩니다. 수채화 스타일의 벚꽃 캠퍼스, 파스텔 색감, 3:4 비율까지 정확히 반영됩니다. DALL-E 3에서 특히 텍스트 렌더링이 정확합니다.

📋 **평가**
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: **텍스트 내용 지정 + 스타일 명시 + 비율 지정 + 색감 지정**이 결정적

---

#### 💡 교훈: 이미지 프롬프트의 마법

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 주제 | △ (너무 넓음) | ✓ | ✓ |
| 스타일 | ✗ | △ (밝은 느낌) | ✓ (수채화) |
| 텍스트 내용 | ✗ | ✗ | ✓ |
| 색감 | ✗ | ✗ | ✓ |
| 비율 | ✗ | ✗ | ✓ |
| 결과 품질 | 15% | 45% | 95% |

> 📌 **핵심 원칙**: 이미지 프롬프트는 **"말이 아니라 설계도를 쓴다"**고 생각하세요. 건축가가 "좋은 집 지어줘"라고 하지 않듯, AI에게도 구체적인 사양을 전달해야 합니다.

스타일: [수채화 / 미니멀 / 팝아트 / 사진 느낌]
색감: [파스텔 / 네이비+골드 / 따뜻한 톤 / 차가운 톤]
분위기: [밝고 활기찬 / 차분하고 학술적 / 힙하고 트렌디한]
비율: 3:4 (인스타그램 최적)

텍스트는 반드시 정확하게 렌더링해줘.
```

> 💡 **커스텀 포인트**: `[대괄호]` 안의 내용을 본인 상황에 맞게 수정하세요.
> 예시: 주제: "2026 사진 동아리 신입부원 모집", 스타일: "미니멀", 색감: "네이비+화이트"

✅ **체크**: ChatGPT가 이미지를 생성하여 화면에 표시하면 성공. 약 10~30초 소요.

## 6. 실전 사례

### 사례 1: Canva for Education — 무료로 전문가 수준 디자인

Canva는 교육용 계정에 대해 **100% 무료** Pro 기능을 제공합니다(K-12 교사/학생 대상). 대학생의 경우 .edu 이메일로 가입하면 Canva Free를 사용할 수 있고, 일부 대학은 **Canva for Campus**를 통해 Pro 기능을 무료로 이용할 수 있습니다.

Pro 기능에는 **Magic Design**(AI 자동 디자인), **Magic Eraser**(배경 제거), 1억 개 이상의 프리미엄 에셋, 1TB 클라우드 스토리지가 포함됩니다. LMS 연동(Google Classroom, Blackboard, Teams)도 지원하여 과제 제출까지 원스톱으로 가능합니다.

> 📌 **실행 방법**: canva.com/education에서 학교 이메일로 신청. 승인 후 즉시 활성화.
> — 출처: Canva 공식 Education 페이지, StyleFactory (2026)

### 사례 2: AI 이미지 저작권 분쟁 — Andersen v. Stability AI

2023년 아티스트 그룹이 Stability AI, Midjourney, DeviantArt를 상대로 집단 소송을 제기했습니다. 핵심 쟁점은 **LAION 데이터셋(50억 개 이미지)**을 스크래핑하여 AI 학습에 사용한 것이 저작권 침해인지 여부입니다.

미국 연방 판사가 기각 신청을 **거부**하고 디스커버리 단계로 진행을 허용했습니다. 이는 AI 이미지 생성의 법적 프레임워크가 아직 확립되지 않았음을 보여줍니다.

> 📌 **학생 시사점**: AI로 만든 이미지를 **공모전, 출판물, 상업적 용도**로 사용할 때는 저작권 리스크를 반드시 고려하세요. 대학 과제용은 대부분 허용되지만, "AI 생성"이라고 명시하는 것이 안전합니다.
> — 출처: NYU Journal of IP & Entertainment Law (2025)

### 사례 3: 미리캔버스 — 한국 대학생의 대안

미리캔버스는 한국 교육 현장에 특화된 디자인 도구입니다. 팬데믹 이후 급격히 채택되었으며, 교사 무료 프리미엄을 제공합니다. **최대 50명 동시 편집**이 가능해 팀플 홍보물 공동 제작에 최적입니다.

한국어 템플릿, 한국 대학 맞춤 레이아웃, 한글 폰트가 풍부하여 **한국어 포스터에는 Canva보다 미리캔버스가 더 자연스러운 결과**를 줍니다.

### 사례 4 (Before/After): 디자인 비전공자의 포스터 제작

**Before** (AI 없이, 디자인 비전공자)
- 무료 이미지 사이트에서 사진 검색(1시간) → 파워포인트로 텍스트 배치(1시간) → 조악한 결과물

## 7. 꿀팁 & 주의사항

### 전문가 팁

**팁 1: "DALL-E로 이미지, Canva로 완성" 릴레이**

DALL-E 3는 이미지 생성에 강하지만, 정확한 텍스트 배치나 로고 삽입은 제한적입니다. AI로 메인 비주얼을 만들고, Canva에서 텍스트·로고·레이아웃을 조합하는 것이 가장 효율적입니다.

**팁 2: 도구별 프롬프트 차이를 기억하세요**

| 도구 | 프롬프트 스타일 | 예시 |
|------|--------------|------|
| DALL-E 3 | 자연어 설명, 한국어 OK | "벚꽃이 만개한 대학 캠퍼스를 수채화 스타일로 그려줘" |
| Midjourney | 영어 키워드 나열 + 파라미터 | "cherry blossom university campus, watercolor --ar 3:4 --v 6.1" |
| Gemini | 자연어, 한국어 OK | "봄 느낌의 대학교 캠퍼스 사진을 만들어줘" |

**팁 3: AI 이미지 저작권 체크리스트**

과제나 홍보물에 AI 이미지를 사용할 때 반드시 확인하세요
- [ ] "특정 아티스트 스타일" 프롬프트를 사용하지 않았는가?
- [ ] "AI 생성 이미지"라고 명시했는가?
- [ ] 공모전/출판물이면 해당 도구의 이용약관을 확인했는가?
- [ ] AI 이미지에 인간의 편집·변형을 추가했는가? (저작권 강화)

### ❌ Anti-Pattern 1: "○○ 스타일로 그려줘" — 시각적 표절의 위험

**상황**: 동아리 홍보물에 유명 아티스트 스타일의 이미지를 넣으면 눈에 띌 것 같아서 "미야자키 하야오 스타일로 그려줘"라고 프롬프트를 입력합니다.

**잘못된 방법**
```
미야자키 하야오 스타일로 대학 캠퍼스를 그려줘.
지브리 애니메이션처럼 따뜻한 색감으로.
```

**결과**: IEEE Spectrum에 따르면, 특정 아티스트의 고유 스타일을 AI로 모방하는 것은 **시각적 표절(visual plagiarism)**에 해당할 수 있습니다. Andersen v. Stability AI 소송에서 이 문제가 핵심 쟁점입니다. 공모전 출품이나 상업적 사용 시 법적 리스크가 있습니다.

**올바른 방법**
```
대학 캠퍼스를 따뜻한 수채화 스타일로 그려줘.
파스텔 톤, 부드러운 선, 자연광 느낌.
(특정 아티스트 이름 없이 스타일만 묘사)
```

**차이**: 특정 아티스트 이름 대신 **스타일 요소를 묘사**(수채화, 파스텔, 자연광)하면 법적 리스크 없이 비슷한 분위기를 얻을 수 있습니다.

### ❌ Anti-Pattern 2: AI 이미지의 편향을 검수하지 않는 실수

**상황**: 동아리 신입부원 모집 포스터에 "대학생 그룹이 캠퍼스에서 활동하는 모습"을 AI로 생성했습니다. 바로 사용합니다.

**잘못된 방법**
```
대학생 그룹이 캠퍼스에서 활동하는 사진을 만들어줘.
[→ 생성 결과를 검수 없이 바로 사용]
```

**결과**: AI가 생성한 "대학생" 이미지는 특정 인종, 성별, 외모에 편향될 수 있습니다. 다양성을 반영하지 못한 포스터는 의도치 않게 **배타적인 인상**을 줄 수 있습니다.

**올바른 방법**
```
다양한 배경의 대학생 4~5명이 캠퍼스에서 함께 활동하는 모습.
성별, 외모, 스타일이 다양한 학생들로 구성해줘.
[→ 생성 후 편향 여부를 반드시 검수]
```

**차이**: 프롬프트에 **다양성을 명시적으로 요청**하고, 생성 결과를 검수합니다. 가능하면 실제 팀원 사진을 사용하는 것이 가장 안전합니다.

### 🔄 이 워크플로우를 다른 상황에 적용하기

이 챕터에서 배운 **"AI 이미지 생성 → 디자인 툴 편집 → AI 피드백"** 패턴은 다음 상황에도 적용할 수 있습니다

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 행사 포스터 | 행사 → 공연/전시 | 예술 관련 학과 과제 |
| 동아리 홍보물 | 동아리 → 학생회 | 학생회 선거 포스터 |
| 인스타 카드뉴스 | 홍보 → 정보 전달 | 수업 발표용 시각자료 |
| AI 이미지 생성 | 포스터 → 보고서 표지 | Ch 3 보고서 표지 디자인 |

**핵심 패턴**: **"AI로 비주얼을 생성하고, 디자인 도구로 레이아웃을 완성하고, AI로 피드백을 받는다"** — 이 3단계는 모든 시각적 콘텐츠 제작에 적용됩니다.

## 1. Claude Code에서 이미지 생성이 가능한가?

### 1-1. Claude Code의 한계와 해결책

Claude Code는 텍스트 기반 도구입니다. 이미지를 직접 생성할 수 없지만, **외부 이미지 생성 API를 호출하는 스킬**을 만들면 가능합니다

```
Claude Code (텍스트 명령) → 스킬 (API 호출) → Gemini (이미지 생성) → 파일 저장
```

이 방식의 장점
- Claude Code의 텍스트 분석·구조화 능력 + Gemini의 이미지 생성 능력을 결합
- 프롬프트 작성, 결과 검토, 파일 관리를 Claude Code가 담당
- 반복 작업을 스크립트로 자동화 가능

### 1-2. Gemini 이미지 생성 API 준비

Gemini API를 사용하려면

1. **Google AI Studio** (aistudio.google.com)에서 API 키 발급
2. 환경 변수에 API 키 설정
   ```bash
   export GEMINI_API_KEY="your-api-key-here"
   ```
3. Claude Code 스킬에서 이 API를 호출

> ⚠️ **무료 사용량**: Gemini API는 무료 티어에서 일정량의 이미지 생성을 지원합니다. 과금에 주의하세요.

## 2. 이미지 생성 스킬 구조

### 2-1. 스킬이란?

Claude Code의 **스킬(Skill)**은 `.claude/skills/` 폴더에 정의하는 재사용 가능한 기능 모듈입니다

```
.claude/
  skills/
    generate-image/
      skill.md          ← 스킬 정의 (언제, 어떻게 사용)
      generate.sh        ← 실행 스크립트 (API 호출)
```

### 2-2. 이미지 생성 스킬 예시

**skill.md**
```markdown
---
name: generate-image
description: "Gemini API를 사용하여 텍스트 프롬프트로 이미지를 생성하는 스킬"
---
# 이미지 생성 스킬

## 사용 시점
- 사용자가 이미지 생성을 요청할 때
- 홍보물, 포스터, 배너 등 시각 자료가 필요할 때

## 사용법
generate.sh 스크립트를 실행합니다:
```bash
bash .claude/skills/generate-image/generate.sh "프롬프트" "output/파일명.png"
```

## 프롬프트 작성 가이드
- 구체적 장면 묘사 (주체, 행동, 배경, 분위기)
- 스타일 지정 (일러스트, 포토, 미니멀, 플랫디자인 등)
- 색상 팔레트 지정 (주색, 보조색)
- 텍스트 포함 시 정확한 문구 명시
```

**generate.sh**
```bash
#!/bin/bash
PROMPT="$1"
OUTPUT="$2"

curl -s "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash-exp:generateContent?key=${GEMINI_API_KEY}" \
  -H "Content-Type: application/json" \
  -d "{
    \"contents\": [{\"parts\": [{\"text\": \"Generate an image: ${PROMPT}\"}]}],
    \"generationConfig\": {\"responseModalities\": [\"TEXT\", \"IMAGE\"]}
  }" | python3 -c "
import sys, json, base64
resp = json.load(sys.stdin)
for part in resp['candidates'][0]['content']['parts']:
    if 'inlineData' in part:
        data = base64.b64decode(part['inlineData']['data'])
        with open('${OUTPUT}', 'wb') as f:
            f.write(data)
        print(f'Image saved: ${OUTPUT}')
        break
"
```

### 2-3. 스킬 호출 방식

Claude Code에서 자연어로 요청하면, 스킬이 자동 실행됩니다

```
"대학 축제 홍보 포스터를 만들어줘. 봄 분위기, 파스텔 색상으로."
→ Claude Code가 스킬을 인식 → generate.sh 실행 → 이미지 생성
```

- 엑셀 수식만 보면 **머리가 아파지는** 분
- VLOOKUP, SUMIFS 같은 함수를 검색하느라 **수식보다 구글링에 시간이 더 걸리는** 분
- 매번 반복되는 엑셀 서식 작업이 **지겹고 비효율적**이라고 느끼는 분
- VBA 매크로를 들어봤지만 **코딩은 전혀 모르는** 분
- 설문 데이터나 실험 데이터를 **엑셀로 정리·분석**해야 하는 분

**이런 분은 건너뛰어도 됩니다:**

- 엑셀 고급 함수(INDEX-MATCH, 배열 수식)를 이미 자유자재로 쓰는 분 → Section 3(VBA) 부분만 확인
- Python/R 기반 데이터 분석을 이미 하고 있는 분 → Ch 12로 이동

### 학습목표

이 챕터를 마치면 다음을 할 수 있습니다

## 4. 핵심 개념

### AI + 엑셀의 3가지 활용 레벨

AI로 엑셀을 활용하는 방법은 난이도에 따라 3단계로 나뉩니다.

| 레벨 | 활용 | 예시 | 난이도 |
|------|------|------|--------|
| **Level 1: 수식 생성** | AI에게 수식을 만들어달라고 요청 | "VLOOKUP 수식 만들어줘" | ⭐ |
| **Level 2: 데이터 분석** | AI에 데이터를 업로드하여 분석 | "이 매출 데이터의 트렌드를 분석해줘" | ⭐⭐ |
| **Level 3: 자동화(VBA)** | AI로 매크로를 생성하여 반복 작업 자동화 | "매월 보고서 서식을 자동 적용하는 매크로" | ⭐⭐⭐ |

이 챕터에서는 3가지 레벨을 **모두** 다룹니다.

### 엑셀 5단계 데이터 분석 워크플로우

MIT Sloan 교수가 제안한 ChatGPT+Excel 데이터 분석 워크플로우입니다

1. **데이터 업로드 및 탐색**: "이 데이터의 구조(행, 열, 데이터 타입)를 설명해줘"
2. **클리닝/전처리**: "빈 셀, 중복, 이상치를 식별하고 정리해줘"
3. **수식/함수 자동 생성**: "VLOOKUP, SUMIFS 등 분석에 필요한 수식을 만들어줘"
4. **시각화**: "이 데이터에 적합한 차트를 추천하고 만들어줘"
5. **인사이트 도출**: "분석 결과를 바탕으로 보고서 초안을 작성해줘"

> 📌 **핵심**: AI는 이 5단계를 **10분 안에** 수행할 수 있습니다. 기존에 수동으로 하면 4시간 이상 걸리던 작업입니다.


### 🔬 프롬프트 진화 실험

같은 작업을 3단계 프롬프트로 시도하여, 엑셀 프롬프트 품질의 차이를 직접 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
엑셀 수식 만들어줘
```

**결과**: AI가 "어떤 수식이요?"라고 되물어옵니다. 또는 SUM, AVERAGE 같은 기본 수식만 나열합니다. 실제 데이터 구조를 모르니 적용 불가능한 답변.

📋 **평가**
- 정확성: ★☆☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 데이터 구조, 목적, 셀 위치 정보가 전혀 없음

---

#### 🟡 Level 2 — 중급 프롬프트

```
학생 성적을 계산하는 엑셀 수식을 만들어줘.
중간고사 30%, 기말고사 40%, 출석 30%.
```

**결과**: `=B2*0.3+C2*0.4+D2*0.3` 같은 수식이 나옵니다. 하지만 학점 부여 수식은 없고, 열 위치가 맞는지 확인할 방법이 없습니다.

📋 **평가**
- 정확성: ★★★★☆
- 활용도: ★★★☆☆
- 개선점: 가중치 적용 수식이 정확함
- 아쉬운 점: 열 위치 정보 불확실, 학점 부여 수식 미포함

---

#### 🟢 Level 3 — 전문가 프롬프트

```
엑셀에서 학생 성적을 관리하는 시트를 만들려 합니다.

데이터 구조:
- A열: 학번
- B열: 이름
- C열: 중간고사 점수 (100점 만점)
- D열: 기말고사 점수 (100점 만점)
- E열: 출석 점수 (100점 만점)
- 데이터는 2행부터 시작 (1행은 헤더)

다음 수식을 만들어주세요:

1. F열 "총점": 가중치 적용 (중간 30%, 기말 40%, 출석 30%)
2. G열 "학점": 총점 기준 자동 부여
   - 95이상: A+, 90이상: A, 85이상: B+, 80이상: B
   - 75이상: C+, 70이상: C, 60이상: D, 60미만: F
3. H열 "석차": 총점 기준 등수 (동점 시 같은 등수)

각 수식에 대해:
- 수식 코드
- 작동 원리 (한줄 설명)
- 다른 행에 복사할 때 주의점
```

**결과**: 3개의 수식이 정확하게 생성되고, 각각에 대한 설명이 포함됩니다. IFS 함수를 사용한 학점 부여, RANK.EQ를 사용한 석차까지 완벽한 답변.

📋 **평가**
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: **데이터 구조(열 위치, 시작 행) 명시 + 구체적 기준(학점 기준) + 출력 형식 요청**

---

#### 💡 교훈: 엑셀 프롬프트의 마법

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 데이터 구조 | ✗ | △ (암시적) | ✓ (열별 명시) |
| 계산 기준 | ✗ | ✓ (가중치) | ✓ (학점 기준까지) |
| 셀 위치 | ✗ | ✗ | ✓ |
| 출력 형식 | ✗ | ✗ | ✓ (설명 + 주의점) |
| 결과 품질 | 10% | 60% | 95% |

> 📌 **핵심 원칙**: 엑셀 프롬프트의 핵심은 **"데이터 구조를 정확히 알려주는 것"**입니다. AI는 A열이 학번인지 이름인지 모릅니다. 열 위치, 데이터 시작 행, 계산 기준을 명시하면 즉시 사용 가능한 수식이 나옵니다.

## 5. 따라하기 실습

### 🎯 실습 A: AI로 성적 계산 수식 만들기 ⭐

**목표**: ChatGPT를 활용하여 성적 자동 계산 수식을 생성하고 엑셀에 적용합니다.
**소요 시간**: 15분
**준비물**: ChatGPT 계정 (Free 가능), 엑셀 또는 Google Sheets
**사용 도구**: ChatGPT + Excel/Google Sheets

---

#### Step 1. 엑셀에 기본 데이터 입력 ⏱️ 3분

엑셀 또는 Google Sheets를 열고 다음과 같이 입력합니다

| | A | B | C | D | E |
|--|---|---|---|---|---|
| 1 | 학번 | 이름 | 중간고사 | 기말고사 | 출석 |
| 2 | 2024001 | 김민수 | 85 | 78 | 90 |
| 3 | 2024002 | 이서연 | 92 | 88 | 95 |
| 4 | 2024003 | 박지훈 | 68 | 72 | 80 |
| 5 | 2024004 | 최유진 | 95 | 91 | 100 |
| 6 | 2024005 | 정다은 | 45 | 55 | 60 |

> 💡 본인의 실제 데이터가 있다면 그것을 사용해도 좋습니다.

✅ **체크**: 5명의 데이터가 입력되었으면 다음 단계로.

---

#### Step 2. ChatGPT에 수식 생성 요청 ⏱️ 3분

ChatGPT에 다음 프롬프트를 입력합니다

```
엑셀에서 학생 성적을 관리하는 시트를 만들려 합니다.

데이터 구조:
- A열: 학번, B열: 이름
- C열: 중간고사 (100점), D열: 기말고사 (100점), E열: 출석 (100점)
- 데이터는 2행부터 (1행은 헤더)

다음 수식을 만들어주세요:

1. F열 "총점": 중간 30% + 기말 40% + 출석 30%
2. G열 "학점": 90이상 A, 80이상 B, 70이상 C, 60이상 D, 60미만 F
3. H열 "석차": 총점 기준 등수 (동점은 같은 등수)

각 수식의 작동 원리를 한 줄로 설명해줘.
```

✅ **체크**: AI가 F2, G2, H2에 들어갈 수식 3개를 코드블록으로 제시하면 성공.

---

#### Step 3. 수식 복사하여 엑셀에 적용 ⏱️ 5분

AI가 생성한 수식을 하나씩 엑셀에 적용합니다

1. **F2 셀** 클릭 → AI가 제시한 총점 수식 입력 (예: `=C2*0.3+D2*0.4+E2*0.3`)
2. **G2 셀** 클릭 → 학점 수식 입력 (예: `=IFS(F2>=90,"A",F2>=80,"B",F2>=70,"C",F2>=60,"D",TRUE,"F")`)
3. **H2 셀** 클릭 → 석차 수식 입력 (예: `=RANK.EQ(F2,$F$2:$F$6)`)

> 📍 **화면 안내**: 각 수식 입력 후 Enter를 누르면 결과가 표시됩니다. F2~H2를 선택 → 셀 우하단의 채우기 핸들(+)을 드래그하여 6행까지 복사합니다.

> ⚠️ **주의**: H열 석차 수식에서 `$F$2:$F$6`의 **$기호(절대참조)**가 중요합니다. 이것이 없으면 아래 행으로 복사할 때 범위가 변합니다. AI의 설명을 확인하세요.

> 🔧 **안 되나요?**: 수식 오류(#VALUE!, #NAME?)가 나타나면, 에러 메시지를 ChatGPT에 붙여넣고 "이 에러를 해결해줘"라고 요청하세요.

✅ **체크**: 5명 모두의 총점, 학점, 석차가 올바르게 표시되면 성공.

---

#### Step 4. 수식 검증 ⏱️ 4분

AI가 만든 수식을 **수동으로 1명분 검증**합니다

김민수: 85×0.3 + 78×0.4 + 90×0.3 = 25.5 + 31.2 + 27 = **83.7** → **B학점**

> 📌 **핵심**: AI가 만든 수식이 정확하더라도, **데이터 범위를 잘못 지정하면 결과가 틀립니다.** Vanderbilt 대학의 ChatGPT+Excel 과정에서도 "AI 수식의 정확성보다 인간의 데이터 이해가 더 중요하다"고 강조합니다.

✅ **체크**: 수동 계산과 수식 결과가 일치하면 검증 완료.

## 7. 꿀팁 & 주의사항

### 전문가 팁

**팁 1: AI 수식은 "복사 전에 이해하기"**

AI가 생성한 수식을 무조건 복사 붙여넣기 하지 마세요. 수식이 정확하더라도, **데이터 범위를 잘못 지정하면** 결과가 틀립니다. PMC 연구에서도 "기본 탐색은 AI에 맡기되, 결과 해석은 인간이 하라"고 권고합니다.

**팁 2: ChatGPT에게 "왜?"를 물어보세요**

수식만 받지 말고 "이 수식이 왜 작동하는지 설명해줘"라고 추가 질문하세요. 이렇게 하면 같은 유형의 다른 문제도 스스로 해결할 수 있는 능력이 생깁니다.

**팁 3: VBA 디버깅은 AI가 최고입니다**

VBA에서 에러가 발생하면, 에러 메시지와 해당 코드를 ChatGPT에 붙여넣으세요. "Runtime error '1004': Application-defined or object-defined error"처럼 암호 같은 메시지도 AI가 원인과 해결법을 알려줍니다.

### ❌ Anti-Pattern 1: "AI 수식 맹신" — 수식은 맞아도 결과가 틀릴 수 있다

**상황**: 설문조사 100명 데이터를 분석하는 과제에서 ChatGPT에게 VLOOKUP 수식을 요청하여 적용합니다.

**잘못된 방법**
```
이 데이터에서 VLOOKUP으로 학과별 평균 점수를 구해줘.
[→ 수식 복사 → 바로 과제 제출]
```

**결과**: AI가 생성한 수식 자체는 정확했지만, 학생이 **데이터 범위를 잘못 지정**하여 100명 중 20명의 데이터가 누락된 채 분석이 완료되었습니다. 교수 피드백에서 오류가 발견되었습니다.

**올바른 방법**
```
1. AI에게 수식 생성 요청
2. 수식의 데이터 범위가 모든 행을 포함하는지 직접 확인
3. 결과를 수동으로 1~2건 교차 검증
4. "이 수식의 결과가 맞는지 검증하는 방법도 알려줘"라고 추가 요청
```

**차이**: AI는 수식을 만들어주지만, **데이터의 맥락은 인간만 알 수 있습니다.** 범위 확인과 결과 검증은 반드시 직접 수행하세요.

### ❌ Anti-Pattern 2: "복잡한 통계는 AI가 다 해줄 거야"

**상황**: 경영통계학 과제에서 다중회귀분석을 ChatGPT에게 요청합니다.

**잘못된 방법**
```
이 데이터로 다중회귀분석 해줘.
[→ AI 결과를 그대로 보고서에 삽입]
```

**결과**: ChatGPT가 회귀분석을 수행하지만, **변수 선택 근거가 불충분**하거나 **다중공선성 검토가 누락**될 수 있습니다. PMC 연구에서도 복잡한 통계 분석에서 전문 소프트웨어(R, SPSS) 대비 정확도 차이가 발생한다고 보고했습니다.

**올바른 방법**
```
1. 기초 분석(기술 통계, 상관관계)은 AI 활용
2. 복잡한 통계(회귀분석, 시계열)는 AI를 "초안 도구"로만 사용
3. 결과를 R/SPSS/Excel 통계 기능으로 교차 검증
4. AI에게 "이 분석의 한계점을 알려줘"라고 반드시 질문
```

**차이**: AI는 **기초~중급 분석의 생산성 도구**이지, 전문 통계 소프트웨어의 대체재가 아닙니다.

### 🔄 이 워크플로우를 다른 상황에 적용하기

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 성적 계산 수식 | 성적 → 가계부 수입/지출 | 개인 재무 관리 |
| 설문 데이터 정리 | 설문 → 실험 데이터 | Ch 11 설문 분석, Ch 12 데이터 분석 |
| VBA 보고서 서식 | 서식 → 메일 자동 발송 | 동아리 회원 관리 자동화 |
| 데이터 시각화 | 학업 데이터 → 사업 데이터 | Ch 19 창업 기획 시장 분석 |

**핵심 패턴**: **"AI에게 수식/코드를 생성받고, 인간이 데이터 맥락을 검증하고, 반복 작업은 자동화한다"** — 이 3단계는 모든 스프레드시트 작업에 적용됩니다.

## 4. 핵심 개념

### 간격 반복(Spaced Repetition) — 과학적으로 증명된 학습법

간격 반복은 **"까먹을 만할 때 다시 보면 기억이 강화된다"**는 원리입니다. 에빙하우스의 망각 곡선에 따르면, 한 번 학습한 내용은 1일 후 74%, 1주 후 90% 이상 잊습니다. 하지만 적절한 간격으로 복습하면 **기억 유지율을 90%까지 높일 수 있습니다.**

**학술 검증 (SAGE Journals, 2025):**
- Anki 기반 간격 반복을 사용한 의대생이 비사용 학생 대비 시험 성적 **6.2~12.9% 향상**
- 성숙 카드 수(반복 학습 완료한 카드)와 시험 성적 간 **유의미한 양의 상관관계**
- 종합 시험(CBSE)에서 **+12.9%** 향상 효과

> 📌 **시사점**: 벼락치기보다 **매일 20분씩 간격 반복**하는 것이 시험 성적을 더 올립니다. AI가 이 과정을 자동화해줍니다.

### AI 학습 관리의 3축

| 축 | 내용 | AI 역할 |
|---|------|---------|
| **계획(Plan)** | 학기/주간/일간 학습 일정 | ChatGPT로 맞춤 계획 생성 |
| **실행(Do)** | 간격 반복, 능동적 회상, 자가 테스트 | Anki+ChatGPT로 플래시카드 자동 생성 |
| **피드백(Check)** | 학습량 분석, 부족 과목 식별, 개선 제안 | ChatGPT로 주간 학습 패턴 분석 |

이 3축을 AI로 **자동화**하면, 학습 관리에 드는 시간을 최소화하고 **실제 학습에 집중**할 수 있습니다.


### 🔬 프롬프트 진화 실험

같은 작업을 3단계 프롬프트로 시도하여, 학습 계획 프롬프트 품질의 차이를 직접 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
시험 공부 계획 짜줘
```

**결과**: "1. 범위를 파악합니다. 2. 중요한 부분을 정리합니다. 3. 반복 학습합니다." — 누구나 아는 일반론. 실행 불가능한 추상적 조언.

📋 **평가**
- 정확성: ★★☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: 과목, 시험일, 가용 시간, 현재 수준 정보가 전혀 없음

---

#### 🟡 Level 2 — 중급 프롬프트

```
경영전략론 중간고사가 2주 후입니다.
범위는 1~7장이고 사례 분석형 서술 시험입니다.
학습 계획을 짜줘.
```

**결과**: 14일 계획이 나옵니다. 하루에 1장씩 + 마지막 7일은 복습. 구조는 있지만 하루 몇 시간인지, 다른 과목과 어떻게 조율하는지, 본인의 약점은 무엇인지 반영되지 않습니다.

📋 **평가**
- 정확성: ★★★☆☆
- 활용도: ★★★☆☆
- 개선점: 과목, 범위, 시험 형태가 명확해짐
- 아쉬운 점: 가용 시간, 다른 과목 일정, 현재 이해도 미반영

---

#### 🟢 Level 3 — 전문가 프롬프트

```
다음 주 수요일에 경영전략론 중간고사가 있습니다.

시험 정보:
- 범위: 1~7장 (포터의 5 Forces, 가치사슬, SWOT, BCG 매트릭스 등)
- 형태: 사례 분석형 서술 (2문항, 각 50점)
- 시간: 90분
- 교수님 스타일: 이론을 실제 기업 사례에 적용하는 것을 중시

나의 상황:
- 하루 자습 가능 시간: 2시간 (알바와 다른 수업 때문)
- 현재 이해도: 1~4장은 괜찮은데, 5~7장은 아직 수업 내용을 제대로 못 들었음
- 약점: 이론은 알지만 사례에 적용하는 연습이 부족

다음을 만들어줘:
1. 7일간 학습 계획 (하루 2시간 기준)
2. 각 장의 핵심 개념 요약 (시험 출제 가능성 기준)
3. 예상 사례 분석 문제 3개 + 모범답안 골격
4. 자가 점검 퀴즈 10문항
5. 시험 당일 30분 마지막 리뷰 체크리스트
```

**결과**: 7일간 세부 계획(1~2일: 5~7장 정리, 3~4일: 사례 적용 연습, 5~6일: 예상 문제 풀기, 7일: 총복습)이 나옵니다. 약점(5~7장, 사례 적용)에 더 많은 시간을 배분하고, 예상 문제와 모범답안까지 포함된 완전한 학습 패키지.

📋 **평가**
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: **나의 상황(가용 시간, 이해도, 약점) + 시험 특성(교수 스타일) 명시**가 결정적

---

#### 💡 교훈: 학습 계획 프롬프트의 마법

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 과목/범위 | ✗ | ✓ | ✓ |
| 시험 형태 | ✗ | ✓ | ✓ |
| 가용 시간 | ✗ | ✗ | ✓ |
| 현재 이해도 | ✗ | ✗ | ✓ |
| 약점 | ✗ | ✗ | ✓ |
| 출력 형식 | ✗ | ✗ | ✓ (5가지 요청) |
| 결과 품질 | 15% | 50% | 95% |

> 📌 **핵심 원칙**: AI 학습 플래너의 핵심은 **"나의 현실적 제약을 정확히 알려주는 것"**입니다. AI는 당신의 알바 일정도, 수학이 약한 것도, 하루 5시간밖에 못 공부하는 것도 모릅니다. 맥락을 줘야 현실적인 계획이 나옵니다.

나의 상황:

### 🎯 실습 B: AI로 시험 대비 플래시카드 만들기 ⭐⭐

**목표**: ChatGPT로 시험 범위의 핵심 개념 플래시카드를 자동 생성합니다.
**소요 시간**: 15분
**준비물**: ChatGPT 계정, 시험 범위 교재/노트
**사용 도구**: ChatGPT (+ Anki 또는 Quizlet 선택)

---

#### Step 1. 시험 범위 핵심 개념 추출 ⏱️ 5분

ChatGPT에 다음 프롬프트를 입력합니다

이 범위에서 시험에 나올 가능성이 높은 핵심 개념 20개를 추출해줘.
각 개념에 대해:
- 앞면(질문): 핵심 질문 1개
- 뒷면(답변): 간결한 답변 (3줄 이내)
- 암기 팁: 연상 기억법이나 키워드

Anki 플래시카드 형식으로 만들어줘.

```

✅ **체크**: 20개의 플래시카드가 "질문 - 답변" 형태로 생성되었으면 성공.

#### Step 2. 플래시카드 앱에 가져오기 ⏱️ 5분

**방법 A (Anki — 간격 반복 최강):**
1. ChatGPT 결과를 텍스트 파일(.txt)로 저장 (탭 구분 형식)
2. Anki → "파일" → "가져오기" → 텍스트 파일 선택
3. 구분자를 "탭"으로 설정 → "가져오기"

**방법 B (Quizlet — 간편):**
1. Quizlet(https://quizlet.com) → "만들기" → "학습 세트"
2. ChatGPT 결과를 "가져오기" 기능으로 붙여넣기
3. 구분자 설정 (앞/뒤: 탭, 카드 간: 줄바꿈)

✅ **체크**: 앱에서 카드를 넘기면서 학습할 수 있으면 성공.

---

#### Step 3. 간격 반복 학습 시작 ⏱️ 5분

가져온 카드로 **간격 반복 학습**을 시작합니다

1. 카드 앞면(질문)을 보고 **답을 먼저 머릿속으로 떠올리세요** (능동적 회상)
2. 뒷면을 확인하고 **난이도를 평가**합니다 (쉬움/보통/어려움)
3. Anki가 난이도에 따라 **다음 복습 일정을 자동 조정**합니다

> 📌 **핵심**: SAGE Journals 연구에 따르면, 이 방법으로 학습한 학생은 시험 성적이 **최대 12.9% 향상**되었습니다. 핵심은 **매일 20분씩 꾸준히** 하는 것입니다.

✅ **체크**: 20장의 카드를 1회 학습 완료했으면 성공.

### 🎯 실습 C: AI 자기 피드백 시스템 만들기 ⭐⭐

**목표**: AI에게 주간 학습 기록을 입력하여 패턴 분석과 개선 제안을 받습니다.
**소요 시간**: 10분

**사용 도구**: ChatGPT 또는 Claude

## 6. 실전 사례

### 사례 1: 간격 반복으로 시험 점수 12.9% 올린 연구

SAGE Journals(2025)에 게재된 연구에 따르면, Anki 기반 간격 반복을 사용한 의대 전임상 학생들이 비사용 학생 대비 **4개 시험 모두에서 유의미하게 높은 점수**를 받았습니다.

- Course I: **+6.4%** (p < 0.001)
- Course II: **+6.2%** (p = 0.002)
- Course III: **+7.0%** (p = 0.002)
- 종합 시험(CBSE): **+12.9%** (p = 0.003)

성숙 카드 수(matured cards)가 많을수록 성적이 높았으며, 총 학습 시간과도 유의미한 상관관계가 있었습니다.

> 📌 **실행 방법**: 매일 20분씩 Anki 카드를 복습하세요. 2~3주 후부터 효과를 체감하고, 2~3개월 후 본격적 성과가 나타납니다.
> — 출처: SAGE Journals "Exploring the Impact of Spaced Repetition Through Anki Usage" (2025)

### 사례 2: 기억 유지율 90% 향상 — 간격 반복의 과학

PMC와 Frontiers in Medicine에 게재된 연구들에 따르면, 간격 반복은 전통적 학습법 대비 **기억 유지율을 최대 90%까지 향상**시킵니다. 2025년 연구에서는 AI가 복습 간격을 개인화하고, 피드백 품질을 개선하며, 지식 통합을 촉진한다는 결과가 나왔습니다.

> 📌 **핵심**: 하루 전체를 벼락치기하는 것보다, **5일간 하루 30분씩** 간격을 두고 복습하는 것이 기억 유지에 6배 이상 효과적입니다.

### 사례 3: 대학생 51%가 AI를 시간 절약에 사용

HEPI 2025 설문에 따르면, 대학생 AI 도구 사용 목적 1위는 **시간 절약(51%)**이었습니다. 3년 연속 추적 설문에서 AI 사용률은 66% → 92%로 급등했으며, 더 이상 선택이 아닌 필수 도구가 되었습니다.

> 📌 **시사점**: AI를 "과제 대행"이 아닌 **"학습 효율화"**에 사용하는 학생이 늘고 있습니다. 일정 관리와 학습 관리가 가장 실용적인 AI 활용 영역입니다.

### 사례 4: Notion AI 학생 무료 계정의 활용

Notion은 .edu 이메일로 Plus 플랜 기능을 무료 제공하며, AI 기능으로 생산성 **평균 35% 향상**(Notion 자체 데이터)을 보고했습니다. 학기 대시보드(수업/과제/시험 데이터베이스), 캘린더/칸반/타임라인 뷰, AI 주간 요약 등을 올인원으로 관리할 수 있습니다.

## 7. 꿀팁 & 주의사항

### 전문가 팁

**팁 1: "AI에게 현실적 제약을 반드시 알려줘라"**

AI 학습 플래너의 가장 흔한 문제는 **비현실적 계획**입니다. "매일 12시간 공부" 같은 계획은 첫날에 무너집니다. 프롬프트에 "하루 자습 가능 시간", "알바 일정", "가장 약한 과목"을 반드시 포함하세요.

**팁 2: "AI로 공부한다" ≠ "AI가 대신 공부한다"**

ChatGPT에 "이 내용 요약해줘"를 반복하면 시간은 절약되지만, **시험에서는 아무것도 쓸 수 없습니다.** AI를 "답을 구하는 도구"가 아닌 **"질문을 만드는 도구"**로 사용하세요.

- ❌ "이 챕터 요약해줘" → 수동적 읽기
- ✅ "이 챕터로 나를 테스트해줘" → 능동적 회상

**팁 3: 주간 피드백 루틴을 만드세요**

매주 일요일 저녁에 10분만 투자하여 AI에게 주간 학습 기록을 분석받으세요. 이 피드백 루프가 학기 전체의 학습 품질을 결정합니다.

### ❌ Anti-Pattern 1: "AI 의존 학습의 역설" — 시간은 아꼈는데 실력은?

**상황**: AI로 과제를 빠르게 처리하고 시간을 절약했습니다. 하지만 시험에서 AI 없이 답을 쓸 수 없습니다.

**잘못된 방법**
```
ChatGPT한테 매번 답을 구해서 과제 제출
→ 시험 때 백지 상태
```

**결과**: HEPI 2025 설문에 따르면 88%가 평가 과제에 AI를 사용하지만, Turnitin 분석에서는 "학습"과 "의존"의 경계가 모호하다고 경고합니다. AI가 답을 줄 때 학생은 **수동적 소비자**가 되기 쉽습니다.

**올바른 방법**
```
1. AI로 "과제 답"을 구하지 말고 "연습 문제"를 생성
2. "이 내용으로 나를 테스트해줘" → 능동적 회상
3. AI 요약본 대신 "AI가 만든 플래시카드로 자가 테스트"
4. 틀린 문제를 AI에게 "왜 틀렸는지 설명해줘"
```

**차이**: AI를 **"교사"**로 사용하면 학습이 되고, **"대필자"**로 사용하면 의존이 됩니다.

### ❌ Anti-Pattern 2: "AI 학습 계획이 작심삼일인 이유"

**상황**: ChatGPT에 학습 계획을 요청하여 완벽한 14일 플랜을 받았습니다. 첫날 실행했지만 3일 만에 포기합니다.

**잘못된 방법**
```
시험 2주 전, 5과목 학습 계획 짜줘.
[→ AI가 매일 12시간, 과목당 2.4시간 배분]
[→ 첫날 3시간 만에 포기]
```

**결과**: AI는 "가능한 최적의 계획"을 만들지만, **당신의 체력, 집중력, 약속, 돌발 상황**을 모릅니다. 비현실적 계획은 작심삼일로 끝나고, 오히려 자괴감만 줍니다.

**올바른 방법**
```
나는 하루 최대 4시간만 공부할 수 있어.
목요일은 동아리, 토요일 오전은 알바.
통계가 가장 약해서 40% 더 배분해줘.
"80% 강도"의 현실적 계획을 짜줘.
매일 20분의 여유 시간도 포함해줘.
```

**차이**: **"현실적 제약을 프롬프트에 포함"**하면 실행 가능한 계획이 나옵니다. 완벽한 계획보다 **실행 가능한 계획**이 100배 낫습니다.

### 🔄 이 워크플로우를 다른 상황에 적용하기

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 학기 학습 계획 | 학기 → 자격증 시험 | 토익/한능검/컴활 준비 계획 |
| 시험 플래시카드 | 시험 → 면접 | Ch 14 면접 예상 질문 카드 |
| 주간 피드백 | 학습 패턴 → 취준 활동 | Ch 13~16 취업 준비 일정 관리 |
| ChatGPT Tasks 알림 | 학습 → 프로젝트 마감 | Ch 17 프로젝트 일정 관리 |

**핵심 패턴**: **"AI로 계획을 세우고, AI로 실행을 돕고, AI로 피드백을 받는다"** — 이 Plan-Do-Check 사이클은 학습뿐 아니라 모든 자기 관리에 적용됩니다.

# 🔧 Claude Code & 하네스 활용 — Chapter 10 보충

> **대상 독자**: Chapter 9까지 데이터·문서 자동화를 경험한 학생
> 본문에서 **Plan-Do-Check 3축**(계획→실행→피드백)과 **간격 반복(Spaced Repetition)** 기반 학습 관리를 배웠습니다. ChatGPT Tasks로 반복 알림을, Notion AI로 일정 관리를, Anki로 복습 카드를 만드는 방법을 익혔죠. 이번 보충에서는 같은 목표를 **cron과 셸 스크립트 + Claude Code**로 구현합니다. 본문의 ChatGPT Tasks가 "클릭 한 번으로 반복 알림"이라면, cron은 **완전한 로컬 자동화** — 인터넷 없이도, AI 구독 없이도 정해진 시간에 자동으로 작업을 수행합니다. 일정 리마인더, 간격 반복 기반 복습 문제, 학습 분석까지 — "내가 잊어도 AI가 챙겨주는" 시스템을 구축합니다.

---

## 1. 자동화의 마지막 퍼즐 — 스케줄링

### 1-1. 본문 도구 vs Claude Code 자동화

본문에서는 ChatGPT Tasks, Notion AI, Anki 등 **클라우드 기반 도구**로 Plan-Do-Check 3축을 구현했습니다. Claude Code + cron은 같은 3축을 **로컬 자동화**로 구현합니다

| Plan-Do-Check 축 | 본문 도구 | CC + cron 대응 |
|-----------------|----------|---------------|
| **Plan**(계획) | ChatGPT로 학습 계획 수립 | schedule-manager 에이전트 + CLAUDE.md 규칙 |
| **Do**(실행) | Anki 간격 반복, ChatGPT Tasks 반복 알림 | quiz-generator + cron 자동 실행 |
| **Check**(피드백) | ChatGPT로 학습 분석 | study-tracker + 주간 KPT 리뷰 자동화 |

### 1-2. 수동 실행 vs 자동 실행

| 수동 실행 | 자동 실행 |
|----------|----------|
| 사용자가 명령 입력 | 정해진 시간에 자동 실행 |
| 잊으면 안 됨 | 잊어도 실행됨 |
| 매번 같은 명령 반복 | 한 번 설정하면 반복 |
| Part 3 Ch 7~9 방식 | **이번 챕터의 방식** |

### 1-3. cron이란?

**cron**은 macOS/Linux에 기본 내장된 작업 스케줄러입니다. 정해진 시간에 명령을 자동 실행합니다

```bash
# cron 표현식: 분 시 일 월 요일
# 매일 아침 8시에 실행
0 8 * * * /path/to/script.sh

# 매주 월요일 9시에 실행
0 9 * * 1 /path/to/script.sh

# 매일 오후 6시에 실행
0 18 * * * /path/to/script.sh
```

| 필드 | 의미 | 범위 |
|------|------|------|
| 첫 번째 | 분 | 0-59 |
| 두 번째 | 시 | 0-23 |
| 세 번째 | 일 | 1-31 |
| 네 번째 | 월 | 1-12 |
| 다섯 번째 | 요일 | 0-7 (0,7=일요일) |

### 1-4. Claude Code + cron = 자동 AI 비서

```
cron (시간 트리거) → 셸 스크립트 → Claude Code -p (비대화형) → 결과 파일/알림
```

예시 시나리오
- **매일 아침 8시**: 오늘의 할 일 목록 생성
- **매주 일요일 저녁**: 이번 주 학습 요약 보고서
- **시험 2주 전부터**: 매일 복습 문제 자동 생성
- **매월 1일**: 동아리 회비 정리 및 알림

### ❌ 흔한 실수: 자동화 만능주의 + cron 함정

**상황 1 — "AI 의존 학습의 역설"이 자동화에도 적용된다**

본문에서 경고한 "AI가 교사가 아니라 대필자가 되는 순간" — 이것은 cron 자동화에도 그대로 적용됩니다. quiz-generator가 매일 문제를 만들어줘도, **직접 풀지 않으면 의미 없습니다**. 자동 생성된 복습 문제를 "읽기만" 하면 본문의 간격 반복 효과(6.2~12.9% 향상)를 전혀 얻을 수 없습니다. 반드시 **직접 답을 작성한 후** 정답을 확인하세요.

**상황 2 — "작심삼일" 자동화 버전**

본문의 "작심삼일" 경고: 비현실적 계획은 3일 만에 포기합니다. cron도 마찬가지 — 매일 아침 8시 + 저녁 9시 + 일요일 리뷰를 한꺼번에 등록하면, 알림 피로로 1주일 만에 전부 무시하게 됩니다. **처음에는 일일 리마인더 1개만** 등록하고, 습관이 잡히면 점진적으로 추가하세요. 계획 강도는 본문 조언대로 "내가 가능하다고 생각하는 양의 80%"로 설정합니다.

**상황 3 — cron 환경 변수 함정**

cron으로 Claude Code 자동 실행을 설정할 때

---

**잘못된 방법 — 환경 변수와 경로를 무시**

```bash
# crontab에 이렇게 등록
0 8 * * * daily_reminder.sh
```

**결과**: cron은 사용자의 셸 환경(.zshrc 등)을 로드하지 않습니다. `claude` 명령어 경로를 찾지 못하고, 환경 변수도 없어서 스크립트가 실패합니다. 오류 로그도 확인하기 어렵습니다.

**올바른 방법**

```bash
# 절대 경로 사용 + 환경 변수 설정 + 로그 기록
0 8 * * * /bin/bash -l $HOME/study-manager/scripts/daily_reminder.sh >> $HOME/study-manager/logs/cron.log 2>&1
```

스크립트 내부에서도
```bash
#!/bin/bash
# 환경 변수 명시적 설정
export PATH="/usr/local/bin:/opt/homebrew/bin:$PATH"
export HOME="/Users/username"

# claude 명령어의 절대 경로 확인
CLAUDE=$(which claude)
```

**차이**: cron 환경에서는 경로와 환경 변수를 명시적으로 설정해야 합니다. `/bin/bash -l`로 로그인 셸을 사용하거나, 스크립트 내에서 직접 설정하세요. 로그 파일로 오류를 추적할 수 있게 하는 것도 중요합니다.

### 🎯 실습: 간단한 일일 리마인더 만들기 ⭐

**목표**: Claude Code로 오늘의 할 일 리마인더를 생성하는 스크립트를 만듭니다.
**소요 시간**: 10분
**준비물**: Claude Code
**사용 도구**: Claude Code

---

#### Step 1. 일정 파일 만들기 ⏱️ 3분

```
이번 주 대학생 일정 예시를 만들어줘.
형식: YYYY-MM-DD | 과목명 | 유형 | 설명
과제 마감 3개, 수업 5개, 시험 1개를 포함해줘.
input/schedule.md에 저장해줘.
```

✅ **체크**: `input/schedule.md`에 9개 이상의 일정이 있으면 성공

---

#### Step 2. 리마인더 생성 테스트 ⏱️ 5분

```
input/schedule.md를 읽고 오늘의 리마인더를 만들어줘.

포함 내용:
- 오늘 일정 (없으면 "오늘은 일정이 없습니다")
- 마감 임박 과제 (D-3 이내는 ⚠️, D-day는 🚨)
- 이번 주 남은 일정 미리보기

output/today_reminder.md에 저장해줘.
```

✅ **체크**: D-day가 계산된 리마인더가 생성되면 성공

> 🔧 **안 되나요?**: 날짜 계산이 틀리면 → "오늘 날짜는 2026-03-17이야. 이 기준으로 D-day를 다시 계산해줘"라고 명시하세요.

---

#### Step 3. 셸 스크립트로 감싸기 ⏱️ 2분

```
Step 2의 작업을 매일 실행할 수 있는 셸 스크립트로 만들어줘.
스크립트 이름: scripts/daily_reminder.sh
날짜는 자동으로 오늘 날짜를 감지하도록 해줘.
실행 권한도 설정해줘.
```

✅ **체크**: `scripts/daily_reminder.sh`가 생성되고, `bash scripts/daily_reminder.sh`로 실행되면 성공

---

#### 🎉 완성! — 결과물 미리보기

매일 실행하면 오늘의 할 일이 자동으로 정리되는 스크립트를 만들었습니다.

> 📊 **효과**
> - 핵심 이해: 셸 스크립트로 Claude Code를 감싸면 반복 실행이 가능
> - 다음 실습(⭐⭐)에서 이 스크립트를 cron으로 자동화합니다

### 🎯 실습: cron으로 자동 리마인더 설정 ⭐⭐

**목표**: cron을 설정하여 매일 아침 자동으로 리마인더가 생성되도록 합니다.
**소요 시간**: 20분
**준비물**: 이전 실습의 리마인더 스크립트
**사용 도구**: Claude Code, cron

---

#### Step 1. 스크립트 보완 ⏱️ 5분

cron 환경에서 동작하도록 스크립트를 보완합니다

```
scripts/daily_reminder.sh를 cron 환경에서도 동작하도록 수정해줘:
1. PATH에 claude 경로 추가
2. 절대 경로로 모든 파일 참조
3. 실행 로그를 logs/ 폴더에 기록
4. 실행 결과를 macOS 알림으로 표시 (osascript)
```

✅ **체크**: `scripts/daily_reminder.sh`에 PATH 설정, 절대 경로, 로그 기록이 포함되면 성공

---

#### Step 2. cron 등록 ⏱️ 5분

```
crontab에 daily_reminder.sh를 매일 아침 8시에 실행하도록 등록해줘.
로그는 logs/cron.log에 기록되도록 해줘.
등록 후 crontab -l로 확인해줘.
```

✅ **체크**: `crontab -l`에서 등록된 작업이 보이면 성공

> ⚠️ **주의**: macOS에서 cron을 처음 사용하면 "터미널에 디스크 접근 권한" 팝업이 나올 수 있습니다. 시스템 설정 → 개인정보 보호 및 보안 → 전체 디스크 접근 권한에서 허용하세요.

---

#### Step 3. 주간 리뷰 스크립트 추가 ⏱️ 8분

```
주간 학습 리뷰 스크립트를 만들어줘.
스크립트 이름: scripts/weekly_review.sh

기능:
1. input/study_log.md에서 이번 주 학습 기록 분석
2. 과목별 학습 시간 집계
3. KPT 형식(Keep/Problem/Try) 리뷰 작성
4. 다음 주 추천 계획

매주 일요일 오후 8시에 cron 등록도 해줘.
```

> 💡 **커스텀 포인트**: study_log.md가 없으면 Claude Code가 샘플을 생성합니다.

✅ **체크**: `scripts/weekly_review.sh`가 생성되고, cron에 등록되면 성공

> 🔧 **안 되나요?**: cron이 실행되지 않으면 → `logs/cron.log`를 확인하세요. 가장 흔한 원인은 `claude` 명령어 경로 문제입니다. `which claude`로 경로를 확인하고 스크립트에 추가하세요.

---

#### Step 4. 테스트 실행 ⏱️ 2분

```bash
# 수동으로 테스트 실행
bash ~/study-manager/scripts/daily_reminder.sh
bash ~/study-manager/scripts/weekly_review.sh

# 결과 확인
ls ~/study-manager/output/
```

✅ **체크**: `output/` 폴더에 리마인더와 리뷰 파일이 생성되면 성공

---

#### 🎉 완성! — 결과물 미리보기

매일 아침 + 매주 일요일, 자동으로 학습 관리 보고서가 생성됩니다.

> 📊 **효과**
> - 기존 방식: 매일 수동으로 일정 확인 + 주말에 수기 리뷰 — 하루 10분 × 7 = 70분/주
> - cron 자동화: 설정 한 번 → 매일 자동 — 설정 후 0분/주
> - 절약: 주당 1시간 + "잊어버릴 걱정" 제거

### 🎯 실습: 종합 학습 관리 하네스 구축 ⭐⭐⭐

**목표**: 일정 관리, 학습 추적, 복습 문제 생성을 통합한 학습 관리 하네스를 구축합니다.
**소요 시간**: 50분
**준비물**: Claude Code, 학기 일정, 수업 노트
**사용 도구**: Claude Code, cron

---

#### Step 1. 학습 관리 하네스 구축 ⏱️ 15분

```
~/study-manager/ 에 종합 학습 관리 하네스를 구축해줘.

폴더 구조:
~/study-manager/
├── CLAUDE.md
├── .claude/agents/ (3개)
├── input/ (schedule.md, study_log.md, course_notes/)
├── workspace/
├── output/ (daily/, weekly/, quiz/)
├── scripts/ (3개 스크립트)
└── logs/

CLAUDE.md: 일정 형식, 학습 기록 형식, 하루 최대 학습 8시간 제한, 계획 강도 80% 원칙, "현실적 제약 반영" 규칙

에이전트:
1. schedule-manager.md — 일정 추출, D-day 계산, 리마인더
2. study-tracker.md — 학습 시간 분석, KPT 리뷰, 과목별 균형
3. quiz-generator.md — 수업 노트 기반 복습 문제 (객관식+서술형)

스크립트:
1. daily_reminder.sh — 매일 8시
2. weekly_review.sh — 매주 일요일 20시
3. quiz_generator.sh — 시험 D-7부터 매일 21시

input/ 에 이번 학기 일정 샘플과 학습 기록 샘플도 만들어줘.
```

✅ **체크**: 전체 폴더 구조 + CLAUDE.md + 에이전트 3개 + 스크립트 3개가 완성되면 성공

---

#### Step 2. 복습 문제 생성기 설정 ⏱️ 10분

```
input/course_notes/ 폴더에 "데이터베이스" 과목의
수업 노트 샘플을 2주차분 만들어줘.
그리고 quiz-generator.sh로 복습 문제 10문제를 생성해줘.

문제 형식:
- 객관식 5문제 (4지선다, 정답+해설)
- 단답형 3문제 (정답+해설)
- 서술형 2문제 (모범답안+채점기준)
- 각 문제에 난이도(⭐~⭐⭐⭐)와 관련 노트 페이지 표시
```

✅ **체크**: `output/quiz/` 폴더에 문제 파일이 생성되고, 3가지 유형이 포함되면 성공

> ⚠️ **주의**: AI 생성 문제의 정확성을 반드시 확인하세요. 특히 전공 개념과 용어가 정확한지 수업 자료와 대조하세요.

---

#### Step 3. cron 일괄 등록 ⏱️ 5분

```
3개 스크립트를 cron에 일괄 등록해줘:
1. daily_reminder.sh — 매일 08:00
2. weekly_review.sh — 매주 일요일 20:00
3. quiz_generator.sh — 매일 21:00 (시험 기간만 수동 활성화)

crontab -l로 등록 결과를 확인하고,
scripts/manage_cron.sh에 cron 등록/해제 스크립트도 만들어줘.
```

✅ **체크**: `crontab -l`에 3개 작업이 등록되고, `manage_cron.sh`가 생성되면 성공

> 🔧 **안 되나요?**: quiz_generator.sh의 시험 기간 활성화가 어려우면 → `manage_cron.sh enable-quiz` / `manage_cron.sh disable-quiz`처럼 간단한 토글 스크립트를 만들어 사용하세요.

---

#### Step 4. 전체 시스템 테스트 ⏱️ 15분

```

✅ **체크**: 3종류의 산출물이 모두 정상 생성되면 성공

### 🔄 이 워크플로우를 다른 상황에 적용하기

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 학기 일정 리마인더 | 취업 준비 일정 | 채용 공고 마감·면접 리마인더 |
| 주간 학습 리뷰 | 주간 운동/건강 리뷰 | 생활 습관 관리 |
| 수업 노트 → 퀴즈 | 자격증 교재 → 문제 | 자격증 시험 대비 |
| 매일 아침 리마인더 | 매일 저녁 일기 프롬프트 | AI 활용 저널링 |
| cron + Claude Code 분석 | Notion AI + cron 연동 | .edu 무료 Notion Plus 활용 |

**핵심 패턴**: `입력 데이터 + 에이전트 + 스크립트 + cron = 자동 반복 시스템` — 정기적으로 반복되는 모든 작업에 이 패턴을 적용할 수 있습니다.

> 💡 **Hybrid Tip — 본문 도구 + CC 자동화 조합**
> - **빠른 시작**: 본문의 ChatGPT Tasks로 간단한 반복 알림 설정 (클릭 한 번, 설정 2분)
> - **심화 자동화**: Claude Code + cron으로 커스텀 학습 관리 시스템 구축 (설정 50분, 이후 완전 자동)
> - **최적 조합**: Notion AI(.edu 무료 Plus)로 일정 관리 + cron으로 Claude Code 분석 자동화 — Notion에 기록하고, cron이 매주 분석 보고서를 자동 생성
> - **복습 조합**: Anki/Quizlet로 모바일 플래시카드 + quiz-generator로 서술형·응용 문제 자동 생성 — 암기는 Anki, 이해도 테스트는 Claude Code

> **핵심 정리**: 본문에서 배운 Plan-Do-Check 3축과 간격 반복 원리를 Claude Code + cron으로 **완전 자동화**했습니다. 본문의 핵심 교훈 — "AI에게 현실적 제약을 알려줘라", "요약해줘보다 테스트해줘" — 은 CLAUDE.md 규칙과 에이전트 설계에 그대로 녹아 있습니다. Part 3의 여정을 돌아보면, Ch 7(슬라이드)·Ch 8(이미지)·Ch 9(데이터·문서)에서 Claude Code의 **변환 능력**을 배웠고, 이번 Ch 10에서 **스케줄링**을 더해 완전한 자동화를 달성했습니다. HEPI 2025 보고서에 따르면 AI 활용 학생의 51%가 시간 절약을 경험했는데, cron 자동화까지 갖추면 그 효과는 더욱 극대화됩니다. 이제 여러분은 단순히 "AI에게 질문하는 사람"이 아니라, **"AI가 일하는 시스템을 설계하는 사람"**이 되었습니다. 다음 파트에서는 이 모든 능력을 데이터 분석과 취업 준비에 본격적으로 활용합니다.
