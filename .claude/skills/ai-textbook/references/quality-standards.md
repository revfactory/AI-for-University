# 교재 품질 기준서 — 시중 교안 대비 차별화 전략

## 품질 철학

> "따라하면 누구나 할 수 있고, 결과물은 전문가 수준이어야 한다."

이 교재는 시중의 AI 활용 교안들과 **근본적으로 다른 수준**을 목표로 한다.
기존 교안의 문제점을 정확히 진단하고, 각각을 체계적으로 극복한다.

## 시중 교안의 한계와 우리의 차별화

### 1. 피상적 도구 소개 → 깊이 있는 실전 활용

**시중 교안의 문제:**
- "ChatGPT에 질문을 입력하면 답변이 나옵니다" 수준의 표면적 설명
- 도구 기능 나열에 그침, 실제 활용 맥락 부재
- 스크린샷 몇 장으로 끝나는 얕은 실습

**우리의 기준:**
- 도구를 **왜** 그 상황에서 쓰는지 의사결정 근거 제시
- 동일 작업에 **여러 도구를 비교 실험**한 결과 제시 (벤치마크)
- 프롬프트를 **반복적으로 개선**하는 과정을 그대로 보여줌 (1차 → 2차 → 3차 프롬프트)
- 실패 사례도 포함 — "이렇게 하면 안 됩니다" (Anti-pattern)

### 2. 비현실적 예시 → 실제 대학생 맥락

**시중 교안의 문제:**
- "마케팅 전략을 수립하세요" 같은 직장인 대상 예시
- 대학생 실생활과 동떨어진 시나리오
- "AI로 소설을 써보세요" 같은 비실용적 실습

**우리의 기준:**
- 모든 예시가 **실제 대학 수업 과제** 기반 (경영학 레포트, 공학 실험보고서, 인문학 에세이 등)
- **학과별/학년별** 맞춤 시나리오 (1학년 교양, 3학년 전공, 4학년 졸업논문)
- 실제 **대학교 양식과 요구사항**을 반영한 과제 시뮬레이션
- 취업 준비 예시도 **실제 채용공고**와 **실제 기업** 기반

### 3. 일회성 스킬 → 재현 가능한 워크플로우

**시중 교안의 문제:**
- "이렇게 해보세요" 한 번 하고 끝
- 다른 주제/상황에 적용하는 방법 미제시
- 파편적 팁의 나열

**우리의 기준:**
- 모든 실습이 **일반화 가능한 워크플로우**로 구성
- 워크플로우 다이어그램으로 전체 과정을 시각화
- "이 패턴을 다른 과제에 적용하려면?" 섹션 포함
- **도전 과제 3단계** (기초/응용/심화)로 워크플로우를 다른 맥락에 적용하는 훈련

### 4. 출처 없는 주장 → 실증 기반 사례

**시중 교안의 문제:**
- "AI를 활용하면 효율이 높아집니다" — 어디서 어떻게?
- 검증되지 않은 효과 주장
- 가상의 성공 사례

**우리의 기준:**
- **실제 대학/교육기관의 AI 도입 사례** 조사 (출처 명시)
- **정량적 데이터**: 시간 절약률, 성적 변화, 만족도 등
- **실패 사례와 한계**도 정직하게 포함
- 다각도 조사: 학술 논문, 뉴스 기사, 블로그, 커뮤니티, 공식 보고서

### 5. 텍스트 위주 → 비주얼 리치 콘텐츠

**시중 교안의 문제:**
- 글자로만 가득 찬 슬라이드/문서
- 저해상도 스크린샷
- 디자인 없는 무미건조한 레이아웃

**우리의 기준:**
- **챕터당 5~7장의 고품질 이미지** (Gemini 3 Pro로 생성)
- 워크플로우 다이어그램, 인포그래픽, 개념도 필수 포함
- 일관된 비주얼 아이덴티티 (색상, 스타일, 캐릭터)
- Before/After를 **시각적으로** 비교

## 따라하기 실습 품질 기준

### Step-by-Step 작성 규칙

1. **한 스텝에 한 동작만**: "로그인하고 새 채팅을 열고 프롬프트를 입력하세요" ✗ → 3개 스텝으로 분리
2. **화면 기준 안내**: "왼쪽 사이드바 하단의 '+' 버튼을 클릭합니다" (위치 명시)
3. **예상 결과 명시**: 각 스텝 후 "이렇게 보이면 정상입니다" 포함
4. **분기 처리**: "만약 ~가 보이지 않으면" 트러블슈팅 포함
5. **프롬프트는 복사 가능**: 코드블록으로 감싸고, "이 프롬프트를 그대로 복사하여 붙여넣으세요" 안내

### 실습 난이도 기호

| 기호 | 난이도 | 소요 시간 | 설명 |
|------|--------|----------|------|
| ⭐ | 기초 | 5~10분 | AI 처음 써보는 분도 OK |
| ⭐⭐ | 응용 | 15~30분 | 기본기를 다른 상황에 적용 |
| ⭐⭐⭐ | 심화 | 30~60분 | 여러 도구/기법을 결합 |

### 프롬프트 품질 기준

**S급 프롬프트 (교재에 실리는 모든 프롬프트):**
- 복사하여 즉시 사용 가능 (placeholder 최소화, 있으면 `[여기에 본인의 주제 입력]` 형태로 명시)
- 맥락 제공 (역할, 상황, 제약조건)
- 출력 형식 명시 (구조, 분량, 스타일)
- "이 프롬프트가 좋은 이유" 해설 포함
- "이렇게 바꾸면 이런 결과가" 변형 가이드 포함

**프롬프트 진화 패턴 (3단계 개선):**
```
🔴 1차 프롬프트 (흔한 실수): "레포트 써줘"
→ 결과: 너무 일반적, 맥락 없음

🟡 2차 프롬프트 (개선): "경영학 레포트 개요를 작성해줘. 디지털 트랜스포메이션 주제로."
→ 결과: 나아졌지만 아직 구체성 부족

🟢 3차 프롬프트 (최적): [교재의 S급 프롬프트]
→ 결과: 바로 활용 가능한 품질
```

## 실증 사례 조사 기준 (다각도 조사)

### 조사 채널 (최소 3개 이상 교차검증)

| 채널 | 조사 대상 | 기대 자료 |
|------|----------|----------|
| **학술 논문** | AI 교육 활용 연구 | 정량적 데이터, 교수법 분석 |
| **뉴스/미디어** | 대학 AI 도입 기사 | 정책, 프로그램, 트렌드 |
| **대학 공식** | AI 교육 프로그램, 가이드라인 | 공식 정책, 커리큘럼 |
| **블로그/유튜브** | 학생/교수 경험담 | 1인칭 활용 사례, 팁 |
| **커뮤니티** | 에브리타임, 레딧, X | 비공식 활용 패턴, 솔직한 평가 |
| **AI 도구 공식** | ChatGPT/Claude/Gemini 블로그 | 교육 분야 활용 사례, 기능 업데이트 |
| **해외 대학** | MIT OCW, Stanford AI 등 | 글로벌 베스트 프랙티스 |

### 사례 품질 등급

| 등급 | 기준 | 표시 |
|------|------|------|
| **A급** | 출처 명확 + 정량 데이터 + 재현 가능 | ✅ 검증 완료 |
| **B급** | 출처 있음 + 정성적 설명 | 📝 출처 확인 |
| **C급** | 합리적 추론 기반 구성 사례 | 💡 구성 사례 |

A급 사례를 최대한 확보하되, 부족한 영역은 B/C급으로 보완. C급은 챕터당 1개 이하로 제한.

## 최종 품질 검증 체크리스트

### 필수 통과 기준 (PASS 조건)

- [ ] **따라하기 재현성**: 프롬프트를 그대로 복사하여 동일/유사 결과를 얻을 수 있는가?
- [ ] **도구 정보 정확성**: 도구의 기능, 가격, 접근 방법이 2025년 기준으로 정확한가?
- [ ] **실증 사례 신뢰성**: 최소 2개 이상의 A/B급 사례가 포함되어 있는가?
- [ ] **비주얼 완성도**: 챕터당 5장 이상의 고품질 이미지가 포함되어 있는가?
- [ ] **워크플로우 일반화**: 실습이 다른 주제/상황에도 적용 가능한 패턴으로 제시되었는가?
- [ ] **프롬프트 3단계 진화**: 최소 1개의 프롬프트 진화 패턴 (🔴→🟡→🟢)이 포함되었는가?
- [ ] **실패 사례 포함**: "이렇게 하면 안 됩니다" Anti-pattern이 1개 이상 포함되었는가?
- [ ] **CC 섹션 실행 가능성**: CC 섹션의 모든 명령어가 실제 실행 가능한가?

### 차별화 기준 (시중 교안 대비)

- [ ] **깊이**: 단순 기능 소개가 아닌, 의사결정 근거와 비교 분석 포함
- [ ] **현실성**: 모든 예시가 실제 대학생 과제/상황 기반
- [ ] **재현성**: 독자가 책만 보고 따라할 수 있는 구체성
- [ ] **비주얼**: 전문 디자인 수준의 이미지와 인포그래픽
- [ ] **정직성**: 성공 사례뿐 아니라 한계와 실패도 다룸
- [ ] **확장성**: 일회성 팁이 아닌 재사용 가능한 워크플로우 패턴
