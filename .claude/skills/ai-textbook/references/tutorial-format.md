# 따라하기 실습 형식 가이드

## 핵심 원칙

> "독자가 책 옆에 노트북을 펴고, 한 스텝씩 따라하면 멋진 결과물이 나온다."

따라하기 실습은 이 교재의 핵심 자산이다. 다음 형식을 엄격히 준수한다.

## 따라하기 실습 표준 포맷

```markdown
### 🎯 실습: [실습 제목] [난이도 기호]

**목표**: [이 실습을 마치면 무엇을 얻는지]
**소요 시간**: [예상 시간]
**준비물**: [필요한 도구, 계정, 파일 등]
**사용 도구**: [ChatGPT / Claude / Gemini / Perplexity / 기타]

---

#### Step 1. [동작] ⏱️ [소요 시간]

[구체적 지시사항. 한 스텝에 한 동작만.]

> 📍 **화면 안내**: [어디를 클릭/입력하는지 정확한 위치]

[필요시 이미지 삽입 위치]
<!-- IMAGE: Step 1 화면 안내 - [구체적 설명] -->

✅ **체크**: [이 단계가 성공했을 때 보이는 화면/결과]

---

#### Step 2. 프롬프트 입력 ⏱️ [소요 시간]

다음 프롬프트를 **그대로 복사**하여 붙여넣으세요:

```
[프롬프트 내용]
```

> 💡 **커스텀 포인트**: `[대괄호 안의 내용]`은 본인 상황에 맞게 수정하세요.
> 예시: [실제 변형 예시]

✅ **체크**: [기대되는 AI 응답의 핵심 특징]

---

#### 🎉 완성! — 결과물 미리보기

[기대 결과물 예시 또는 스크린샷]

> - AI 활용: [시간/노력]
> - 절약: [구체적 수치]
```

## 프롬프트 진화 패턴 포맷

각 챕터의 핵심 실습에는 반드시 프롬프트 진화 패턴을 포함한다:

```markdown
### 🔬 프롬프트 진화 실험

같은 작업을 3단계 프롬프트로 시도하여, 프롬프트 품질의 차이를 직접 체감합니다.

#### 🔴 Level 1 — 초보 프롬프트

```
[흔히 쓰는 단순한 프롬프트]
```

**결과**: [실제 AI 응답 요약 또는 예시]

📋 **평가**:
- 정확성: ★★☆☆☆
- 활용도: ★☆☆☆☆
- 문제점: [구체적 문제]

---

#### 🟡 Level 2 — 중급 프롬프트

```
[맥락과 조건을 추가한 프롬프트]
```

**결과**: [실제 AI 응답 요약 또는 예시]

📋 **평가**:
- 정확성: ★★★★☆
- 활용도: ★★★☆☆
- 개선점: [Level 1 대비 무엇이 나아졌는지]
- 아쉬운 점: [아직 부족한 점]

---

#### 🟢 Level 3 — 전문가 프롬프트

```
[완전하고 구조화된 최적 프롬프트]
```

**결과**: [실제 AI 응답 요약 또는 예시]

📋 **평가**:
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: [Level 2 대비 결정적으로 달라진 점]

---

#### 💡 교훈: 프롬프트의 마법

| 요소 | Level 1 | Level 2 | Level 3 |
|------|---------|---------|---------|
| 맥락 제공 | ✗ | △ | ✓ |
| 조건 명시 | ✗ | △ | ✓ |
| 출력 형식 | ✗ | ✗ | ✓ |
| 예시/참고 | ✗ | ✗ | ✓ |
| 결과 품질 | 20% | 60% | 95% |
```

## Anti-Pattern (이렇게 하면 안 됩니다) 포맷

```markdown
### ❌ 흔한 실수: [실수 이름]

**상황**: [어떤 상황에서 이 실수를 하는지]

**잘못된 방법**:
```
[잘못된 프롬프트 또는 방법]
```

**결과**: [왜 이것이 문제인지 — 실제 결과 예시]

**올바른 방법**:
```
[수정된 프롬프트 또는 방법]
```

**차이**: [무엇을 바꿨고 왜 나아졌는지]
```

## 실습 배치 전략

각 챕터의 실습은 다음 순서로 배치:

1. **도구 비교 실험** (Section 3: AI 도구 소개와 연계) — 도구의 차이를 체감
2. **프롬프트 진화 패턴** (Section 4: 핵심 개념과 연계) — 프롬프팅 스킬 학습
3. **메인 따라하기 실습** (Section 5: 따라하기 실습) — 핵심 워크플로우 체득
4. **Anti-Pattern** (Section 7: 꿀팁 & 주의사항과 연계) — 실수 방지
5. **도전 과제** (Section 8) — 배운 것을 새로운 맥락에 적용

## 워크플로우 패턴화

모든 따라하기 실습은 "이 과제에서만 쓰는 스킬"이 아닌 "패턴"으로 제시한다:

```markdown
### 🔄 이 워크플로우를 다른 상황에 적용하기

이 실습에서 배운 패턴은 다음 상황에도 적용할 수 있습니다:

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| [실습 작업] | [바꿀 부분] | [새로운 활용] |
| [실습 작업] | [바꿀 부분] | [새로운 활용] |
| [실습 작업] | [바꿀 부분] | [새로운 활용] |

**핵심 패턴**: [일반화된 워크플로우 패턴 한 줄 요약]
```
