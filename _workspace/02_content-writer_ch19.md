## 📚 학습 목표

이 챕터를 마치면 다음을 할 수 있습니다:

1. **AI 도구로 시장 조사**를 수행하고, 데이터 기반으로 사업 기회를 평가할 수 있다

4. **AI로 피치덱을 제작**하고 발표를 준비할 수 있다
5. 창업 기획의 **Anti-Pattern**(기술 우선 함정, 비용 간과)을 인식하고 피할 수 있다

## 2. 핵심 개념 이해

### 2.1 린 캔버스(Lean Canvas)란?

린 캔버스는 에릭 리스(Eric Ries)의 린 스타트업 방법론을 Ash Maurya가 1페이지 프레임워크로 정리한 것입니다. 기존 사업계획서가 수십 페이지를 요구하는 것과 달리, **9개 블록에 핵심만 담습니다.**

| 블록 | 질문 | 핵심 |
|------|------|------|
| 1. 문제 (Problem) | 고객의 상위 3가지 문제는? | 가장 아픈 곳 |
| 2. 고객 세그먼트 | 누가 이 문제를 가지고 있는가? | 얼리어답터 특정 |
| 3. 고유 가치 제안 | 왜 이 솔루션을 써야 하는가? | 한 줄 차별점 |
| 4. 솔루션 | 각 문제를 어떻게 해결하는가? | 핵심 기능 3개 |
| 5. 채널 | 고객에게 어떻게 도달하는가? | 마케팅 경로 |
| 6. 수익 모델 | 어떻게 돈을 버는가? | 과금 방식 |
| 7. 비용 구조 | 무엇에 돈이 드는가? | 고정/변동 비용 |
| 8. 핵심 지표 | 성공을 어떻게 측정하는가? | KPI 3개 |
| 9. 경쟁 우위 | 쉽게 따라할 수 없는 것은? | 해자(moat) |

### 2.2 MVP(Minimum Viable Product)란?

MVP는 **핵심 가설을 검증할 수 있는 최소 기능의 제품**입니다. "완벽한 제품"이 아니라 "학습을 위한 제품"입니다.

**MoSCoW 기법**으로 기능의 우선순위를 정합니다:

- **M**ust Have: MVP에 반드시 포함 (핵심 가치 검증용)
- **S**hould Have: 가능하면 포함
- **C**ould Have: 시간 여유가 있으면
- **W**on't Have: V2에서 개발

### 2.3 피치덱(Pitch Deck)이란?

투자자나 심사위원에게 사업을 설명하는 **10~15장짜리 프레젠테이션**입니다. 구조는 보통 다음 10장을 따릅니다:

1. 표지 (서비스명 + 한줄 소개)
2. 문제 (고객의 핵심 페인포인트)
3. 솔루션 (우리의 해결 방법)
4. 시장 규모 (TAM/SAM/SOM)
5. 비즈니스 모델 (수익 구조)
6. 경쟁 우위 (차별화 포인트)
7. 제품 데모/목업 (핵심 화면)
8. 팀 소개 (왜 우리 팀이 해야 하는지)
9. 재무 계획 (3년 매출 예측)
10. Ask (투자 요청 금액 / 대회 목표)

## 3. 따라하기 실습: AI로 4주 창업 기획 완성하기

> **실습 시나리오:** "대학생 대상 AI 학습 도우미 앱"을 기획합니다. 시장 조사부터 피칭까지 AI 도구만으로 전 과정을 진행합니다.

### 실습 A: Perplexity로 시장 조사하기 ⭐

**사용 도구:** Perplexity (perplexity.ai)

**Step 1:** Perplexity에 접속하여 로그인합니다.

**Step 2:** 다음 프롬프트를 입력합니다:

```
"대학생 AI 학습 도우미" 서비스의 시장성을 분석해주세요:

1. TAM/SAM/SOM 시장 규모 추정 (한국 기준)
2. 시장 성장률 및 트렌드 (최근 3년)
3. 주요 경쟁사 5곳 비교표 (제품, 가격, 강점, 약점)
4. 타겟 고객 세그먼트 및 규모
5. 진입 장벽 및 기회 요인

모든 데이터에 출처를 포함해주세요.
```

**Step 3:** Perplexity의 답변에서 **출처가 포함된 데이터**를 확인합니다.

✅ **체크포인트:** TAM/SAM/SOM 수치와 출처가 포함되어 있나요? 경쟁사 비교표(5곳 이상)와 시장 성장률 수치가 출력에 있나요?

**Step 4:** 후속 질문으로 깊이를 더합니다:

```
위 분석에서 한국 대학생 시장에 집중해서 추가 분석해주세요:
- 한국 대학생 약 300만 명 중 AI 학습 도구 사용 비율
- 월 지불 의향(WTP) 추정
- 학기 중 vs 방학 중 사용 패턴 차이
```

🔧 **트러블슈팅:** Perplexity가 한국 데이터를 충분히 제공하지 않으면, "한국어로 검색해서" 또는 "site:kr" 키워드를 추가로 안내해보세요.

> **왜 Perplexity인가?** 시장 조사에서 가장 중요한 것은 **데이터의 출처**입니다. ChatGPT나 Claude는 출처를 생성(hallucination)할 수 있지만, Perplexity는 실시간 검색 결과에 **인라인 출처(Citation)**를 제공하므로 신뢰도가 높습니다.

### 실습 B: ChatGPT로 린 캔버스 작성하기 ⭐⭐

**사용 도구:** ChatGPT (chatgpt.com)

**Step 1:** ChatGPT에 다음 프롬프트를 입력합니다:

```
다음 사업 아이디어로 린 캔버스(Lean Canvas)를 작성해주세요:

사업 아이디어: 대학생 AI 학습 도우미 — 강의 노트를 업로드하면
AI가 핵심 요약, 퀴즈, 플래시카드를 자동 생성하는 앱
타겟 고객: 한국 대학생 (특히 시험 기간에 효율적 학습이 필요한 학생)
핵심 문제: 학습 자료 정리에 시간이 너무 많이 걸림

린 캔버스 9개 블록 모두 채워주세요:
1. 문제 (Problem) — 상위 3가지 문제
2. 고객 세그먼트 (Customer Segments) — 얼리어답터 포함
3. 고유 가치 제안 (Unique Value Proposition)
4. 솔루션 (Solution) — 문제별 핵심 기능
5. 채널 (Channels) — 고객 도달 경로
6. 수익 모델 (Revenue Streams)
7. 비용 구조 (Cost Structure)
8. 핵심 지표 (Key Metrics)
9. 경쟁 우위 (Unfair Advantage)

각 블록에 구체적 근거/수치를 포함하고,
표 형태로 한눈에 볼 수 있게 정리해주세요.
```

**Step 2:** 결과를 복사해 **Notion이나 Google Docs**에 붙여넣습니다.

**Step 3:** Claude에 같은 린 캔버스를 요청하여 **교차 검증**합니다:

```
[ChatGPT가 생성한 린 캔버스를 붙여넣고]

위 린 캔버스를 비판적으로 검토해주세요:
1. 각 블록에서 약한 점이나 누락된 부분
2. 가설(Hypothesis)이 명확하지 않은 블록
3. 검증 우선순위 Top 3
4. MVP로 검증 가능한 실험 3가지 제안
```

✅ **체크포인트:** 9개 블록이 모두 채워져 있나요? 수익 모델에 구체적 가격/과금 방식이 포함되어 있나요? 경쟁 우위가 "AI 기술" 같은 모호한 표현이 아닌, 구체적인가요?

> **핵심 팁:** AI가 생성한 린 캔버스는 **초안**입니다. 가장 중요한 작업은 "이 가설이 맞는가?"를 실제 사용자에게 물어보는 것입니다. AI는 구조를 잡아주지만, **검증은 사람이 합니다.**

### 실습 C: Lovable로 MVP 만들기 ⭐⭐

**사용 도구:** Lovable (lovable.dev)

<!-- IMAGE: workflow - "Lovable MVP 제작 과정 스크린샷 시퀀스: (1) 프롬프트 입력 → (2) AI가 코드 생성 → (3) 실시간 미리보기 → (4) Supabase 연동 → (5) 배포". 테마 컬러: #E74C3C 레드 -->

**Step 1:** lovable.dev에 접속하여 "Start building"을 클릭합니다.

**Step 2:** 다음 프롬프트를 입력합니다:

```
대학생 AI 학습 도우미 웹앱을 만들어줘.

핵심 기능 (Must Have):
1. 로그인/회원가입 (이메일)
2. 강의 노트 텍스트 입력
3. AI 요약 생성 (핵심 포인트 5개)
4. 퀴즈 자동 생성 (객관식 5문제)
5. 대시보드 (내 학습 기록)

디자인:
- 깔끔한 모던 UI
- 모바일 반응형
- 메인 컬러: #4A90D9 (파란색 계열)

기술 스택: React + TypeScript + Supabase
```

**Step 3:** Lovable이 코드를 생성하는 동안 실시간 미리보기를 확인합니다.

**Step 4:** Supabase 연동 설정:
- Lovable 대시보드에서 "Connect Supabase" 클릭
- Supabase 프로젝트 생성 (무료)
- 데이터베이스 테이블 자동 생성 확인

**Step 5:** 수정 요청:

```
회원가입 페이지에 "대학교 선택" 드롭다운을 추가해줘.
주요 대학교 20개 목록을 포함해줘.
```

✅ **체크포인트:** 웹앱이 실시간으로 미리보기에 표시되나요? 로그인/회원가입이 작동하나요? AI 요약 기능이 동작하나요? (API 키 설정 필요)

🔧 **트러블슈팅:**
- **"빌드 에러" 발생:** Lovable 채팅에 에러 메시지를 그대로 붙여넣으면 AI가 자동으로 수정합니다
- **Supabase 연동 실패:** Supabase URL과 anon key가 올바른지 확인하세요
- **AI 기능 미작동:** OpenAI API 키를 환경 변수에 추가해야 합니다

**MVP 도구 비교 — 어떤 도구를 선택할까?**

| 상황 | 추천 도구 | 개발 기간 | 비용 |
|------|----------|----------|------|
| 랜딩 페이지 (검증용) | v0 + Vercel | 1일 | 무료 |
| 웹앱 MVP (DB 포함) | **Lovable** | 1~2주 | $25/월 |
| 빠른 프로토타입 | Bolt.new | 2~3일 | ~$20/월 |
| 복잡한 앱 (AI 연동) | Replit Agent | 2~4주 | $25/월 |
| 모바일 앱 MVP | Replit + React Native | 3~4주 | $25/월 |

> **Lovable vs Bolt.new — 핵심 차이:**
> - **Lovable:** 구조적으로 계획 → 코드 생성, Supabase 내장 통합, 초보자 친화도 ★★★★★
> - **Bolt.new:** 빠른 즉시 코드 생성(diff 방식), 다중 프레임워크 지원, 속도 최우선

### 실습 D: Gamma로 피치덱 만들기 ⭐⭐

**사용 도구:** Gamma (gamma.app)

**Step 1:** gamma.app에 접속하여 "Create" → "Presentation"을 선택합니다.

**Step 2:** 다음 내용을 입력합니다:

```
대학생 AI 학습 도우미 "StudyMate AI" 피치덱.

투자자/심사위원용 10장 슬라이드:
1. 표지: StudyMate AI — "강의 노트가 시험 대비 자료로 변하는 60초"
2. 문제: 대학생의 학습 자료 정리 시간 — 평균 주 8시간
3. 솔루션: AI 자동 요약 + 퀴즈 생성 + 플래시카드
4. 시장 규모: 한국 대학생 300만 × EdTech 시장 성장률
5. 비즈니스 모델: Freemium — 기본 무료, Pro 월 9,900원
6. 경쟁 우위: 한국어 특화 + 대학 강의 맥락 이해
7. 제품 데모: MVP 스크린샷 3장
8. 팀: 팀원 소개 (역할, 역량)
9. 재무: 3년 매출 예측 (시나리오별)
10. Ask: 초기 투자 5,000만 원 / 대회 수상 목표
```

**Step 3:** Gamma가 60초 내로 피치덱을 생성합니다. 각 슬라이드의 레이아웃과 디자인을 확인합니다.

**Step 4:** 발표 스크립트를 ChatGPT로 작성합니다:

```
다음 피치덱에 맞는 발표 스크립트를 작성해주세요.
발표 시간: 10분 (슬라이드당 1분)
청중: VC + 교수 심사위원

각 슬라이드에:
- 핵심 메시지 1줄
- 발표 스크립트 (60초 분량)
- 전환 멘트 (다음 슬라이드로 넘어가는 연결)
```

**Step 5:** Q&A 대비:

```
위 피치덱에 대해 VC 심사위원이 할 수 있는
까다로운 질문 10개와 각각에 대한 모범 답변을 준비해주세요.

예상 질문 유형:
- 시장 규모의 근거
- 경쟁사 대비 차별점
- 수익화 전략
- 팀 역량
- 확장 계획
```

✅ **체크포인트:** 피치덱 10장이 모두 생성되었나요? 각 슬라이드에 데이터/수치가 포함되어 있나요? 발표 스크립트가 10분 분량인가요?

## 4. 프롬프트 진화 패턴

### 창업 아이디어 프롬프트의 진화

**🔴 Level 1 — 단순 요청:**

```
대학생 창업 아이디어 추천해줘
```

→ 일반적이고 누구에게나 해당되는 답변. 실행 불가능.

**🟡 Level 2 — 맥락 추가:**

```
대학생 대상 AI 기반 서비스 창업 아이디어 5개와
각각의 비즈니스 모델을 제안해줘.
```

→ 분야(AI)와 산출물(BM)을 지정했으나, 구체적 제약 조건이 없음.

**🟢 Level 3 — 완전한 맥락 + 제약 조건:**

```
대학생 창업 경진대회(예비창업패키지) 준비:
■ 분야: EdTech (AI 학습 도우미)
■ 요청:
1. 린 캔버스 9블록 (구체적 수치 포함)
2. TAM/SAM/SOM (한국 대학생 600만명 기준)
3. 경쟁사 5곳 포지셔닝 맵 (기능 × 가격)
4. MVP 기능 MoSCoW 분류 (Lovable 개발 기준)
5. 3년 재무 모델 (MAU 시나리오별)
6. 피치덱 10장 구조 + 슬라이드별 스크립트
■ 대회: 발표 10분 + Q&A 5분, 심사위원 = VC + 교수
```

→ 대회명, 분야, 도구, 청중까지 모두 지정. AI가 맞춤형 결과를 생성.

## 5. 도구 비교 실험: 같은 프롬프트, 4개 도구

> **실험 프롬프트:** "대학생 대상 AI 학습 도우미 서비스의 린 캔버스를 작성해줘. 9개 블록을 모두 채우고, 각 블록에 구체적 수치를 포함해줘."

| 평가 항목 | ChatGPT | Claude | Gemini | Perplexity |
|----------|---------|--------|--------|------------|
| **구조 완성도** | ★★★★★ | ★★★★★ | ★★★★☆ | ★★★☆☆ |
| **수치/데이터** | ★★★★☆ | ★★★★★ | ★★★★☆ | ★★★★★ |
| **비판적 관점** | ★★★☆☆ | ★★★★★ | ★★★☆☆ | ★★★☆☆ |
| **출처 포함** | ★★☆☆☆ | ★★☆☆☆ | ★★★☆☆ | ★★★★★ |
| **실행 가능성** | ★★★★★ | ★★★★☆ | ★★★★☆ | ★★★☆☆ |

**도구별 특징:**

- **ChatGPT:** 가장 완성도 높은 캔버스. 빠르게 구조를 잡고 실행 가능한 수준으로 제시. 다만 수치의 출처가 불명확할 수 있음.
- **Claude:** 가설을 비판적으로 분석. "이 가정이 약합니다"라고 직접 지적. BM 설계에서 논리 검증에 최적.
- **Gemini:** Google 검색 연동으로 최신 데이터 반영. 다만 캔버스 포맷이 다소 느슨할 수 있음.
- **Perplexity:** 출처가 가장 풍부. 시장 규모 등 **데이터 검증**에 최적이나, 캔버스 구조 완성도는 낮음.

**최적 조합:** Perplexity(시장 데이터) → ChatGPT(캔버스 초안) → Claude(비판적 검증)

## 6. 실전 사례: AI 창업은 이미 일어나고 있다

### 사례 1: Y Combinator — 코드의 95%가 AI 생성

Y Combinator 2025년 Winter 배치에서 **스타트업의 25%가 코드베이스의 95% 이상을 AI로 생성**했습니다. 이 스타트업들의 공통점은:

- 기술이 아닌 **문제**에서 출발함
- MVP를 **2~4주 내**에 구축함
- 빠른 반복(iteration)으로 **제품-시장 적합성(PMF)**을 검증함
- 수직형(vertical) AI — 범용이 아닌 **도메인 특화** 솔루션

2026년 핵심 트렌드는 **수직형 AI**입니다. 범용 챗봇을 만드는 것이 아니라, 특정 도메인(의료, 법률, 교육, 물류 등)의 문제를 AI로 해결하는 서비스가 투자를 받고 있습니다.

### 사례 2: Columbia AI Startup Challenge — 대학생 AI 창업 대회

Columbia Business School이 주관하는 AI Startup Challenge에는 Columbia, Cornell Tech, NYU, Yale 등 명문 대학원생이 참가하며, **$40,000 이상의 상금**이 걸려 있습니다. "Applied AI and the Future of Business"를 주제로, 실제 시장에서 작동하는 AI 비즈니스 모델을 피칭합니다.

한국에서도 유사한 기회가 풍부합니다:

| 프로그램 | 대상 | 특징 |

## 7. 꿀팁 & 주의사항

### 꿀팁 5선

**Tip 1: 도구 조합의 황금 공식**

```
시장 데이터: Perplexity (출처 포함)
      ↓
BM 설계: ChatGPT (초안) → Claude (검증)
      ↓
MVP: Lovable (비개발자) 또는 Cursor (개발자)
      ↓
피치덱: Gamma (60초 생성)
      ↓
발표 준비: ChatGPT (스크립트 + Q&A)
```

**Tip 2: MVP는 "최소"에 집중하라**

기능을 5개 이하로 줄이세요. Lovable이나 Bolt.new는 기능이 많아질수록 에러가 급증합니다. Must Have만 구현하고 나머지는 V2로 미루세요.

**Tip 3: 린 캔버스는 1주일마다 업데이트하라**

AI가 처음 생성한 린 캔버스는 가설(hypothesis)입니다. 매주 실제 피드백을 반영하여 업데이트하세요. 특히 "문제"와 "고객 세그먼트" 블록이 가장 많이 바뀝니다.

**Tip 4: Perplexity 시장 조사에서 PEST 분석을 추가하라**

시장 규모(TAM/SAM/SOM)만으로는 부족합니다. "PEST 분석(정치/경제/사회/기술)"을 함께 요청하면 심사위원이 감탄하는 시장 분석이 됩니다.

**Tip 5: 피치덱은 "문제 슬라이드"가 80%를 결정한다**

심사위원은 첫 2장(표지 + 문제)에서 관심 여부를 결정합니다. 문제가 공감되지 않으면 나머지 8장은 의미가 없습니다. 문제 슬라이드에 가장 많은 시간을 투자하세요.

### ⚠️ Anti-Pattern 1: "기술이 있어서 만든 것 vs 문제가 있어서 만든 것"

**상황:** 팀 K가 "GPT API가 있으니까 뭔가 만들자"로 시작.

**잘못된 방법:** 기술(GPT API)에서 출발 → "이걸로 뭘 할 수 있지?" → 범용 챗봇 제작.

**결과:**
- 차별점 없는 서비스
- 심사위원: "기존 ChatGPT와 뭐가 다른가요?"
- 사용자도 ChatGPT를 직접 쓰면 되므로 전환 이유 없음

**올바른 방법:**
1. 문제에서 출발 → "대학생이 학습 자료 정리에 주 8시간 쓴다"
2. 해결 방법으로 AI를 적용 → 강의 노트 자동 요약 + 퀴즈 생성
3. **수직형(vertical) AI** — 범용이 아닌 도메인 특화

**차이:** "기술이 있어서 만든 것"은 시장이 없고, "문제가 있어서 만든 것"은 시장이 있습니다. Getting Smart(2025)의 조언: **"실제 문제를 해결하라."** Y Combinator 데이터도 동일합니다 — 코드의 95%를 AI로 생성할 수 있어도, **문제 정의와 시장 검증은 여전히 인간의 몫**입니다.

---

### ⚠️ Anti-Pattern 2: "AI API 비용을 간과한 서비스 기획"

**상황:** 팀 J가 ChatGPT API로 "대학생 맞춤 학습 도우미" 프로토타입 완성.

**잘못된 방법:** 무료 서비스로 출시 → 사용자 100명 확보 → API 비용 월 50만 원 발생.

**결과:**
- 수익 모델 없이 운영 → 2개월 만에 서비스 중단
- "사용자가 늘수록 적자가 커지는" 구조

**올바른 방법:**
1. 기획 단계에서 비용 구조를 설계: API 호출당 비용 × 예상 사용량 = 월 비용
2. 과금 모델을 역산하여 Freemium이라도 비용 상한(cap)을 설계
3. 무료 사용자에게는 응답 횟수 제한, Pro 사용자에게만 무제한 제공

**차이:** AI API 비용은 사용량에 비례합니다. 무료 서비스로는 지속 불가능하며, 기획 단계에서 반드시 비용 구조를 설계해야 합니다.

**비용 계산 예시:**
- GPT-4o API: 입력 $2.5/100만 토큰, 출력 $10/100만 토큰
- 사용자 1명이 하루 평균 5,000 토큰 사용 시
- 100명 × 30일 × 5,000 토큰 = 1,500만 토큰/월
- 비용: 약 $15~$150/월 (모델에 따라 다름)
- **반드시 기획 단계에서 계산하세요**

## 8. 워크플로우 일반화: AI 창업 기획의 보편 원칙

지금까지의 실습에서 도출한 창업 기획의 보편 원칙입니다. **어떤 사업 아이디어에든 적용할 수 있습니다.**

### 원칙 1: 문제 → 솔루션 → 기술 순서를 지켜라

```
❌ 기술(AI가 있으니까) → 솔루션(뭔가 만들자) → 문제(누가 쓸까?)
✅ 문제(누구의 어떤 고통?) → 솔루션(어떻게 해결?) → 기술(AI가 최선인가?)
```

### 원칙 2: AI는 속도의 도구이지, 판단의 도구가 아니다

| AI가 해주는 것 | AI가 해줄 수 없는 것 |
|---------------|-------------------|
| 시장 데이터 수집 | 이 시장에 진입할지 결정 |
| 린 캔버스 구조화 | 가설이 맞는지 검증 |
| MVP 코드 생성 | 사용자가 원하는지 확인 |
| 피치덱 디자인 | 청중을 설득하는 열정 |

### 원칙 3: 4주 파이프라인

<!-- IMAGE: infographic - "AI 창업 기획 4주 파이프라인: Week1(발견: Perplexity+ChatGPT) → Week2(기획: ChatGPT+Claude+CI) → Week3(MVP: Lovable/Replit) → Week4(발표: Gamma+ChatGPT)" 타임라인 다이어그램. 테마 컬러: #E74C3C 레드 -->

| 주차 | 활동 | 핵심 도구 | 산출물 |
|------|------|----------|--------|
| **Week 1** | 시장 조사 + 문제 정의 | Perplexity, ChatGPT | 시장 분석 보고서, 페르소나 |
| **Week 2** | BM 설계 + 재무 모델 | ChatGPT, Claude, CI | 린 캔버스, 재무 모델 |
| **Week 3** | MVP 개발 | Lovable / Replit | 작동하는 프로토타입 |
| **Week 4** | 피치덱 + 발표 준비 | Gamma, ChatGPT | 피치덱 10장, 스크립트 |

### 원칙 4: 검증 우선 순위

모든 가설을 한꺼번에 검증할 수 없습니다. 다음 순서로 검증하세요:

1. **문제 가설:** "이 문제가 진짜 존재하는가?" → 잠재 사용자 10명 인터뷰
2. **솔루션 가설:** "우리 방법이 이 문제를 해결하는가?" → MVP 사용 테스트
3. **비즈니스 가설:** "돈을 낼 의향이 있는가?" → 가격 실험 (A/B 테스트)

## 9. 도전 과제

### ⭐ 기본 과제: AI로 린 캔버스 작성하기

**목표:** 자신만의 사업 아이디어로 린 캔버스를 작성하세요.

1. 주변에서 느끼는 불편함(페인포인트) 3가지를 적으세요
2. ChatGPT에 린 캔버스 프롬프트(실습 2 참고)를 입력하세요
3. Claude에 교차 검증을 요청하세요
4. 두 결과를 비교하여 최종 린 캔버스를 완성하세요

**제출물:** 린 캔버스 1장 (9개 블록 모두 완성)

### ⭐⭐ 심화 과제: 노코드 MVP 만들기

**목표:** Lovable 또는 Bolt.new로 작동하는 MVP를 구현하세요.

1. 기본 과제의 린 캔버스에서 Must Have 기능 3개를 선정하세요
2. Lovable에 MVP 프롬프트를 입력하세요 (실습 3 참고)
3. Supabase를 연동하여 데이터 저장 기능을 추가하세요
4. 최소 3명의 친구에게 사용하게 하고 피드백을 수집하세요
