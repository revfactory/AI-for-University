## 1. 디자인 씽킹 × AI 도구 매핑

### 1.1 디자인 씽킹 5단계별 AI 활용

| 단계 | 목적 | 추천 AI 도구 | 활용법 |
|------|------|------------|-------|
| **1. 공감(Empathize)** | 사용자 이해 | Perplexity, ChatGPT | 사용자 리서치, 인터뷰 질문 생성, 페르소나 |
| **2. 정의(Define)** | 문제 정의 | Claude, ChatGPT | HMW 질문, 문제 재정의 |
| **3. 아이디어(Ideate)** | 해결책 발상 | ChatGPT, Claude, Gemini | SCAMPER, 역발상, 아이디어 매트릭스 |
| **4. 프로토타입(Prototype)** | 시제품 제작 | v0, Lovable, Bolt.new, Replit | 노코드 웹앱/UI 빠른 제작 |
| **5. 테스트(Test)** | 검증 | ChatGPT, Google Forms+Gemini | 테스트 시나리오, 피드백 분석 |

### 1.2 노코드 프로토타입 도구 비교

| 도구 | 가격 | 핵심 기능 | 코딩 필요 | 추천 상황 |
|------|------|----------|----------|----------|
| **v0 (Vercel)** | Free 제한 / 유료 | 자연어→React UI 생성 | ⚠️ 수정 시 도움됨 | **UI 프로토타입 1순위** |
| **Lovable** | Free / Pro $25/월 | 풀스택 앱, Supabase DB 통합 | ❌ 노코드 가능 | **비개발자, DB 필요 앱** |
| **Bolt.new** | Free / ~$20/월 | 빠른 프로토타이핑, 다중 프레임워크 | ⚠️ 편집 가능 | **래피드 프로토타입** |
| **Replit** | Free / Core $25/월 | AI Agent로 앱 빌드, 즉시 배포 | ❌ Agent가 코딩 | **학생, 협업 프로젝트** |
| **Figma + AI** | Free / $15/유저/월 | UI/UX 디자인, 인터랙션 | ❌ | **디자인 프로토타입** |

> **Lovable vs Bolt.new:** Lovable은 구조적 계획 → 코드 생성, Supabase 내장으로 비개발자에게 최적. Bolt.new는 빠른 diff 방식으로 속도가 핵심인 래피드 프로토타입에 적합.

## 2. 핵심 개념: AI가 창의적 문제 해결에 미치는 영향

### 2.1 연구가 입증한 효과

**ScienceDirect(2024) 실험 연구:**
- ChatGPT가 대학생의 **창의적 문제 해결 성과를 향상**
- **유창성(fluency)과 유연성(flexibility)** 증가
- 단, **참신성(novelty)**은 상대적으로 감소 — AI가 "안전한 답"을 선호하는 경향

**Taylor & Francis(2025) RCT:**
- 비즈니스 전공 학생 대상: ChatGPT 사용 그룹이 **유의미하게 높은 창의적 성과**
- ChatGPT + 개념맵(concept mapping) → **비판적 사고·문제 해결 능력 향상**

### 2.2 SPARK 프레임워크 — 디자인 씽킹 + AI

디자인 씽킹과 ChatGPT를 결합한 **SPARK 5단계 프레임워크**:

| 단계 | 의미 | AI 활용 |
|------|------|--------|
| **S**cenario | 현재 상황 파악 | Perplexity로 배경 데이터 수집 |
| **P**roblem | 해결할 문제 정의 | ChatGPT로 HMW 질문 생성 |
| **A**spiration | 목표 설정 | Claude로 목표 구체화 |
| **R**esources | 가용 자원 확인 | 도구·시간·팀원 역량 정리 |
| **K**eys | 핵심 성공 요인 도출 | AI로 우선순위 매트릭스 |

### 2.3 AI 문제 해결의 핵심 한계

**AI는 공감하지 못합니다.** 디자인 씽킹의 출발점은 "사용자의 고통을 이해하는 것"이지만, AI는 데이터 기반 최적화를 할 뿐 **사용자의 감정과 맥락을 진정으로 이해하지 못합니다**.

> **ResearchGate(2023) 논문:** "AI는 아이디어 생성(ideation)과 프로토타입 단계에서 가장 효과적이지만, 공감(empathy) 단계는 인간이 주도해야 한다."

## 3. 따라하기 실습

### 실습 A: AI로 캠퍼스 문제 해결 기획하기 ⭐

> **목표:** 캠퍼스에서 겪는 실제 문제를 디자인 씽킹 + AI로 해결합니다.
> **소요 시간:** 40분
> **준비물:** ChatGPT 계정

---

**Step 1: 문제 발견 + HMW 질문 생성**

```
대학 캠퍼스에서 학생들이 겪는 일상적인 문제 10가지를 나열해줘.
각 문제에 대해:
1. 문제 설명 (1줄)
2. 영향받는 학생 수 (추정)
3. 현재 해결 방법의 한계
```

가장 공감가는 문제 1개를 선택한 후:

```
다음 문제 상황을 HMW(How Might We) 질문으로 변환해주세요:

문제 상황: [선택한 문제]
사용자: 대학생
맥락: [문제가 발생하는 구체적 상황]

요구:
1. HMW 질문 5가지 (관점을 달리하여)
2. 각 질문에 대한 해결 아이디어 3가지씩
3. 실현 가능성 × 영향력 매트릭스로 아이디어 평가
4. 상위 3개 아이디어로 프로토타입 방향 제안
```

📍 **화면 가이드:** HMW 질문이 5가지 관점으로 생성되고, 각각에 해결 아이디어가 매핑됩니다.

✅ **체크포인트:** HMW 질문 5개와 아이디어 매트릭스가 완성되었나요? 상위 3개 아이디어가 선정되었나요?

---

**Step 2: AI 브레인스토밍**

상위 아이디어를 선택하고 심화합니다:

```
다음 아이디어를 심화해주세요: [선택한 아이디어]

다음 기법으로 각각 확장해주세요:
1. SCAMPER (대체/결합/적용/수정/다른용도/제거/뒤집기)
2. 역브레인스토밍 (이 문제를 더 악화시키려면? → 반대로)
3. 6-3-5 시뮬레이션 (6명의 가상 전문가: 학생/교수/개발자/디자이너/심리학자/행정담당자)

최종적으로 상위 3개 아이디어를 선정해 구체화해주세요:
- 아이디어명
- 핵심 기능 3가지
- 타겟 사용자
- 차별화 포인트
```

📍 **화면 가이드:** SCAMPER, 역발상, 6-3-5 기법이 각각 적용된 아이디어가 나옵니다.

✅ **체크포인트:** 3가지 기법으로 확장된 아이디어가 있나요? 상위 3개가 구체화되었나요?

🔧 **트러블슈팅:**
- 아이디어가 너무 비현실적이다면 → "대학생 2~3명이 4주 안에 프로토타입을 만들 수 있는 범위로 축소해줘"
- 모든 아이디어가 비슷하다면 → "완전히 다른 분야(게임, 소셜미디어, 오프라인)의 관점에서 해결 방법을 제안해줘"

---

### 실습 B: 노코드 프로토타입 만들기 ⭐⭐

> **목표:** 아이디어를 노코드 도구로 실제 프로토타입으로 구현합니다.
> **소요 시간:** 60분
> **준비물:** v0 또는 Lovable 계정

---

**Step 1: MVP 기능 정의**

```
다음 서비스의 MVP 기능을 정의해주세요:

서비스: [선택한 아이디어 설명]
타겟: 대학생
목표: [검증하고 싶은 가설]

MoSCoW 기법으로 분류:
- Must Have: MVP에 반드시 포함 (핵심 가치 검증)
- Should Have: 가능하면 포함
- Could Have: 시간 여유 시
- Won't Have: V2에서 개발

각 기능에:
- 기능명 + 1줄 설명
- 유저 스토리
- 예상 개발 시간 (노코드 기준)
```

📍 **화면 가이드:** Must Have 기능만으로 프로토타입을 만듭니다. Should/Could/Won't는 무시하세요.

✅ **체크포인트:** Must Have 기능이 2~3개로 명확히 정의되었나요?

---

**Step 2: v0로 UI 프로토타입 생성**

v0(v0.dev)에 접속하여:

```
[서비스명] 의 메인 페이지를 만들어줘:

기능:
1. [Must Have 기능 1]
2. [Must Have 기능 2]

디자인: 깔끔하고 모던, 모바일 반응형
색상: [선호하는 컬러 스킴]
React + TypeScript + shadcn/ui 기반
```

📍 **화면 가이드:** v0가 React 코드와 함께 프리뷰를 생성합니다. 수정이 필요하면 자연어로 추가 요청하세요.

✅ **체크포인트:** 프로토타입 프리뷰가 작동하나요? Must Have 기능이 UI에 반영되었나요?

---

**Step 3: 테스트 시나리오 생성**

```
다음 프로토타입의 사용자 테스트 시나리오를 만들어줘:

프로토타입: [서비스 설명]
타겟: 대학생 5명

5가지 테스트 시나리오:
1. [시나리오: 사용자가 어떤 상황에서 어떤 행동을 하는지]
2. 각 시나리오의 성공 기준
3. 관찰할 포인트
4. 피드백 질문 3가지
```

✅ **체크포인트:** 5가지 테스트 시나리오와 성공 기준이 완성되었나요?

🔧 **트러블슈팅:**
- v0가 원하는 UI를 못 만든다면 → Lovable로 전환하여 "자연어로 전체 앱을 만들어줘" 요청
- 프로토타입이 너무 복잡해진다면 → Must Have 1개 기능만 남기고 나머지 삭제

---

### 실습 C: 4개 도구로 아이디어 확장 비교 ⭐⭐

> **목표:** 동일한 문제에 대해 4개 AI의 아이디어를 비교합니다.

## 4. 프롬프트 진화 패턴

**🔴 Level 1 (초보):**
```
대학 캠퍼스 문제를 해결하는 앱 아이디어 줘
```
→ 막연한 아이디어 나열. 실행 방법이 없습니다.

**🟡 Level 2 (중급):**
```
대학 캠퍼스 식당 혼잡 문제를 해결하는 앱을 기획해줘.
디자인 씽킹 5단계로 진행하고, 프로토타입은 v0로 만들 예정이야.
```
→ 구조가 잡히지만, 데이터와 구체적 제약 조건이 부족합니다.

**🟢 Level 3 (고급):**
```
캠퍼스 식당 혼잡 문제 해결 프로젝트:
■ 공감: 학생 300명 설문 결과 첨부 [CSV]
■ 정의: "점심시간(11:30~13:00) 5개 식당의 대기시간 평균 23분"
■ 요청:
1. HMW 질문 5개 + 아이디어 매트릭스
2. 상위 아이디어로 v0 프로토타입 UI 설계 (와이어프레임)
3. MVP 기능 정의 (Must/Should/Could/Won't)
4. 2주 스프린트 계획
5. 테스트 시나리오 5개 + 성공 기준
```
→ 실제 데이터와 제약 조건이 포함되어 바로 실행 가능한 프로젝트가 됩니다.

## 5. 실전 사례

### 사례 1: ChatGPT가 창의적 문제 해결 능력을 향상시킨다 (A급)

ScienceDirect(2024) 실험 연구에 따르면, ChatGPT를 사용한 학생은 창의적 문제 해결에서 **유창성(fluency)과 유연성(flexibility)**이 증가했습니다. 학생들은 AI 사용 시 **호기심과 침착함**을 가장 많이 느꼈습니다. 다만 참신성(novelty)은 상대적으로 감소하여, AI의 "안전한 답" 선호 경향에 주의가 필요합니다.

### 사례 2: ChatGPT + 개념맵 — 비판적 사고 향상 (A급)

Taylor & Francis(2025) 연구: ChatGPT 지원 개념맵(concept mapping) 접근법으로 학생의 **학업 성취도, 비판적 사고, 문제 해결 능력**이 모두 유의미하게 향상되었습니다.

### 사례 3: 비즈니스 문제 해결 — RCT로 입증 (A급)

Taylor & Francis(2025) RCT: 비즈니스 전공 학생 대상 실험에서 ChatGPT 사용 그룹이 **유의미하게 높은 창의적 성과**를 보였습니다. 실제 비즈니스 문제 해결 과제에서 검증된 결과입니다.

### 사례 4: AI Lean Canvas — 85% 더 빠른 의사결정 (B급)

AI 기반 BMC 도구를 사용한 스타트업의 **85%**가 더 빠른 의사결정을 보고했습니다. F/MS BMC Tool은 2개 질문만으로 AI가 맞춤 Lean Canvas + PDF + Google Sheet를 생성합니다.

## 6. 꿀팁 & 주의사항

### 💡 꿀팁 5가지

1. **문제 정의는 현장에서:** AI에게 문제를 정의해달라고 하지 말고, 직접 관찰·인터뷰한 후 AI에게 구조화를 요청하세요
2. **MoSCoW로 범위를 잡아라:** AI가 아무리 많은 기능을 제안해도, Must Have 2~3개만 프로토타입에 포함하세요
3. **v0는 UI 프로토타입 최강:** 자연어 한 문장으로 React UI를 생성하고, 수정도 자연어로 가능합니다
4. **Lovable은 비개발자 최강:** DB가 필요한 앱을 코딩 없이 만들 수 있습니다. Supabase가 자동 연결됩니다
5. **테스트는 AI가 아닌 실제 사람에게:** AI에게 "이 아이디어 어때?"라고 묻지 말고, 실제 사용자 5명에게 프로토타입을 보여주세요

### ⚠️ Anti-Pattern 1: "AI는 공감하지 못한다" — 문제 해결의 출발점은 현장

**상황:** 팀 프로젝트에서 ChatGPT에게 "대학생이 겪는 문제 10가지"를 물어봄. AI가 추천한 "캠퍼스 식당 혼잡" 문제를 선택.

**잘못된 방법:** 현장 관찰 없이 AI가 정의한 문제를 그대로 수용. AI가 "평균 대기시간 20분"이라고 했으므로 그것을 팩트로 사용.

**결과:**
- 실제 관찰: 식당마다 혼잡 패턴이 완전히 다름 (A식당 12분, B식당 35분)
- AI가 놓친 것: 학생들은 "대기시간"보다 "메뉴 정보 부족"을 더 큰 불편으로 느낌
- 잘못된 문제를 풀었기 때문에, 솔루션이 사용자에게 외면받음

**올바른 방법:**
1. **먼저 현장 관찰** — 직접 식당에 가서 대기 패턴, 학생 행동, 불편 사항 관찰
2. **인터뷰 5명 이상** — "가장 불편한 점이 뭐예요?" 직접 질문
3. 관찰·인터뷰 데이터를 **AI에게 입력** → 패턴 분석 + HMW 질문 생성
4. AI는 **데이터 분석과 아이디어 확장**에 활용, 문제 정의는 현장에서

**차이:** AI는 데이터 기반 최적화를 하지만, **사용자의 진짜 고통**은 현장에서만 발견됩니다. 공감 없는 솔루션은 기술적으로 완벽해도 외면받습니다.

---

### ⚠️ Anti-Pattern 2: "완벽한 AI 계획서, 2주차부터 지연" — 이상 vs 현실

**상황:** ChatGPT로 12주 프로젝트 계획서를 만듦. 역할 분담, 마일스톤, 리스크 관리까지 포함된 "완벽한" 계획.

**잘못된 방법:** AI가 만든 계획을 팀원들의 실제 역량과 가용 시간을 고려하지 않고 그대로 실행.

**결과:**
- AI 계획: "Week 2에 데이터 수집 완료" → 실제: 설문 응답이 30명밖에 안 모임
- AI 계획: "Week 4에 분석 완료" → 실제: 팀원 2명이 중간고사 기간과 겹침
- 2주차부터 계획 대비 지연 → 팀 사기 저하 → 프로젝트 품질 하락

**올바른 방법:**
1. AI 계획을 **초안**으로만 활용
2. 팀원들과 함께 **현실적 일정으로 조정** — 각자의 시간표, 중간고사 등 고려
3. **버퍼 시간** 포함 (계획의 70%만 채워넣기)
4. 매주 **체크포인트**에서 일정 재조정

**차이:** AI는 이상적 조건에서 계획을 세우지만, 팀의 현실(역량, 시간, 동기)은 인간이 조정해야 합니다.

## 7. 워크플로우 일반화: AI 문제 해결 파이프라인

```
[Phase 1: 공감] — 인간 주도
  → 현장 관찰 + 인터뷰 (최소 5명)
  → 관찰 데이터를 ChatGPT에 입력 → 패턴 분석
  → 페르소나 생성

[Phase 2: 정의] — 인간 + AI 협업
  → 관찰 데이터 기반 HMW 질문 생성 (ChatGPT)
  → 문제 재정의 (Claude로 논리 검증)
  → SPARK 프레임워크 적용

[Phase 3: 아이디어] — AI 주도
  → SCAMPER + 역발상 + 6-3-5 (ChatGPT)
  → 실현가능성 × 영향력 매트릭스
  → 상위 3개 아이디어 선정

[Phase 4: 프로토타입] — AI 도구 활용
  → MVP 기능 정의 (MoSCoW)
  → v0/Lovable/Bolt.new로 노코드 프로토타입
  → Must Have 기능만 구현

[Phase 5: 테스트] — 인간 주도
  → 실제 사용자 5명에게 프로토타입 테스트
  → Google Forms로 피드백 수집
  → ChatGPT로 피드백 분석 → 개선 방향
```

핵심 원칙: **공감과 테스트는 인간이, 아이디어와 프로토타입은 AI가** 주도합니다.

## 8. 도전 과제

### ⭐ 기본 과제: HMW 질문 + 아이디어 매트릭스

1. 캠퍼스에서 겪는 문제 1개를 선택하세요
2. 실습 A를 따라 HMW 질문 5개를 생성하고 아이디어 매트릭스를 완성하세요
3. 상위 3개 아이디어를 구체화하세요

**제출물:** HMW 질문 5개 + 아이디어 매트릭스 + 상위 3개 아이디어 상세

### ⭐⭐ 심화 과제: 노코드 프로토타입 제작

1. 실습 B를 수행하여 v0 또는 Lovable로 프로토타입을 만드세요
2. Must Have 기능 2~3개가 작동하는 프로토타입을 완성하세요
3. 테스트 시나리오 5개를 작성하세요

**제출물:** 프로토타입 URL + MVP 기능 정의서 + 테스트 시나리오

### ⭐⭐⭐ 도전 과제: 문제 해결 프로젝트 완주

1. 전체 5단계(공감→정의→아이디어→프로토타입→테스트) 파이프라인을 실행하세요
2. 실제 사용자 5명에게 프로토타입을 테스트하고 피드백을 수집하세요
3. 피드백 기반으로 프로토타입을 1회 이상 개선하세요
4. 최종 발표 자료를 Gamma로 제작하세요

**제출물:** 5단계 과정 문서화 + 프로토타입 URL(개선 전/후) + 사용자 피드백 분석 + 발표 자료
