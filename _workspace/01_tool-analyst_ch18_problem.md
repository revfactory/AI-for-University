## 1. 디자인 씽킹 × AI 도구 매핑

### 1.1 디자인 씽킹 5단계별 AI 도구

| 단계 | 목적 | 추천 AI 도구 | 활용법 |
|------|------|------------|-------|
| **1. 공감(Empathize)** | 사용자 이해 | Perplexity, ChatGPT | 사용자 리서치, 인터뷰 질문 생성, 페르소나 작성 |
| **2. 정의(Define)** | 문제 정의 | Claude, ChatGPT | HMW(How Might We) 질문 생성, 문제 재정의 |
| **3. 아이디어(Ideate)** | 해결책 발상 | ChatGPT, Claude, Gemini | 브레인스토밍, SCAMPER 기법, 아이디어 평가 매트릭스 |
| **4. 프로토타입(Prototype)** | 시제품 제작 | v0, Replit, Lovable, Bolt.new | 노코드 웹앱/UI 프로토타입 빠른 제작 |
| **5. 테스트(Test)** | 검증 | ChatGPT, Google Forms+Gemini | 테스트 시나리오 생성, 피드백 수집·분석 |

### 1.2 AI 브레인스토밍 프롬프트

```
다음 문제에 대해 브레인스토밍을 도와주세요:

문제: [구체적 문제 설명]
대상 사용자: [타겟 사용자]
제약 조건: [시간/비용/기술적 제약]

다음 기법으로 각각 아이디어를 생성해주세요:

1. SCAMPER 기법 (대체/결합/적용/수정/다른용도/제거/뒤집기)
2. 역브레인스토밍 (이 문제를 더 악화시키려면? → 반대로)
3. 랜덤 연결 (무관한 단어 3개와 문제 연결)
4. 6-3-5 시뮬레이션 (6명의 가상 전문가 관점)

각 아이디어에 실현 가능성(H/M/L)과 영향력(H/M/L)을 평가하고,
상위 3개 아이디어를 선정해 구체화해주세요.
```

## 2. 노코드 프로토타입 도구 비교

### 2.1 주요 도구 비교표

| 도구 | 가격 | 핵심 기능 | 코딩 필요 | 적합한 프로토타입 |
|------|------|----------|----------|----------------|
| **v0 (Vercel)** | Free 제한 / Premium 유료 | 자연어→React UI 코드 생성, shadcn/ui | ⚠️ 코드 수정 가능하면 유리 | **UI/프론트엔드 프로토타입** |
| **Lovable** | Free / Pro $25/월 (무제한 유저) | 자연어→풀스택 앱, Supabase 통합, 구조적 계획 | ❌ 노코드 가능 | **초보자 친화, DB 연동 앱** |
| **Bolt.new** | Free / Pro ~$20/월 (토큰 기반) | 빠른 프로토타이핑, 다중 프레임워크, 직접 코드 편집 | ⚠️ 코드 편집 가능 | **빠른 래피드 프로토타입** |
| **Replit** | Free / Core $25/월 / Pro $100/월 | AI Agent로 앱 빌드, 브라우저 IDE, 즉시 배포 | ❌ Agent가 코딩 | **학생 프로젝트, 협업** |
| **Cursor** | Free / Pro $20/월 | AI 코드 에디터, 코드베이스 이해, 서브에이전트 | ✅ 코딩 필요 | **개발 프로젝트** |
| **Figma + AI** | Free / Pro $15/유저/월 | UI/UX 디자인, 프로토타입 인터랙션 | ❌ | **디자인 프로토타입** |

### 2.2 v0 (Vercel) 심층 분석

**핵심 기능:**
- 자연어 프롬프트 → React + TypeScript + shadcn/ui 코드 자동 생성
- 실시간 프리뷰 + 반복 수정
- Vercel로 원클릭 배포
- Next.js 생태계 네이티브 통합

**적합한 상황:** UI 컴포넌트, 랜딩 페이지, 대시보드, 프론트엔드 프로토타입
**한계:** 백엔드 로직은 제한적, React/Next.js 외 프레임워크 미지원

### 2.3 Lovable vs Bolt.new 비교

| 항목 | Lovable | Bolt.new |
|------|---------|---------|
| 접근 방식 | 구조적 계획 → 코드 생성 | 빠른 즉시 코드 생성 |
| 코드 품질 | React + TypeScript, 깔끔 | 다중 프레임워크, 유연 |
| 백엔드 | **Supabase 내장 통합** | 유연한 백엔드 선택 |
| 속도 | 상대적으로 느림 (재작성 방식) | **빠름** (diff 방식) |
| 초보자 | ★★★★★ 친화적 | ★★★☆☆ 약간의 기술 이해 필요 |
| 가격 | Pro $25/월 (무제한 유저) | ~$20/월 (토큰 기반) |
| 추천 | **비개발자, DB 필요 앱** | **빠른 프로토타입, 개발자** |

### 2.4 Replit Agent 심층

**Agent 3 (2025.09~):**
- 자연어 → 풀스택 앱 빌드 (프론트+백엔드+DB)
- 자동 테스트 + 자동 수정 반복 루프
- 최대 200분 자율 작업
- 다른 에이전트를 생성하는 능력
- 2~3배 속도 향상 (v2 대비)

**Design Mode (2025.11~):**
- 비주얼 디자인 → 코드 변환

**가격:**
| 플랜 | 가격 | AI 기능 |
|------|------|---------|
| Starter | 무료 | 기본 AI, 1개 앱 배포 |
| Core | $25/월 (연간 $20/월) | Agent 사용, 앱 배포 |
| Pro | $100/월 (15 빌더) | 우선 지원, 크레딧 롤오버 |

## 3. 문제 해결 프로세스 AI 프롬프트

### 3.1 HMW(How Might We) 질문 생성

```
다음 문제 상황을 HMW(How Might We) 질문으로 변환해주세요:

문제 상황: [구체적 문제 설명]
사용자: [대상 사용자]
맥락: [문제가 발생하는 상황]

요구:
1. HMW 질문 5가지 (관점을 달리하여)
2. 각 질문에 대한 해결 아이디어 3가지씩
3. 실현 가능성 × 영향력 매트릭스로 아이디어 평가
4. 상위 3개 아이디어로 프로토타입 방향 제안
```

### 3.2 노코드 프로토타입 프롬프트 (v0/Lovable)

```
다음 기능을 가진 웹앱 프로토타입을 만들어줘:

[앱 설명]

요구사항:
1. 페이지 구성: [메인/상세/입력 등]
2. 핵심 기능: [기능 리스트]
3. 디자인: 깔끔하고 모던한 UI, 모바일 반응형
4. 데이터: [필요한 데이터 구조]

React + TypeScript + shadcn/ui 기반으로 생성해줘.
```

### 3.3 프롬프트 진화 패턴

**🔴 Level 1 (초보):**
```
대학 캠퍼스 문제를 해결하는 앱 아이디어 줘
```

**🟡 Level 2 (중급):**
```
대학 캠퍼스 식당 혼잡 문제를 해결하는 앱을 기획해줘.
디자인 씽킹 5단계로 진행하고, 프로토타입은 v0로 만들 예정이야.
```

**🟢 Level 3 (고급):**
```
캠퍼스 식당 혼잡 문제 해결 프로젝트:
■ 공감: 학생 300명 설문 결과 첨부 [CSV]
■ 정의: "점심시간(11:30~13:00) 5개 식당의 대기시간 평균 23분"
■ 요청:
1. HMW 질문 5개 + 아이디어 매트릭스
2. 상위 아이디어로 v0 프로토타입 UI 설계 (와이어프레임)
3. MVP 기능 정의 (Must/Should/Could/Won't)
4. 2주 스프린트 계획
5. 테스트 시나리오 5개 + 성공 기준
```

## 4. 상황별 도구 선택 가이드

| 상황 | 추천 도구 | 이유 |
|------|----------|------|
| 아이디어 발상 | ChatGPT / Claude | 브레인스토밍, SCAMPER, 역발상 |
| 사용자 리서치 | Perplexity + Google Forms | 트렌드 조사 + 설문 수집 |
| UI 프로토타입 | v0 | 자연어→React UI 가장 빠름 |

| 학생 프로젝트 | Replit | 무료 tier, 브라우저 IDE, 협업 |
| 디자인 프로토타입 | Figma + AI | UI/UX 전문 디자인 |
| 테스트·검증 | ChatGPT + Google Forms | 테스트 시나리오 + 피드백 분석 |
