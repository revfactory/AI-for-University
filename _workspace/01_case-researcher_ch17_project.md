## 1. 학술 논문 채널

### 사례 1-1: AI 강화 PBL — 교사 인식과 교육적 효과 (✅ A급)

**출처:** MDPI Education Sciences (2025)
- URL: https://www.mdpi.com/2227-7102/15/2/150

**핵심 데이터:**
- AI 통합 프로젝트 기반 학습(PBL) vs 일반 PBL 비교 연구
- AI-PBL이 일반 PBL보다 **유의미하게 높은 효과** — 대규모 효과 크기(large effect size)
- AI 기반 **실시간 학습 성과 반영 + 맞춤형 콘텐츠 제공** → 학업 불안 감소, 성과 향상
- AI가 프로젝트 제작·리뷰 전 과정에서 **연속적 피드백과 자기평가** 수단으로 활용

**교재 활용:** "AI를 접목한 프로젝트 수업이 일반 프로젝트보다 효과적인 이유"

## 7. 실패 사례 / Anti-Pattern

### 사례 7-1: AI 프로젝트의 '도구 의존' 함정 — "코드는 돌아가지만 이해는 못 한다" (✅ A급 — 교차검증)

**출처:** 사례 1-1, 1-2 교차검증

**핵심 교훈:**
- AI-PBL이 효과적이지만, AI에 **과도하게 의존**하면 학습 효과 감소 (사례 1-1)
- Harvard 연구: AI 튜터로 2배 학습 가능 — 하지만 **자기주도 학습 없이는 지속 불가**
- AI가 코드를 생성하고 결과를 만들어주지만, **"왜 이 코드가 동작하는가"를 이해 못하면** 프로젝트 발표에서 실패

**학생이 주의할 점:**
1. AI 생성 코드를 **한 줄씩 이해** — 복사만 하지 않기
2. 프로젝트 과정 **문서화** — AI와의 대화 기록, 의사결정 과정 포함
3. **"AI가 못하는 부분"을 직접 해결**하는 경험이 핵심 학습

**교재 활용:** "AI가 만든 프로젝트, 면접에서 설명 못하면 끝" — 핵심 Anti-Pattern

---

### 사례 7-2: AI 프로젝트의 '범위 초과' 실패 (💡 C급)

**출처:** 합리적 추론 기반 구성 사례

**시나리오:**
- 대학생 I가 AI를 활용해 "건강 관리 앱" 졸업 프로젝트 기획
- ChatGPT가 추천한 기능: 식단 분석 + 운동 추천 + 수면 추적 + 감정 분석 + 커뮤니티
- 4개월 안에 모든 기능 구현 불가 → 핵심 기능 하나도 완성 못함
- **교훈:** AI는 아이디어를 무한히 생성하지만, **MVP(최소 기능 제품) 범위를 정하는 것은 인간의 판단**
- AI에게 "이 기능 중 4개월 안에 혼자서 구현 가능한 핵심 1~2개를 골라줘"가 올바른 프롬프트

**교재 활용:** "AI 아이디어 폭발 → MVP 범위 설정 실패"
