## 7. 실패 사례 / Anti-Pattern

### 사례 7-1: 바이브 코딩의 치명적 약점 — "코드는 돌아가지만 품질은 위험" (✅ A급 — 교차검증)

**출처:** 사례 1-1, 2-2 교차검증

**핵심 교훈:**
- AI 공동 작성 코드: 주요 이슈 **1.7배**, 로직 오류 **75% 더 많음** (사례 2-2, 2025.12 분석)
- Accenture 연구: 제안 수용률 **30%** — 70%는 개발자가 거부 (사례 1-1)
- 전문 개발자 **72%가 바이브 코딩 미사용** — 프로덕션 코드에는 부적합 (사례 2-2)
- **핵심:** AI로 빠르게 만들 수 있지만, **코드 리뷰·테스트·보안 검증은 반드시 필요**

**학생이 주의할 점:**
1. AI 생성 코드를 **반드시 리뷰** — 특히 보안 취약점 (SQL 인젝션, XSS 등)
2. **테스트 코드를 AI에게 먼저 요청** — 코드보다 테스트가 먼저
3. "돌아가니까 됐다"가 아닌 **"왜 이렇게 동작하는지 이해한다"**가 목표
4. 학습 목적: AI를 **코드 설명 도구**로 활용 — "이 코드를 한 줄씩 설명해줘"

**교재 활용:** "바이브 코딩의 빛과 그림자 — AI 코드의 1.7배 이슈" — 핵심 Anti-Pattern

---

### 사례 7-2: AI 코딩 도구의 '과신' — 디버깅 능력 저하 (💡 C급)

**출처:** 합리적 추론 기반 구성 사례

**시나리오:**
- 대학생 K가 Cursor로 웹앱을 완성 — "AI가 다 해줬다"
- 배포 후 에러 발생 — 에러 메시지를 읽을 수 없음
- AI에게 에러를 복사해서 붙여넣기 → AI가 "수정 코드"를 생성 → 다른 에러 발생 → 반복
- **교훈:** AI 코딩 도구는 **"에러가 없는 코드"를 만드는 게 아니라 "에러를 더 빠르게 만나게"** 해줌
- 기초적인 **디버깅 능력**(에러 메시지 읽기, 스택 트레이스 이해)은 AI 시대에도 필수

**교재 활용:** "AI가 코딩해줘도 디버깅은 내가 해야 한다"

- **속도:** Cursor가 **30% 더 빠름**

### Claude Code 상세
- **설치:** `npm install -g @anthropic-ai/claude-code`
- **사용 환경:** 터미널, VS Code 확장, 데스크톱 앱, 브라우저
- **가격:** Claude Pro ($20/월) 또는 Team/Enterprise 구독 — 별도 Claude Code 요금 없음
- **터미널 에이전트** — CLI에서 직접 코딩
- **멀티에이전트 코드 리뷰** 기능
- Anthropic **연매출 $2B+** (2026 기준)

### Replit 학생 플랜
- **Starter (무료):** 기본 AI, 1개 앱 배포 — 학생 입문용 충분
- **Core ($25/월, 연간 $20/월):** Agent 사용, 다수 앱 배포
- 공식 학생 할인은 미확인. Starter 무료로 시작 가능
