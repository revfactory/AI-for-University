# 🔧 Claude Code & 하네스 활용 — Chapter 11 보충

> **대상 독자**: Part 3까지 Claude Code 자동화를 경험한 학생
> 본문에서 **Google Forms → ChatGPT 분석 → Jamovi 검증** 3단계 워크플로우를 배웠습니다. ChatGPT Code Interpreter로 빈도분석·교차분석(χ²)·상관분석을 수행하고, Jamovi로 교차 검증하여 정확성을 확보하는 방법이었죠. 이번 보충에서는 같은 설문 분석을 **Claude Code + pandas + scipy**로 구현합니다. 본문의 웹 UI 기반 분석이 "매번 대화에서 재요청"이라면, CC 접근은 **재현 가능한 분석 파이프라인** — 같은 형식의 설문을 반복 분석할 때 스크립트 한 번으로 끝냅니다. PMC 연구(Koçak, 2025)에서 GPT-4o 기초 통계가 R과 오차 5% 이내로 일치함을 확인했듯, Claude Code도 pandas/scipy 기반으로 동일한 정확도를 로컬에서 달성합니다.

## 1. 설문 데이터 분석 파이프라인

### 1-1. 본문 워크플로우 vs CC 파이프라인

| 항목 | 본문 방식 (ChatGPT + Jamovi) | Claude Code 파이프라인 |
|------|---------------------------|---------------------|
| 데이터 업로드 | ChatGPT에 파일 첨부 (크기 제한) | 로컬 파일 직접 접근 (제한 없음) |
| 분석 과정 | 대화형, 매번 재요청 | 스크립트로 재현 가능 |
| 교차 검증 | Jamovi 별도 설치 → 수동 비교 | CLAUDE.md 규칙으로 자동 검증 포함 |
| 시각화 | AI가 생성한 이미지 다운로드 | 로컬에 PNG/PDF 자동 저장 |
| 반복 분석 | 매번 처음부터 | `bash analyze.sh` 한 줄 |
| 대용량 데이터 | 토큰 제한에 걸림 | 로컬 처리, 제한 없음 |

### 1-2. 분석 파이프라인 구조

```
CSV 파일 → Claude Code → 파이썬 코드 자동 생성·실행
                          ├── 데이터 정제 (결측치, 이상치)
                          ├── 기술 통계 (빈도, 평균, 표준편차)
                          ├── 교차 분석 (그룹별 비교)
                          ├── 시각화 (막대, 파이, 히트맵)
                          └── 보고서 생성 (마크다운)
```

```markdown
# Data Loader — 데이터 로딩·정제

## 핵심 역할
- input/ 폴더의 CSV 파일을 pandas로 읽어들입니다.
- 결측치, 이상치, 중복 응답을 탐지하고 처리합니다.
- 데이터 타입을 자동 변환합니다 (숫자형, 범주형 구분).
- 정제된 데이터를 _workspace/cleaned_data.csv로 저장합니다.

## 작업 원칙
- 결측치 처리: 전체의 5% 미만이면 제거, 이상이면 대체값 사용
- 자유 응답(텍스트)은 별도 컬럼으로 분리
- 리커트 척도(1~5)는 숫자형으로 자동 변환
- 정제 과정을 _workspace/cleaning_log.md에 기록

## 출력
- _workspace/cleaned_data.csv: 정제된 데이터
- _workspace/cleaning_log.md: 정제 과정 기록
- _workspace/data_summary.md: 데이터 개요 (응답 수, 컬럼 목록, 결측치 현황)
```

**survey-analyzer.md**:
```markdown
---
name: survey-analyzer
description: "설문 데이터의 통계 분석과 교차 분석을 수행하는 에이전트"
---
# Survey Analyzer — 통계 분석

## 핵심 역할
- 기술 통계: 각 문항의 빈도, 비율, 평균, 표준편차를 계산합니다.
- 교차 분석: 그룹별(성별, 학년, 전공 등) 응답 차이를 분석합니다.
- 상관 분석: 문항 간 상관관계를 파악합니다.
- 핵심 인사이트를 도출하여 보고서 초안을 작성합니다.

## 작업 원칙
- 통계 수치는 반드시 소수점 둘째 자리까지 표시
- 교차 분석(χ²) 시 기대빈도 5 미만 셀이 20% 초과하면 Fisher 정확 검정으로 전환 (본문 Jamovi 실습 기준)
- 교차 분석 시 그룹별 응답 수가 30 미만이면 "소규모 표본 주의" 경고 표시
- 백분율은 소수점 첫째 자리까지 (예: 45.2%)
- 모든 분석에 해석 코멘트를 함께 작성 ("이 결과는 ~을 의미합니다")
- 본문에서 다룬 7가지 통계 개념(기술통계, 빈도분석, 교차분석, t-검정, ANOVA, 상관분석, Cronbach's α) 중 해당되는 분석을 자동 식별하여 수행

## 출력
- output/report/analysis_report.md: 분석 보고서
```

### 2-2. CLAUDE.md 설문 분석 규칙

```markdown
# 설문 분석 시스템 규칙

## 데이터 형식
- CSV: UTF-8 인코딩, 첫 행 헤더
- 리커트 척도: 1(매우 그렇지 않다) ~ 5(매우 그렇다)
- 범주형 응답: 원본 텍스트 유지

## 분석 원칙
- 결론을 데이터가 지지하는 범위 내에서만 작성
- 인과 관계("~때문에")가 아닌 상관/경향("~과 관련이 있다")으로 표현
- 소규모 표본(N<30)의 교차 분석은 "참고 수준" 명시
- 자유 응답은 키워드 빈도 분석 + 대표 응답 5개 인용
- **AI 분석 신뢰 범위**: 기초통계·빈도·교차분석은 AI가 정확 (Frontiers 2025 확인), 탐색적 요인분석(EFA) 등 복잡한 분석은 전문 도구 필요 (Frontiers: ChatGPT가 반복 루프에 빠짐)
- **표본 대표성 경고**: 분석 결론에 "본 설문 대상자의 N명 기준" 명시 — "대학생 전체"로 일반화 금지 (본문 Anti-Pattern 2)

## 시각화 원칙
- 한글 폰트 설정 필수 (깨짐 방지)
- 색상 접근성: 색약 고려 팔레트 사용
- 차트 하단에 "N=응답수" 표시
```

**잘못된 방법 — 데이터를 보지 않고 분석 지시**:

```bash
claude -p "input/survey.csv를 분석하고 보고서 작성해줘"
```

**결과**: Claude Code가 CSV 구조를 파악하지 못해 잘못된 컬럼을 분석하거나, 리커트 척도를 문자열로 처리하여 평균을 계산하지 못합니다. 특히 한글 컬럼명이 있으면 인코딩 오류가 발생할 수 있습니다.

**올바른 방법**:

```bash
claude -p "
input/survey.csv의 첫 5행과 컬럼 목록을 먼저 확인해줘.
각 컬럼의 데이터 타입과 결측치 수를 알려줘.
그 다음 정제 계획을 _workspace/cleaning_plan.md에 작성해줘.
"
```

**차이**: "먼저 데이터를 본다" — 분석 전에 데이터 구조를 확인하고 정제 계획을 세우는 것이 핵심입니다. 본문에서 강조하는 "설문 설계가 분석의 절반"과 같은 원리입니다.

---

**잘못된 방법 — 본문의 "AI 분석 맹신" 패턴이 CC에도 적용**:

```
claude -p "교차 분석 결과를 그대로 보고서에 넣어줘"
```

**결과**: 본문에서 경고한 "AI 분석 맹신" 그대로 — 기대빈도 5 미만 셀이 40%인데 χ² 검정을 그대로 적용하거나, 소규모 그룹(N=5)의 결과를 "유의한 차이"로 해석합니다. BBC 조사에 따르면 **AI 쿼리의 45%가 오류**를 포함하며, 본문의 "표본 편향 착각" 안티패턴처럼 IT 동아리 50명 = 대학생 전체로 일반화하는 실수도 발생합니다.

**올바른 방법**: CLAUDE.md에 "기대빈도 5 미만 20% 초과 시 Fisher 전환", "N<30 경고", "상관/경향으로만 표현", "표본 대표성 명시" 규칙을 설정합니다. 본문에서 Jamovi로 수동 교차 검증하던 것을, CLAUDE.md 규칙으로 **자동 안전장치화**한 것입니다.

**차이**: 본문의 "ChatGPT 분석 → Jamovi 검증" 2단계가, CC에서는 "CLAUDE.md 규칙 내장 → 분석 시 자동 검증 포함" 1단계로 간소화됩니다.

> 💡 **Hybrid Tip — 본문 도구 + CC 자동화 조합**
> - **빠른 1회 분석**: 본문의 ChatGPT Code Interpreter로 CSV 업로드 → 즉시 분석 (설정 0분)
> - **반복 분석 자동화**: CC 파이프라인으로 스크립트화 → 새 데이터마다 재실행 (설정 25분, 이후 1분)
> - **정확성 확보**: 핵심 수치는 본문의 Jamovi(무료)로 교차 검증 — CC가 χ²=15.67을 출력하면 Jamovi에서 동일 결과인지 확인
> - **전공별 선택**: 사회과학(Jamovi 필수) / 경영학(CC 파이프라인 충분) / 졸업논문(CC 분석 + Jamovi 검증 조합)

> **핵심 정리**: 본문의 "Google Forms → ChatGPT → Jamovi" 3단계를 CC에서는 "CSV → 에이전트 팀 → 스크립트"로 자동화했습니다. 본문이 강조한 통계 7개념(기술통계~Cronbach's α)은 survey-analyzer 에이전트가 자동 식별하여 수행하고, "AI 분석 맹신"과 "표본 편향 착각" 안티패턴은 CLAUDE.md 규칙으로 자동 방지합니다. Frontiers 연구에서 확인한 "기초통계 AI 신뢰 가능, 복잡한 분석은 전문 도구 필요"라는 원칙도 규칙에 반영되어 있습니다. Ch 10(cron)의 스케줄링을 결합하면 **매월 자동 설문 분석 보고서**도 가능합니다.
