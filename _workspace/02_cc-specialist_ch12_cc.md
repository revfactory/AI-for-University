# 🔧 Claude Code & 하네스 활용 — Chapter 12 보충

> **대상 독자**: Ch 11에서 설문 분석 파이프라인을 경험한 학생
> 본문에서 **5단계 노코드 분석 파이프라인**(데이터 획득→탐색→정제→시각화→보고서)을 ChatGPT Code Interpreter와 Claude Artifacts로 체험했습니다. 본문의 핵심 메시지는 **"코딩 능력이 아닌 질문 능력이 데이터 분석의 핵심 역량"** — 이 원칙은 Claude Code에도 그대로 적용됩니다. 핵심 차이: ChatGPT Code Interpreter는 웹 UI에서 파일을 업로드하고 결과를 다운로드하지만, Claude Code는 **로컬 파일을 직접 읽고, 파이썬 코드를 자동 생성·실행하며, 결과를 로컬에 저장**합니다. 그리고 Claude Code만의 독보적 장점 — **코드가 터미널에 항상 보인다**는 것. 본문이 "쉽다≠정확하다"를 경고했다면, CC 보충은 "쉽게 하되 코드를 이해하자"를 제안합니다.

## 1. "파이썬 없이" 분석의 두 가지 경로

### 1-1. 본문 도구 vs Claude Code 비교

본문에서는 ChatGPT Code Interpreter(정확한 분석) + Claude Artifacts(인터랙티브 대시보드)를 조합했습니다. Claude Code는 이 둘의 장점을 로컬에서 통합합니다:

| 항목 | ChatGPT Code Interpreter | Claude Artifacts | Claude Code |
|------|------------------------|-----------------|-------------|
| 접근 방식 | 웹 UI 파일 업로드 | 웹 UI 대화 | 로컬 CLI 자연어 지시 |
| 분석 엔진 | Python (서버사이드) | JavaScript/React | **Python (로컬)** |
| 시각화 | 정적 이미지 | 인터랙티브 대시보드 | 로컬 PNG/PDF + 스크립트 |
| 코드 확인 | 접힌 블록 클릭 | 코드 숨김 | **항상 보임** — 학습 효과 |
| 반복 실행 | 매번 대화에서 요청 | 매번 재생성 | 스크립트로 재실행 |
| 대용량 | 100MB 제한 | 1M 토큰 | **제한 없음** |

### 1-2. Claude Code의 "코드 투명성" 장점

Claude Code는 분석을 수행할 때 **파이썬 코드를 터미널에 그대로 보여줍니다**. 이것이 학습자에게 큰 장점입니다:

```
사용자: "이 CSV의 평균과 표준편차를 구해줘"

Claude Code가 실행하는 코드:
───────────────────────────────
import pandas as pd
df = pd.read_csv('input/data.csv')
print(df.describe())
───────────────────────────────

→ 코드를 보면서 "아, pandas에서 describe()를 쓰면 기술 통계가 나오는구나" 학습
```

> 💡 **팁**: Claude Code가 실행하는 코드를 `_workspace/`에 저장하도록 요청하면, 나중에 파이썬을 배울 때 참고 자료가 됩니다.

```markdown
# Code Writer — 코드 자동 생성기

## 핵심 역할
- 사용자의 자연어 분석 요청을 파이썬 코드로 변환합니다.
- 코드를 실행하고 결과를 보여줍니다.
- 생성한 코드를 _workspace/scripts/에 .py 파일로 저장합니다.

## 작업 원칙
- 코드에 한글 주석을 반드시 포함 (학습자를 위해)
- 각 코드 블록에 "이 코드가 하는 일" 설명을 먼저 출력
- pandas, matplotlib, seaborn만 사용 (복잡한 라이브러리 지양)
- 코드는 5~15줄 단위로 나누어 실행 (한 번에 50줄 이상 금지)
- 오류 발생 시 원인을 쉬운 말로 설명

## 출력
- _workspace/scripts/: 실행된 파이썬 스크립트 모음
```

### 2-2. CLAUDE.md 코드 학습 규칙

```markdown
# 데이터 분석 체험 시스템 규칙

## 코드 투명성 원칙
- 코드를 실행하기 전에 "이 코드가 하는 일"을 한 줄로 설명
- 코드에 한글 주석 포함 (# 데이터 읽기, # 평균 계산 등)
- 핵심 함수/메서드를 처음 사용할 때 "📘 파이썬 배움 포인트" 박스로 설명
- 예: "📘 df.groupby('학년').mean()은 학년별로 그룹을 나누고 평균을 구합니다"

## 분석 원칙
- 결론은 데이터가 지지하는 범위 내에서만
- "~때문에"(인과) 대신 "~과 관련이 있다"(상관) 사용
- 소규모 데이터(N<50)는 "탐색적 분석" 수준임을 명시
- 모든 수치에 기준 기간, 단위, 신뢰구간을 명시 (본문 "쉽다≠정확하다" 원칙)
- 공공데이터(data.go.kr) 사용 시 출처와 기준일 명시

## 코드 저장
- 실행한 모든 코드를 _workspace/scripts/에 날짜_순번.py로 저장
- 각 파일 상단에 "# 이 스크립트의 목적: ..." 주석 포함
```

### ❌ 흔한 실수: "파이썬 없이 분석"의 함정

**상황**: Claude Code로 데이터 분석을 할 때

---

**잘못된 방법 — 본문의 "쉽다≠정확하다"가 CC에도 적용**:

```
데이터 분석해주고 결과만 보여줘. 코드는 필요 없어.
```

**결과**: 분석 결과는 나오지만, "왜 이런 결과가 나왔는지" 검증할 수 없습니다. 본문에서 경고한 **"쉽다≠정확하다"** 그대로 — AI가 잘못된 컬럼을 분석하거나, 집계 기준을 임의로 설정해도 발견하지 못합니다. BBC 조사에 따르면 AI 쿼리의 45%가 오류를 포함하며, "열 이름을 잘못 매핑"하는 것이 가장 흔한 원인입니다.

**올바른 방법**:

```
데이터 분석해주고, 각 단계의 코드도 보여줘.
코드에 한글 주석을 달고, 핵심 함수는 쉽게 설명해줘.
코드를 _workspace/scripts/에 저장해줘.
```

**차이**: 코드를 "이해하려는 노력" 자체가 학습입니다. 파이썬을 배우지 않아도, "df.groupby()가 그룹별 분석이구나"를 알게 되면 다음에 AI에게 더 정확한 지시를 내릴 수 있습니다.

---

**잘못된 방법 — 본문의 "멋진 차트≠좋은 분석" 패턴**:

```
AI가 "매출이 23% 증가했다"는 멋진 차트를 만들어서 그대로 보고서에 썼다
```

**결과**: 본문 Anti-Pattern 2에서 경고한 그대로 — 표본 50명으로 소수점 한 자리까지 제시하는 "과도한 정밀도", 신뢰구간 없는 수치, 검증되지 않은 기준(전월 대비인지 전년 대비인지). Claude Code의 멋진 matplotlib 차트도 **내용이 틀리면 소용없습니다**.

**올바른 방법**: "23% 증가의 기준 기간은? 95% 신뢰구간은? 이상치를 제외하면 달라지는지도 확인해줘"라고 검증 요청을 추가합니다. CLAUDE.md에 "모든 수치에 기준과 신뢰구간 명시" 규칙을 포함합니다.

**차이**: AI 분석 결과에 대한 **"정말?" 습관**이 데이터 리터러시의 핵심입니다. 본문이 "비판적 검토를 AI에게 요청하는 한 줄이 보고서의 질을 결정한다"고 했듯, CC에서는 이 한 줄을 CLAUDE.md에 영구 규칙으로 넣습니다.

#### Step 1. 데이터 탐색 하네스 구축 ⏱️ 15분


### 🔄 이 워크플로우를 다른 상황에 적용하기

| 이 실습의 사례 | → 이렇게 바꾸면 | 다른 활용 |
|--------------|----------------|----------|
| 따릉이 이용 데이터 | 날씨 데이터 | 기상 패턴 분석 |
| 시간대별·요일별 분석 | 지역별·월별 분석 | 공공 데이터 탐색 |
| 자연어 질문→코드 | 과제 질문→코드 | 전공 데이터 분석 과제 |
| 파이썬 학습 노트 | R 학습 노트 | 통계 도구 학습 |

**핵심 패턴**: `자연어 질문 + 코드 자동 생성(한글 주석) + 배움 포인트 = "쓰면서 배우는" 데이터 분석` — 어떤 데이터셋이든 이 패턴으로 탐색하면 분석 능력과 코딩 능력이 동시에 성장합니다.

> - **빠른 탐색**: 본문의 ChatGPT Code Interpreter로 즉시 EDA (설정 0분, 코드 숨김)
> - **인터랙티브 대시보드**: 본문의 Claude Artifacts로 React 기반 시각화 (공유 가능)
> - **코드 학습 + 자동화**: Claude Code로 분석하며 파이썬 자연 습득 + 스크립트화 (재현 가능)
> - **최적 조합**: ChatGPT(빠른 1차 탐색) → CC(코드 학습 + 스크립트화) → Claude Artifacts(최종 대시보드)
> - **공공데이터 활용**: 본문에서 소개한 data.go.kr, data.seoul.go.kr 데이터를 CC 하네스에 input/으로 바로 활용

> **핵심 정리**: 본문의 핵심 메시지 — **"코딩이 아닌 질문 능력이 핵심 역량"** — 은 Claude Code에서도 그대로 적용됩니다. 다만 CC만의 차별점은 **코드 투명성**: 질문 능력으로 분석을 지시하되, 생성된 코드를 보면서 자연스럽게 파이썬을 습득합니다. 본문이 경고한 "쉽다≠정확하다"와 "멋진 차트≠좋은 분석"은 CLAUDE.md 규칙("기준·신뢰구간 명시", "비판적 검토 포함")으로 자동 방지합니다. No-code AI 시장이 $2,361억(2033)으로 성장하고, MIT가 공식 교육과정을 운영하는 시대 — 본문의 5단계 노코드 파이프라인과 CC의 코드 투명성을 결합하면, "데이터를 두려워하는 사람"에서 **"데이터에 질문하고 코드도 이해하는 사람"**으로 성장합니다.
