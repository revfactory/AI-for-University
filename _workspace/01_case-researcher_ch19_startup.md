## 7. 실패 사례 / Anti-Pattern

### 사례 7-1: AI 창업의 '기술만 있고 문제가 없는' 함정 (✅ A급 — 교차검증)

**출처:** 사례 1-1, 1-2 교차검증

**핵심 교훈:**
- Getting Smart: **"실제 문제를 해결하라"** — 기술이 아닌 문제에서 시작 (사례 1-1)
- 2026년 최고 아이디어: **수직형** — 범용 AI가 아닌 도메인 특화 (사례 1-2)
- Y Combinator: 코드의 95%가 AI 생성이어도 **문제 정의와 시장 검증은 인간의 몫**
- **핵심:** AI로 빠르게 만들 수 있다고 해서, 아무거나 만들면 안 된다

**학생이 주의할 점:**
1. **"누구의 어떤 문제를 해결하는가"** 먼저 정의
2. AI로 MVP를 빠르게 만들되, **실제 사용자 피드백** 우선
3. "AI 때문에 빠르게 만들었다"가 아닌 **"이 문제를 AI로 더 잘 풀었다"**가 가치

**교재 활용:** "기술이 있어서 만든 것 vs 문제가 있어서 만든 것" — 핵심 Anti-Pattern

---

### 사례 7-2: AI 서비스 기획의 '수익 모델 부재' (💡 C급)

**출처:** 합리적 추론 기반 구성 사례

**시나리오:**
- 팀 J가 ChatGPT API로 "대학생 맞춤 학습 도우미" 프로토타입 완성
- 사용자 100명 확보 — 하지만 API 비용이 월 50만 원 발생
- 수익 모델 없이 운영 → 2개월 만에 서비스 중단
- **교훈:** AI API 비용은 **사용량에 비례** — 무료 서비스로는 지속 불가
- 기획 단계에서 **비용 구조 + 수익 모델**을 반드시 설계

**교재 활용:** "AI API 비용을 간과한 서비스 기획의 실패"

## tool-analyst 교차 데이터 (2026-03-17)

### MVP 비용·기간 트렌드
- **AI 활용 MVP 비용:** $5K~$20K (전통 $100K+ 대비 **80% 절감**)
- **AI 활용 MVP 기간:** 2~6주 (전통 6개월+)
- **피치덱:** Gamma로 **60초** 자동 생성 가능

### Lovable vs Bolt.new 상세 비교
| 항목 | Lovable | Bolt.new |
|------|---------|---------|
| 접근 방식 | 구조적 계획 → 코드 생성 | 빠른 즉시 코드 생성 (diff 방식) |
| 코드 | React + TypeScript + shadcn/ui | 다중 프레임워크 (Vue, Svelte 등) |
| 백엔드 | **Supabase 내장 통합** | 유연한 백엔드 선택 |
| 가격 | Pro $25/월 (**무제한 유저**) | ~$20/월 (토큰 기반, 10M/월) |
| 초보자 | ★★★★★ | ★★★☆☆ |
| 속도 | 느림 (재작성) | **빠름** (diff) |
| 추천 | 비개발자, DB 필요 앱 | 빠른 프로토타입, 개발자 |

### Replit Agent 3 (MVP 제작)
- **Agent 3 (2025.09~):** 자연어 → 풀스택 앱, 200분 자율작업, 자동 테스트·수정 루프
- **가격:** Starter 무료 / Core $25/월 (연간 $20/월) / Pro $100/월 (15빌더)
- **MVP 적합성:** ★★★★★ — 복잡한 앱(AI 연동, 커스텀 로직)에 최적. 2~4주 내 MVP 구축 가능
