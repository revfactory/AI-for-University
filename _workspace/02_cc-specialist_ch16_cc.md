## 1. 직무 리서치 자동화의 장점

### 1-1. 수동 탐색 vs CC 리서치

| 항목 | 수동 탐색 (웹 검색) | Claude Code 리서치 |
|------|-------------------|-------------------|
| 정보 수집 | 사이트별 수동 검색 | 에이전트가 체계적 분석 |
| 비교 분석 | 엑셀에 수동 정리 | 자동 비교표 생성 |
| 적합도 판단 | 직감에 의존 | 프로필 기반 자동 매칭 |
| 업데이트 | 수시 재검색 | 스크립트로 주기적 갱신 |
| 보고서 | 직접 작성 | 자동 생성 |

> **본문 프레임워크와 CC 에이전트 대응**:
> | 본문 3가지 프레임워크 | CC 에이전트 | 본문 추천 도구 |
> |---------------------|-----------|-------------|
> | 채용공고(JD) 분석 | job-analyst | ChatGPT로 JD 파싱 |
> | 산업 구조 분석 | (Perplexity 연계) | Claude로 심층 분석 |
> | 역량 갭(Gap) 분석 | skill-matcher | ChatGPT로 매칭 |
> | → 전략 수립 | strategy-planner | (본문에 없는 자동화) |

### 1-2. 직무 리서치 파이프라인

```
채용 공고 수집 → 직무 분석 → 역량 매칭 → 전략 수립
      ↓              ↓           ↓           ↓
  job_postings/  직무 요구사항  my_profile   지원 우선순위
  (input)        키워드 추출    매칭 점수    + 준비 계획
```

```markdown
# Job Analyst — 직무 분석가

## 핵심 역할
- 채용 공고에서 필수/우대 역량, 핵심 키워드, 업무 내용을 추출합니다.
- 동일 직무의 여러 공고를 비교 분석합니다.
- 직무의 성장 전망과 필요 역량 트렌드를 정리합니다.

## 작업 원칙
- 채용 공고 키워드 빈도 분석 (상위 20개)
- "필수"와 "우대"를 명확히 구분
- 연봉·복지·성장 경로 정보도 가능한 범위에서 정리
- 동일 직무의 기업 간 요구 역량 차이 비교
- 업계 트렌드와 연결 (성장/축소 직무 판단)

## 출력
- output/analysis/직무분석_[직무명].md: 직무 분석 보고서
```

**skill-matcher.md**:
```markdown

name: skill-matcher

---
# Skill Matcher — 역량 매칭기

## 핵심 역할
- my_profile.md의 역량과 직무 요구사항을 1:1 매칭합니다.
- 적합도 점수(0~100)를 산출합니다.
- "보유 역량"과 "부족 역량"을 식별합니다.
- 부족 역량의 보완 방법을 제안합니다.

## 작업 원칙
- 필수 역량 매칭: 가중치 2배
- 우대 역량 매칭: 가중치 1배
- 매칭 기준: 정확 일치 / 유사 / 미보유 3단계
- 부족 역량: 단기(1~3개월) / 중기(3~6개월) 보완 계획 구분
- 여러 직무 간 적합도 비교 순위

## 출력
- output/matching/매칭결과_[직무명].md: 적합도 보고서
```

```markdown

name: strategy-planner

---
# Strategy Planner — 전략 수립가

## 핵심 역할
- 적합도 분석을 바탕으로 지원 우선순위를 결정합니다.
- 기업별·직무별 지원 전략을 수립합니다.
- 부족 역량 보완을 위한 학습 로드맵을 작성합니다.

## 작업 원칙
- 지원 우선순위: 적합도 + 선호도 + 성장 가능성 종합 평가
- 전략 유형: 안정 지원(높은 적합도) / 도전 지원(성장 잠재력) / 전략 지원(독특한 차별점)
- 로드맵: 주 단위 계획, 구체적 학습 자원 명시
- 현실적 제약 반영 (남은 준비 기간, 병행 가능한 학습량)

## 출력
- output/strategy/지원전략.md: 종합 지원 전략
- output/strategy/학습로드맵.md: 역량 보완 계획
```

```markdown
# 직무 리서치 시스템 규칙

## 분석 원칙
- 채용 공고의 원문 키워드를 보존 (임의 해석 금지)

- 기업 평판은 사실 기반 정보만 (루머 배제)

## 매칭 원칙
- 적합도 점수는 투명하게 산출 근거 공개
- "보유" 판단 기준: 프로필에 명시된 경험 또는 자격

## 전략 원칙
- "모든 곳에 지원" 대신 전략적 선별
- 현실적 제약 반영 (준비 기간, 자격 요건)
- 부족 역량 보완은 구체적 자원 명시 (강의명, 자격증명)
```

**잘못된 방법 — AI 추천만 믿고 지원**:

```
내 프로필로 어디 지원하면 좋을지 추천해줘.
AI가 1순위로 추천한 곳에 바로 지원!
```

**결과**: AI는 채용 공고 텍스트와 프로필 텍스트의 키워드 매칭은 하지만, 기업 문화, 워크라이프밸런스, 실제 업무 환경 등 **텍스트에 드러나지 않는 요소**는 판단하지 못합니다.

**올바른 방법**: AI 적합도 분석은 **"후보군 정리"**에 활용하고, 최종 결정은 현직자 인터뷰, 기업 리뷰, 실제 경험을 종합하여 내립니다.

**차이**: AI는 **"탐색 범위를 좁혀주는 도구"**이지, "인생 결정을 대신하는 도구"가 아닙니다.

---

**잘못된 방법 — 부족 역량에 압도당하기**:

```
갭 분석을 보니 부족한 게 너무 많아. 포기하자.
```

**결과**: 모든 요구사항을 100% 충족하는 지원자는 없습니다. "필수 역량 80% + 학습 의지"가 대부분의 채용에서 요구하는 수준입니다.

**올바른 방법**: strategy-planner가 "필수 역량 중 보유율"과 "단기 보완 가능 여부"를 분석합니다. 필수 역량 70%+ 보유이면 지원 가치가 있고, 부족한 30%는 면접에서 "학습 계획"으로 보완합니다.

**차이**: 갭 분석은 **"포기 이유"**가 아니라 **"준비 계획"**의 출발점입니다.

#### Step 1. 심층 매칭 ⏱️ 8분

