## 1. 왜 마크다운으로 PPT를 만드는가?

### 1-1. 기존 발표자료 제작의 문제점

| 기존 방식 | 문제점 |
|----------|--------|
| PPT에서 직접 디자인 | 내용보다 디자인에 시간 소모 (본문의 "4시간 30분→45분" 비교 참조) |
| Gamma 등 AI PPT 도구 | 빠르지만 PPTX 내보내기 시 레이아웃 깨짐, 세밀한 수정 제한적 |
| 템플릿 활용 | 내용에 맞게 커스터마이즈 번거로움 |

### 1-2. Marp — 마크다운 기반 프레젠테이션 도구

**Marp**(Markdown Presentation Ecosystem)는 마크다운 문서를 슬라이드로 변환하는 도구입니다:

```markdown
---
marp: true
theme: default
paginate: true
---

# 첫 번째 슬라이드

발표 내용을 **마크다운**으로 작성합니다.

---

# 두 번째 슬라이드

- 글머리 기호도 자동 정렬
- 코드 블록, 표, 이미지 모두 지원
```

핵심 장점:
- **내용에 집중**: 디자인은 테마가 처리, 발표자는 내용만 작성
- **버전 관리**: 텍스트 파일이므로 수정 이력 추적 가능
- **자동화 가능**: Claude Code가 마크다운을 생성 → Marp가 PDF/HTML로 변환
- **일관성**: 같은 테마를 적용하면 모든 슬라이드가 통일된 디자인

### 1-3. Claude Code + Marp 워크플로우

```
[보고서/자료] → Claude Code(내용 구조화) → Marp 마크다운 → PDF/HTML 슬라이드
```

Claude Code가 기존 보고서나 노트를 분석하여 Marp 형식의 마크다운을 자동 생성하고, Marp CLI가 이를 PDF나 HTML 슬라이드로 변환합니다.

## 2. Marp 설치와 기본 사용법

### 2-1. Marp CLI 설치

```bash
# Node.js가 설치되어 있다면:
npm install -g @marp-team/marp-cli

# 설치 확인
marp --version
```

> ⚠️ **Node.js가 없다면**: Homebrew로 설치 — `brew install node` 후 위 명령어 실행

### 2-2. 기본 변환 명령어

```bash
# 마크다운 → PDF 변환
marp slide.md --pdf

# 마크다운 → HTML 변환
marp slide.md --html

# 마크다운 → PPTX 변환
marp slide.md --pptx
```

### 2-3. Marp 마크다운 핵심 문법

| 문법 | 설명 | 예시 |
|------|------|------|
| `---` | 슬라이드 구분선 | 새 슬라이드 시작 |
| `<!-- _class: lead -->` | 슬라이드 클래스 | 타이틀 슬라이드 스타일 |
| `<!-- _backgroundColor: #1e3a5f -->` | 배경색 | 슬라이드별 색상 변경 |
| `![bg](image.jpg)` | 배경 이미지 | 전체 배경으로 이미지 삽입 |
| `![bg right:40%](image.jpg)` | 분할 레이아웃 | 오른쪽 40%에 이미지 |
| `![w:300](image.jpg)` | 이미지 크기 | 너비 300px |

## 3. 발표자료 에이전트 팀 설계

### 3-1. 에이전트 팀 구성

```
.claude/
  agents/
    slide-planner.md      ← 슬라이드 구성 기획
    content-extractor.md  ← 보고서/자료에서 핵심 추출
    marp-writer.md        ← Marp 마크다운 작성
    design-reviewer.md    ← 디자인·가독성 검토
```

**slide-planner.md**:
```markdown
---
name: slide-planner
description: "발표 목적과 청중에 맞춰 슬라이드 구성을 기획하는 에이전트"
---
# Slide Planner — 슬라이드 기획자

## 핵심 역할
- 발표 목적, 청중, 시간에 맞는 슬라이드 구성안을 설계합니다.
- 각 슬라이드의 핵심 메시지와 시각 자료 유형을 결정합니다.
- 스토리라인(도입→전개→결론)을 설계합니다.

## 작업 원칙
- 본문의 **1:1:1 법칙** 준수: 1슬라이드 = 1메시지 = 1분 발표
- 발표 시간 기준: 슬라이드 장수 = 발표시간(분) × 0.8
- 텍스트 최소화, 키워드 중심으로 구성
- 데이터는 표/차트로, 프로세스는 다이어그램으로 표현 제안

## 출력
- _workspace/slide_outline.md: 슬라이드 구성안

## 협업
- 완료 후 content-extractor에게 전달합니다.
```

**marp-writer.md**:
```markdown
---
name: marp-writer
description: "구성안과 추출된 내용을 Marp 마크다운으로 작성하는 에이전트"
---
# Marp Writer — Marp 마크다운 작성자

## 핵심 역할
- slide-planner의 구성안을 Marp 형식 마크다운으로 변환합니다.
- 적절한 Marp 디렉티브(테마, 페이지 번호, 클래스)를 적용합니다.
- 텍스트, 표, 코드 블록, 이미지 참조를 Marp 문법으로 작성합니다.

## 작업 원칙
- 첫 슬라이드는 `<!-- _class: lead -->`로 타이틀 슬라이드
- 슬라이드당 텍스트는 6줄 이내 (본문의 6×6 규칙)
- 강조는 **볼드**와 색상 활용, 밑줄은 사용하지 않음
- 모든 슬라이드에 발표자 노트(<!-- 발표자 노트 -->)를 포함

## 출력
- output/presentation.md: Marp 형식 마크다운 파일

## 협업
- content-extractor의 핵심 내용을 반영합니다.
- design-reviewer의 피드백으로 수정합니다.
```

### 3-2. CLAUDE.md 발표자료 규칙 예시

```markdown
# 발표자료 프로젝트 규칙

## Marp 설정
- 테마: default (또는 gaia, uncover)
- 페이지 번호: 항상 표시
- 가로세로비: 16:9

## 슬라이드 작성 원칙
- 슬라이드 1장에 핵심 메시지 1개
- 텍스트 6줄 이내
- 글꼴 크기: 제목 36pt 이상, 본문 24pt 이상
- 표는 5행 이내, 차트 설명 텍스트 포함

## 구조
- 표지 → 목차 → 도입(왜?) → 본론(3~5장) → 결론 → Q&A
- 전체 슬라이드: 발표시간(분) × 0.8 = 최대 장수
```

#### 🟢 Level 3 — 전문가 프롬프트 (에이전트 팀 활용)

```

input/report.md를 10분 수업 발표용 Marp 슬라이드로 변환해줘.

1. slide-planner: 10분 발표 구성안 작성 (8~10장)
   → _workspace/slide_outline.md
2. content-extractor: 보고서에서 슬라이드별 핵심 내용 추출

   → output/presentation.md

```

**결과**: 기획→추출→작성→검토의 4단계를 거쳐 체계적인 슬라이드가 완성됩니다. 각 슬라이드가 핵심 메시지 중심이고, 텍스트 양이 적절하며, PDF로 즉시 변환됩니다.

📋 **평가**:
- 정확성: ★★★★★
- 활용도: ★★★★★
- 핵심 차이: 기획/추출/작성/검토를 분리하여 각 단계의 품질이 높음
