## 1. Claude Code에서 이미지 생성이 가능한가?

### 1-1. Claude Code의 한계와 해결책

Claude Code는 텍스트 기반 도구입니다. 이미지를 직접 생성할 수 없지만, **외부 이미지 생성 API를 호출하는 스킬**을 만들면 가능합니다:

```
Claude Code (텍스트 명령) → 스킬 (API 호출) → Gemini (이미지 생성) → 파일 저장
```

이 방식의 장점:
- Claude Code의 텍스트 분석·구조화 능력 + Gemini의 이미지 생성 능력을 결합
- 프롬프트 작성, 결과 검토, 파일 관리를 Claude Code가 담당
- 반복 작업을 스크립트로 자동화 가능

### 1-2. Gemini 이미지 생성 API 준비

Gemini API를 사용하려면:

1. **Google AI Studio** (aistudio.google.com)에서 API 키 발급
2. 환경 변수에 API 키 설정:
   ```bash
   export GEMINI_API_KEY="your-api-key-here"
   ```
3. Claude Code 스킬에서 이 API를 호출

> ⚠️ **무료 사용량**: Gemini API는 무료 티어에서 일정량의 이미지 생성을 지원합니다. 과금에 주의하세요.

## 2. 이미지 생성 스킬 구조

### 2-1. 스킬이란?

Claude Code의 **스킬(Skill)**은 `.claude/skills/` 폴더에 정의하는 재사용 가능한 기능 모듈입니다:

```
.claude/
  skills/
    generate-image/
      skill.md          ← 스킬 정의 (언제, 어떻게 사용)
      generate.sh        ← 실행 스크립트 (API 호출)
```

### 2-2. 이미지 생성 스킬 예시

**skill.md**:
```markdown
---
name: generate-image
description: "Gemini API를 사용하여 텍스트 프롬프트로 이미지를 생성하는 스킬"
---
# 이미지 생성 스킬

## 사용 시점
- 사용자가 이미지 생성을 요청할 때
- 홍보물, 포스터, 배너 등 시각 자료가 필요할 때

## 사용법
generate.sh 스크립트를 실행합니다:
```bash
bash .claude/skills/generate-image/generate.sh "프롬프트" "output/파일명.png"
```

## 프롬프트 작성 가이드
- 구체적 장면 묘사 (주체, 행동, 배경, 분위기)
- 스타일 지정 (일러스트, 포토, 미니멀, 플랫디자인 등)
- 색상 팔레트 지정 (주색, 보조색)
- 텍스트 포함 시 정확한 문구 명시
```

**generate.sh**:
```bash
#!/bin/bash
PROMPT="$1"
OUTPUT="$2"

curl -s "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash-exp:generateContent?key=${GEMINI_API_KEY}" \
  -H "Content-Type: application/json" \
  -d "{
    \"contents\": [{\"parts\": [{\"text\": \"Generate an image: ${PROMPT}\"}]}],
    \"generationConfig\": {\"responseModalities\": [\"TEXT\", \"IMAGE\"]}
  }" | python3 -c "
import sys, json, base64
resp = json.load(sys.stdin)
for part in resp['candidates'][0]['content']['parts']:
    if 'inlineData' in part:
        data = base64.b64decode(part['inlineData']['data'])
        with open('${OUTPUT}', 'wb') as f:
            f.write(data)
        print(f'Image saved: ${OUTPUT}')
        break
"
```

### 2-3. 스킬 호출 방식

Claude Code에서 자연어로 요청하면, 스킬이 자동 실행됩니다:

```
"대학 축제 홍보 포스터를 만들어줘. 봄 분위기, 파스텔 색상으로."
→ Claude Code가 스킬을 인식 → generate.sh 실행 → 이미지 생성
```

### 3-1. 에이전트 팀 구성

```
.claude/
  agents/
    concept-designer.md  ← 홍보물 컨셉/메시지 기획
    prompt-engineer.md   ← 이미지 생성 프롬프트 작성
    image-generator.md   ← 이미지 생성 스킬 실행

```

**concept-designer.md**:
```markdown
---
name: concept-designer
description: "홍보물의 컨셉, 타겟, 핵심 메시지를 기획하는 에이전트"
---
# Concept Designer — 컨셉 디자이너

## 핵심 역할
- 홍보 대상, 타겟 청중, 핵심 메시지를 정의합니다.
- 시각적 컨셉(분위기, 색상, 스타일)을 결정합니다.
- 홍보물의 정보 계층(제목→부제→상세)을 설계합니다.

## 작업 원칙
- 타겟 청중의 관심사와 언어를 반영합니다.
- 핵심 메시지는 1개로 집중합니다.
- 색상은 3색 이내로 통일합니다.

## 출력
- _workspace/concept_brief.md: 컨셉 기획서

## 협업
- prompt-engineer와 copywriter에게 컨셉을 전달합니다.
```

**prompt-engineer.md**:
```markdown
---
name: prompt-engineer
description: "컨셉을 이미지 생성 프롬프트로 변환하는 에이전트"
---
# Prompt Engineer — 프롬프트 엔지니어

## 핵심 역할
- concept-designer의 기획서를 이미지 생성 프롬프트로 변환합니다.
- Gemini 이미지 생성에 최적화된 프롬프트를 작성합니다.
- 스타일, 구도, 색상을 구체적으로 명시합니다.

## 작업 원칙
- 본문의 **이미지 프롬프트 7요소** 중 5가지 이상 포함: 주체(Subject), 스타일(Style), 구도(Composition), 색감(Color), 분위기(Mood), 조명(Lighting), 비율(Format)
- 부정 프롬프트(원하지 않는 요소)도 명시
- **"○○ 스타일로" 프롬프트 사용 금지** (본문의 시각적 표절 경고 참조) — 스타일 요소를 묘사로 대체
- 한 번에 3가지 변형 프롬프트를 제안 (후보 선택용)

## 출력
- _workspace/image_prompts.md: 이미지 생성 프롬프트 3종

## 협업
- image-generator에게 프롬프트를 전달합니다.
```

📋 **평가**:
- 정확성: ★★★★★

**잘못된 방법 — 추상적이고 모호한 프롬프트**:

```
아름답고 멋진 대학교 포스터를 만들어줘.
다양한 색상으로 화려하게.
```

**결과**: "아름답다", "멋진", "화려하게"는 주관적 표현이어서 AI가 해석하기 어렵습니다. 결과물이 매번 달라지고, 의도와 다른 이미지가 생성됩니다. 본문의 이미지 프롬프트 7요소(주제·스타일·구도·조명·색감·분위기·비율)를 기억하세요.

**올바른 방법**:

```
대학 봄 축제 포스터. 일러스트 스타일.
중앙에 벚꽃 나무 아래 학생 3명이 피크닉하는 장면.
파스텔 핑크(#FFB6C1) + 민트(#98FF98) + 흰색 배경.
상단에 "Spring Festival 2024" 텍스트.
밝고 경쾌한 분위기. 세로 비율(3:4).
```

**차이**: 구체적 장면(누가, 어디서, 무엇을), 정확한 색상 코드, 스타일, 구도, 비율을 명시하면 의도에 가까운 이미지가 생성됩니다.

### 🎯 실습: 시리즈 홍보물 자동화 파이프라인 ⭐⭐⭐

**목표**: 여러 장의 시리즈 홍보물(행사 안내 시리즈)을 일괄 생성하는 파이프라인을 구축합니다.
**소요 시간**: 40분
**준비물**: 이미지 생성 스킬, 행사/프로그램 목록
**사용 도구**: Claude Code (비대화형 모드), Gemini API

#### Step 1. 시리즈 기획 ⏱️ 5분

```

프로그램 목록 (없으면 예시로):
1. [프로그램 1 — 예: 공연 동아리 쇼케이스]
2. [프로그램 2 — 예: 학술 동아리 전시회]
3. [프로그램 3 — 예: 체육 동아리 체험존]

시리즈 통일 요소: 로고 위치, 색상 팔레트, 레이아웃 구조
시리즈 차별 요소: 프로그램별 강조 색상, 이미지 컨셉

결과: _workspace/series_plan.md
```

✅ **체크**: 통일 요소와 차별 요소가 구분된 시리즈 기획서가 생성되면 성공

#### Step 2. 일괄 생성 스크립트 작성 ⏱️ 10분

```

기능:
1. _workspace/series_plan.md를 읽어 프로그램별 프롬프트 생성

3. 각 프롬프트로 이미지 생성

```
